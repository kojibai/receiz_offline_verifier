# Security Policy

## Reporting a vulnerability

Please report security issues privately.

- Email: security@receiz.com

Do **not** open a public GitHub issue for vulnerabilities.

## In scope

- False positive verification (tampered file passes)
- Bundle swapping attacks
- Canonicalization ambiguity
- Parser confusion (chunk parsing / compression handling)
- UI state bugs that can display “Verified” without passing checks
- Dependency supply-chain issues (if any are added later)

## Out of scope

- Social engineering
- Issues requiring physical device compromise

## Response targets

- Acknowledge within 72 hours
- Initial assessment within 7 days
