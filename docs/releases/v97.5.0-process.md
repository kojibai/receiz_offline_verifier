# Receiz v97.5.0 Process

Release date: 2026-06-30

## Goal

Ship `v97.5.0` as the official public app-state, enterprise `@receiz/sdk`, and MCP agent rail release. The release must make Receiz easier for developers and agents to build with while preserving proof-native law: known verified truth first, first admission only, then append forever, and no SDK/MCP authority above proof objects.

## What Worked

- The SDK release contract forced root version, SDK version, SDK runtime constant, and release docs to move together.
- The MCP package tests caught the npm `.bin` symlink startup defect with a direct entrypoint test before implementation.
- The MCP authority boundary remained explicit in code, README, tests, and developer pages.
- The developer page now surfaces MCP without adding client-heavy app behavior.
- The OpenAPI and SDK contracts stayed green after adding MCP discovery.
- MCP package build regenerated `dist` output with the new tools, URI templates, and 97.5.0 initialize metadata.
- Delegated agent tokens were added at the Receiz Connect/OIDC boundary, not inside MCP. This keeps local agents below Receiz identity proof while giving them a clean bearer token for scoped writes.

## What Did Not Work

- The original MCP entrypoint guard compared `import.meta.url` to `process.argv[1]` directly. That worked for the real file path but failed when npm invoked the symlinked binary path.
- The first MCP documentation path used `npx receiz-mcp`, which is not the correct public npm invocation for first-time host setup.
- The MCP route initially documented only the first tool set and needed to be updated after the offline, sports, preview, and URI template tools were added.
- Agent setup would be unsafe if agents copied browser cookies, OIDC client secrets, or Vercel env authority. The correct path is a Receiz-issued delegated token shown once, revocable by token ID, and labeled with `agent:*`.

## Practical Rule For Future Work

For every `@receiz/sdk`/MCP release, move root version, SDK package version, SDK runtime constant, MCP package version, MCP serverInfo version, package docs, developer pages, release docs, and package dist outputs together. SDK and MCP convenience must remain beneath Receiz primitives. Durable proof memory remains first admission only, then append forever.

For every agent-enabled release, verify `/developers/connect` can mint scoped delegated agent tokens, `/api/oidc/agent-tokens` can revoke them by token ID, and MCP documents `RECEIZ_ACCESS_TOKEN` as a local bearer token issued by Receiz auth.
