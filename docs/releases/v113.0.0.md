# Receiz v113.0.0

Release date: 2026-07-19

v113 ships Production Global Reconciliation for one explicitly named coordination domain: `receiz.com/global/v1`.

Exact sealed artifact bytes and independently verified causal history remain the stronger truth. Receiz.com now provides private immutable staging, atomic expected-head acceptance, scoped idempotency, receipts, audit references, effects, and outbox coordination beneath that truth. “Global” means acceptance by this named domain; it does not mean universal consensus.

The SDK exposes a provider-neutral `client.coordination.remoteDomain()` façade with distinct plan, Identity Seal authorization, stage, commit, accepted-head resolution, attempt recovery, and effect-status phases. Connect authenticates the account and tenant only. The exact Identity Seal capability authorizes the completed plan. Every returned artifact is independently reverified before the SDK exposes runtime-custodied artifact truth.

MCP retains the frozen five v112 artifact transition tools and adds exactly four reconciliation tools: global resolve, reconcile plan, reconcile stage, and reconcile commit. AI skills, current manifests, compiler quickstarts, and packed-package documentation use the current v113 matrix and exact nine-tool inventory.

The additive private migration creates coordination-only rows and object storage. It does not replace artifact, identity, ownership, history, or operation authority, and it does not change deterministic product first paint. Applying the migration is a deployment action separate from publishing these packages.

The deterministic unsigned registry lock is `docs/releases/v113.0.0-constitution-registry.json`, digest `4c4aa85f9785d205dcf7e4e5109837a83f8c3bf8e166130ae7e87353f299c637`, chained to v112.
