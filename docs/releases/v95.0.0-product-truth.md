## Frozen Primitives

| Primitive | v95.0.0 truth | Must not regress |
| --- | --- | --- |
| Proof object | Complete Receiz proof objects carry deterministic identity, sealed/verifiable payload, provenance, inspection behavior, and Kai/Klok state. | Do not expose a "proof object" contract without Kai/Klok. |
| Kai Klok | Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit. | Do not call Kai a label, metadata, vibe, timestamp helper, or optional ordering field. |
| Offline proof | Offline proof works because the object carries its own proof coordinate without consensus. | Do not make offline proof depend on server/DB rediscovery before use. |
| Durable proof memory | First admission persists verified truth. Later sync asks only for verified additions after the known head. | Do not rediscover known truth before projecting it. |
| SDK | The SDK is developer convenience over the canonical primitive contract, not a softer model. | Do not loosen SDK proof types beneath internal proof bundle law. |
| Chronos | Chronos is display, bridge, import context, and debugging context. | Do not use Chronos as the primary state machine for Kai-bearing proof objects. |
| Sports proof economy | Sports cards, appends, event proofs, ownership, and score history are proof object projections. | Do not drop known event memory or double-credit provider ID churn. |
| Wallet settlement | Wallet ledger rows are settlement primitives ordered by Kai heads. | Do not collapse older settlement history into wall-clock windows. |

## Frozen Invariants

- Use known verified truth immediately.
- Ask for verified additions after the known proof/Kai head.
- Complete proof objects carry Kai/Klok by default.
- `kaiPulseEternal` / `kai_upulse` is the unit, not an optional tie breaker.
- DB rows, server routes, SDK responses, and UI state are weaker than sealed/admitted proof truth.
- A partial later payload may fill missing fields, but it must not downgrade stronger admitted truth.
- Developer-facing SDK contracts must match internal canonical contracts.

## Release-Lock Requirement

`pnpm release:lock 95.0.0` must require this document for `v95.0.0` and later release cuts. A release that omits the product-truth freeze is incomplete because complete proof object authority, Kai/Klok state-machine doctrine, `kaiPulseEternal` / `kai_upulse` pulse-unit truth, offline proof completeness, SDK canonical contract parity, Chronos limits, Sports proof economy projection, and wallet settlement ordering would not be explicit release truth.
