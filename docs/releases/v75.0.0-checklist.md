## Canonical Product Gates

- Receiz ID login uses the device-held Receiz ID key as local-first account authority.
- Local identity truth projects the signed-in shell and seeds profile and wallet route truth before network reconfirmation.
- Server Receiz ID continuation may publish, bind, mint cookies, and append stronger additions, but it does not replace a stronger local account binding with a weaker or stale session payload for another account.
- Local account bindings persist as durable register truth across Receiz device, Receiz Key, PBI, email, and server-switch sources.
- Switching account bindings projects the selected account and clears account-scoped continuity when the actor changes.
- Identity Record, Identity Seal, and Receiz Key restore project the verified file session directly.
- Verified file restore admits the restored account locally and seeds profile, wallet, and upgrade truth.
- The restored account remains active when import, bind, wallet refresh, or remote reconfirmation is delayed.
- First-time offline Receiz Key restore requires verified portable state.
- Later restores can reuse admitted local key truth.
- Optional local PBI publication is a background append rather than restore permission.
- Unsupported browsers route to `/signin/receiz-id-fallback` with sanitized return paths.
- Primary sign-in surfaces remain Receiz ID pass-through.
- The backup sign-in route owns PBI/passkey, magic-link, and Identity Record / Seal / Receiz Key file restore.
- Account, managed profile, wallet, and upgrade surfaces prefer locally restored account authority over stale old-cookie echoes.
- Profile identity artwork and signed original-download paths preserve deterministic profile truth, owner proof, purchase context, and PBI download authorization without redefining the base profile shell.
- Plans and Upgrade preserve parity for immediate payment, upgrade actions, and old-browser fallback affordances.
- Market snapshots preserve stronger signed-in wallet, owned-item, prediction-history, and share-trade truth across lagging or anonymous payloads.
- Market snapshots refuse to preserve private state across viewer changes.
- Sold share positions remain closed when local sell history is newer than incoming market payloads.
- Market navigation uses stable internal navigation for record cards, verified routes, live players, and asset links.
- Market pointer handling prevents mobile press behavior from double-firing or falling through.
- Market mobile inputs and certificate panels preserve no-zoom viewport behavior, 16px editable controls, and wallet-density typography.
- Explore opens as a card-first, dock-safe mobile surface with bounded swipe rendering.
- Explore uses deterministic preview fallbacks and in-card read / play / save / share / proof actions.
- Explore route entry is local-first and server discovery remains background recovery.
- Already-held Explore view state and deterministic previews are reused instead of replaced by rediscovery payloads that are merely richer.
- Product doctrine explicitly names Identity Record, Identity Seal, and Receiz Key restore parity with PBI recovery.
- HPL continuity review and Receiz identity continuity one-pager are retained as release evidence for zero-regression local-proof upgrades.
- Package metadata, service-worker defaults, public version badges, verifier surfaces, README release pointer, and release docs move together to `v75.0.0`.
