# Receiz v106.0.0

Release date: 2026-07-16

v106.0.0 makes the SDK, MCP server, independent verifier, and AI skills one registry-bound constitutional enforcement system. `@receiz/sdk` remains the source of executable law and command admission; `@receiz/mcp-server` injects constitutional context and exposes safe semantic operations; `@receiz/ai-skills` carries machine-readable implementation contracts; the independently pinned verifier replays the result.

The release also restores owner-local admission for newly published profile proof objects. A canonical server row can no longer short-circuit local PBI signature admission or Live Player projection. Verified proof memory remains **first admission only, then append forever**.

## Application Behavior Scope

This is the only new main-application capability in v106.0.0: for the logged-in owner, a newly published Showcase proof object retains its accepted PBI signature and is admitted immediately to Live Player even when the canonical global row already exists. No other application surface receives a v106 feature or redesign. Version labels and offline-verifier release identifiers advance to `106.0.0`; all remaining release work belongs to the SDK, MCP server, independent verifier, AI skills, migration, and release-enforcement system. Release qualification also restores the already-governing Sports bottom-bar safe-area continuity contract; that is an inherited regression repair, not a new v106 Sports behavior.

This standalone verifier advances its package, public entrypoints, service worker, and release records together through `pnpm release:lock 106.0.0`.
