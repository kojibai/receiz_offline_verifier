# Receiz v95.0.0

Release date: 2026-06-27

## Goal

Ship a clean public table release that presents Receiz exactly: a proof-native artifact system whose complete proof objects carry Kai/Klok state, whose SDK exposes the canonical internal proof bundle contract, and whose public report cannot be mistaken for generic app architecture.

## What Worked

- The internal proof bundle implementation already required `kaiPulseEternal` and `kaiKlok`.
- Sports proof memory proved the durable history model in production experience: admit once, keep forever, append additions.
- Wallet Kai ordering proved that settlement history should resume from known heads, not wall-clock windows.
- The SDK launch created the right public entry point for developers to build Receiz-native apps.
- The branded architecture report gave non-Receiz readers a complete way to understand the system.

## What Did Not Work

- Public SDK typing was looser than the internal proof bundle contract.
- Some docs still used "when present" language around Kai, which made a complete proof object sound like it could exist without its living coordinate.
- The v94 report used "Kai state coordinate" correctly but not strongly enough; it needed to say Kai Klok is the state machine and `kaiPulseEternal` is the unit.

## Practical Rule For Future Work

Never let public developer contracts understate internal proof truth. If internals require a primitive, the SDK, docs, reports, fixtures, schemas, tests, and release language must require it too.
