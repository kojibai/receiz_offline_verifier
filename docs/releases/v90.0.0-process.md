## Release Framing

Title: Receiz `v90.0.0` - Official Stable Proof-Native Artifact System Release

`v90.0.0` locks Receiz as a proof-native artifact system. Sports event proof is a proof object, Sports Vault is an artifact system, Sports card ownership transfer is an ownership append, Sports value movement is a settlement primitive, Sports conformance is release evidence, competition rooms are proof-native game/market coordination surfaces, and metadata projects public proof without defining proof.

## Preparation Steps

1. Preserve `v89.0.0` as the prior current-release archive.
   - Keep `docs/releases/v89.0.0.md`.
   - Keep `docs/releases/v89.0.0-product-truth.md`.
   - Keep `docs/releases/v89.0.0-checklist.md`.
   - Keep `docs/releases/v89.0.0-process.md`.
   - Keep `docs/releases/v89.0.0-regression-lessons.md`.
   - Keep `docs/releases/v89.0.0-commit-history.md`.
   - Preserve `v89.0.0` as historical release-law truth.

2. Publish canonical `v90.0.0` release artifacts.
   - Add `docs/releases/v90.0.0.md`.
   - Add `docs/releases/v90.0.0-product-truth.md`.
   - Add `docs/releases/v90.0.0-checklist.md`.
   - Add `docs/releases/v90.0.0-process.md`.
   - Add `docs/releases/v90.0.0-regression-lessons.md`.
   - Add `docs/releases/v90.0.0-commit-history.md`.

3. Align repository release law and product-truth references.
   - Update `AGENTS.md` release law to `v90.0.0`.
   - Update product-truth reference status markers carried forward for the current release.
   - Preserve the primitive decision procedure, stronger/weaker truth hierarchy, and forbidden downgrade boundary.

4. Align repository release surfaces.
   - Update `package.json` to `90.0.0`.
   - Update root `README` current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.
   - Ensure static service-worker registrations request `sw.js?v=90.0.0`.
   - Ensure `site/sw.js` carries service-worker runtime version `90.0.0`.
   - Ensure local release-surface checks can execute through `pnpm lint`, `pnpm build`, and `pnpm release:lock`.

5. Validate primitive naming gates.
   - Sports event proof is named as a proof object.
   - Sports Vault is named as an artifact system.
   - Sports card ownership transfer is named as an ownership append.
   - Sports value movement is named as a settlement primitive.
   - Sports conformance is named as release evidence.
   - Competition rooms are named as proof-native game/market coordination surfaces.
   - Metadata is named as public proof projection, not proof authority.
   - Foreground anchor publication remains named as proof publication route truth, not outbox replay.
   - Profile original, profile showcase, public profile market append, and route snapshots keep direct primitive names.
   - Service-worker truth is named as release truth, not incidental cache behavior.

6. Validate post-v89 surface gates.
   - Sports Vault owner-row projection and `SportsVaultSurface` remain protected.
   - Vault projection and team-code normalization preserve verified vault truth.
   - Stale Sports single-game entry payload repair remains covered.
   - Server-side lineup admission uses verified game identity and owned Sports Vault proof cards.
   - Claimed/uploaded card image transfer reuses the existing offline-transfer ownership append route before signed-in local vault commit.
   - Transferred-out vault IDs and batch reveal preserve ownership state.
   - Pack purchase shelf, vault projection flags, action shelf progress nudge, and Save all share for pack reveal remain covered.
   - Issuance and Sports conformance tests remain release gates.
   - Event proof hero card, tool dock, event tiering, card impact, headline labels, rarity odds, link actions, and milestone preservation remain covered.
   - Sports conformance control and mode guide remain release evidence.
   - Live value algorithm, market-depth sales, and live market value seed remain settlement/value truth.
   - Competition rooms component and integration remain proof-native coordination surfaces.
   - Absolute URL metadata for Sports event proof sharing remains projection beneath proof truth.

7. Validate inherited surface gates.
   - `/api/receiz/anchor` remains excluded from local signed-action outbox admission.
   - Profile original publish projection obligations remain protected.
   - Sports Vault sealed image/manifest public proof publishing remains sealed-manifest first.
   - Wallet first-paint, ledger receipt, PBI, Receiz session, and service-worker navigation gates remain protected.
   - Lease/market ownership and settlement paths remain direct primitives.

8. Validate runtime lockstep.
   - Confirm all governed version markers show `v90.0.0`.
   - Confirm all current-release docs and indices point to `v90.0.0` artifacts.
   - Confirm historical `v80.0.0` through `v89.0.0` artifacts remain archived release truth.
   - Confirm `release:lock` requires `docs/releases/v90.0.0-regression-lessons.md`.
   - Confirm `release:lock` requires `docs/releases/v90.0.0-commit-history.md`.
   - Run `pnpm release:lock`.
   - Run whitespace diff check.

9. Validate broader platform gates before production promotion.
   - Run release-freeze and page-first-paint checks.
   - Confirm Sports ownership transfer, event proof value, conformance, market-depth, competition-room, metadata projection, foreground anchor/outbox, Profile, Sports Vault, Wallet, runtime/session, security, and release contracts.
   - Confirm target production DB migrations named by the release are applied.

10. Generate signed release attestation.
    - Run `pnpm release:attest` for the final commit in the environment that holds the release attestation signing key.
    - Retain the signed attestation artifact under the governance release-attestation policy.
    - Do not tag or ship without the signed attestation.

Broader platform release validation should include the gates named in `docs/releases/v90.0.0-checklist.md`, including:

- Sports event proof hero card, tool dock, tiering, card impact, headline labels, rarity odds, link actions, and milestone preservation
- Sports conformance control, mode guide, issuance tests, and Sports conformance tests
- live value algorithm, market-depth sales, and live market value seed behavior
- competition rooms component and integration
- absolute URL metadata for Sports event proof sharing
- release freeze, page first paint, TypeScript, release lock, regression lessons, commit history, whitespace diff, and release attestation gates

## Tag Criteria

Do not tag `v90.0.0` until:

- all required `v89.0.0` and `v90.0.0` release artifacts exist
- version surfaces are aligned to `v90.0.0`
- regression lessons are present and required by `release:lock`
- commit history is present and required by `release:lock`
- release gates in `docs/releases/v90.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v90.0.0`
- `pnpm lint` has zero warnings/errors
- `pnpm build` completes
- `pnpm release:lock` confirms the release surfaces
- whitespace diff check passes
- `pnpm release:attest` emits a signed attestation for the final commit

## Release Rule

Do not ship `v90.0.0` unless the release gates in this checklist pass from the exact release candidate, `pnpm release:lock` confirms the release surfaces, regression lessons, and commit history, whitespace diff passes, focused Sports ownership transfer, event proof value, conformance, market-depth, competition-room, metadata projection, foreground anchor/outbox, Profile, Sports Vault, Wallet, runtime/session, security, and release contracts pass, and production migrations named by the release are applied in the target environment.
