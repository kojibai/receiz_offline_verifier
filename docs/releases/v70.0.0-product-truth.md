# Receiz v70.0.0 Product Truth

Release date: April 24, 2026
Status: Locked for `v70.0.0` release prep

## Purpose

This document defines the product-truth freeze for Receiz `v70.0.0`, the first public alpha release cut of Receiz as an end-to-end product system.

For this offline verifier repository, the release is represented as current-release documentation, canonical release artifacts, shipped HTML version markers, public reference docs, and package-metadata alignment only. The verifier proof-format and producer-payload contract remain unchanged relative to `v60.0.0`.

## Core Product Truth

`v70.0.0` locks the laws already proven in the code:

> Identity is local. Proof is in the file. Verified truth is append-only. `/<username>` is canonical profile truth. Settlement and Reserve preserve funded causality. The network propagates and appends; it does not get to downgrade stronger proof already held by the device.

## Locked Release Surface

### 1. One-Click Receiz ID Entry

- Receiz ID is the primary everyday login rail.
- `/signin` is a pass-through route, not a destination page.
- Existing local Receiz ID accounts go directly to the managed profile.
- Cold devices create a local Receiz ID and continue directly to the new managed profile.
- PBI, keyfile, email, and magic-link paths remain recovery and interoperability rails.

### 2. Canonical Profile Truth

- `/<username>` is the profile.
- `/<username>?manage=1` is the same profile with owner powers layered on top.
- Owner data may append controls and hidden panels.
- Owner data cannot redefine first-paint identity, showcase, media, or shell.

### 3. Complete Proof-Native Value Loop

- Settlement remains proof-native value.
- Reserve remains funded value for external rails, notes, sends, positions, and wire transfer.
- Notes stay Reserve-only.
- Certificates and shares bridge live proof-native value into buyer-funded Reserve.
- Market sells preserve funded/unfunded causality and patch visible truth from the known trade result.

### 4. Local Truth Production Baseline

- Receiz ID, PBI/keyfile proof, route snapshots, wallet graphs, market positions, twin state, and proof bundles are real local truth.
- Server work still runs. It publishes, mirrors, mints cookies, recovers, discovers, and syncs verified additions.
- The user does not wait for network permission when the device already holds stronger proof.

### 5. Runtime And Offline Resilience

- Stale runtime, stale service-worker, and stale session states recover through local proof or server resync.
- Hard runtime recovery returns to the route the user was on instead of dumping them to home.
- Offline verifier surfaces, service-worker defaults, package metadata, and public version badges move together to `v70.0.0`.

### 6. World, Live Player, And Twin Identity

- Live twins cannot fall back to a profile picture as an identity substitute.
- Twin images stay locked to the current per-user/per-twin image.
- Mind trail ZIP downloads package every file with Receiz proof material so the downloaded bundle verifies as a proof bundle.
- Home hero proof-card PWA corrections are scoped to standalone PWA display only.

## Important Non-Changes

- The verifier proof-format and producer-payload contract remain unchanged by this documentation cut.
- Trusted-signature success still requires verified `signatureV4`.
- Effective anchor context remains required for verified outcomes.
- Groth16 fields remain required and only real `g16:` proof payloads are accepted.
- File-first verification remains authoritative and deterministic.
- The `v64.0.0` value-loop invariants remain in force and are carried forward by `v70.0.0`.
- Server runtime remains useful for publication, mirroring, recovery, discovery, cookie minting, sync, and verified additions.

## Allowed Evolution

Implementation details may change after `v70.0.0` only if they preserve the authority boundaries and locked behaviors above. If a future change materially alters one of these truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v70.0.0` if it:

- turns `/signin` into a destination page
- makes local Receiz ID identity wait for weaker network permission
- renders owner mode as a different profile truth than public profile truth
- lets owner-only data redefine public first-paint identity, showcase, media, or shell
- downgrades stronger local proof because transport returned stale, missing, or anonymous data
- collapses Settlement and Reserve causality
- credits funded value before funding is actually present
- lets live twin identity fall back to profile-picture identity
- ships version drift between package metadata, docs, public badges, and shipped verifier entrypoints
