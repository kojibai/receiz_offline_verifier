# Receiz v51.0.0

Release date: April 12, 2026

## Overview

`v51.0.0` is the release where the product framing catches up to what the system has actually become.

For this offline verifier repository specifically, `v51.0.0` is the release-surface alignment release for that broader product shift. The public ladder, upgrade language, and operator hardening live in the broader Receiz application; this repository carries the canonical release note, product-truth freeze, and visible version-surface bump so the public release artifacts stay locked together.

Receiz is no longer presented as a smaller proof utility with a few add-ons above it. The public plan ladder is now the real product model:

- Free = try it
- Authorship = prove it
- Business = own it
- Automation = run it
- Enterprise = scale it

This release rewrites the upgrade surface, plan language, surrounding upgrade copy, and key entitlement-facing UI so the product reads in that order. It also carries forward the operator hardening work that closed major truth gaps in Contacts, world campaigns, activity history, and outreach pacing after `v50.0.0`.

The trust model is unchanged. The file remains the authority. Deterministic manifests remain the authority for route truth. Ledger-backed settlement remains the authority for payment truth. `v51.0.0` changes the framing and operator quality around that core; it does not weaken the core.

The product-truth freeze for this release is defined in `docs/releases/v51.0.0-product-truth.md`.

## What Defines v51.0.0

- The pricing page now presents the real ladder from Free to Enterprise in plain product language, without centering Pro around WireProof.
- Public plan language is now `Authorship`, `Business`, and `Automation`, while internal billing keys remain safely mapped to `premium`, `business`, and `pro`.
- Upgrade, billing, account, explainer, and tier-adjacent surfaces now speak in the same voice about proof, ownership, and automation.
- Free remains fully usable. Paid tiers open deeper operating layers instead of turning the free product into a crippled teaser.
- Contacts, world campaign visibility, placeholder cleanup, guest/profile continuity, user-facing history, and calmer world operator behavior remain part of the release, not a separate side note.
- Release-governed version surfaces move together to `v51.0.0`.

## Since v50.0.0

### 1. The Product Ladder Now Matches The Product

The main change in `v51.0.0` is not cosmetic. It fixes a product-truth mismatch.

Before this release, the plan page still reflected an older story:

- `Premium` as the first paid tier
- `Pro` as the top tier
- WireProof as the dominant anchor of the top paid plan

That was no longer the real shape of the system. Receiz had already become a ladder from proof, to ownership, to active operation. `v51.0.0` makes that visible:

- `Authorship` is the proof layer
- `Business` is the ownership layer
- `Automation` is the active operating layer
- `Enterprise` is the scaled organizational form of Automation

This matters because product framing is not neutral. If the pricing page understates the system, the product becomes harder to understand, harder to buy correctly, and harder to upgrade through honestly.

### 2. Upgrade Surfaces Became Clearer, More Premium, And More Ordered

`/upgrade` is now structured around the actual value ladder rather than legacy tier naming.

- the plan cards are ordered by value from Free to Enterprise
- the comparison section explains the shift from presence to proof to ownership to active operation
- the upgrade path explains what changes at each layer
- the FAQ now answers the real tier questions: why Automation costs more, how Business differs from Automation, where WireProof belongs, and who Enterprise is for
- signed-out users can now understand the pricing story before committing to login
- signed-in users still get embedded checkout, billing, and direct upgrade paths

The page should now read like a mature product category, not a feature inventory.

### 3. Contacts And World Truth Stayed Part Of The Release

This release also carries the operator hardening work that made the live system more trustworthy after `v50.0.0`.

- `/contacts` now pages through the real owner CRM instead of stopping at an arbitrary backend chunk
- world campaign contacts already linked into the owner CRM stay visible from the Contacts frontend
- placeholder rows like bare `Receiz contact` records with no real identity are archived out of the active CRM view
- guest continuity and progressive identity merge paths are stronger
- world activity/history surfaces now prefer user-facing copy over provider or schema plumbing
- outreach pacing is more disciplined, especially around same-day repeat touches

`v51.0.0` is therefore both a framing release and a truth release.

### 4. Tier Language Is More Consistent Across The Product

The public naming shift does not stop at `/upgrade`.

Visible plan references now align more closely across:

- account and billing surfaces
- authorship and AssetSeal prompts
- WireProof upgrade prompts
- FAQ and explainer surfaces
- profile/authorship gating copy

Internal billing, subscription sync, and entitlement keys remain stable for safety. The product language changed without forcing a subscription-schema reset.

## Since v48.0.0

`v48.0.0` established Contacts and world as serious operator surfaces. `v50.0.0` turned that foundation into the first world revenue OS cut. `v51.0.0` should be understood as the maturation release after that jump:

- `48.0.0` built the operator substrate
- `50.0.0` connected that substrate into live revenue motion
- `51.0.0` makes the system easier to understand, easier to trust on screen, and more honest about where the real value lives

That continuity matters. The product did not change direction. The framing simply caught up to the system that was already there.
