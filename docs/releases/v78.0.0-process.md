## Preparation Steps

1. Preserve `v77.0.0` as the prior release baseline.
   - Keep `docs/releases/v77.0.0.md`.
   - Keep `docs/releases/v77.0.0-product-truth.md`.
   - Keep `docs/releases/v77.0.0-checklist.md`.
   - Keep `docs/releases/v77.0.0-process.md`.
   - Preserve `v77.0.0` as historical release truth.

2. Publish canonical `v78.0.0` release artifacts.
   - Add `docs/releases/v78.0.0.md`.
   - Add `docs/releases/v78.0.0-product-truth.md`.
   - Add `docs/releases/v78.0.0-checklist.md`.
   - Add `docs/releases/v78.0.0-process.md`.

3. Align repository release surfaces.
   - Update `package.json` to `78.0.0`.
   - Update root `README` current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update product-truth reference status markers carried forward for the current release.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.

4. Validate Signal Card World and Vault export gates.
   - Card fronts, share pages, QR payloads, and OG/Twitter images carry sealed proof and ownership truth.
   - `/game/cards` merges synced ownership with local card storage while preserving owner scope.
   - Global Vault sync publishes eligible owner rows into discovery without rewriting card truth.
   - Vault Card PNG export embeds manifest, archive hash, chunking, Merkle integrity, KAI timing, roster counts, claim paths, art paths, and restore rules.
   - Vault export and restore begin only from explicit user intent.

5. Validate mobile/PWA, transfer, arena, auction, and exchange gates.
   - Mobile Safari, installed PWA, and in-app browser downloads do not navigate, refresh, collide with headers, or run hidden-frame exports.
   - Transfer packages move ownership explicitly and avoid duplicate ownership ambiguity.
   - Signal Arena battle requests, queues, loadouts, AI twin authorization, settlements, stakes, action ledgers, transfer integration, and append verification preserve original card claims.
   - Card auctions require wallet holds and coherent reserve/account settlement.
   - `/signal-circuit` and `/exchange` present public ownership/value law from the same card and settlement records.

6. Validate conformance, dashboard, service-worker, and value-loop gates.
   - Signal Circuit conformance sources, badges, route inventory, and status entries cover real files and exclude high-risk private paths.
   - Dashboard first paint uses a smaller server/page boundary with data loading delegated to the client surface and API route.
   - Service-worker registration, PWA header behavior, runtime versioning, and release-worker marker coherence prevent stale installed/private-browser state.
   - Value-loop copy remains tied to reserve, settlement, ownership, and proof.

7. Validate release-surface lockstep.
   - Confirm all governed version markers show `v78.0.0`.
   - Confirm all current-release docs and indices point to `v78.0.0` artifacts.
   - Confirm `v77.0.0` artifacts remain historical and untouched.

## Focused Verification Matrix

Broader platform release validation should include the gates named in `docs/releases/v78.0.0-checklist.md`, including:

- Signal Card front/download/share/QR/OG/Twitter routes.
- Signal Card World global sync, owner rows, local merge, discovery, sale, auction, challenge, and proof state.
- Signal Vault Card PNG archive export, restore, and explicit user-intent download behavior.
- mobile Safari, installed PWA, in-app browser, header, and service-worker continuity.
- transfer-download packages and duplicate ownership prevention.
- Signal Arena request, loadout, AI twin authorization, stake settlement, action ledger, and transfer integration.
- auction wallet holds and reserve/account settlement.
- `/signal-circuit` and `/exchange` public routing.
- conformance source repair, badge/status inventory, and public/private route exclusions.
- dashboard first-paint boundary and runtime/service-worker release-marker coherence.
- value-loop copy contract coverage.
- `pnpm lint`
- `pnpm build`
- release-surface lock checks where available

## Tag Criteria

Do not tag `v78.0.0` until:

- all required `v77.0.0` and `v78.0.0` release artifacts exist
- version surfaces are aligned to `v78.0.0`
- release gates in `docs/releases/v78.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v78.0.0`
- `pnpm lint` has zero warnings/errors
- `pnpm build` completes

## Release Rule

Do not ship `v78.0.0` unless the release gates in this checklist pass from the exact release candidate, `pnpm lint` has zero warnings/errors, `pnpm build` completes, and the governed surfaces all show `v78.0.0`.
