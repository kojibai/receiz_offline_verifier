# Receiz v93.0.0 Invariant Register

Release date: June 25, 2026

This register makes explicit the v93 invariants proven by the work after `v92.0.0`.

## 1. Sports Economy Is A Public Proof Surface

Sports Economy publicly projects proof-native Sports card ownership, value, score memory, issue state, market state, and settlement-adjacent value. It is not a dashboard cache.

Violation: a route or component treats Sports Economy rows as generic analytics and drops proof IDs, score ledger basis, ownership identity, or value basis.

## 2. Bounded Proof-Read Projection Is A Primitive

Bounded proof-read projection is the database primitive for hot proof reads. It pages identity first and hydrates authoritative proof payloads second.

Violation: a hot path returns to broad PostgREST JSON aggregation over proof payload rows when an equivalent bounded RPC can preserve the same rows.

## 3. Proof Payload Authority Stays In The Ledger

Ledger payloads remain authority. RPCs, summaries, indexes, and rollups are projection mechanics.

Violation: a summary/RPC/index result is used to overwrite or outrank `append_json`, `proof_json`, sealed file truth, or embedded proof truth.

## 4. Same Proof Requires Same Completeness

If public card, QR card, Sports Economy, logged-in console, export image, and event page carry the same proof object, they must not diverge in proof completeness unless explicitly scoped by law.

Violation: one surface shows full lineage/history/value basis while another route-local projection drops it.

## 5. Deterministic Value Is Not Seller Ask

Sports card deterministic value is proof/settlement projection. Seller ask, best bid, internal cost, margin, and customer price are separate mechanics.

Violation: buyer UI shows internal cost/margin as proof value, or proof value is replaced by listing price.

## 6. Foreground Proof Viewing Outranks Background Cooldown

Background cooldowns reduce duplicate work. Active proof viewing may request scoped append warming immediately.

Violation: an active user waits for live proof appends because a background lease or stale guard suppresses visible hydration.

## 7. Full History Comes Before Summary

History surfaces must resolve available proof chains before showing compressed history.

Violation: only the latest proof appears when prior proof hashes or event rows prove a longer chain.

## 8. Command Proofs Must Open Exact Witnesses

Play Command and Pitch Command actions must bind to the selected event, pitch, game, source date, witness, and archive material when available.

Violation: a historical event pill opens current live command state instead of the event witness.

## 9. Physical Activation Is Proof And Settlement

Physical activation binds object intake, media, activation asset, checkout, and settlement state to the activated proof object.

Violation: activation is framed or implemented as generic ecommerce without object proof identity.

## 10. Deploy Order Is Part Of Release Truth

DB-backed release behavior is not fully locked until migrations exist, route code is live, and runtime logs show no missing function/schema-cache errors.

Violation: code using a new RPC is deployed before the RPC exists or before schema cache visibility is checked.

## 11. Visual Evidence Beats Claimed Passes

Screenshots, browser state, and user-visible observations are release evidence. Contracts must be strengthened when visual truth exposes a missing invariant.

Violation: tests pass while visible value, card media, history, layout, or first-paint behavior regresses.

## 12. Speed Must Preserve Ontology

Performance work may reduce work, bound reads, defer enrichment, and reuse stronger truth. It may not change what the object is.

Violation: proof object becomes a card, ownership surface becomes dashboard state, settlement route becomes checkout, or offline/archived proof becomes fallback cache.

## Release-Lock Requirement

`pnpm release:lock 93.0.0` must require this document for `v93.0.0` and later release cuts. A release that omits the invariant register is incomplete because the Sports Economy, bounded proof-read, value, lineage, command witness, physical activation, deployment, visual-evidence, and ontology-preservation invariants would not be explicit release truth.
