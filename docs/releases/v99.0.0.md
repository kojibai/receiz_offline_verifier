## Title

Receiz `v99.0.0` — Durable Local Proof, Whole-Market Public Register, Deterministic Live Player, and Identity Continuity Release

## Summary

`v99.0.0` locks the complete working set after the annotated `v98.0.0` release tag: 39 committed changes across 184 paths, with 17,733 insertions and 2,084 deletions before the v99 release-lock edits.

The release makes durable local truth operational across identity, authorship, public proof, market membership, profile/player continuity, portable proof memory, Sports proof state, and service-worker projection. It adds the complete whole-Market public proof register, local PBI authorship and custody appends, Receiz Key portable `receiz.account.state.v3` export, deterministic live-player first paint, canonical Explore projection, governed market tombstones, and Kai-moment derivation from sealed pulse state.

This is an artifact-system release. It does not promote network, database, session, service-worker, SDK, or MCP mechanics above sealed artifacts, deterministic proof objects, verified local truth, verified appends, identity proofs, ownership state, settlement state, or public proof surfaces.

## What Changed Since v98.0.0

- The live player now first-paints from deterministic public proof truth and composes stronger held Market/profile/player truth without waiting for session, private owner state, world preparation, or player API rediscovery.
- Whole-Market public membership became a durable complete proof register with revision admission, tombstones, owner-scoped projection, full original/media preservation, and rejection of partial, older, malformed, or archive payloads as replacement truth.
- Profile publish and visibility paths now own public projection into profile, live player, and Market membership; a DB row or owner-local view alone is not global public proof completion.
- Market tombstones now bind removal to confirmed current membership and held route truth, so weak or pending payloads cannot erase stronger admitted public proof.
- Explore now has a canonical complete projection, exact-route continuity, bounded grid windowing, and background append behavior beneath settled first paint.
- Explore now renders its unchanged deterministic proof bundle per request so Next bootstrap scripts carry the enforced CSP nonce; the two-card feed hydrates instead of entering browser recovery, and live-player handoff preserves its computed queue-completeness truth.
- Already loaded same-origin showcase images are re-admitted through the existing verified public-proof-media cache when the service worker becomes ready, closing the first-uncontrolled-page gap that made image proof media intermittently disappear offline while later-requested audio/video remained available.
- Receiz Key export now carries complete portable proof memory for profile originals, wallet/ledger, calendar, Sports vault and event state, and other `receiz.account.state.v3` domains without allowing partial online projection to truncate stronger held truth.
- Receiz ID local creation/publication now preserves strongest local identity projection, cold-flushes verified publication appends, distinguishes publication from session issuance, and keeps local identity truth active through transient server/session gaps.
- PBI authorship now verifies locally, stores a durable append-only outbox, binds reconnect to exact credential/account/source custody, publishes idempotently, and never caps or evicts admitted pending proofs.
- PBI signature history is permanent proof append memory. Profile snapshot writes preserve it, proven global signing remains an additive fallback, and service-worker epochs invalidate runtime mechanics without redefining admitted proof truth.
- Account, Profile, wallet, bottom chrome, and recovery surfaces now compose the strongest local identity/session proof instead of letting stale or absent cookie state downgrade the recovered account.
- Sports arena/vault paths gained append-only vault admission, normalized game primary keys, dated Fight snapshots/archives, portable export coverage, and finalized-entry replay prevention.
- Kai moment projection now resolves from sealed pulse state, preserving Kai Klok as the proof object state machine rather than deriving causal truth from Chronos transport labels.
- Market first-paint Chronos labels now use an explicit UTC display boundary, preventing server/device timezone drift from regenerating the settled proof surface during hydration.
- The measured root runtime is 435,003 bytes; the v99 guard ceiling is 437,000 bytes so strongest-local-identity continuity remains in first-paint runtime with a narrow documented margin.
- App, `@receiz/sdk`, `@receiz/mcp-server`, runtime, service worker, public labels, offline verifiers, and active release documents move together to `99.0.0`.

## Product Law Preserved

- Durable proof memory remains **first admission only, then append forever**.
- Verified local truth is admitted before weaker session, DB, server, or transport state.
- Public proof surfaces first-paint deterministic proof truth before authenticated/private enrichment.
- Whole-Market membership is a public proof register; it is not a mutable UI cache.
- PBI authorship proof is locally recomputable and publication is a verified append, not permission for local proof to exist.
- Receiz Key portable state is file-carried proof memory, not a convenience backup.
- Receiz ID and PBI retain their distinct identity boundaries; binding/publication remains additive.
- Kai Klok remains the proof object state machine; Chronos remains display/bridge time.
- SDK and MCP remain convenience rails beneath implemented Receiz primitives.
