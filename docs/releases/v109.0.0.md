# Receiz v109.0.0

Release date: 2026-07-18

Receiz v109 is the coordinated local-offline developer release for `@receiz/sdk`, `@receiz/mcp-server`, and `@receiz/ai-skills`.

## Release outcome

- `receiz.verification.verifyArtifact(file)` verifies complete `.receizbundle` proof objects locally from their bytes.
- `receiz.artifacts.verifyAndOpen(file)` verifies before extraction and returns the enclosing artifact separately from the verified payload.
- Public verification/opening performs zero network calls and requires no access token, cookie, database, registry lookup, or running Receiz.com process.
- Signature V4 roots, the document-seal Groth16 verification key, and the `snarkjs` runtime are packaged with the SDK.
- One-byte mutation, malformed carrier, artifact-binding mismatch, Signature V4 failure, Groth16 failure, and invalid portable continuity fail closed.
- MCP artifact tools compose the same SDK verifier and remain orchestration beneath sealed artifact truth.
- Artifact-related AI skills require `Network calls during verification: 0` and a recorded local-verifier result before a production-ready claim.
- `ARTIFACT-011` makes zero-network complete-artifact verification constitutional release law.

## Compatibility

- The maintained public imports remain unversioned: `@receiz/sdk`, `@receiz/sdk/react`, `@receiz/sdk/compiler`, `@receiz/sdk/testing`, and `@receiz/mcp-server`.
- Existing v108 method names and artifact/payload result separation remain source-compatible.
- Native issuance remains authenticated Record -> Seal and may use Receiz.com; v109 changes verification and opening, not issuance authority.
- Historical sealed artifacts, receipts, registry records, and release evidence remain immutable historical truth.
- Obsolete version-specific SDK/MCP implementations, exports, generators, and migration commands are no longer shipped or installable.
