## Title

Receiz `v98.0.0` - Post-v97 Proof Graph, Developer Runtime, Ledger Settlement, Sports Performance, and Pitch Archive Release

## Summary

`v98.0.0` locks the full working set after `v97.0.0`: SDK and MCP developer-runtime rails, delegated agent tokens, signed merchant public-store publishing, developer webhooks, the public proof graph, transaction and ledger proof surfaces, market-cap and settlement projections, sports vault and arena performance hardening, profile/public proof continuity, Receiz AI skills, and the Pitch Command day-proof archive catch-up.

This is a release lock over `v97.0.0..HEAD`, not only over the last minor tag. The review scope contains 65 commits and 535 changed paths before the `98.0.0` lock edits.

The release preserves the authority hierarchy. SDK, MCP, OIDC, webhooks, public app-state rows, service worker state, charts, developer pages, and database migrations are mechanics beneath proof objects, sealed artifacts, verified local truth, Kai/Klok state, ownership appends, settlement primitives, public proof surfaces, and offline verification.

- Signed merchant public-store publishing was promoted to first-class SDK and API behavior through Identity Seal / Receiz Key proof authority, no delegated bearer token required for that lane.
- Developer webhooks added endpoint create/list/update/delete, secret rotation, test events, signature helpers, OpenAPI coverage, SDK/MCP support, and portal UI.
- `/proof-graph` became a public proof surface with live Kai pulse state, proof/economy/flow/asset/agent views, detailed panels, mobile graph navigation, public ledgers, market-cap projections, settlement rollups, transaction volume, asset pulse charts, and visual contract coverage.
- `/transactions/[slug]/[code]/[pulse]` added public transaction proof inspection with wallet ledger links, proof-bundle projection, copy affordance, and route contracts.
- Ledger and wallet routes gained proof-bundle extraction, transaction prioritization, transfer value snapshots, cursor repair, public ledger pagination, `kai_upulse` / Kai Klok display, and stronger sorting/rollup handling.
- Market-cap and LiveChart work added downsampling, OHLC candle handling, sampling budgets, chart controls, asset pulse projections, and only-open asset chart construction so rich proof graph views remain light beneath the visible primitive.
- Market/profile write paths gained profile first-paint fallback and projection repairs so public profile and market proof membership remain write-side responsibilities rather than read-path rediscovery.
- Sports vault and arena work added `owner_current` projection mode, sport-aware portrait/card image behavior, base-card `BASE 1:1` language, deferred surfaces that preserve route first paint, bounded rollup workers, active-job dedupe, transfer routes, value snapshots, and closeout/performance contracts.
- Pitch Command archive work added day-proof bundles through July 7, 2026, manifest/projection updates, strict archive parsing, workflow catch-up behavior, and lint-safe proof-register typing.
- Receiz AI skills were added for builder, commerce, distribution, MCP agent, offline verifier, proof, skill bundle, and sports card workflows, all with canonical language that keeps proof primitives above generic app categories.
- Public release identity moved to `98.0.0` across root package, SDK package, MCP package, runtime/service-worker defaults, public footer labels, offline verifier labels, and release documents.

## Developer Outcome

Developers can now build and operate Receiz apps with two distinct rails:

```ts
import { createReceizClient } from "@receiz/sdk";

const receiz = createReceizClient({ baseUrl: "https://receiz.com" });

await receiz.publicStore.publishWithIdentityProof({
  keyFile,
  tenantHost: "bjklock.receiz.app",
  merchantReceizId: "bjklock.receiz.id",
  storeStateRecord,
});
```

Merchant-owned public-store state is signed by the merchant proof object. Delegated agent/server work uses scoped OIDC tokens:

```json
{
  "mcpServers": {
    "receiz": {
      "command": "npx",
      "args": ["-y", "@receiz/mcp-server"],
      "env": {
        "RECEIZ_BASE_URL": "https://receiz.com",
        "RECEIZ_ACCESS_TOKEN": "<delegated agent token>"
      }
    }
  }
}
```

Those rails are convenience and publication mechanics. They do not replace proof objects, verified appends, durable local truth, ownership appends, settlement primitives, or offline verification.

## Product Law Preserved

- Proof objects remain the source of truth.
- Kai Klok remains the proof object state machine.
- Durable proof memory remains first admission only, then append forever.
- Public proof surfaces first paint deterministic proof truth before authenticated/private enrichment.
- App-state and public-store rows remain projections beneath proof objects and verified appends.
- SDK, MCP, OIDC, webhooks, charts, service worker, and database migrations remain mechanics beneath Receiz primitives.
- Merchant public-store publishing uses Identity Seal / Receiz Key proof authority and does not require delegated token authority.
- Developer webhooks publish signed delivery events and test events beneath Connect/OIDC scopes; they do not create proof authority.
- Ledger, wallet, transaction, and market-cap surfaces preserve Kai/proof ordering and settlement language.
- Sports arena and vault changes reduce work beneath the visible primitive without removing proof, ownership, settlement, or first-paint truth.
- Pitch Command day proofs are release-carried proof archive material, not generic static content.
