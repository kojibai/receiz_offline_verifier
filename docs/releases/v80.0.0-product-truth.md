## Locked Product Truth

- Known verified truth moves first. Server and network paths append verified additions, and visible surfaces do not visibly rediscover what the system already knows.
- Local identity proof, file restore, profile truth, market/wallet state, card ownership, vault export, and creator production all follow the same verified-truth-first law.
- `/<username>` remains canonical profile truth, and `?manage=1` remains a permission overlay only.
- Identity Record, Identity Seal, Receiz Key, PBI, email, server session, and Receiz ID continuity converge into the same signed-in product experience after proof acceptance.
- Wallet, Settlement, Reserve, market shares, cards, auctions, packs, transfers, and battle stakes preserve funded causality and append-only value truth.
- Signal Cards are owned Receiz assets across game, world, vault, discovery, transfer, auction, battle, market, export, share, and conformance surfaces.
- Signal Card share-card and proof popovers stay bounded and cannot become full-screen replacement surfaces.
- Vault Verify enters an explicit opening state only after user click; opening a panel is not download, export, or restore intent.
- Arena append proof manifests expose card battle/appended proof without mutating original card truth.
- Creator OS and World agent loops produce durable artifacts, charge or hold wallet value explicitly, and keep first paint settled while private work enriches behind it.
- Public Proof, Signal Circuit conformance, Status, and Exchange make the public system inspectable without publishing private/admin/worker routes.
- Runtime and service-worker coherence are release surfaces. A stale worker, stale header, stale chunk, or stale session must not create a second-load feeling.
- Release labels, service-worker defaults, offline verifier labels, README release pointer, release docs, and world resident worker markers all move together to `v80.0.0`.

## Out Of Bounds

A change is out of bounds for `v80.0.0` if it:

- weakens file-first verifier authority or changes verifier trust semantics in this repository.
- treats accepted local proof as provisional cache after it has established account, card, vault, wallet, or profile truth.
- allows owner mode to redefine public canonical profile truth.
- detaches Signal Card share/download/public surfaces from sealed ownership proof.
- starts Vault export, restore, download, timers, or hidden navigation without explicit user intent.
- allows transfer, battle, auction, listing, or sale behavior to mutate an original card claim instead of appending truth.
- permits auction, purchase, transfer, reserve, market, or battle settlement without wallet/account/reserve coherence.
- leaks private vault, pack purchase, admin, webhook, worker, private session, privileged issuer, or legacy keyring surfaces into public status/conformance inventory.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker defaults, badges, or release-worker markers.
