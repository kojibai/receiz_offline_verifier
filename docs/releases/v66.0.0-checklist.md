## Canonical Product Gates

- Local verified-register proof outranks weaker offline or legacy evidence.
- Receipt rendering classifies proof authority.
- Receipt details and proof popovers expose stronger local authority without letting older payloads reframe it.
- Wallet transaction receipts verify deterministic ledger proof bundles before treating ledger claims as truth.
- Account surfaces preserve a valid bootstrap/keyfile/PBI-backed local identity when the session API is unavailable or weaker.
- Anonymous or missing network session payloads do not erase locally held identity proof.
- Network session is enrichment over a local identity root, not permission to exist.
- Local twin register projection wins for the visible reply when present.
- Online stream remains available when no local register projection exists, but it does not silently replace settled local projection.
- WebLLM local model support has deterministic fallback when the model is unavailable.
- Twin mind PNG import requires portable public-key verification instead of server-secret HMAC authority.
- World profile/public/me snapshots opt into service-worker offline persistence where deterministic or private route truth requires it.
- Explore snapshots are durable verified local truth and reconcile into the visible view without replacing useful already-visible content.
- Wallet projects the durable wallet graph during first render.
- Market snapshot continuity preserves known local positions.
- Tracked positions require a transport payload to prove a stronger state before removing visible local position truth.
- Offline persistence is explicit through `x-receiz-offline-persist` response headers.
- Deterministic public snapshots and private account snapshots are stored as local truth carriers.
