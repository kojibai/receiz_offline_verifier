# Receiz v88.0.0 Regression Lessons

Release date: June 3, 2026
Status: Required release-lock artifact for `v88.0.0`

## Purpose

This document records the post-v87 regression lessons that `v88.0.0` turns into release gates. It is release truth, not an activity feed. Each lesson names the stronger primitive that must keep authority during first paint, hydration, route entry, runtime migration, or database hardening.

## Lessons Locked

1. Live Sports field truth cannot be treated as generic display state.
   - Stronger primitive: live Sports proof truth for selected-game field state, locked entries, per-player scores, live cockpit totals, and finalized reward recovery.
   - Regression boundary: hydration and source-date appends may refresh or append, but they may not erase stronger selected-game proof/local truth.

2. `/baseball` cannot fork canonical baseball arena truth.
   - Stronger primitive: canonical Sports baseball arena route truth.
   - Regression boundary: `/baseball` is a direct entry route to `/game/sports/baseball` behavior, not an independent game surface.

3. Fallback field data cannot replace known detailed live feed truth.
   - Stronger primitive: detailed arena field truth, including inning, score, base state, pitch-by-pitch recap, last play, batter, and pitcher.
   - Regression boundary: fallback is allowed only when the real feed is genuinely unavailable.

4. Market/Profile first paint cannot be rediscovered after stronger public proof truth is known.
   - Stronger primitive: canonical public profile truth and market proof object truth.
   - Regression boundary: session, owner permissions, RLS-constrained asset lookup, and server reconciliation may append controls or assets, not redefine public first paint.

5. Public Live Cockpit cannot be downgraded to chat UI.
   - Stronger primitive: public witness/live proof surface truth for `/live` rooms and public live cockpit routes.
   - Regression boundary: chat, SFU, and notification mechanics support the cockpit; they do not rename or define it.

6. Service-worker/runtime version drift can become accidental authority.
   - Stronger primitive: service-worker release truth and deterministic runtime projection key truth.
   - Regression boundary: package version, public labels, service-worker version, and default runtime coherence must move together.

7. Explicit session clear/logout must remove stale runtime-scoped authority.
   - Stronger primitive: deterministic first-paint session truth after explicit clear/logout.
   - Regression boundary: legacy durable keys must not resurrect stale signed-in first paint after a user clears or logs out.

8. DB timeout fixes must stay testable and privilege-bounded.
   - Stronger primitive: DB policy/index/migration evidence for Supabase timeout hardening.
   - Regression boundary: production application requires the exact target Supabase project and privileged deployment path; repository tests prove only the migrations and guards present here.

## Release-Lock Requirement

`pnpm release:lock` must require this document for `v88.0.0` and later release cuts. A release that omits regression lessons is incomplete because post-regression truth has not been made explicit.

## No Downgrade Statement

The lessons above preserve proof-native artifact system truth, deterministic first paint, public proof/witness surfaces, service-worker release truth, and append-only verified history. They do not downgrade proof objects to cards, public proof surfaces to share pages, live proof surfaces to generic UI, settlement primitives to balance display, or route truth to framework convention.
