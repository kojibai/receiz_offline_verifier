# Receiz v92.1.0 Regression Lessons

Release date: June 22, 2026

This document is part of the release lock. It records the regression classes found or fixed during the `v92.1.0` arc and the rules that must prevent the same mistakes.

## What Regressed

### Foreground Live Appends Waited Behind A Background Guard

Some new Sports Arena events appeared late even though older events hydrated correctly. The active signed-in foreground view was not missing proof permanently; it was relying on slower catch-up because the append warming path was held behind a 75-second stale guard.

### Historical Pitch Pills Could Open Current Pitch State

The card event panel Pitch proof action could open the current Pitch Command context instead of the exact pitch for the selected historical event. That violated pitch witness truth.

### Public Event Pages Missed Video Available In Card Popovers

The logged-in card popover could show real event video while the public event proof page did not. That meant two surfaces for the same event proof object projected different media truth.

### Cycle Completion Was Not Projected As Milestone Truth

A triple that completed a Bryce Harper cycle was not automatically recorded as milestone proof on the event page. The event renderer showed the base play result without projecting the milestone implication from verified game/event truth.

### Play Command Mobile Hierarchy Became Too Dense

The field of play did not occupy enough of the Play Command surface, and official outcome geometry data competed with the 3D telemetry view. The product surface must make the actual play primary and move detail into secondary inspection.

### Pack Derby Could Force Close On Open

The recent Derby first-scene/drop stabilization work touched the opening path and could cause a refresh or forced close when the game opened.

### Vercel Build Failed On Nullable Proof/Video Types

The release candidate hit TypeScript failures around nullable selected game pk, playable video, and selected event proof. These were real release blockers because build failure prevents proof deployment.

## Why It Regressed

- Background pressure protection was reused for a foreground active-view path.
- Event proof action state did not carry the exact historical witness binding all the way into Pitch Command.
- Public event page projection and logged-in card popover projection diverged instead of sharing the same event proof video resolver.
- Milestone projection was treated as optional enrichment instead of proof-bearing event truth.
- Play Command detail controls were allowed to compete with the primary 3D play surface.
- Derby stabilization work changed runtime entry behavior without locking the open path.
- TypeScript nullability exposed real proof-boundary ambiguity and had to be narrowed at the call site.

## Locked Rules

- Active foreground Sports Arena views must not wait behind background stale guards for live append warming.
- Pitch proof pills must open exact pitch witnesses for the selected event, including historical source date and game pk.
- Event proof pages and card popovers must project the same media proof when the event proof carries playable video.
- Milestone truth must be projected when event/game proof establishes it.
- Play Command puts the 3D field/play telemetry first; dense data belongs in compact controls or secondary inspection.
- Derby opening behavior is part of the proof-game surface and must not refresh, close, or reset without user action.
- Nullable proof, pitch, play, and video inputs must be narrowed before crossing proof-action boundaries.
