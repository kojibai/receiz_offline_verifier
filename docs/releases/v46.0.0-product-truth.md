# Receiz v46.0.0 Product Truth

Release date: March 30, 2026
Status: Locked for `v46.0.0` release prep

## Purpose

This document defines the product-truth freeze for Receiz `v46.0.0`. It records which post-`v45.0.0` behaviors are now treated as explicit non-regression product truth at release time.

## Core Authority Boundaries

The release does not relax the core Receiz authority model:

- The file remains the authority for artifact truth.
- Deterministic manifests remain the authority for historical route truth.
- Replayable append-only ledger math remains the authority for settlement truth.
- Viewers, players, Public Twin shells, world stages, and storage routes remain transport around those boundaries, not replacements for them.

## Locked Product Surface

### 1. Original and Document Delivery

The original-file and document experience is now treated as a first-class product surface.

- Documents must classify deliberately across PDF, DOCX, text, JSON, CSV, and code-like assets.
- Truthful reading flows must stay openable inline and fullscreen instead of collapsing into attachment-only fallback behavior.
- Download precedence between original, derivative, and moment-record assets must remain explicit.
- Profile/original delivery and verified-player delivery must stay coordinated.

### 2. Verified-Original Player

The live verified-player route is now a release-governed destination.

- Profile identity framing must stay explicit.
- Purchase and paid-download behavior must remain deliberate rather than implicit.
- Kairos, pulse, queue, timeline, and direct profile-navigation surfaces must stay coherent.
- Route-specific social/share framing for the live-player route is part of the locked product surface.

### 3. Public Twin Photoreal Forge and Portrait Authority

Photoreal generation and shell behavior now operate under an explicit portrait contract.

- Weak renders must not silently pass as acceptable photoreal outputs.
- Feature-map metadata must persist with authority information.
- Face-feature alignment may use backend, native, or heuristic paths, but fallback behavior must stay explicit.
- Compact and mobile rendered-portrait overlays must use deterministic fit primitives rather than ad hoc placement.
- Owner setup, embed behavior, and inactive-twin handling remain deliberate product states.

### 4. World Stage and Delivery Truth

World presentation remains constrained by real delivery state.

- Stage and dock are separate surfaces and must not collapse into one another.
- Video renders only when a real delivery exists.
- Presentation layers must not fake embodiment or media delivery that is not backed by a real transport path.

### 5. Streaming and Storage Correctness

Delivery correctness is part of product truth, not an implementation detail.

- Signed storage proxy routes must preserve byte-range streaming behavior.
- `HEAD` requests must preserve partial-content semantics.
- Inline document viewing must stay openable without bypassing delivery rules.
- Service-worker versioning remains tied to release-version truth.

### 6. Release Freeze Mechanics

The release contract itself is now more explicit.

- A release-scoped product-truth document is required.
- A canonical product-surface inventory contract is required.
- Profile/world release-contract coverage is part of the freeze boundary.
- The broader application release groups major local blockers under `pnpm test:release-freeze`; this documentation repo records the public-facing freeze statement and current-release version surfaces.

## Allowed Evolution

Implementation details may change after `v46.0.0` only if they preserve the authority boundaries and locked behaviors above. If a future change materially alters one of these product truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v46.0.0` if it:

- weakens file/manifold/ledger authority boundaries
- turns deliberate original/document viewing back into attachment passthrough
- reduces the verified-original player to a thin wrapper with weaker purchase or metadata framing
- permits weak or authority-unclear photoreal outputs to pass silently
- collapses stage/dock separation or renders video without real delivery
- relaxes streaming/storage correctness expectations
- changes release truth without updating the versioned freeze layer
