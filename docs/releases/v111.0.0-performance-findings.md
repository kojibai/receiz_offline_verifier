# Receiz v111.0.0 Performance Findings

`v111.0.0` removes circular and structural authority shortcuts while preserving exact-byte local admission. Authority-bearing recovery work is recomputed from retained artifact bytes, independently rooted proof history, a single verified head, complete identity binding, current capabilities, and a complete recovery-plan basis.

## Findings

- Canonical domain-separated encoding makes authority digests deterministic across SDK, MCP, clean-room installs, and conforming receivers.
- Admission retains and reverifies exact artifact bytes; comparing a stored admission digest is insufficient and structural JavaScript objects are rejected.
- Each authority-bearing proof-history node requires its own independently verified stronger evidence root. Normalized graphs remain inspection material until rooted.
- Divergent verified heads fail closed rather than selecting a convenient latest branch, preventing reconciliation work from manufacturing recovery state.
- Canonical identity proves private-key control through a live domain-separated signing challenge bound to the verified public identity.
- Recovery plans bind the complete authority basis—artifact, admission, history roots, head, identity proof, state, capability basis, and registry—so commit does not rediscover or weaken inputs.
- Current-registry capability enforcement rejects historical operational capabilities while preserving historical sealed artifacts as inspectable evidence.
- Expected failures return structured immutable results without writes, outbox intent, receipts, audit references, or partial state.
- MCP separates deterministic plan identity from consumable attempt identity; committed and failed attempts require a fresh confirmation digest while an actively pending identical attempt may resume.
- Receiz.com application/API behavior and database schema/migrations remain unchanged, so v111 adds no application foreground, query, or production-data work.

## Performance Boundary

- Exact verified artifact truth and independently rooted history precede planning, explanation, session, server, registry, network, UI, or projection work.
- Receiver conformance rejects weakened authority without restricting application freedom or elevating runtime custody into truth.
- Fail-closed structural, divergence, identity, capability, plan-basis, and attempt errors change nothing.
- This repository validates local release surfaces and constitutional records; GitHub attestation, package publication, deployment, production verification, and Git push remain external.
