# Receiz v105.0.0 Regression Lessons

`@receiz/sdk` preserves first admission only, then append forever.

## What Regressed

No Receiz proof behavior regression was accepted. During development, the universal root's v104 compiler exports were proven to pull Node built-ins into browser graphs.

## Why It Regressed

Compiler and runtime exports previously shared one barrel, so package resolution—not application usage—created the browser incompatibility.

## Locked Rules

- Root and React graphs cannot reach compiler, CLI, testing, filesystem, or Node built-ins.
- Developer-owned extension siblings are never overwritten.
- Inspection, MCP, compiler, conformance, and emulator output cannot claim Receiz verification.
- Versioned releases must not be packed from an in-progress next-version tree.

## Contract Gates

AST emitted-graph inspection, exact export maps, external tarball install, stable error redaction, conformance failure injection, MCP confirmation tests, AI-skill distribution tests, and full release freeze enforce these rules.

## Release-Lock Requirement

`pnpm release:lock 105.0.0` must require these regression lessons in the standalone offline-verifier checkout. Omitting them would sever the browser-safe release from the emitted-graph Node leakage and in-progress-next-version packing correction that its gates are designed to prevent.
