# Receiz v75.0.0 HPL Continuity Review

Release date: May 2, 2026
Status: Evidence artifact for `v75.0.0`

## Purpose

This review records the `v75.0.0` continuity check against the product law carried forward from `v70.0.0` through the `v74.0.0` Receiz Key / identity-artwork baseline.

The release is allowed only if local proof, file-backed recovery, deterministic route state, and value-loop causality remain stronger than weaker network, cookie, cache, or rediscovery payloads.

## Continuity Findings

- Receiz ID login remains local-first. A device-held Receiz ID key may project signed-in truth before server continuation finishes.
- Server continuation may append stronger truth, mint cookies, and bind accounts, but it may not downgrade a stronger local account binding.
- Account bindings are durable register truth, not disposable UI state.
- Identity Record, Identity Seal, Receiz Key, and PBI recovery remain parity restore rails that bind back into the local Receiz identity root.
- Verified file restore may seed account, profile, wallet, and upgrade truth without waiting for remote reconfirmation.
- Unsupported browsers use the explicit `/signin/receiz-id-fallback` backup route with sanitized return paths.
- Profile identity artwork and signed original-download hardening append proof without redefining `/<username>` profile truth.
- Market continuity preserves stronger signed-in wallet, owned-item, prediction-history, share-trade, and sold-position truth across weaker snapshots.
- Explore continuity reuses already-held view state and deterministic previews while server discovery remains background recovery.
- Plans and Upgrade preserve the same immediate payment and old-browser fallback behavior.

## Blocking Regressions

A candidate fails this review if it:

- makes local Receiz ID proof wait for network permission before projecting known identity truth
- lets stale cookies or stale session payloads replace a stronger local account binding
- requires PBI publication as permission for verified Identity Record, Identity Seal, or Receiz Key restore
- blanks restored account authority while import, bind, wallet refresh, or remote reconfirmation is delayed
- carries account-scoped private continuity across actor changes
- resurrects closed market ownership from weaker incoming snapshots
- replaces already-held Explore state with richer but non-authoritative rediscovery data
- lets any governed version surface drift away from `v75.0.0`

## Review Result

The `v75.0.0` release law preserves the HPL continuity boundary as a release-blocking invariant. Broader platform runtime validation must still pass the focused suites listed in `docs/releases/v75.0.0-checklist.md` from the exact release candidate.
