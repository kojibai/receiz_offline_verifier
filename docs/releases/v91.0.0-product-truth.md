`v91.0.0` preserves the `v90.0.0` proof-native artifact system and locks the scale law learned after v90: rich deterministic primitives stay rich, and unnecessary weaker work beneath them is removed.

## Frozen Primitives

| Primitive | v91.0.0 truth | Must not regress |
| --- | --- | --- |
| Receiz product identity | Receiz remains a proof-native artifact system. | Do not describe Receiz as a generic app with proof features. |
| Scale reasoning | Receiz scales by using implemented deterministic primitives and removing weaker work beneath them. | Do not simplify a visible primitive to hide timer, scan, stale-row, or routing problems. |
| Deterministic value chart | Sports card value series are deterministic Kai pulse math when the state machine has the inputs. | Do not fetch, accumulate, poll, or timer-drive chart data that can be reconstructed from deterministic state. |
| LiveChart value surface | Sports value panels keep range controls, tooltip, overlay dock, pulse/time rows, y-overflow, and full chart behavior. | Do not remove controls or visual depth as a performance fix. |
| Sports card proof object | Sports cards carry deterministic issue, owner/vault state, sport/player identity, event proof context, market value, and verification. | Do not let client game payloads or metadata outrank verified card ownership/proof truth. |
| Public Sports proof surface | Public card, QR, vault, event proof, OpenGraph, and Twitter routes project Sports proof objects. | Do not downgrade them into share pages or social previews. |
| Event proof media | Event reels and proof docks project exact event proof media when available and separate broad highlights beneath it. | Do not let broad media enrichment replace exact proof media. |
| Pack Derby game | Pack Derby is a Sports reward game with proof-backed score, run, challenge, and settlement contracts. | Do not treat it as disposable animation or client-only game state. |
| Pack Derby challenge settlement | Challenges settle when every entrant has used attempts or the deadline passes. | Do not require attempts-only settlement when the server contract allows deadline settlement. |
| Pack Derby run admission | Challenge run receipts are rejected after deadline. | Do not let ended challenges keep changing settlement truth. |
| Sports entry closeout | Soccer and UFC entries reuse the shared Sports entry proof primitive and finalize through sport-specific routes. | Do not route FIFA/UFC claim/finalize through baseball. |
| FIFA Main Event closeout | Soccer tournament reads run closeout and sweep signed-in unresolved old dates. | Do not return `closeout: null` for a tournament route whose settlement primitive is ready to close. |
| Reward opened truth | Opened reward receipts close stale finalized game recovery on first paint. | Do not resurrect opened rewards as active reveal or claim pressure. |
| Settlement conformance | Settlement conformance is its own required public conformance suite. | Do not alias settlement conformance to economy conformance. |
| Sports market depth | Shared sports card proof surfaces use sport-aware market-depth projection. | Do not hardcode baseball for soccer or UFC card value/depth proof surfaces. |
| Event reel playback | Player card/event reels must start the active proof media without requiring an extra click when browser policy allows or muted fallback is needed. | Do not suppress autoplay rejection without falling back to a playable state. |

## Frozen Invariants

- `v90.0.0` proof-native artifact truth remains the base release truth.
- `v91.0.0` adds scale law; it does not weaken v90.
- Advanced deterministic surfaces are allowed to remain visually rich.
- Performance work must remove weaker work beneath primitives, not remove primitive capability.
- Deterministic chart data is math, not fetched state.
- Known proof/value/ownership/settlement truth must paint before weaker refresh or sync.
- Old active Sports rows must close when game, slate, reward, tournament, or challenge truth is final.
- Sport-specific primitives must route through the sport they carry.
- Settlement conformance is release evidence, not documentation.
- Metadata and social previews project proof truth; they do not define proof truth.
- Challenge settlement truth cannot keep changing after the deadline.
- Public Sports proof surfaces must remain proof surfaces.
- Release docs, public labels, service worker version, runtime defaults, offline verifier labels, and package version must move together.
