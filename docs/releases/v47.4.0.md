# Receiz v47.4.0

Release date: April 9, 2026

## Overview

`v47.4.0` is the native business-entitlement and embedded-surface continuity release. It keeps the current `v47.x` product truth intact while fixing the exact places where Business could still be flattened back into older Premium-era assumptions, replacing remaining Calendly upgrade / WireProof scheduling surfaces with the canonical Receiz calendar embed, and making the public payments widget keep Stripe checkout inside a Receiz-hosted popup instead of redirecting buyers away from the page.

For this offline verifier repository specifically, `v47.4.0` is a release-surface alignment patch. The effective-plan/session fixes, Receiz-native scheduling surfaces, and Receiz-hosted Stripe embedded checkout live in the broader Receiz application; this repository carries the current release note, product-truth freeze, and visible version-surface bump so the public release artifacts stay locked together.

The product-truth freeze for this release is defined in `docs/releases/v47.4.0-product-truth.md`.

## What Defines v47.4.0

- Business is treated as a first-class plan in native session-derived surfaces instead of being collapsed into legacy Premium-only session output.
- Affiliate, profile bootstrap, and world session payloads now preserve effective entitlement truth so Business survives older compatibility layers.
- Upgrade and WireProof scheduling surfaces use the Receiz calendar embed instead of Calendly.
- The public payments embed opens secure Stripe embedded checkout inside a Receiz-hosted modal / popup shell instead of navigating buyers directly to a hosted Stripe URL.
- Release-governed version surfaces stay locked together across package metadata, service worker defaults, public badges, offline verifier surfaces, and release documents.

## Targeted Release Outcomes

### 1. Business Entitlement Survives Native Session Flows

The core entitlement correction in this release is that Business is no longer lost when a surface depends on session-shaped plan data:

- effective plan calculation now preserves Business over legacy Premium-compatible session values
- `/api/session`, `/api/profile/bootstrap`, and `/api/world/me` now expose the effective entitlement truth instead of flattening Business into Premium
- affiliate session handling now consumes the effective plan directly so Business participates natively in affiliate-gated surfaces

### 2. Scheduling Surfaces Are Receiz-Native

Receiz-owned scheduling now replaces the remaining external scheduling dependency:

- upgrade CTAs use the Receiz calendar embed
- WireProof scheduling CTAs use the Receiz calendar embed
- the user stays inside a consistent Receiz scheduling surface instead of crossing into Calendly-branded flows

### 3. Public Payment Checkout Stays On-Surface

The public payment widget now behaves like a Receiz-controlled checkout surface instead of a handoff:

- the public payments embed requests Stripe embedded checkout sessions
- the widget opens a Receiz-hosted modal / popup shell and mounts checkout inside it
- redirect-required Stripe methods return back into the same Receiz-hosted checkout surface rather than dumping the buyer onto a direct Stripe-hosted page

### 4. Release Truth Moves With The Patch

This release also rolls the visible release surfaces to `v47.4.0`, so the live app, offline surfaces, and release docs present one consistent version truth.

In this repository, that alignment shows up as updated release documentation plus the shipped version markers in the published verifier, studio, and settlement entrypoints.
