# Receiz v55.0.0

Release date: April 15, 2026

## Overview

`v55.0.0` is the first full release after the `v54.0.0` operating baseline. It absorbs the narrower `v54.1.0` continuity patch, then broadens the release surface in five concrete directions: owner profile setup becomes a guided, sticky workspace flow; profile avatar and cover uploads become sealed Receiz originals; world twin rendering prefers the freshest saved avatar across route handoffs; market navigation and live-player handoff move onto a shared stable-route contract; and performance analytics now canonicalize route groups so admin performance reporting can roll up real product paths instead of noisy per-URL variants.

Since `v54.0.0`, the repository state has added a fast-entry profile workspace shell, username claim/editing inside the owner setup rail, one-tap twin-photo handoff from profile into World forge, persisted guided-setup completion, profile media minting with anchored verification paths, proof-manifest media-hash fallback, world preview locking and avatar refresh fanout, compact-device safeguards around background twin hydration, market snapshot warming without a route-level loading shell, seeded live-player navigation from market, batched prediction-book hydration, and canonical web-vitals route grouping with an admin backfill path.

The core product rules still do not change. File-backed proof stays authoritative. Deterministic route state still owns first paint. Wallet and ledger state still own money truth. Live deltas still reconcile on top of those deterministic surfaces instead of replacing them.

The product-truth freeze for this release is defined in `docs/releases/v55.0.0-product-truth.md`.

## What Defines v55.0.0

- Owner profile setup is now a guided checklist with sticky completion, inline username claiming, twin-photo capture/upload handoff, and a fast-entry rehydration path for `/profile`.
- Profile avatar and cover updates now process into canonical images, upload to public profile media storage, and mint sealed Receiz originals with anchor-backed verify paths.
- World twin freshness is now explicit: the freshest saved render wins, preview locks fan out across tabs, forged avatars sync back into the profile avatar, and compact touch devices avoid background-hydrating the heavy twin module.
- Market route continuity is broader: bottom-bar navigation prewarms market and profile, market keeps warm snapshots without a dedicated route loading shell, and live-player entry can navigate from a seeded local payload before the authoritative fetch settles.
- Performance analytics now normalize route groups centrally so web-vitals ingestion, admin reporting, and historical backfill use the same canonical route map.
- Large market snapshots can hydrate prediction books in bounded batches instead of one oversized query.
- Release-governed version surfaces move together to `v55.0.0`.

## Since v54.0.0

### 1. Profile Workspace Became The Guided Owner Setup Surface

The signed-in profile workspace is no longer only an editor. It now behaves like the activation path for the product itself.

The current repository state adds:

- a five-step owner setup checklist centered on the "place that proves"
- inline username claim/editing with availability feedback and suggestions
- sticky completion that survives reloads and route transitions once all setup steps are done
- a fast-entry `/profile` path that can rehydrate from a primed client seed instead of waiting for the full workspace payload every time
- clearer owner actions around twin setup, proof posting, cover upload, and copying the live link

This matters because the signed-in workspace now teaches the intended product journey instead of assuming the user already knows it.

### 2. Profile Media Became Sealed Product State

Avatar and cover uploads are no longer just mutable profile decoration.

The current release state introduces:

- canonical avatar/banner image processing before save
- public profile-media storage with deterministic file naming and size guards
- Receiz minting for profile avatar and cover updates, including anchored verify URLs
- proof-manifest rows that can recover a usable media SHA-256 when the visible original did not carry one directly

This keeps profile presentation aligned with the broader product rule: high-value state should be explainable and verifiable after the fact.

### 3. World Twin Freshness Became A Cross-Route Contract

The twin no longer relies on a single "current" flag or a stale preview surviving too long.

The current release state adds:

- freshest-avatar selection on the server instead of trusting only `is_current`
- preview locks that prefer render-locked shell faces over older source-backed payloads
- browser refresh fanout over window events, storage, and broadcast channel
- forged-avatar syncing back into the default profile avatar
- route revalidation for profile, public twin, and world embed surfaces after avatar saves
- background hydration heuristics that keep phones and low-memory touch devices on on-demand twin loading
- retrying lazy imports and bootstrap fetches for world profile startup

This turns "the twin is updated" into a real route-continuity guarantee, not just a best-effort refresh.

### 4. Market Navigation And Player Handoff Hardened

The market route now shares more of the app's navigation discipline instead of keeping bespoke exceptions.

The current release state adds:

- shared stable internal navigation helpers for market exits and bottom-bar routing
- market route snapshot warming from auth chrome without introducing a route-level loading shell
- seeded local live-player handoff from market before the authoritative player payload finishes loading
- profile navigation from market that stays on the app-router path
- prediction-book loading that chunks large market-item sets into bounded queries

The important behavior change is that market no longer needs to choose between speed and continuity. It can move immediately while still preserving deterministic route state.

### 5. Performance Reporting Became Route-Aware

Web-vitals ingestion and admin analytics now agree on what a route means.

The current release state adds:

- shared path normalization and canonical route grouping for perf ingestion
- admin rollups for route, device, and time-bucket performance summaries
- a backfill script for older `perf_web_vitals` rows whose `route_group` was recorded before canonicalization

This matters because performance regressions can now be traced to stable product surfaces instead of being diluted across one-off URLs and user-specific paths.

## Public Summary

Receiz `v55.0.0` turns the `v54.0.0` operating baseline into a fuller production release: the profile workspace now behaves like a guided owner setup lane, profile media is sealed instead of merely uploaded, world twin surfaces prefer the freshest avatar render across route handoffs, market navigation and live-player entry follow a shared continuity contract, and performance reporting rolls up stable route groups that match the actual product map.
