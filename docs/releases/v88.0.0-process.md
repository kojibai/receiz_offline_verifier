## Preparation Steps

1. Preserve `v87.0.0` as the prior current-release archive.
   - Keep `docs/releases/v87.0.0.md`.
   - Keep `docs/releases/v87.0.0-product-truth.md`.
   - Keep `docs/releases/v87.0.0-checklist.md`.
   - Keep `docs/releases/v87.0.0-process.md`.
   - Preserve `v87.0.0` as historical release-law truth.

2. Publish canonical `v88.0.0` release artifacts.
   - Add `docs/releases/v88.0.0.md`.
   - Add `docs/releases/v88.0.0-product-truth.md`.
   - Add `docs/releases/v88.0.0-checklist.md`.
   - Add `docs/releases/v88.0.0-process.md`.
   - Add `docs/releases/v88.0.0-regression-lessons.md`.

3. Align repository release law and product-truth references.
   - Update `AGENTS.md` release law to `v88.0.0`.
   - Update product-truth reference status markers carried forward for the current release.
   - Preserve the primitive decision procedure, stronger/weaker truth hierarchy, and forbidden downgrade boundary.

4. Align repository release surfaces.
   - Update `package.json` to `88.0.0`.
   - Update root `README` current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.
   - Ensure static service-worker registrations request `sw.js?v=88.0.0`.
   - Ensure `site/sw.js` carries service-worker runtime version `88.0.0`.
   - Ensure local release-surface checks can execute through `pnpm lint`, `pnpm build`, and `pnpm release:lock`.

5. Validate primitive naming gates.
   - Offline proof is named as proof, not fallback mode.
   - Compact proof token is named as transport, not authority.
   - Live Sports truth is named as live proof truth, not scoreboard display.
   - Locked entry, player score, finalized reward, and append truth keep direct primitive names.
   - Public Live Cockpit is named as a public witness/live proof surface.
   - Service-worker truth is named as release truth, not incidental cache behavior.
   - Public proof, identity, ownership, settlement, route, and verified-history truth retain direct primitive names.

6. Validate live Sports gates.
   - Selected live games keep detailed field data.
   - `/baseball` routes directly into the canonical baseball arena without forking logic.
   - Locked entries first-paint from stored proof/local truth.
   - Public feed updates append and refresh without erasing stronger selected-game truth.
   - Single-game proof projection restores locked entries when selected game state is stale.
   - Field layout preserves score plate, center pitch-by-pitch recap, and last-play ticker truth.
   - Individual card rows use proof-backed card scores while appends hydrate.
   - Finalized reward recovery remains claimable until opened.

7. Validate post-v87 surface gates.
   - Market desk buy/sell continuation remains repaired.
   - Market value pill chart/live popover does not force `/market` to rediscover known truth.
   - Public profile originals and assets remain available under tightened RLS/security constraints.
   - Public profile route remains canonical profile truth.
   - Public Live Cockpit and `/live` room routes retain public witness/live proof framing.
   - Notification delivery, push subscription, and chat call/message routes pass focused contracts.
   - Supabase timeout hardening migrations and RLS guard tests remain tracked.

8. Validate runtime lockstep.
   - Confirm all governed version markers show `v88.0.0`.
   - Confirm all current-release docs and indices point to `v88.0.0` artifacts.
   - Confirm historical `v80.0.0` through `v87.0.0` artifacts remain archived release truth.
   - Confirm `release:lock` requires `docs/releases/v88.0.0-regression-lessons.md`.
   - Run `pnpm release:lock`.
   - Run whitespace diff check.

9. Validate broader platform gates before production promotion.
   - Run release-freeze and page-first-paint checks.
   - Confirm Sports, Public Live Cockpit, Market, Profile, runtime/session, notification/chat, Supabase timeout, security, and release contracts.
   - Confirm target production DB migrations named by the release are applied.

10. Generate signed release attestation.
    - Run `pnpm release:attest` for the final commit in the environment that holds the release attestation signing key.
    - Retain the signed attestation artifact under the governance release-attestation policy.
    - Do not tag or ship without the signed attestation.

## Focused Verification Matrix

Broader platform release validation should include the gates named in `docs/releases/v88.0.0-checklist.md`, including:

- direct primitive naming and no forbidden downgrade
- offline proof semantics unchanged from `v87.0.0`
- live Sports selected-game, field, player score, locked-entry, and reward truth
- `/baseball` no-fork direct-route behavior
- Public Live Cockpit public witness/live proof behavior
- Market/Profile first-paint and RLS contracts
- service-worker/runtime version coherence
- session clear/logout durable-key migration
- notification delivery and chat push/call contracts
- Supabase timeout hardening migration guards
- release freeze, page first paint, TypeScript, release lock, regression lessons, whitespace diff, and release attestation gates

## Tag Criteria

Do not tag `v88.0.0` until:

- all required `v87.0.0` and `v88.0.0` release artifacts exist
- version surfaces are aligned to `v88.0.0`
- regression lessons are present and required by `release:lock`
- release gates in `docs/releases/v88.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v88.0.0`
- `pnpm lint` has zero warnings/errors
- `pnpm build` completes
- `pnpm release:lock` confirms the release surfaces
- whitespace diff check passes
- `pnpm release:attest` emits a signed attestation for the final commit

## Release Rule

Do not ship `v88.0.0` unless the release gates in this checklist pass from the exact release candidate, `pnpm release:lock` confirms the release surfaces and regression lessons, whitespace diff passes, focused Sports, Market, Profile, Public Live Cockpit, runtime/session, notification/chat, Supabase timeout, security, and release contracts pass, and production migrations named by the release are applied in the target environment.
