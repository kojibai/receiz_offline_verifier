# Receiz v76.0.0 - World Agent Revenue Loop, Public Proof, Market Twin, And Developer Trust Surface Release

Release date: May 6, 2026

## Summary

`v76.0.0` locks the post-`v75.0.0` product candidate as a value-production release. The release preserves `v70.0.0` profile law, `v74.0.0` identity-artwork law, and `v75.0.0` local-proof/file-restore law while making Receiz World more agentic, Market Twin more habit-forming and mandate-safe, Public Proof a first-class registry surface, Wallet settlement more reliable for non-subscription value, and Status more useful for public users and developers without exposing private attack surfaces.

This is not a label-only bump. The diff from `v75.0.0` to the pre-bump `v76` candidate includes Agents SDK integration, protected concierge response continuity, public-safe agent trace UX, Sales Brain, daily/autopilot brief loops, campaign follow-up repairs, email-default delivery, Market Twin authority and persistence, Public Proof observe/claim/verify/seal flows, wallet reserve settlement corrections, public/developer status inventory expansion, public profile/auth continuity, Explore packaging, and release-contract coverage for the new world, market, proof, status, wallet, auth, and profile behavior.

## What This Release Locks

- World Concierge keeps replying through the direct Responses streaming path while the Agents SDK is used for agentic work, tool execution, handoffs, tracing, and orchestration where those capabilities add value.
- The World agent toolbelt exposes robust business actions: lead research, contact enrichment, CRM sync, outreach drafting, outreach sending, follow-up campaigns, reply classification, payment path generation, booking path generation, proof ledger entries, handoffs, and Sales Brain routing.
- Public-safe agent trace becomes product UX through task-level summaries such as searched leads, imported contacts, drafted messages, sent messages, held actions, and next revenue action without exposing raw model traces.
- World revenue loops turn concierge work into visible missions and outcomes: research, qualify, draft, send, follow up, close, value ledger, revenue attribution, autopilot brief, Sales Brain, and next commercial action.
- Campaign follow-up delivery defaults to email unless the user explicitly configured a valid connected-app webhook destination, and legacy invalid app-webhook campaigns can heal back to email.
- Market Twin is a fast-entry, saved-state, mandate-safe money loop. It opens from cached market truth, hydrates saved authority without remounting the shell, explains why a move matters, gates actions through mandate checks, and keeps a value ledger.
- Market Twin authority persists owner mandate settings, approval/auto mode, order caps, daily caps, profit sweep, and append-only intent rows; repair migration handles environments where an earlier `create table if not exists` left missing columns.
- Market Twin replay uses deterministic market snapshot history, labels simulated/replay results truthfully, and fixes win-rate math to divide by actual replayed outcomes.
- Public Proof is a first-class surface for observing URLs, creating proof records, resolving records by URL or ID, creator lookup, claim/verify/seal/dispute flows, identity verification, OAuth channel control, rights assertions, and registry feed discovery.
- Public Proof UI presents a mobile-first proof command surface with compact headers, claim/dispute panels, tier ladder, graph/source views, and deterministic proof/state contracts.
- Wallet settlement fixes non-subscription reserve/payment credit handling, running reserve balances, public/private ledger behavior, and wallet value-state framing.
- Public/developer Status now displays high-value safe endpoints: OIDC/OAuth discovery, all conformance suites, core Receiz proof routes, public profile, Public Proof, Market read surfaces, World public/profile/stream, Calendar embed, signer keyring, public wallet/action ledgers, payment embeds, and Status itself.
- Status continues to exclude admin, webhook, internal worker, private session-only, privileged issuer, and other routes that would increase attacker value without improving public/developer confidence.
- Account surfaces replace product framing with operating surfaces and expose Market Twin and Public Proof as first-class user tools.
- World and Market mobile polish protects no-zoom inputs, four-across compact pills, popover command surfaces, bottom-chrome spacing, and settled first-paint continuity.
- Payments embed checkout session handling, profile paid original purchase paths, Stripe billing subscription handling, and wallet deposit settlement move forward without redefining public profile truth.
- Auth/profile continuity work keeps PBI, Receiz Key, passkey, bridge redemption, and local Receiz ID binding aligned with `v75.0.0` law: restored or locally proven truth cannot be downgraded by stale transport echoes.
- Release contracts now cover Agents SDK migration boundaries, concierge transport, agent toolbelt, World Sales Brain, revenue attribution, autopilot brief, campaign email defaults, Market Twin historical/persistence/value-loop behavior, Public Proof, public status inventory, wallet settlement, and value-state framing.

## Developer And Public Value

- Developers get a richer, safer Status page that proves the public integration surface is alive without advertising admin or internal controls.
- Users get more obvious value loops: World produces business artifacts, Market Twin produces guarded market actions, Public Proof produces shareable proof, Wallet records value movement, and Status makes platform capability legible.
- Operators get clearer release gates around the exact behavior that matters: no regressions in direct concierge replies, saved Market Twin settings, email-default follow-up delivery, public proof registry behavior, wallet settlement, and public/developer endpoint inventory.
