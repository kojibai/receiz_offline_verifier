## Locked Product Truth

- Receiz is a proof-native artifact system, not a generic app with proof features.
- Files, identities, ownership states, settlement states, public witness surfaces, verification routes, Sports cards, Series Passports, wallet images, Cash Images, OTC upgrades, and market positions carry product truth.
- Series Passport is a proof object and Series access primitive. It is not a generic pass, checkout product, badge, or marketing card.
- Sports Arena is a live proof economy. It is not a generic game UI.
- Main Event is a settlement primitive. It is not a balance display or prize-card treatment.
- Market and wallet ownership surfaces must paint known owned shares and owned assets immediately.
- Local market execution changes visible ownership and settlement projection immediately.
- Lagging server/global sync snapshots cannot reopen a locally settled position unless they prove an equal-or-newer authoritative state.
- Session, owner, DB, analytics, global sync, SEO, cache, and external provider payloads may hydrate, append, index, reconcile, and distribute. They do not outrank stronger local, deterministic, sealed, verified, artifact, ownership, or settlement truth.
- Hidden hydration may append verified additions only. It may not replace settled visible truth to prove something already known.
- Series public sharing uses the implemented affiliate/referral system. It must not create a duplicate authority.
- Series Passport pricing language, outcome language, payout split, rank path, and daily lineup contribution language are release truth.
- The Arena strip must show Series-specific action truth: `$25 Passport` and `Every lineup counts`.
- Release labels, package metadata, service-worker runtime version, offline verifier labels, README release pointer, release docs, and public markers all move together to `v85.0.0`.

## Forbidden Downgrades

A change is out of bounds for `v85.0.0` if it:

- downgrades proof object to card.
- downgrades Series Passport to pass, checkout product, badge, or generic CTA.
- downgrades Sports Arena to game UI without proof economy truth.
- downgrades Main Event settlement primitive to prize display or balance display.
- downgrades Market ownership state to visual badge or delayed server display.
- downgrades deterministic first paint to loading state.
- downgrades verified history to activity feed.
- downgrades route truth to SEO payload.
- downgrades service-worker release truth to incidental cache behavior.
- lets external systems define Receiz truth instead of providing implementation mechanics.
- treats generic app convention as stronger than implemented Receiz primitives.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker runtime, badges, or release markers.
