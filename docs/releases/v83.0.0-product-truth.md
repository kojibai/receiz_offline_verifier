## Locked Product Truth

- Sports must feel already there. The first visible shell must be truthful, immediate, and stable while hidden hydration warms richer data.
- Main Event and single-game selection must share the same scoreboard selection path, including immediate score projection for the first selected single game.
- Known game state must be used before hydration. Rediscovery must not block first paint when deterministic local or route truth already exists.
- Live scores and lock state must stay scoped to the selected game. Active entry lock state must not rewrite unrelated field, scoreboard, or selected-game truth.
- Known zero scores are legitimate zero scores and must not be hidden behind loading doubt.
- Card reveal remains a game surface. Click, popover, drag, drop, selection, deselection, reward reveal, and 1-card through 10-card spread behavior must preserve the same interaction contract.
- Saved Sports cards must keep issue number, pack origin, card identity, proof object, current owner, append history, vault visibility, and market history together.
- Player availability is product truth for Pick 5. Starter, bench/role, backup, injured/out, Main Event, and single-game availability language must stay consistent.
- The profile live-player route must open from iMessage and first-paint from canonical public profile truth without SEO, manifest, or structured-proof work blocking the shell.
- SEO enrichment may describe truth, but it must not make first paint wait for truth the route already has. Public profile remains canonical at `/<username>`, and `?manage=1` remains owner permission layered over canonical public truth.
- Explore must avoid white-screen races when build work is already in flight. Three.js, remote media, and heavier discovery enrichment must stay behind the immediate shell.
- `/market` must keep the fast entry path. Market conformance must check transport truth, proof, provenance, value, and asset surfaces without reintroducing removed hot-path JSON-LD work.
- Payment and embed callback URL policy must remain centralized in one allowlist path, with Vercel trace inclusion covering the policy users rely on.
- OTC and Cash Image proof objects must upgrade chain asset or wallet state into offline-verifiable Receiz proof before optional later reconciliation.
- Wallet proof objects must preserve image-native Receiz ID wallet creation, private-key import, connected-wallet add, `receiz.wallet.bundle-proof.v1`, proof export/import, and wallet-panel action placement.
- Account Brain, Calendar, profile showcase, profile analytics, account feedback, bottom navigation, and owner/manage hydration must avoid all-or-nothing page failure and preserve one-viewport contracts.
- Release conformance is a boundary. Market, Sports, interoperability, profile first-paint, service-worker version coherence, release freeze, page first paint, TypeScript, release lock, and release attestation gates must protect product law before tagging.
- Release labels, service-worker defaults, offline verifier labels, README release pointer, release docs, and public release markers all move together to `v83.0.0`.

## Out Of Bounds

A change is out of bounds for `v83.0.0` if it:

- slows first paint or replaces a truthful visible shell after the page feels loaded.
- rediscoveries before using known deterministic route, account, profile, game, or market truth.
- lets Sports active entry lock state, live score, field data, or scoreboard state leak across game contexts.
- hides known zero scores as loading ambiguity.
- breaks card reveal interaction contracts after viewer, popover, drag/drop, reward-card, or deselection paths.
- drops issue number, owner, pack origin, provenance, proof object, card career, market history, or append history from Sports cards.
- treats player availability as decoration instead of lineup truth.
- blocks live-player first paint on SEO, JSON-LD, manifest, structured proof, or owner/manage hydration work.
- slows `/market` to satisfy stale conformance assumptions.
- scatters callback policy across route logic instead of the centralized allowlist path.
- creates wallet, Cash Image, or OTC proof objects that are not offline-verifiable Receiz proof before optional reconciliation.
- updates conformance to ratify a visible regression.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker defaults, badges, or release-worker markers.
