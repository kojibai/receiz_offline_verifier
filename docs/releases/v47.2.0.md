# Receiz v47.2.0

Release date: April 9, 2026

## Overview

`v47.2.0` is the custom-domain routing and setup-surface release. It keeps the current `v47.x` product truth intact while fixing the exact production path that matters for Business domains: the custom hostname root now resolves to the bound public profile instead of falling through to the Receiz 404 surface, and the DNS setup panel is easier to execute from because each record value can be copied directly.

For this offline verifier repository specifically, `v47.2.0` is a release-surface alignment patch. The custom-domain routing and `/account` setup work live in the broader Receiz application; this repository carries the current release note, product-truth freeze, and visible version-surface bump so the public release artifacts stay locked together.

The product-truth freeze for this release is defined in `docs/releases/v47.2.0-product-truth.md`.

## What Defines v47.2.0

- Custom-domain root traffic resolves from the effective forwarded host, not only the internal request hostname.
- A connected Business custom domain can land visitors on the bound public profile at `/` without requiring the `/<username>` fallback path.
- DNS setup guidance in `/account` is easier to execute because each DNS name and value has a direct copy control.
- Release-governed version surfaces stay locked together across package metadata, service worker defaults, public badges, and release documents.

## Targeted Release Outcomes

### 1. Bound Custom Domains Open On The Profile At Root

The key patch fixes the production edge-routing gap:

- `https://custom.host/` now rewrites from the forwarded host to the bound `/_business-domain/...` profile route.
- Custom-domain requests no longer depend on the internal request URL already matching the custom hostname.
- Existing domains that were already attached and DNS-live do not need new registrar changes for this routing fix.

### 2. DNS Setup Is Faster To Execute

The Business account panel now exposes direct copy controls for each DNS record field:

- record name is copyable
- record value is copyable
- the setup panel reduces registrar copy/paste mistakes without changing provider behavior

### 3. Release Truth Moves With The Patch

This release also rolls the visible release surfaces and service worker version to `v47.2.0`, so the live app, offline surfaces, and release docs present one consistent version truth.

In this repository, that alignment shows up as updated release documentation plus the shipped version markers in the published verifier, studio, and settlement entrypoints.
