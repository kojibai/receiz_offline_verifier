## Preparation Steps

1. Preserve `v73.0.0` as the prior release baseline.
   - Keep `docs/releases/v73.0.0.md`.
   - Keep `docs/releases/v73.0.0-product-truth.md`.
   - Keep `docs/releases/v73.0.0-checklist.md`.
   - Keep `docs/releases/v73.0.0-process.md`.
   - Preserve v73 as historical release truth once current pointers move to `v74.0.0`.

2. Publish canonical `v74.0.0` release artifacts.
   - Add `docs/releases/v74.0.0.md`.
   - Add `docs/releases/v74.0.0-product-truth.md`.
   - Add `docs/releases/v74.0.0-checklist.md`.
   - Add `docs/releases/v74.0.0-process.md`.

3. Align repository release surfaces.
   - Update `package.json` to `74.0.0`.
   - Update root README current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.

4. Validate Receiz Key and recovery gates.
   - Receiz Key restore is proven by fresh key signatures and active key registration.
   - PBI is not required to restore from an already-held key.
   - Account Receiz Key controls include export, optional file password, import, key listing, refresh, and two-step revocation.
   - Offline Receiz Key projection requires verified portable state at first projection and preserves admitted local identity proof on later server misses.

5. Validate identity artwork and PBI-signed identity gates.
   - Profile identity artwork derives from account creation Kai truth.
   - Runtime randomness, upload timing, and remount timing do not alter identity artwork.
   - KaiSigil glyph geometry and size math remain canonical.
   - PBI-signed identity seal and identity record artifacts append verified authorship state without replacing the base profile shell.
   - Signed witness emphasis uses trust-blue chrome.

6. Validate profile media, link, and timestamp gates.
   - Profile creation timestamps remain stable local truth.
   - Long-form profile originals preserve full text, title, hash, word count, and a compact Read affordance.
   - Attached Receiz URLs preserve exact href values and long valid links survive composer and anchor transport.
   - Profile video derivatives improve playback while original proof downloads remain canonical.

7. Validate market and portable-state gates.
   - Sold share positions remain local truth across market snapshots and route hydration.
   - Closed positions do not resurrect as live ownership.
   - Portable state import preserves profile media/state fields.
   - Partial ledger entries are admitted only through explicit partial-import handling.
   - Integrity failures remain visible.

8. Validate release-surface lockstep.
   - Confirm package metadata resolves to `74.0.0`.
   - Confirm shipped HTML version markers display `v74.0.0`.
   - Confirm current release docs and indexes point to `v74.0.0` artifacts.
   - Confirm `v73.0.0` artifacts remain historical release archive docs.

## Focused Verification Matrix

Broader platform release validation should include the gates named in `docs/releases/v74.0.0-checklist.md`, including:

- TypeScript compile: `pnpm exec tsc --noEmit --pretty false`
- Deterministic surfaces: `pnpm test:deterministic-surfaces`
- Release freeze: `pnpm test:release-freeze`
- Release lock: `pnpm release:lock`
- Receiz Key restore, active-key listing, refresh, and revocation suites
- Identity Record PNG export/import and legacy key JSON suites
- offline Receiz Key projection and admitted-local-identity suites
- deterministic profile identity artwork and KaiSigil geometry suites
- PBI-signed identity seal and identity record suites
- profile timestamp, long-form original, attached-link, and video derivative suites
- sold-position continuity and portable-state import suites
- business/custom-domain hardening suites touched by the release candidate

This offline verifier repository does not contain those broader platform test suites. Its local release gates are version-surface, documentation, package metadata, and shipped static-entrypoint checks.

## Tag Criteria

Do not tag `v74.0.0` until:

- all required `v73.0.0` and `v74.0.0` release artifacts exist
- version surfaces are aligned to `v74.0.0`
- the release gates in `docs/releases/v74.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v74.0.0`
- broader platform Receiz Key, identity artwork, PBI-signed identity, profile media, link preservation, video playback, sold-position continuity, portable-state import, and release-contract gates have passed against the release candidate

## Release Rule

Do not ship `v74.0.0` unless the release gates in `docs/releases/v74.0.0-checklist.md` pass from the exact release candidate and the governed surfaces all show `v74.0.0`.
