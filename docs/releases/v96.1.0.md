## Title

Receiz `v96.1.0` - Twin And World SDK Release Lock

## Summary

`v96.1.0` is the patch release that keeps the public app, `@receiz/sdk`, OpenAPI/status catalog, Connect docs, and release evidence aligned after the SDK gained first-class Twin and World integration surfaces.

This release does not create a new authority layer. It exposes existing Receiz primitives to developers through typed convenience clients, Connect-scoped routes, OpenAPI documentation, and copy-paste integration docs. The authority remains the proof object, identity primitive, ownership/settlement append, durable local truth, Kai/Klok coordinate, and verified append path.

## What Changed Since v96.0.0

- `@receiz/sdk@96.1.0` exposes World public convenience clients.
- `@receiz/sdk@96.1.0` exposes delegated Twin clients for Connect-authorized applications.
- Connect now names `receiz:twin.read` and `receiz:twin.write` as scoped capability grants.
- Twin and World routes are documented in the developer API surface and status catalog.
- The SDK docs now include a dedicated Twin and World integration guide.
- Identity conformance records the new Connect/Twin surface so public developer contracts stay visible.
- App and SDK release identity are locked together as `96.1.0`.

## Product Law Preserved

- Proof objects remain the source of truth.
- Kai Klok remains the proof object state machine.
- Durable proof memory remains first admission only, then append forever.
- Connect remains delegated API access, not the identity proof root.
- SDK clients remain convenience beneath existing proof, identity, ownership, settlement, Twin, and World primitives.
- Server, database, SDK, CLI, and OpenAPI docs do not become truth authority.
