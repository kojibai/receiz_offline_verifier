## Release Framing

Title: Receiz `v81.0.0` - Signal Circuit Provenance Release

`v81.0.0` locks Signal Circuit as a more inspectable proof-native asset network. Signal Cards gain DNA/history, rewarded-card seal payment becomes funded and idempotent before reveal, card activity becomes visible in wallet and calendar history, transfer receive activity deduplicates, discovery preserves explicit expansion intent, and mobile/PWA chrome stays stable around asset inspection.

## Preparation Steps

1. Preserve `v80.0.0` as the prior current-release archive.
   - Keep `docs/releases/v80.0.0.md`.
   - Keep `docs/releases/v80.0.0-product-truth.md`.
   - Keep `docs/releases/v80.0.0-checklist.md`.
   - Keep `docs/releases/v80.0.0-process.md`.
   - Preserve `v80.0.0` as historical release truth.

2. Publish canonical `v81.0.0` release artifacts.
   - Add `docs/releases/v81.0.0.md`.
   - Add `docs/releases/v81.0.0-product-truth.md`.
   - Add `docs/releases/v81.0.0-checklist.md`.
   - Add `docs/releases/v81.0.0-process.md`.

3. Align repository release surfaces.
   - Update `package.json` to `81.0.0`.
   - Update root `README` current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update product-truth reference status markers carried forward for the current release.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.
   - Ensure local release-surface checks can execute through `pnpm lint`, `pnpm build`, and `pnpm release:lock`.

4. Validate Signal Card DNA gates.
   - `/game/cards/history` renders card origin, owner context, current listing, auctions, bids, sales, arena appends, metrics, and price points.
   - Card selector filters, search, focus URLs, and progressive hydration preserve selected card and claim-hash context.
   - DNA/history remains additive and never rewrites sealed card truth.

5. Validate rewarded-card payment gates.
   - Submitted card claims are verified before charge.
   - Seal price remains fixed at 99 USD cents in the route.
   - Eligible wallet lanes spend first.
   - Insufficient balance returns the precise missing funding delta and respects Stripe minimum checkout behavior.
   - Compact idempotency returns the same purchase result on repeated seal attempts.
   - Wallet action-ledger activity is appended for sealed reward cards.

6. Validate activity, transfer, and discovery gates.
   - Wallet and calendar views surface purchased packs, rewarded card purchases, rewarded-card seals, card transfers, and card receives.
   - Receive activity deduplicates repeated rows.
   - Card history reads across all discoverable cards.
   - Card World remembers explicit same-session "More cards" expansion without silently changing the initial surface.

7. Validate mobile, PWA, and visual gates.
   - Game bottom toolbar does not cover top-bar popovers.
   - Dark game route styling does not bleed into white routes.
   - Game nav buttons remain easy to hit.
   - Card-share animation completes before transition.
   - Popover loaders, rewarded-card stages, Signal Card art, orb realism, rim treatment, and proof-manifest layout stay readable and stable.

8. Validate release-surface lockstep.
   - Confirm all governed version markers show `v81.0.0`.
   - Confirm all current-release docs and indices point to `v81.0.0` artifacts.
   - Confirm `v80.0.0` artifacts remain historical and untouched.
   - Run `pnpm release:lock`.

9. Generate signed release attestation.
   - Run `pnpm release:attest` for the final commit in the environment that holds the release attestation signing key.
   - Retain the signed attestation artifact under the governance release-attestation policy.
   - Do not tag or ship without the signed attestation.

## Focused Verification Matrix

Broader platform release validation should include the gates named in `docs/releases/v81.0.0-checklist.md`, including:

- Signal Card DNA route, metadata, social image, selector, filters, search, focus URLs, event rows, price charting, owner context, and progressive hydration
- rewarded-card seal API, claim verification, wallet charge, insufficient-balance funding metadata, compact idempotency, Stripe minimum behavior, and wallet action-ledger emission
- wallet and calendar card activity for pack purchases, rewarded card purchases, rewarded-card seals, transfers, and receives
- card transfer receive deduplication
- all-card history and explicit same-session discovery memory
- card pages, World cards, share-card DNA entry, Find Cards entry, and card-world navigation
- explorer density, proof-manifest card-back space, popover behavior, Find Cards bottom alignment, and fallback entry behavior
- mobile game bottom chrome, white-route isolation, nav hit targets, card-share animation, loaders, rewarded-card stage fit, and PWA continuity
- Signal Card art, orb, artifact, rim, and card-back proof layout
- release-surface lock checks and signed release attestation

## Tag Criteria

Do not tag `v81.0.0` until:

- all required `v80.0.0` and `v81.0.0` release artifacts exist
- version surfaces are aligned to `v81.0.0`
- release gates in `docs/releases/v81.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v81.0.0`
- `pnpm lint` has zero warnings/errors
- `pnpm build` completes
- `pnpm release:lock` confirms the release surfaces
- `pnpm release:attest` emits a signed attestation for the final commit

## Release Rule

Do not ship `v81.0.0` unless the release gates in this checklist pass from the exact release candidate, `pnpm release:lock` confirms the release surfaces, and `pnpm release:attest` emits a signed attestation for the final commit.
