# Receiz v73.0.0 - Ownership, Vault, And Continuity Release

Release date: April 29, 2026

## Summary

`v73.0.0` locks the product growth and continuity work accumulated after the `v72.0.0` stable continuity baseline. The baseline for this release diff is recorded by the release brief as commit `61ae0ef2b` (`v72.0.0 Stable Continuity Release`) because this checkout does not contain a `v72.0.0` tag.

This release freezes visible product expansions without weakening the `v70.0.0` public alpha law:

- Business account entity ownership and certificates now have first-class API, server, and contract coverage.
- Receipt ownership provenance and transfer flows now carry stronger proof bundles and route-level continuity.
- Vault and original-download paths preserve portable verified records instead of treating recovered artifact state as disposable UI state.
- Account brain selections expose compact proof-bundle data in a dismissible mobile-friendly popover, with Open shown only for nodes that have a real route target.
- Account authority navigation names the offline surface as Offline Verifier and consolidates Privacy/Terms into Legal while keeping Security one tap away.
- Account, profile, composer, chat, discovery, and world twin surfaces keep post-v72 UI improvements under explicit continuity and fit contracts.

For this offline verifier repository, the release work is publication of the canonical `v72.0.0` archive, `v73.0.0` release-prep artifacts, and outward version-surface alignment. The verifier proof-format and producer-payload contract remain unchanged.

## What This Release Locks

- `/<username>` remains canonical profile truth; `?manage=1` remains owner permissions only.
- Owner/private enrichment must not make signed-in profile first paint slower than public profile truth.
- Business account ownership is account truth with explicit server APIs, certificate generation, holdings reads, and contract coverage.
- Receipt ownership provenance and transfer proof are verified additions; they must not rewrite artifact identity or canonical receipt truth.
- Vault decode, verifier, and download state must preserve portable verified record truth instead of degrading to transient transport state.
- Account page and one-viewport continuity fixes remain product law, not cosmetic layout preference.
- Account brain proof-bundle detail is selected-node data, not first-paint loading theater; every selected node shows proof data and only openable nodes show Open.
- World twin face feature authority and photoreal shell fit remain bounded by contracts, not broad viewport guesses.
- Package metadata, service-worker defaults, public version badges, verifier surfaces, README release pointer, and release docs move together to `v73.0.0`.
