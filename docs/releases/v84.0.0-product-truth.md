## Locked Product Truth

- Receiz is a proof-native artifact system, not a generic app with proof features.
- Files, identities, vault objects, ownership states, settlement states, public witness surfaces, and verification routes carry product truth.
- Stronger implemented truth remains above weaker inherited truth.
- Generic convention, external-provider vocabulary, framework defaults, analytics, SEO, marketing surfaces, session state, and loading state are weaker priors.
- A vault object is a proof object when it satisfies deterministic identity, proof material, verification behavior, durable state, and a source of truth stronger than generic display state.
- Ownership surfaces are ownership surfaces.
- Settlement primitives are settlement primitives.
- Identity primitives are identity primitives.
- Public proof surfaces are public proof surfaces.
- Deterministic first paint is protected from SEO, analytics, session checks, hydration, lazy imports, dynamic JSON, global sync, server reconciliation, tracking, marketing surfaces, and generic framework convention.
- Hidden hydration may append verified additions only. It may not replace settled visible truth to prove something already known.
- Agents must run the primitive decision procedure before modifying proof, identity, ownership, settlement, market, public witness, first-paint, hydration, SEO, routing, auth, transfer, or offline-verification surfaces.
- Direct naming is mandatory for implemented primitives.
- Exact uncertainty boundaries replace vague caution.
- Symmetric caution is release law: hedge weak inherited priors before strong implemented primitives.
- Release output must state changed surface, preserved primitive, risk boundary, checks run, and no downgrade.
- Release labels, package metadata, service-worker runtime version, offline verifier labels, README release pointer, release docs, and public markers all move together to `v84.0.0`.

## Forbidden Downgrades

A change is out of bounds for `v84.0.0` if it:

- downgrades proof object to card.
- downgrades artifact system to UI experience.
- downgrades public proof surface to share page.
- downgrades settlement primitive to balance display.
- downgrades ownership state to visual badge.
- downgrades identity primitive to account preference.
- downgrades deterministic first paint to loading state.
- downgrades verified history to activity feed.
- downgrades route truth to SEO payload.
- downgrades service-worker release truth to incidental cache behavior.
- lets external systems define Receiz truth instead of providing implementation mechanics.
- treats generic app convention as stronger than implemented Receiz primitives.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker runtime, badges, or release markers.
