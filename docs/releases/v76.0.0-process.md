## Preparation Steps

1. Preserve `v75.0.0` as the prior release baseline.
   - Keep `docs/releases/v75.0.0.md`.
   - Keep `docs/releases/v75.0.0-product-truth.md`.
   - Keep `docs/releases/v75.0.0-checklist.md`.
   - Keep `docs/releases/v75.0.0-process.md`.
   - Preserve `v75.0.0` as historical release truth.

2. Publish canonical `v76.0.0` release artifacts.
   - Add `docs/releases/v76.0.0.md`.
   - Add `docs/releases/v76.0.0-product-truth.md`.
   - Add `docs/releases/v76.0.0-checklist`.
   - Add `docs/releases/v76.0.0-process.md`.

3. Align repository release surfaces.
   - Update `package.json` to `76.0.0`.
   - Update root `README` current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.

4. Validate World and Market gates.
   - Direct concierge response continuity remains uninterrupted while agent tool execution runs through the SDK.
   - World toolbelt actions and mission progression are surfaced by task-level summaries.
   - Sales Brain and revenue-attribution traces remain within intended, deterministic pathways.
   - Public Proof observe/claim/verify/seal/dispute flows remain first-class and safe on public surfaces.
   - Market Twin authority and mandate behavior persists across edits and saves.
   - Market replay, simulated/replay labeling, and win-rate math use deterministic histories.

5. Validate status/auth and wallet gates.
   - Public/developer Status inventory includes only safe endpoints.
   - PBI/Receiz Key/passkey/boundary continuity stays aligned to the strongest local truth.
   - Wallet reserve settlement and non-subscription payment-credit handling are corrected and stable for public/private surfaces.
   - Campaign follow-up defaults and repairs preserve business continuity and avoid broken webhook fallback states.

6. Validate release-surface lockstep.
   - Confirm all governed version markers show `v76.0.0`.
   - Confirm all current-release docs and indices point to `v76.0.0` artifacts.
   - Confirm `v75.0.0` artifacts remain historical and untouched.

## Focused Verification Matrix

Broader platform release validation should include the gates named in `docs/releases/v76.0.0-checklist.md`, including:

- World mission continuity and concierge direct-response continuity.
- World agent toolbelt action evidence and Sales Brain attribution.
- Public-safe task summaries and raw-trace hiding.
- Campaign follow-up default/repair behavior.
- Campaign autopilot brief loops.
- Market Twin authority persistence, mandate checks, replay integrity, and win-rate math.
- Public Proof registry and claim/verify/seal/dispute flow behavior.
- Wallet non-subscription reserve and payment-credit handling.
- Public/dev status inventory and route exposure boundaries.
- Auth/profile continuity against stale transport echoes.
- `pnpm lint`
- `pnpm build`
- `pnpm release:lock` / release-surface lock checks where available

## Tag Criteria

Do not tag `v76.0.0` until:

- all required `v75.0.0` and `v76.0.0` release artifacts exist
- version surfaces are aligned to `v76.0.0`
- release gates in `docs/releases/v76.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v76.0.0`
- `pnpm lint` has zero warnings/errors
- `pnpm build` completes

## Release Rule

Do not ship `v76.0.0` unless the release gates in this checklist pass from the exact release candidate, `pnpm lint` has zero warnings/errors, `pnpm build` completes, and the governed surfaces all show `v76.0.0`.
