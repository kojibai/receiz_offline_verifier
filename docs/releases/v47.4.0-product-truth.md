# Receiz v47.4.0 Product Truth

Release date: April 9, 2026
Status: Locked for `v47.4.0` release prep

## Purpose

This document defines the patch-level product-truth freeze for Receiz `v47.4.0`. It preserves the current `v47.x` product truth while recording the native Business entitlement continuity, Receiz-native scheduling, and on-surface embedded checkout behaviors that are now treated as explicit non-regression behavior at release time.

For this offline verifier repository, the patch is represented as current-release documentation and version-surface alignment only. The verifier trust contract remains unchanged relative to `v47.0.0`.

## Core Authority Boundaries

The release does not relax the core Receiz authority model:

- The file remains the authority for artifact truth.
- Deterministic manifests remain the authority for historical route truth.
- Effective entitlement truth must remain derived from the real plan state, not from legacy Premium-compatible compatibility views.
- Public readers, verified-player surfaces, storage proxies, session compatibility layers, scheduling embeds, checkout shells, world embodiment, and release metadata remain transport and governance around those boundaries, not replacements for them.

## Locked Patch Surface

### 1. Business Must Survive Native Session-Derived Surfaces

Business entitlement is now an explicit governed compatibility condition.

- Business must remain a first-class plan in native session-derived surfaces.
- Effective plan calculation must preserve Business over legacy Premium-compatible session values.
- `/api/session`, `/api/profile/bootstrap`, and `/api/world/me` must expose effective entitlement truth instead of flattening Business into Premium.
- Affiliate session handling must consume the effective plan directly so Business participates natively in affiliate-gated surfaces.

### 2. Scheduling Must Stay Receiz-Native

Receiz-owned scheduling is now part of the governed continuity surface.

- Upgrade CTAs must use the Receiz calendar embed.
- WireProof scheduling CTAs must use the Receiz calendar embed.
- Governed scheduling paths must not send the user through Calendly-branded flows when the Receiz scheduling surface is authoritative.

### 3. Public Payment Checkout Must Stay On-Surface

The public payment widget now has an explicit checkout-surface contract.

- The public payments embed must request Stripe embedded checkout sessions.
- Checkout must open inside a Receiz-hosted modal / popup shell.
- Redirect-required Stripe methods must return to the same Receiz-hosted checkout surface rather than switching to a direct hosted-checkout handoff.

### 4. Release Governance and Version Truth

The release contract remains locked across visible version surfaces.

- Release-scoped note and product-truth documents are required for the patch.
- Release-governed version surfaces must move together across package metadata, service worker defaults, public badges, offline verifier surfaces, and release documents.
- This repository records the public release documents and shipped HTML version markers for that lockstep requirement.

## Allowed Evolution

Implementation details may change after `v47.4.0` only if they preserve the authority boundaries and locked patch behaviors above. If a future change materially alters one of these truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v47.4.0` if it:

- flattens Business back into Premium through session, bootstrap, affiliate, or world compatibility layers
- consumes legacy compatibility plan truth instead of effective entitlement truth on governed Business surfaces
- routes upgrade or WireProof scheduling CTAs through Calendly or another non-Receiz scheduling surface
- redirects buyers away from a Receiz-hosted checkout shell instead of keeping Stripe embedded checkout on-surface
- changes release truth without updating the versioned release docs and aligned version surfaces
