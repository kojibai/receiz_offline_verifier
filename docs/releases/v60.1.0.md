# Receiz v60.1.0

Release date: April 18, 2026

## Overview

`v60.1.0` is the continuity-and-market-proof release on top of the `v60.0.0` stable baseline. The work since the last cut concentrates on one rule: once a surface is already settled, Receiz should not visually rediscover it, and once market witness truth exists, it should come from real holder activity instead of decorative fallback timestamps.

This release makes public profile asset markets holder-aware, retunes the market desk and chart for compact/public presentation, keeps home and profile twin shells visually stable while background preload warms hidden code and data, lazy-splits public profile route assembly without weakening SSR truth, and removes hydration drift in market sigil SVG namespaces.

There is no dedicated `v60.0.1` release note or tag in the repository. This `v60.1.0` note therefore rolls up the full repository delta after `v60.0.0`, including the intervening version-surface prep state, into one coherent release record.

The product-truth freeze for this release is defined in `docs/releases/v60.1.0-product-truth.md`.

## What Defines v60.1.0

- Public market witness truth now derives holder count and latest witness timing from live public positions and buy activity, not just creator timestamps or weak engagement fallbacks.
- The market desk, chart dock, and public market metadata surfaces are tighter and more legible on compact devices, especially in signed-out and public-profile flows.
- Home and profile twin shells now treat background preload as hidden preparation only: preload can warm code and data, but it cannot visually remount or upgrade the settled shell before explicit open intent.
- `MarketValuePill` sigils now use deterministic surface-scoped IDs across profile, player, hero, and artifact surfaces, closing SSR/client hydration drift in SVG namespaces.
- Public profile route assembly is lighter: the heavy client and trusted-stats merge path are lazy-loaded without weakening the initial rendered route truth.
- Release-governed version surfaces now move together to `v60.1.0` across package metadata, service worker defaults, public verifier artifacts, home, make, footer, legal, trust, plans, upgrade, and the active release docs.

## Since The Last Bumped Version Surfaces

### 1. Public Market Witness Truth Became Holder-Aware

Public profile asset markets now pull two new facts from live market state:

- holder count from public asset positions with non-zero quantity
- latest purchase witness time from public buy activity, owner position open time, and matching share-trade history

That feeds the desk in three ways:

- holder count now reflects actual public holders instead of generic envelope counts
- witnessed duration no longer gets stranded at zero when there is real purchase or engagement activity
- the witness detail and latest witness timestamp now prefer real holder or trade activity over creator seal or post time

This matters because the market is no longer narrating demand from a weaker creator-side clock when the system already knows the holder-side truth.

### 2. The Market Desk Was Retuned For Real Compact Usage

Most of the patch concentration sits in `MarketDeskExperience.tsx`, and it is product-facing:

- compact hero and public stat cards are denser and easier to read
- chart dock spacing, bottom margins, and overlay fit were retuned
- timestamp presentation is more precise and more legible in narrow widths
- proof metadata tiles now fit better in expanded public chart mode
- artifact labeling and UTC chronos lines are clearer in the manifest
- signed-out and public header copy, caption sizing, and market summary tiles no longer waste space

This is not new market behavior. It is the same market rendered with less friction on actual devices.

### 3. Twin Continuity Stopped Late Visible Promotion

The twin/runtime fixes in this release separate visible shell from background preparation more aggressively:

- the visible home and profile dock shell stays on the stable `WorldAgentDockRig`
- the full world profile module can warm in the background, but it only takes over after explicit open intent
- remembered twin session state no longer auto-promotes the full module on load
- home-route auth prewarming backs off so the settled front door is not disturbed

The user-facing effect is simple: once the page looks loaded, it stays loaded. Background hydration still happens, but it stays hidden until the user asks for the richer surface.

### 4. Profile Route And Sigil Hydration Became More Deterministic

The profile route and market pill path were tightened in two separate ways:

- `UserProfileRouteClient` now lazy-loads the heavy client and trusted-stats merge path, keeping the route lighter without dropping SSR parity
- `MarketValuePill` no longer seeds SVG IDs from `useId()`; callers now provide stable surface-scoped IDs so hydration order cannot change the SVG namespace
- the stable sigil scope is now threaded through profile cards, verified player surfaces, hero video surfaces, and original viewer surfaces
- profile copy around extra showcase hydration now stays honest and no longer implies silent full-surface completion after settlement

This release treats already-rendered truth as something to preserve, not something to quietly rebuild.
