# Receiz v97.0.0 Performance Findings

Release date: 2026-06-30

`v97.0.0` keeps the `v96.1.0` SDK ecosystem law intact and extends it to the durable public app-state projection rail. Developers and AI agents can recover public app first paint from Receiz projection reads, while proof objects, identity primitives, ownership appends, settlement rows, durable proof memory, Kai/Klok state, and verified appends remain stronger truth.

## Findings

- Public app-state projection reads improve cold-start behavior by letting apps recover public storefronts, marketplaces, games, rewards, profiles, directories, blogs, access systems, and custom-domain applications from Receiz rails.
- `receiz.appState.publishRecord()` improves developer speed by generating the canonical one-record projection envelope instead of making every integration hand-assemble feed boilerplate.
- `receiz.appState.byHost()` improves custom-domain and subdomain recovery by normalizing host input through the same public URL projection route.
- URL normalization prevents duplicate public projection rows for `https://example.app` and `https://example.app/`.
- OIDC consent readability and redirect correctness are performance concerns because a broken delegated login blocks third-party app launch before proof recovery can happen.
- Public proof projection responses must return canonical absolute verification URLs so downstream apps do not recover weaker route truth.
- Admin performance analytics must measure the user-visible app path, not internal/admin surfaces that misstate real public performance.

## Preserved Rules

- First paint must project known proof truth before hydration, analytics, SEO, session checks, lazy imports, global sync, server reconciliation, or SDK convenience work.
- Hidden work may append verified additions only.
- App-state projections are public recovery/index rows beneath proof objects and durable proof memory.
- Connect grants delegated route access only; it must not become an identity proof root.
- SDK package evidence must remain release-blocking when developer-facing primitives change.
- Public app-state recovery must not teach developers to add Supabase, Redis, Vercel KV, Shopify, or a custom database as proof authority.
