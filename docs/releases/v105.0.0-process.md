## Goal

Split `@receiz/sdk` architecture cleanly and ship developer runtime improvements without changing Receiz proof primitives. Preserve first admission only, then append forever.

## What Worked

Emitted-graph tests exposed Node leakage; TDD locked stable errors, capability discovery, exact import migration, protected extensions, emulator boundaries, conformance, MCP semantics, and external tarball installation.

## What Did Not Work

The initial v105 branch temporarily interrupted a v104 pack attempt. The work was isolated, main was restored to v104.0.0 for publication, and v105 returned only after v104 shipped.

## Practical Rule For Future Work

Publish the settled release before opening the next version on main. Verify packed external consumers, not only monorepo builds. Compiler code belongs in `@receiz/sdk/compiler`; sandbox evidence never becomes verification.
