Release date: May 2, 2026

## Summary

`v75.0.0` locks every governed release surface to the post-`v74.0.0` product candidate. The release preserves the `v70.0.0` public alpha law and the `v74.0.0` Receiz Key / identity-artwork baseline while making local Receiz ID proof, file restore parity, old-browser backup sign-in, market snapshot continuity, Explore packaging, profile identity downloads, and Plans / Upgrade parity first-class release truth.

This is not a label-only bump. The diff from the `v74.0.0` tag to the pre-bump v75 candidate includes local-first identity work, backup account-access routes for browsers that cannot hold a Receiz ID key, restored-account truth that outranks stale server echoes, deterministic market and Explore continuity repairs, profile identity download hardening, and release-contract coverage that must move together with the governed version surfaces.

For this offline verifier repository, the release work is publication of canonical `v75.0.0` release-prep artifacts and outward version-surface alignment. The verifier proof-format and producer-payload contract remain unchanged.

## What This Release Locks

- Receiz ID login is local-first: the device-held Receiz ID key admits local identity truth, projects the signed-in shell, seeds profile and wallet route truth, and starts server cookie / account propagation in the background.
- Server Receiz ID continuation may publish, bind, mint cookies, and return stronger additions, but it cannot replace a stronger local account binding with a weaker or stale session payload for a different account.
- Local account bindings are durable register truth across Receiz device, Receiz Key, PBI, email, and server-switch sources; switching bindings projects the selected account and clears account-scoped continuity when the actor changes.
- Identity Record, Identity Seal, and Receiz Key restore now project the verified file session directly, admit the restored account locally, seed profile / wallet / upgrade truth, and keep the restored account active even when import, bind, wallet refresh, or remote reconfirmation is delayed.
- First-time offline Receiz Key restore requires verified portable state; later restores can reuse admitted local key truth, and optional local PBI publication is a background append rather than restore permission.
- Browsers that cannot support the Receiz ID crypto path route to `/signin/receiz-id-fallback` with sanitized return paths. The primary sign-in surfaces stay Receiz ID pass-through; the backup route owns PBI/passkey, magic-link, and Identity Record / Seal / Receiz Key file restore.
- Account, managed profile, wallet, and upgrade surfaces recognize locally restored account authority before stale old-cookie echoes, so restored identity does not require a manual browser refresh to become visible.
- Profile identity artwork and signed original-download paths preserve deterministic profile truth, owner proof, purchase context, and PBI download authorization without redefining the base profile shell.
- Plans and Upgrade now keep parity for immediate payment / upgrade actions, with tests proving the two surfaces do not diverge on the payment path or old-browser fallback affordance.
- Market snapshots preserve stronger signed-in wallet, owned-item, prediction-history, and share-trade truth across lagging or anonymous payloads, while refusing to preserve private state across viewer changes.
- Sold share positions remain closed when local sell history is newer than incoming market payloads, preventing weaker snapshots from resurrecting closed ownership.
- Market navigation uses stable internal navigation for record cards, verified routes, live players, and asset links, with explicit pointer handling so mobile press behavior does not double-fire or fall through.
- Market mobile inputs and certificate panels keep no-zoom viewport behavior, 16px editable controls, and wallet-density typography.
- Explore opens as a card-first, dock-safe mobile surface with bounded swipe rendering, deterministic preview fallbacks, in-card read / play / save / share / proof actions, and local-first route entry.
- Explore server discovery remains a background recovery source; already-held view state and deterministic previews are reused instead of replaced by richer discovery rediscovery.
- Product doctrine now explicitly names Identity Record, Identity Seal, and Receiz Key restore parity with PBI recovery, and the v75 release preserves that law as a release-blocking invariant.
- The release includes the HPL continuity review and Receiz identity continuity one-pager as evidence artifacts for zero-regression local-proof upgrades.
- Package metadata, service-worker defaults, public version badges, verifier surfaces, README release pointer, and release docs move together to `v75.0.0`.
