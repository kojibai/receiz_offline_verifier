# Receiz v55.0.0 Product Truth

Release date: April 15, 2026
Status: Locked for `v55.0.0` release prep

## Purpose

This document defines the product-truth freeze for Receiz `v55.0.0`. It carries forward the existing file-authoritative, deterministic-route, wallet-and-ledger-backed, and governed-market model from `v54.0.0` while recording the guided owner-setup profile workspace, sealed profile media state, explicit world-twin freshness, shared market navigation continuity, canonical performance route grouping, and bounded prediction-book hydration as explicit non-regression behavior at release time.

For this offline verifier repository, the release is represented as current-release documentation, canonical release artifacts, and visible version-surface alignment only. The verifier trust contract remains unchanged relative to `v54.0.0`.

## Core Authority Boundaries

The release does not relax the core Receiz authority model:

- File-backed proof remains the authority for artifact truth.
- Deterministic route state remains the authority for first paint and route continuity truth.
- Wallet and ledger state remain the authority for money truth.
- Live session, market, media, and world deltas may hydrate on top of deterministic baseline state, but they must not replace that baseline as the source of truth.
- Public product routes, admin surfaces, release metadata, analytics rollups, and offline verifier entrypoints remain operating or governance surfaces around those authorities, not replacements for them.

## Locked Release Surface

### 1. `/profile` Must Stay The Guided Owner Setup Surface

- The signed-in profile workspace must remain a guided activation surface rather than regressing into a plain editor.
- The five-step owner checklist, inline username claim or edit flow, twin-photo handoff into World forge, fast-entry seed rehydration path, and sticky completion state must remain part of the profile setup experience.
- Once all setup steps are complete, completion must continue to survive reloads and route transitions.
- The signed-in workspace must continue to teach the intended owner journey instead of assuming prior product knowledge.

### 2. Profile Media Must Stay Sealed Product State

- Avatar and cover uploads must continue to process into canonical images before save.
- Public profile-media storage must continue to use deterministic naming and size-guard behavior.
- Profile avatar and cover updates must continue to mint sealed Receiz originals with anchor-backed verify paths.
- Proof-manifest and adjacent media records must continue to preserve or recover a usable media SHA-256 when direct visible-original metadata is incomplete.

### 3. World Twin Freshness Must Stay A Cross-Route Contract

- World surfaces must continue to prefer the freshest saved avatar render rather than relying only on a stale `is_current` marker.
- Preview locks must continue to prefer render-locked shell faces over older source-backed payloads when freshness is ambiguous.
- Refresh fanout across tabs and contexts must continue to work through window events, storage, and broadcast-channel style updates where available.
- Forged avatars must continue to sync back into the default profile avatar.
- Profile, public twin, and world embed surfaces must continue to revalidate after avatar saves.
- Compact touch devices and low-memory contexts must continue to avoid background hydration of the heavy world module until explicit user intent.

### 4. Market Navigation And Player Entry Must Stay On Shared Stable Routes

- Market exits, bottom-bar routing, and profile navigation from market must continue to use the shared stable-route helpers instead of bespoke route exceptions.
- Market route warming from auth chrome must continue without introducing a dedicated route-level loading shell.
- Live-player entry from market must continue to be able to navigate from seeded local payload before authoritative fetches settle.
- Deterministic route continuity must remain the first-paint contract even when live player data hydrates later.

### 5. Performance Reporting Must Stay Canonical And Route-Aware

- Web-vitals ingestion, admin reporting, and historical backfill must continue to share the same canonical route-group normalization.
- Admin performance rollups must continue to aggregate by stable product route, device class, and time bucket rather than noisy raw URLs.
- Historical `perf_web_vitals.route_group` backfill paths must remain available when legacy rows need canonical grouping.

### 6. Large Market Snapshots Must Stay Bounded

- Prediction-book hydration for large market snapshots must continue to load in bounded batches instead of one oversized query.
- Snapshot warming and player handoff behavior must not regress into unbounded hydration work that blocks route continuity.

### 7. Release Governance And Version Truth

- Release-scoped note and product-truth documents are required for `v55.0.0`.
- Release-governed version surfaces must move together across current-release docs, verifier-contract docs, and shipped offline HTML entrypoints.
- This repository records the public release documents and shipped HTML version markers for that lockstep requirement.

## Allowed Evolution

Implementation details may change after `v55.0.0` only if they preserve the authority boundaries and locked release behaviors above. If a future change materially alters one of these truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v55.0.0` if it:

- regresses `/profile` from a guided, sticky owner-setup flow into a plain editor or breaks the fast-entry seeded workspace path
- turns profile avatar or cover state back into unsealed mutable decoration without canonical processing, anchored verify paths, or usable media-hash recovery
- allows stale or non-fresh world avatar state to win across profile, public twin, world embed, or forged-avatar handoffs
- reintroduces heavy background world-module hydration by default on compact touch devices that should stay on-demand
- breaks stable-route market navigation, reintroduces bespoke route exceptions, or makes player entry wait on the full authoritative network round trip before navigating
- dilutes performance analytics back into raw per-URL variants instead of canonical route groups
- regresses large prediction-book hydration into an unbounded one-shot query path
- changes release truth without updating the versioned release docs and aligned version surfaces
