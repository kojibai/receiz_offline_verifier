## Canonical Product Gates

- Signal Card downloaded fronts, share pages, share actions, QR payloads, and OG/Twitter image routes describe and carry sealed card proof without detaching ownership.
- Signal Card World at `/game/cards` reads synced Vault ownership, merges local IndexedDB/localStorage cards, preserves owner scope, and publishes eligible local batches without rewriting card truth.
- Global Signal Vault sync moves server-known owner rows into public discovery while preserving account scope and proof state.
- Signal Vault Card export is a receized PNG archive carrier with manifest, archive hash, chunked archive payload, Merkle integrity, KAI timing, roster counts, card claim paths, art paths, and restore rules.
- Vault card restore and download behavior is explicit user intent only.
- Mobile and installed PWA card downloads do not navigate, refresh, collide with headers, or start hidden-frame work.
- Transfer-download packages make ownership movement explicit and avoid ambiguous duplicate ownership state.
- Signal Arena request, queue, loadout, AI twin authorization, settlement, XP/money/fun/keeps stake modes, action ledger, transfer integration, and append verification preserve original card claims.
- Card auctions are backed by wallet holds and settle against coherent reserve/account state.
- `/signal-circuit` exposes the public card law for earn, own, find, move, hold, and use against the same card record.
- `/exchange` ties Circuit, Market, settlement, wallet, payments, transfers, ledger, offline notes, card discovery, and conformance into one public ownership/value route.
- Signal Circuit conformance sources, badges, route inventory, and public status entries match real files and exclude high-risk private/worker/admin paths.
- Dashboard first paint is split through a smaller server/page boundary with dashboard data loaded through a dedicated client surface and API route.
- Service-worker registration, PWA header behavior, and runtime versioning do not preserve stale worker/header state in installed or private-browser sessions.
- Mobile chrome, global auth chrome, chat dock, composer host, profile/card CSS, discovery controls, first-paint card artwork, and public card popover/footer placement stay usable on mobile and PWA surfaces.
- Value-loop copy stays tied to reserve, settlement, ownership, and proof.
- Release surfaces move together to `v78.0.0`.
