## Frozen Primitives

| Primitive | v92.0.0 truth | Must not regress |
| --- | --- | --- |
| Receiz product identity | Receiz is a proof-native artifact system. | Do not describe Receiz as a generic app with proof features. |
| Receiz reasoning kernel | Work starts from primitive, strongest truth, weaker layers, forbidden downgrade, append path, and contract/visual evidence. | Do not start product decisions from route, DB, API, session, cache, component state, hydration, or generic convention. |
| Deterministic first paint | Known proof/profile/player/value/history truth paints before weaker enrichment. | Do not make known truth wait for session, private route work, API hydration, DB refresh, or route loading boundaries. |
| Public profile truth | `/<username>` remains canonical public profile truth; `?manage=1` layers owner permission on top. | Do not let owner/session/local fallback state redefine the base profile shell when canonical public truth exists. |
| Public proof live player | The live player first-paints from deterministic public proof/media truth for signed-in and signed-out viewers. | Do not let signed-in hydration or private profile work block or replace public proof first paint. |
| Vault object / artifact system | Vault objects carry identity, custody, provenance, media, history, ownership, and action affordances. | Do not flatten vault truth into DB rows, generic card UI, or hydration cache. |
| Sports ownership surface | Transfer-aware current-owner filtering projects custody beneath the vault object. | Do not show transferred-out collectibles as owned because stale owner rows still exist. |
| Sports settlement/value surface | The value chart remains visible and complete; event memory scrolls; opened event proof desk stacks above the value panel. | Do not let events consume the chart bottom or hide an opened event proof behind the value surface. |
| Event proof desk | The card-viewer event proof desk is the existing proof/event surface with a higher scoped stack when opened over value. | Do not duplicate it into a second event panel or change generic event popover behavior to solve a local stacking conflict. |
| Sports append proof | Source date comes from `append_json.sourceProof.sourceDate` when that is the proof payload truth. | Do not invent a DB authority or physical column when append proof already carries the truth. |
| Pitch Command proof object | Pitch Command carries pitch witness truth, geometry, edge projection, miss attribution, review signal, day proof, and proof-card output. | Do not downgrade it into a generic pitch UI, scoreboard widget, or fetched feed. |
| Historical pitch proof | Admitted historical pitch snapshots paint immediately and refresh behind known truth. | Do not force historical proof to wait for fresh full-slate rediscovery. |
| Pack Derby proof game | Pack Derby is proof-native Sports gameplay with runs, outs, strikes, rewards, and settled field geometry. | Do not treat Pack Derby as client-only animation or let visual geometry contradict gameplay truth. |
| Source age / next game | Source freshness projects the coherent current state: live age when live, next-game countdown when no game is live. | Do not render meaningless live-only age copy during post-slate or between-game gaps. |
| Local artifact save | User-held blob/artifact save is local/native first; server staging is not proof authority. | Do not make server staging permission for a file the user already holds. |

## Frozen Invariants

- Receiz law and implemented primitives outrank generic app convention.
- Stronger truth beats fresher weaker transport.
- DB/session/API/hydration/UI state may project, index, hydrate, sync, decorate, or append; they may not define product truth above the primitive.
- Public proof routes first-paint public proof truth before private enrichment.
- Owner mode appends controls and owner-only data; it does not redefine canonical public truth.
- Verified historical proof paints immediately when held.
- Verified appends carry source truth; database fields may accelerate lookup only beneath that truth.
- Settlement/value surfaces must remain legible when event memory exists.
- Visual evidence that contradicts a claimed pass is failing product evidence.
- Contracts must be strengthened after visual truth, not weakened to bless the visible mismatch.
- Release docs, public labels, service worker version, runtime defaults, offline verifier labels, and package version must move together.

## Residual Risks

- Local browser route validation for `/sports`, `/game/sports`, and `/arena` returned the sealed-map 404 in the current dev context; authenticated sports-card value/event visual proof must be captured in a route-enabled preview or production-equivalent session.
- Production DB state for individual sports append rows and vault ownership rows was not queried during the release-doc pass.
- Existing installed PWAs need the `92.0.0` service-worker update before stale `91.x` shells are guaranteed gone.
- Full release attestation requires build output and the production release-signing environment.
- Live streaming is not included in this release boundary; if it becomes a proof/live witness primitive, it needs its own product-truth doc and release gates.
