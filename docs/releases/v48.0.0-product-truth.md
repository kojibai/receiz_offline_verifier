# Receiz v48.0.0 Product Truth

Release date: April 10, 2026
Status: Locked for `v48.0.0` release prep

## Purpose

This document defines the product-truth freeze for Receiz `v48.0.0`. It carries forward the existing file-authoritative and continuity-safe platform truth while recording the operator-first contacts shell, durable `linked_user_id` upgrades, and cleaner world atlas layering as explicit non-regression behavior at release time.

For this offline verifier repository, the release is represented as current-release documentation and version-surface alignment only. The verifier trust contract remains unchanged relative to `v47.0.0`.

## Core Authority Boundaries

The release does not relax the core Receiz authority model:

- The file remains the authority for artifact truth.
- Deterministic manifests remain the authority for historical route truth.
- Contacts prioritization, continuity context, linked-account merge upgrades, and world atlas layering must operate on top of the existing trust and continuity primitives, not weaken or replace them.
- Signed-in account state and server-truth persistence remain the authority for contact identities, `linked_user_id`, notes, stages, invite state, timeline history, latest-touch memory, and direct-handoff enrichment.
- Mobile operator overlays, atlas HUD layers, offline verifier entrypoints, and release metadata remain transport and governance around those boundaries, not replacements for them.

## Locked Release Surface

### 1. `/contacts` Must Remain Operator-First

- The initial mobile frame must keep the command header, dock, and active relationship shell visible.
- The roster must rank by what needs action through best-move, continuity, latest-touch, and queue-style prioritization rather than only recent memory.
- Dense detail sections must move into overlays instead of forcing immediate stacked scrolling.

### 2. Mobile Contacts Must Preserve Full Operator Access

- Operators must retain access to overview, next-step, continuity context, recent activity, edit, invite, log, schedule, and full history from mobile.
- Compact presentation may reorganize these surfaces, but it must not remove or strand them behind missing state.

### 3. Identity Upgrades Must Persist Through Merge

- Imported or merged contacts must persist `linked_user_id` upgrades.
- Existing contact rows must be able to upgrade into authenticated continuity even when the rest of the payload is unchanged.
- The merge contract must remain covered by a source-level release test.

### 4. `/world` Must Remain A Clean Authoritative Atlas On Mobile

- Atlas naming must remain authoritative across the full surface.
- Map readability must be preserved by separating signal layers instead of stacking overlapping pills and duplicate floating labels.
- Lower panels must stay differentiated, with atlas focus explaining the district and return signal explaining why to come back.

### 5. Release Governance And Version Truth

- Release-scoped note and product-truth documents are required for `v48.0.0`.
- Release-governed version surfaces must move together across package metadata, service worker defaults, public badges, offline verifier surfaces, and release documents.
- This repository records the public release documents and shipped HTML version markers for that lockstep requirement.

## Allowed Evolution

Implementation details may change after `v48.0.0` only if they preserve the authority boundaries and locked release behaviors above. If a future change materially alters one of these truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v48.0.0` if it:

- regresses `/contacts` from an operator-first shell back into a long stacked record that hides command context or prioritization
- removes mobile access to overview, next-step, continuity context, recent activity, edit, invite, log, schedule, or full history
- drops `linked_user_id` merge upgrades or requires unrelated payload changes before authenticated continuity can persist
- makes `/world` mobile atlas naming non-authoritative or reintroduces overlapping signal layers and redundant lower panels
- changes release truth without updating the versioned release docs and aligned version surfaces
