## Locked Product Truth

- Sports is a first-class Receiz destination at `/game/sports` and `/game/sports/baseball`, with sport navigation, live MLB rows, Pick 5, live field, cockpit, leaders, proofs, cards, profile access, and mobile chrome.
- MLB cards are playable inventory. Owned Sports cards must carry deterministic issuance, issue numbers, pack origin, owner, history, live career, and market paths without fake rarity labels on base-set catalog cards.
- Player strength and card rarity are separate scarcity gates:

```txt
player strength is a player scarcity gate
card rarity is a variant issuance gate
both can compound, but neither replaces the other
```

- Pack odds must be auditable through published odds, observed simulation output, player scarcity, duplicate pressure, top-player concentration, rarity availability, spread, and registry basis hashes.
- Pick 5 mode boundaries must stay clear: per-game play uses owned matchup-team cards, Main Event uses the user's best five across the slate, and final games cannot be entered as live games.
- Live cockpit data must be scoped to the selected context and expose selected scoring cards, live deltas, total, rank, payout proximity, scoring story, line-chart movement, and value cues without wasting mobile space.
- Live game state must be deterministic:

```txt
entry lock state is not field state
selected game field data must match the selected game
active entries cannot rewrite live scoreboards
finalized proof fallback cannot hijack live play
```

- Pregame state is a real matchup surface. It may show venue, first-pitch readiness, projected cards, ratings, projected scores, stats, and card access, but it must not pretend a live play has happened.
- Daily tournaments must preserve funded entry, Stripe delta recovery, minimum-participant refund logic, top-percent payout, joined first paint, finality, settlement payout, result sharing, and play-again reset semantics.
- Per-game play and Main Event are separate lock domains. Users can enter one without mutating the other, see the correct cockpit, claim final rewards, and move forward after claimed finals clear.
- Finality is a product moment: entries can finalize, rank, issue rewards, open reward packs, update proofs, append card-career memory, and clear claimed proof surfaces without refresh rituals.
- Sports card memory must be append-only and long-term, including arena entry memory, rank memory, reward memory, score totals, source date, game key, lineup scope, append hashes, provenance rows, and value/ownership context.
- Sports market surfaces must support card sales, wallet-first purchase, Stripe delta funding, provenance, value panels, bids, asks, floors, spread, volume, movement, listings, sale history, and market conformance without collapsing play identity into market state.
- Sports trading must guard accept, decline, cancel, and counterproposal flows. Decline or cancel on non-open rows must return `409 sports_trade_not_open`; unexpected lookup failures must stay `500 sports_trade_lookup_failed`.
- Sports conformance is official under Game and must include `/game/sports/conformance`, `/api/game/sports/conformance`, badge, history, rollups, rarity scoring, odds, source checks, hot fanout, checkout delta recovery, and signed history checks.
- Scale hardening must prefer hot public snapshots, warmed state, runtime cache, account overlay throttling, route backpressure, worker jobs, observability, and build-cost controls over forcing every active user into fresh live/tournament work.
- PWA and first-paint truth must project known account, entered, cockpit, catalog, signing, and optimistic post-anchor state early instead of requiring refresh or an Arena press.
- Release labels, service-worker defaults, offline verifier labels, README release pointer, release docs, and world resident worker markers all move together to `v82.0.0`.

## Out Of Bounds

A change is out of bounds for `v82.0.0` if it:

- weakens file-first verifier authority or changes verifier trust semantics in this repository.
- treats Sports as a visual skin rather than a proof/value/game loop.
- shows fake rarity labels on unowned base-set cards or conflates player strength with card rarity.
- publishes pack odds that cannot be traced to deterministic issuance math and observed simulation output.
- lets per-game play and Main Event share a lock state, cockpit context, finality path, or reward claim state incorrectly.
- lets entry lock state, finalized proof fallback, or first-live-game fallback override selected-game field truth.
- pretends pregame has live play data.
- silently succeeds stale trade actions instead of returning guarded non-open status.
- hides tournament underfill refunds, payout finality, reward packs, or card-career appends from proof/value history.
- makes every active user force fresh live/tournament work when warmed public state can be used.
- requires refresh rituals where known local, account, or optimistic truth can be projected safely.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker defaults, badges, or release-worker markers.
