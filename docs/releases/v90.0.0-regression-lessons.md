# Receiz v90.0.0 Regression Lessons

Release date: June 14, 2026
Status: Required release-lock artifact for `v90.0.0`

## Purpose

This document records the post-v89 regression lessons that `v90.0.0` turns into release gates. It is release truth, not an activity feed. Each lesson names the stronger primitive that must keep authority during Sports proof, vault projection, ownership transfer, event proof value, conformance, market-depth, competition-room, metadata projection, first paint, or release-surface alignment.

## Lessons Locked

1. Sports Vault projection cannot be reduced to a shelf or owner row.
   - Stronger primitive: Sports Vault artifact system truth, including owner-row projection, `SportsVaultSurface`, vault projection flags, team-code normalization, and batch reveal.
   - Regression boundary: shelves, rows, filters, and batch UI may project verified vault truth, but may not replace the vault object or sealed manifest truth.

2. Sports card ownership transfer is an ownership append.
   - Stronger primitive: ownership transfer proof for Sports cards, including transferred-out vault IDs and the existing offline-transfer ownership append route for claimed/uploaded card images.
   - Regression boundary: account display state, gallery movement, or signed-in local vault commit may not replace the ownership append.

3. Sports event proof is a proof object.
   - Stronger primitive: Sports event proof identity with hero card, tool dock, event tiering, card impact, headline labels, rarity odds, link actions, and route-bound proof projection.
   - Regression boundary: event UI may present proof, but it may not rename Sports event proof as a generic card detail or share page.

4. Negative-score events still carry milestone history.
   - Stronger primitive: verified Sports event history and event milestone preservation.
   - Regression boundary: score sign, display filtering, and event summarization may not erase verified milestones.

5. Sports live value and market depth are settlement/value primitives.
   - Stronger primitive: live value algorithm, live UI integration, market-depth sales, and live market value seed evidence.
   - Regression boundary: display formatting, charting, and market UI may present value but may not define settlement truth.

6. Sports conformance is release evidence.
   - Stronger primitive: Sports conformance control, mode guide, issuance tests, and Sports conformance tests.
   - Regression boundary: conformance controls may not be reduced to copy, help text, or optional documentation.

7. Competition rooms are proof-native game/market coordination surfaces.
   - Stronger primitive: competition-room coordination over proof-native game, market, ownership, and value state.
   - Regression boundary: room, lobby, chat, or game UI mechanics may assist coordination but may not define the primitive.

8. Metadata projects public proof but does not define it.
   - Stronger primitive: route-bound payloads, public proof surfaces, signed proof material, and Sports event proof objects.
   - Regression boundary: absolute URL metadata, SEO payloads, previews, and sharing cards may project public proof but may not outrank proof truth.

9. Post-v89 locks remain inherited gates.
   - Stronger primitive: foreground anchor publication, profile projection, Sports Vault sealed-manifest proof, Sports/FIFA/UFC/fight proof objects, wallet/runtime continuity, lease/market ownership surfaces, DB pressure repairs, regression lessons, and commit-history boundary.
   - Regression boundary: v90 additions may append stronger proof behavior, but may not weaken v89 release law.

## Release-Lock Requirement

`pnpm release:lock` must require this document for `v90.0.0` and later release cuts. A release that omits regression lessons is incomplete because post-regression truth has not been made explicit.

## No Downgrade Statement

The lessons above preserve proof-native artifact system truth, Sports event proof objects, Sports Vault artifact-system truth, ownership append behavior, settlement/value primitives, public proof projection, deterministic first paint, service-worker release truth, and append-only verified history. They do not downgrade proof objects to cards, Sports Vault to a shelf, public proof surfaces to share pages, event proof value to balance display, ownership transfer to visual movement, conformance evidence to copy, competition rooms to generic UI, route truth to metadata, or foreground anchor publication to generic outbox replay.
