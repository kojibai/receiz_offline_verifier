# Receiz v107.0.0 developer guide

Install exact coordinated versions:

```bash
pnpm add @receiz/sdk@107.0.0 @receiz/ai-skills@107.0.0
pnpm add -D @receiz/mcp-server@107.0.0
```

## Client and identity profile

```ts
import { createReceizClient } from "@receiz/sdk";

const receiz = createReceizClient({ accessToken: process.env.RECEIZ_ACCESS_TOKEN });
const current = await receiz.identity.getProfile();
const availability = await receiz.identity.checkUsernameAvailability("new_name");

if (!availability.available && !availability.isCurrent) throw new Error("username_taken");

const committed = await receiz.identity.updateProfile({
  identity: current.identity,
  expectedHead: current.head,
  idempotencyKey: crypto.randomUUID(),
  profile: {
    username: "new_name",
    displayName: "New Name",
    avatar: avatarFile,
  },
});

if (!(await receiz.receipts.verify(committed.receipt))) throw new Error("receipt_invalid");
```

Pass an existing `ReceizMediaProofReference` as `avatar` instead of `avatarFile` to reuse prepublished proof media. The two forms share the same atomic operation. Availability is advisory; the commit-time unique claim decides the winner.

## Portable restore and cross-app state

```ts
const account = await receiz.identity.restoreAccount(identityRecordBytes, {
  artifactKind: "identity-record",
});

const appended = await receiz.identity.appendAccountState(account, {
  expectedHead: account.head,
  idempotencyKey: "scores-2026-07-17-1",
  allowedNamespaces: ["com.example.game"],
  additions: { appState: { "com.example.game": { score: 42 } } },
});

await saveFile(appended.exportedState);
```

Restore projects verified carried truth before network discovery. App namespaces append independently and cannot overwrite another namespace. A Vault restore requires an enclosing-artifact verifier.

## Generic bearer claim

```ts
const result = await receiz.ownership.claimBearerAsset({
  artifactBytes,
  filename: "certificate.receiz.json",
  mimeType: "application/json",
  claimantKeyId: current.identity.keyId,
  expectedOwnershipHead,
  idempotencyKey: crypto.randomUUID(),
});

if (!(await receiz.receipts.verify(result.receipt))) throw new Error("receipt_invalid");
await saveFile(result.artifactBytes);
```

Do not pre-authorize from a self-hashable embedded payload. Receiz verifies the enclosing proof object before interpreting that payload.

## Offline commands

```ts
const result = await receiz.offline.executeOrQueue({
  command: {
    operation: "identity.updateProfile",
    proofObjectId: current.head.aggregateId,
    payload: { username: "new_name" },
  },
  identity: deviceIdentity,
  expectedHead: current.head,
  idempotencyKey: "profile-rename-1",
}, {
  storage,
  connected: () => navigator.onLine,
  execute: sendSignedEnvelope,
});

if (result.status === "queued") {
  console.assert(result.globallyCommitted === false);
}
```

Never project a queued proposal as global state. Resume verifies receipts and processes FIFO per proof object.

## Stable errors

Handle the exact codes: `username_taken`, `username_invalid`, `stale_proof_head`, `identity_key_mismatch`, `proof_invalid`, `media_rejected`, `network_unavailable`, `idempotency_conflict`, `ownership_head_stale`, `custody_conflict`, `continuity_conflict`, `offline_command_queued`, and `receipt_invalid`.

Repair stale-head and ownership-head conflicts by rereading the canonical head, reviewing the verified additions, and issuing a new idempotency key. Do not silently overwrite. Repair receipt failure by stopping projection and re-verifying the stronger artifact/admission chain.

## MCP sequence

For a profile mutation:

1. Call `receiz_identity_profile_get`.
2. Call `receiz_identity_profile_update_plan` with identity, expected head, idempotency key, and profile.
3. Present the plan’s authority, consequences, and confirmation digest.
4. Call `receiz_identity_profile_update_execute` with identical `planDigest` and `confirmation`.
5. Accept success only when `receiptVerification.status` is `verified`.

Media, bearer ownership, and continuity use the same plan/execute pattern. MCP plans are not admission.

## Generator and emulator

```bash
receiz generate --framework nextjs \
  --features identity,profile,media,ownership,offline \
  --root ./my-app

receiz generate --framework nextjs \
  --features identity,profile,media,ownership,offline \
  --root ./my-app \
  --confirm <preview-digest>
```

Run emulator scenarios through `createReceizV107Emulator({ seed })`. Determinism and schema compatibility make the emulator conformance evidence; `productionVerified: false` prevents it from masquerading as production proof.

For v106 projects, follow [the forward-only migration](v107.0.0-migration.md). Receiz proof memory is first admission only, then append forever.
