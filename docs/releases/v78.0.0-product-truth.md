## Locked Product Truth

- Signal Cards remain sealed ownership records. Downloaded fronts, share pages, QR payloads, external share actions, and OG/Twitter image routes must carry card proof without detaching ownership state.
- `/game/cards` must behave as a Signal Card World backed by synced Vault ownership, local IndexedDB/localStorage cards, owner scope, sale panels, auction paths, challenge paths, and proof state.
- Global Signal Vault sync may publish eligible ownership rows into discovery, but it must not rewrite card truth or leak account-scoped private vault data.
- Signal Vault Card export is a receized PNG archive carrier with manifest, archive hash, chunked archive payload, Merkle integrity, KAI timing, roster counts, card claim paths, art paths, and restore rules.
- Vault card download and restore are user-intent-only flows. Opening the panel must not silently start sealing, timers, background export, or hidden-frame navigation.
- Mobile and installed PWA card downloads must prefer safer click/download behavior and must not turn export into route navigation, app refresh, or visible header collision.
- Transfer-download packages must make ownership movement explicit so sender and receiver state cannot drift into ambiguous duplicate ownership.
- Signal Arena battle request, queue, loadout, AI twin authorization, settlement, XP/money/fun/keeps stake modes, action-ledger append, transfer integration, and card append verification must not mutate the original card claim.
- Card auctions require wallet-hold backing and account/reserve-coherent settlement.
- `/signal-circuit` is the public standard page for earned cards becoming owned assets. Earn, own, find, move, hold, and use must resolve against the same card record.
- `/exchange` is the public ownership/value route connecting Circuit, Market, settlement, wallet, payments, transfers, ledger, offline notes, card discovery, and conformance.
- Signal Circuit conformance sources, badges, route inventory, and public status entries must cover the real files and exclude high-risk private, worker, webhook, admin, and legacy keyring paths.
- Dashboard first paint must come through a smaller server/page boundary while dashboard data loads through a dedicated client surface and API route.
- Service-worker registration, PWA header behavior, and runtime versioning must keep installed and private-browser sessions from preserving stale worker/header state.
- Value-loop copy must stay tied to reserve, settlement, ownership, and proof.
- Release surfaces must move together: package version, service-worker defaults, public badges, offline verifier pages, README pointer, v78 release docs, and world surface-worker release markers all identify `v78.0.0`.

## Out Of Bounds

A change is out of bounds for `v78.0.0` if it:

- weakens file-first verifier authority or changes verifier trust semantics in this repository.
- detaches Signal Card share/download/public surfaces from sealed ownership proof.
- starts Vault export, restore, download, timers, or hidden navigation without explicit user intent.
- allows transfer, battle, auction, listing, or sale behavior to mutate an original card claim instead of appending truth.
- permits auction settlement without wallet-hold/account/reserve coherence.
- leaks private vault, pack purchase, admin, webhook, worker, private session, or legacy keyring surfaces into public status/conformance inventory.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker defaults, badges, or release-worker markers.
