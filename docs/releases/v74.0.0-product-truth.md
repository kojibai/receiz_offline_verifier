# Receiz v74.0.0 Product Truth

Release date: April 30, 2026
Status: Locked for `v74.0.0` release prep

## Purpose

This document defines the product-truth freeze for Receiz `v74.0.0`, the Receiz Key, identity artwork, and profile media release after the `v73.0.0` ownership/vault continuity baseline.

`v74.0.0` extends product coverage for sovereign Receiz Key restore, deterministic identity artwork, PBI-signed identity artifacts, stable profile creation truth, richer profile originals, exact attached-link preservation, video playback derivatives, sold-position continuity, and portable profile state import. It does not weaken the `v70.0.0` public alpha law or the `v73.0.0` ownership/vault continuity baseline.

For this offline verifier repository, the release is represented as current-release documentation, canonical release artifacts, shipped HTML version markers, and package-metadata alignment only. The verifier proof-format and producer-payload contract remain unchanged relative to `v60.0.0`.

## Core Product Truth

`v74.0.0` carries forward the `v70.0.0` public alpha law and the `v73.0.0` ownership/vault continuity baseline:

> Identity is local. Proof is in the file. Verified truth is append-only. `/<username>` is canonical profile truth. Settlement and Reserve preserve funded causality. The network propagates and appends; it does not get to downgrade stronger proof already held by the device.

## Locked Release Surface

### 1. Receiz Key Restore Is Sovereign

- Receiz Key restore is a sovereign key-file recovery path.
- Fresh key signatures and active key registration prove restore.
- PBI may attest new key export but must not be required to restore from an already-held key.
- First-time offline Receiz Key projection requires verified portable state.
- Later offline login may reuse an admitted local identity block.
- A server miss must not erase local identity proof already admitted by the device.

### 2. Account Owns Receiz Key Control

- Account exposes Receiz Key controls where account protection lives.
- The owner can download a private Identity Record PNG containing the embedded Receiz Key payload.
- Optional file-password protection is available for key-file export.
- Identity Record PNGs and legacy key JSON imports remain accepted restore paths.
- Active and revoked keys are visible, refreshable, and revocable.
- Revocation requires a two-step owner action.

### 3. Profile Identity Artwork Is Deterministic

- Profile identity artwork derives from account creation Kai truth.
- Runtime randomness, storage upload timing, and profile remount timing must not affect identity artwork.
- Receiz card chrome may frame the seal.
- KaiSigil glyph geometry and size math remain canonical and unstyled by the card layer.
- Profile creation timestamps are stable local truth and cannot move forward across route hydration, workspace hydration, or remounts.

### 4. PBI-Signed Identity Artifacts Append Truth

- PBI-signed identity seal and identity record artifacts append verified authorship state.
- They must not replace the base profile shell.
- They must not invent a separate showcase coordinate.
- PBI-signed identity surfaces seed premium material from both the receipt hash and the PBI signing Kai moment.
- Signed witness emphasis uses trust-blue chrome rather than gold.

### 5. Profile Originals Preserve Full Payloads

- Long-form profile originals preserve full text, title, hash, and word count.
- Long-form profile originals provide a compact Read affordance.
- URL, read, and value pills must not be squeezed into one row.
- Attached Receiz URLs preserve the exact href.
- Front-card labels show only the domain.
- Composer and anchor transport must not silently cap valid long links at 280 characters.

### 6. Profile Video Playback Can Optimize Without Replacing Proof

- Profile video originals may use storage-backed playback derivatives.
- Range-aware proxying may be used for fast playback.
- Original proof downloads remain canonical.
- Playback derivatives must not replace original proof truth.

### 7. Market And Portable State Preserve Local Truth

- Sold share positions remain local truth across market snapshots and route hydration.
- Closed positions cannot resurrect as live ownership.
- Portable state import preserves profile media/state fields.
- Valid partial ledger entries are admitted only under explicit partial-import handling.
- Integrity failures remain visible.

### 8. Version Surfaces Move Together

- Package metadata, service-worker defaults, public version badges, verifier surfaces, README release pointer, release notes, changelog, docs indexes, and release docs must agree on `v74.0.0`.
- Historical release documents preserve their own version truth.

## Non-Regression Rule

A change is out of bounds for `v74.0.0` if it:

- requires PBI to restore from an already-held Receiz Key
- erases admitted local identity proof because the server misses it
- hides Receiz Key controls away from account protection
- makes profile identity artwork depend on runtime randomness, storage timing, or remount timing
- lets card chrome alter KaiSigil glyph geometry or size math
- replaces the base profile shell with PBI-signed identity artifacts or invents a separate showcase coordinate
- moves stable profile creation timestamps forward during hydration or remount
- drops long-form original text, title, hash, or word count
- truncates valid attached Receiz URLs through composer or anchor transport caps
- treats video playback derivatives as canonical proof downloads
- resurrects sold share positions as live ownership
- hides portable-state integrity failures
- allows version drift between package metadata, README, public badges, verifier surfaces, and release docs
