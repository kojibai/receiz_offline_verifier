## Preparation Steps

1. Preserve `v76.0.0` as the prior release baseline.
   - Keep `docs/releases/v76.0.0.md`.
   - Keep `docs/releases/v76.0.0-product-truth.md`.
   - Keep `docs/releases/v76.0.0-checklist.md`.
   - Keep `docs/releases/v76.0.0-process.md`.
   - Preserve `v76.0.0` as historical release truth.

2. Publish canonical `v77.0.0` release artifacts.
   - Add `docs/releases/v77.0.0.md`.
   - Add `docs/releases/v77.0.0-product-truth.md`.
   - Add `docs/releases/v77.0.0-checklist.md`.
   - Add `docs/releases/v77.0.0-process.md`.

3. Align repository release surfaces.
   - Update `package.json` to `77.0.0`.
   - Update root `README` current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.

4. Validate Signal Circuit and Signal Card gates.
   - `/game` first-paints without fake loaders.
   - Signal Cards issue through existing proof primitives.
   - Signal Card art, sealed claims, lineage, battle profiles, and verification paths remain durable and portable.
   - Pack purchase, reveal, and recovery keep exact-rate wallet settlement and durable opening records.
   - Vault sync stays account-scoped, paginated, retryable, and protected from account bleed.

5. Validate market, battle, conformance, and status gates.
   - Transfer, listing, auction, purchase, and battle surfaces append truth without mutating original card claims.
   - Card public/share/proof/OpenGraph/Twitter surfaces render from sealed card truth.
   - Signal Pack Foundry preserves Signal Card primitive isolation.
   - Signal Circuit conformance publishes requirements, digests, bounded simulation, badges, history, rollups, and status inventory together.
   - Public/developer Status includes only safe public card/conformance endpoints.

6. Validate Creator OS, Vault, document, and runtime gates.
   - `/creator` is first-class and first-paints settled workspace truth.
   - Creator OS durable queues, wallet metering, asset/publish jobs, safety reviews, and managed launch proof are enforced.
   - Verified Data Vault seals linked bytes, restores originals, and previews restored bytes.
   - Old Safari document sealing uses Signature V4 online and fails closed offline without legacy V3 fallback.
   - Runtime stale recovery clears stale state silently without automatic root interstitials or stale-guard redirects.
   - Market share quote bootstrap batches long asset lists before Supabase `in` filters.

7. Validate release-surface lockstep.
   - Confirm all governed version markers show `v77.0.0`.
   - Confirm all current-release docs and indices point to `v77.0.0` artifacts.
   - Confirm `v76.0.0` artifacts remain historical and untouched.

## Focused Verification Matrix

Broader platform release validation should include the gates named in `docs/releases/v77.0.0-checklist.md`, including:

- Signal Circuit route, mobile chrome, deterministic local game logic, and Signal Card issue flow.
- Signal Card sealed claim, artwork, export, lineage, share proof, listing, auction, purchase, transfer, and battle behavior.
- Signal Pack wallet settlement, opening durability, reveal continuity, and recovery.
- Signal Vault account scope, pagination, retry, transfer/listing/audit rows, ZIP restore, and native preview.
- Signal Pack Foundry campaign, odds, caps, manifest seal, and verification isolation.
- Signal Circuit conformance requirements, source digest, scarcity proof, bounded simulation, badge, history, rollup, and public Status behavior.
- Creator Workspace first paint, Streamdown output, durable Creator OS queues, wallet metering, paid asset/publish jobs, safety review, and managed launch provider proof.
- document sealing old-Safari Signature V4 behavior and offline guidance.
- runtime stale-recovery removal and stable-path return behavior.
- market quote batching, wallet value-state, profile/original continuity, and v76->v77 release-contract coverage.
- `pnpm lint`
- `pnpm build`
- `pnpm test:game-release-contracts`
- `pnpm test:signal-circuit-conformance`
- `pnpm test:release-freeze`
- `pnpm release:lock` / release-surface lock checks where available

## Tag Criteria

Do not tag `v77.0.0` until:

- all required `v76.0.0` and `v77.0.0` release artifacts exist
- version surfaces are aligned to `v77.0.0`
- release gates in `docs/releases/v77.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v77.0.0`
- `pnpm lint` has zero warnings/errors
- `pnpm build` completes

## Release Rule

Do not ship `v77.0.0` unless the release gates in this checklist pass from the exact release candidate, `pnpm lint` has zero warnings/errors, `pnpm build` completes, and the governed surfaces all show `v77.0.0`.
