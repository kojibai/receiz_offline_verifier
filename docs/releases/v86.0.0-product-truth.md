## Locked Product Truth

- Receiz is a proof-native artifact system, not a generic app with proof features.
- Files, identities, ownership states, settlement states, public witness surfaces, verification routes, profiles, Market, Explore, Sports cards, Series Passports, wallet images, Cash Images, OTC upgrades, and card QR proof routes carry product truth.
- Public profile posting appends visible proof objects to canonical public profile truth.
- Visible public profile proof objects belong in the deterministic Market shell.
- Hidden profile proof objects are removed from the local/public market snapshot, and unhidden proof objects return to Market.
- Cold `/market` first-paints from deterministic proof truth before server discovery or private graph hydration.
- Signed-in Market private graph hydration preserves public truth, owned assets, positions, and history.
- Null or anonymous private snapshots cannot erase signed-in ownership history after the shell paints.
- Explore first-paints from deterministic market proof truth instead of waiting on server discovery.
- Market value, chart, certificate/share, wallet, position, and history surfaces reuse deterministic market math and local settlement projection.
- Sports user-live hydration covers every relevant active, zero-append, and recovery source date.
- Live appends apply to every game that needs them for the day.
- Sports single-card QR public proof pages exist without modifying existing vault links.
- Main Event entry remains payment-confirmed; checkout dismissal clears pending tournament funding and does not enter.
- Profile OpenGraph, live-player sharing, Calendar wallet events, World profile/twin surfaces, and public twin bootstrap remain beneath stronger first-paint truth.
- Release labels, package metadata, service-worker runtime version, offline verifier labels, README release pointer, release docs, and public markers all move together to `v86.0.0`.

## Forbidden Downgrades

A change is out of bounds for `v86.0.0` if it:

- downgrades proof object to card.
- downgrades public proof surface to share page.
- downgrades Series Passport to pass, checkout product, badge, or generic CTA.
- downgrades Sports Arena to game UI without proof economy truth.
- downgrades Main Event settlement primitive to prize display or balance display.
- downgrades Market ownership state to visual badge or delayed server display.
- downgrades public profile proof objects to posts detached from proof truth.
- downgrades deterministic first paint to loading state.
- downgrades verified history to activity feed.
- downgrades route truth to SEO payload.
- downgrades service-worker release truth to incidental cache behavior.
- lets external systems define Receiz truth instead of providing implementation mechanics.
- treats generic app convention as stronger than implemented Receiz primitives.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker runtime, badges, or release markers.
