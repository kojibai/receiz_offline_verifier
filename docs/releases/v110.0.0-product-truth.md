# Receiz v110.0.0 Product Truth

Receiz v110 is a proof-native SDK/MCP/AI release. Exact sealed artifact bytes are verified before payload classification or recovery. The admission verdict and permitted actions are explicit; neither a plan nor explanation is authority.

For `@receiz/sdk`, durable proof memory remains **first admission only, then append forever**.

The standard proof history preserves parent links, Kai ordering, ownership transitions, terminal events, receipts, and unknown namespaces. Deterministic conflict handling rejects missing parents, cycles, stale branches, mixed identities, receipt failures, divergence, and terminal reversal.

`admitAndRecover` is local, read-only, zero-network, and zero-write. `commitRecovery` is separately explicit, capability-bound, idempotent, expected-head-bound, and atomic through the caller-provided `ReceizAdmissionStore`.

Bearer recovery never grants signing, canonical identity, publishing, or settlement. Legacy and foreign-owner results remain read-only. Card-only payloads are invalid because the enclosing proof object is the admission boundary.
