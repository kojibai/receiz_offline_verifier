## Preparation Steps

1. Preserve `v88.0.0` as the prior current-release archive.
   - Keep `docs/releases/v88.0.0.md`.
   - Keep `docs/releases/v88.0.0-product-truth.md`.
   - Keep `docs/releases/v88.0.0-checklist.md`.
   - Keep `docs/releases/v88.0.0-process.md`.
   - Keep `docs/releases/v88.0.0-regression-lessons.md`.
   - Preserve `v88.0.0` as historical release-law truth.

2. Publish canonical `v89.0.0` release artifacts.
   - Add `docs/releases/v89.0.0.md`.
   - Add `docs/releases/v89.0.0-product-truth.md`.
   - Add `docs/releases/v89.0.0-checklist.md`.
   - Add `docs/releases/v89.0.0-process.md`.
   - Add `docs/releases/v89.0.0-regression-lessons.md`.
   - Add `docs/releases/v89.0.0-commit-history.md`.

3. Align repository release law and product-truth references.
   - Update `AGENTS.md` release law to `v89.0.0`.
   - Update product-truth reference status markers carried forward for the current release.
   - Preserve the primitive decision procedure, stronger/weaker truth hierarchy, and forbidden downgrade boundary.

4. Align repository release surfaces.
   - Update `package.json` to `89.0.0`.
   - Update root `README` current-release references.
   - Update `RELEASE_NOTES.md` and `CHANGELOG.md`.
   - Update `docs/README.md`, `docs/governance/README.md`, and `docs/FORMAT.md`.
   - Update shipped version markers in `site/index.html`, `apps/offline-verifier.html`, `apps/offline-record-seal.html`, and `apps/offline-settlement.html`.
   - Ensure static service-worker registrations request `sw.js?v=89.0.0`.
   - Ensure `site/sw.js` carries service-worker runtime version `89.0.0`.
   - Ensure local release-surface checks can execute through `pnpm lint`, `pnpm build`, and `pnpm release:lock`.

5. Validate primitive naming gates.
   - Foreground anchor publication is named as proof publication route truth, not outbox replay.
   - Profile original, profile showcase, public profile market append, and route snapshots keep direct primitive names.
   - Sports Vault sealed image/manifest is named as a proof object.
   - Sports FIFA/UFC/fight cards, packs, lineups, vault objects, and event proofs keep proof-object names.
   - Wallet, Receiz ID, PBI, and settlement primitives keep direct primitive names.
   - Lease, market ownership, sale, claim, and transaction room paths keep ownership/settlement primitive names.
   - Service-worker truth is named as release truth, not incidental cache behavior.
   - Public proof, identity, ownership, settlement, route, and verified-history truth retain direct primitive names.

6. Validate anchor/outbox gates.
   - `/api/receiz/anchor` is excluded from local signed-action outbox admission.
   - No broad `/api/receiz` proof classifier re-admits the foreground anchor route.
   - Anchor success returns readable JSON with `{ ok, anchorId, anchoredUrl }`.
   - Anchor failures return readable JSON errors.
   - Generic outbox mechanics do not sign, replay, inspect, wrap, or consume the foreground anchor response before the composer reads it.

7. Validate post-v88 surface gates.
   - Profile original publish continuity and first-view/live-player handoff remain repaired.
   - Canonical public profile snapshot appends after visible profile writes.
   - Public profile market append source refreshes after visible original writes.
   - Local route snapshots update for surfaces made visible by the write path.
   - Sports Vault sealed image preparation runs ahead of server publication.
   - Compact QR publish and showcase publish paths stay separated from full browser payload upload.
   - Public owner vault routes and public event proofs remain proof surfaces.
   - Sports live/vault/FIFA/UFC/fight additions remain proof objects.
   - Wallet first-paint, ledger receipt, PBI, Receiz session, and service-worker navigation gates remain protected.
   - Lease/market ownership and settlement paths remain direct primitives.
   - SQL proof/source payload and high-pressure user-live route repairs remain covered.

8. Validate runtime lockstep.
   - Confirm all governed version markers show `v89.0.0`.
   - Confirm all current-release docs and indices point to `v89.0.0` artifacts.
   - Confirm historical `v80.0.0` through `v88.0.0` artifacts remain archived release truth.
   - Confirm `release:lock` requires `docs/releases/v89.0.0-regression-lessons.md`.
   - Confirm `release:lock` requires `docs/releases/v89.0.0-commit-history.md`.
   - Run `pnpm release:lock`.
   - Run whitespace diff check.

9. Validate broader platform gates before production promotion.
   - Run release-freeze and page-first-paint checks.
   - Confirm anchor/outbox, Profile, Sports Vault, Sports, Market, Wallet, runtime/session, DB pressure, security, and release contracts.
   - Confirm target production DB migrations named by the release are applied.

10. Generate signed release attestation.
    - Run `pnpm release:attest` for the final commit in the environment that holds the release attestation signing key.
    - Retain the signed attestation artifact under the governance release-attestation policy.
    - Do not tag or ship without the signed attestation.

## Focused Verification Matrix

Broader platform release validation should include the gates named in `docs/releases/v89.0.0-checklist.md`, including:

- direct primitive naming and no forbidden downgrade
- offline proof semantics unchanged from `v87.0.0`
- v88 live Sports, runtime, market/profile, DB hardening, and regression lessons preserved
- foreground anchor publication route and local signed-action outbox exclusion
- profile original projection into showcase, public profile market append, and visible route snapshots
- Sports Vault sealed image/manifest public proof publishing
- Sports FIFA/UFC/fight cards, packs, vaults, event proofs, and public showcase behavior
- wallet first-paint, ledger receipts, PBI, Receiz session, and service-worker navigation behavior
- lease lifecycle register, marketplace handoff, selected transaction room, and market ownership behavior
- SQL proof/source payload, high-pressure user-live route, and append discovery behavior
- release freeze, page first paint, TypeScript, release lock, regression lessons, commit history, whitespace diff, and release attestation gates

## Tag Criteria

Do not tag `v89.0.0` until:

- all required `v88.0.0` and `v89.0.0` release artifacts exist
- version surfaces are aligned to `v89.0.0`
- regression lessons are present and required by `release:lock`
- commit history is present and required by `release:lock`
- release gates in `docs/releases/v89.0.0-checklist.md` pass from the exact release candidate
- governed surfaces all show `v89.0.0`
- `pnpm lint` has zero warnings/errors
- `pnpm build` completes
- `pnpm release:lock` confirms the release surfaces
- whitespace diff check passes
- `pnpm release:attest` emits a signed attestation for the final commit

## Release Rule

Do not ship `v89.0.0` unless the release gates in this checklist pass from the exact release candidate, `pnpm release:lock` confirms the release surfaces, regression lessons, and commit history, whitespace diff passes, focused anchor/outbox, Profile, Sports Vault, Sports, Market, Wallet, runtime/session, DB pressure, security, and release contracts pass, and production migrations named by the release are applied in the target environment.
