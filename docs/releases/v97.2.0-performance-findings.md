# Receiz v97.2.0 Performance Findings

Release date: 2026-06-30

`v97.2.0` keeps the `v97.0.0` public app-state projection law intact and extends it to a broader enterprise app runtime SDK. Developers and AI agents can build richer public apps with typed Receiz rails, while proof objects, identity primitives, ownership appends, settlement rows, durable proof memory, Kai/Klok state, public projections, and verified appends remain stronger truth.

## Findings

- Tenant-scoped customer sessions and portal helpers reduce app startup glue while preserving identity proof boundaries.
- Merchant onboarding helpers reduce the time to publish a Commerce Cloud grade storefront without turning onboarding state into proof authority.
- Commerce primitives for refunds, subscriptions, shipping, tax, discounts, gift cards, access passes, inventory, fulfillment, and payouts make the app runtime useful without replacing ownership appends or settlement rows.
- Idempotency material on write helpers improves retry safety after serverless cold starts, browser interruptions, and offline queue recovery.
- Offline proof queues improve resilience by replaying verified additions after known heads instead of forcing rediscovery-first loading.
- Audit logs, event replay, jobs, release pinning, search, notifications, media transforms, risk checks, compliance exports, and store portability improve operational recovery when they remain projections or append surfaces beneath proof truth.
- `@receiz/sdk/react` improves React integration speed, but hosted UI controls must never hide the primitive names or imply authority above proof objects.

## Preserved Rules

- First paint must project known proof truth before hydration, analytics, SEO, session checks, lazy imports, global sync, server reconciliation, SDK convenience work, or hosted React controls.
- Hidden work may append verified additions only.
- Enterprise app runtime helpers are typed convenience rails beneath proof objects and durable proof memory.
- App-state projections are public recovery/index rows beneath proof objects and durable proof memory.
- Connect grants delegated route access only; it must not become an identity proof root.
- SDK package evidence must remain release-blocking when developer-facing primitives change.
- Enterprise app runtime recovery must not teach developers to add Supabase, Redis, Vercel KV, Shopify, or a custom database as proof authority.
