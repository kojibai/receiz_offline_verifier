# Receiz v47.0.0

Release date: April 5, 2026

## Overview

`v47.0.0` is the release where Receiz turns the broad platform expansion of `v46.0.0` into a more explicit, governed, and productized trust surface. The platform is now sharper at the places people actually touch it: original and document delivery, verified-player routing and sharing, byte-range media access, Public Twin/world embodiment, and release-governed version truth.

The core rule does not change. The file remains the authority. Deterministic manifests remain the authority for historical route truth. Public readers, verified-player surfaces, storage proxies, world embodiment, and release metadata are transport and governance layers around those truths, not replacements for them.

The product-truth freeze for this release is defined in `docs/releases/v47.0.0-product-truth.md`.

## What Defines v47.0.0

- Originals and documents now behave like first-class product surfaces instead of attachment passthroughs.
- Verified-player routes are stronger as destinations: better continuity, richer metadata, and clearer public sharing behavior.
- Storage and streaming correctness are harder to accidentally break: byte-range semantics, signed object delivery, and release-versioned client surfaces are explicitly guarded.
- Public Twin and world rendering now operate under more explicit authority, stage-separation, and freshness rules.
- Release truth is more machine-checked: version surfaces, product-surface inventory, governance artifacts, and freeze gates now move together.

## Major Accomplishments Since v46.0.0

### 1. Original and Document Delivery Became a Deliberate Surface

Receiz materially upgraded how originals are delivered and read:

- richer document classification and reader-mode routing
- inline document reading that stays openable without collapsing to attachment-only behavior
- stronger original-versus-derivative download precedence
- richer verified-original and profile-original delivery coordination
- more explicit storage-object delivery behavior for large media

This matters because a deterministic artifact system should present truthful reading and playback behavior directly, not force users back into blunt download-only flows.

### 2. Verified Player and Route Continuity Hardened

The verified-player surface matured into a more coherent destination:

- stronger route hydration and deterministic continuity between profile, artifact, and live-player paths
- richer live-player share metadata and artwork behavior
- better queue handling and playback handoffs
- stronger profile-to-player and player-to-artifact navigation contracts
- clearer paid-download and ownership-aware viewer behavior

The result is a player surface that feels product-grade instead of a thin wrapper over raw media URLs.

### 3. Storage, Streaming, and Client Freshness Tightened

The release also hardens the less visible but critical delivery layer:

- signed storage-object routes preserve `HEAD` and byte-range semantics for large media
- profile original delivery contracts are explicitly guarded
- service worker version truth is tied to release-version truth
- stale asset replay across releases is now treated as a release-governed risk rather than an incidental browser quirk
- upload and storage paths are kept closer to the known-good original bucket contract

These are the kinds of changes that stop "works in a demo" behavior from turning into silent corruption or stale-client regressions in production.

### 4. Public Twin and World Surfaces Became More Authority-Aware

World and Public Twin behavior continued to mature after `v46.0.0`:

- render-locked preview freshness is treated as authoritative
- photoreal shell fit and face-feature alignment now carry stronger authority and fallback rules
- public world/profile payloads are more explicitly sanitized
- stage and dock rendering stay separated
- public live-video presentation is harder to fake or degrade accidentally

This makes Public Twin/world presentation feel more like governed embodiment and less like best-effort media composition.

### 5. Release Governance Became More Explicit

`v47.0.0` sharpens the release process itself:

- release-scoped product-truth document
- canonical product-surface inventory contract
- release-freeze gate that groups the major local release blockers
- release attestation workflow tied to versioned docs and signing requirements
- version badges, package version, release docs, and service worker defaults checked together

This is the release where "don't regress the surface" becomes a machine-checkable rule rather than an informal expectation.
