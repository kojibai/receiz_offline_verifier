# Receiz v96.1.0 Invariant Register

Release date: 2026-06-30

`v96.1.0` carries forward the proof-native artifact system and SDK evidence invariants from `v96.0.0` and adds the Twin and World SDK invariants needed for the expanded developer surface.

## Locked Invariants

1. Receiz remains a proof-native artifact system. Product truth is carried by files, identities, vault objects, ownership states, settlement states, public witness surfaces, verification routes, Twin surfaces, and World projections.
2. Complete proof objects carry deterministic identity, sealed/verifiable payload, provenance, inspection behavior, and Kai/Klok state.
3. Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit.
4. Durable proof memory is not cache. First admission creates known truth, and later work appends verified additions.
5. Connect is delegated API access. It must not become the identity proof root or replace Receiz ID, Identity Record, Identity Seal, Receiz Key, PBI, or artifact proof.
6. Twin SDK clients are convenience beneath authorized user identity, durable local truth, verified appends, and existing Twin route contracts.
7. World SDK clients are convenience beneath public World/Twin projection routes. Public World reads must not become a new proof issuer or database authority.
8. OpenAPI and status catalog entries are route-discovery and contract evidence. They must not outrank implemented route contracts or proof verification.
9. Identity conformance records the Connect/Twin surface as public developer evidence beneath Receiz proof authority.
10. SDK clients, docs, CLI, OpenAPI, status catalog, server, database, SEO, analytics, session, and framework routing conventions remain weaker than admitted proof truth.
11. `@receiz/sdk` release version, app release version, SDK runtime constants, release documents, and public labels must not diverge before promotion.
12. The operating law remains first admission only, then append forever.

## Forbidden Regressions

- proof object to card
- artifact system to UI experience
- durable proof memory to cache
- Connect delegated scope to identity proof root
- Twin convenience to proof authority
- World read to proof issuer
- OpenAPI/status docs to route or proof authority
- known-head sync to rediscovery-first loading
- release evidence to optional package hygiene

## Release-Lock Requirement

`pnpm release:lock 96.1.0` must require this document for `v96.1.0` and later release cuts. A release that omits the invariant register is incomplete because proof-native system identity, complete proof object Kai/Klok state, durable proof memory, Connect delegated access, Twin SDK convenience, World SDK convenience, OpenAPI/status boundaries, identity conformance evidence, version lock, and broader package evidence would not be explicit release truth.
