# Receiz v72.0.0 Product Truth

Release date: April 28, 2026
Status: Locked for `v72.0.0` release prep

## Purpose

This document defines the product-truth freeze for Receiz `v72.0.0`, the stable continuity release after the `v70.0.0` public alpha boundary.

`v72.0.0` does not replace the `v70.0.0` law. It locks stabilization work that protects continuity and mobile Live Twin fit while preserving the same authority model: local identity, file-backed proof, append-only verified truth, canonical profile truth, funded value causality, and network-as-propagation.

For this offline verifier repository, the release is represented as archived release documentation and governed version-surface expectations. The verifier proof-format and producer-payload contract remain unchanged relative to `v60.0.0`.

## Core Product Truth

`v72.0.0` carries forward the `v70.0.0` public alpha law:

> Identity is local. Proof is in the file. Verified truth is append-only. `/<username>` is canonical profile truth. Settlement and Reserve preserve funded causality. The network propagates and appends; it does not get to downgrade stronger proof already held by the device.

## Locked Release Surface

### 1. Canonical Profile Truth Remains Stable

- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains the same profile with owner powers layered on top.
- Owner permissions do not create a competing profile truth.
- Owner/private enrichment must not make signed-in profile first paint slower than public profile truth.

### 2. Useful Hydration Repairs Stay

- Stale app-open state may refresh in the background.
- Background repair must not replace a settled visible shell with a weaker or unrelated state.
- Hydration is useful only when it appends, repairs, or proves stronger truth.

### 3. Account Switching Clears Actor-Scoped Warmth

- Account switch on `/account` must immediately project the new account to every warmed route surface.
- Market, wallet, world, and profile route continuity are actor-scoped.
- Actor-scoped continuity must be cleared when the active account uid changes.
- Market or private wallet state must not be preserved unless the viewer user id matches.

### 4. Mobile Twin Face Fit Is Contractual

- Compact mobile Live Twin rendered-portrait overlays must keep eyes and mouth on the person's face.
- The bounded lower-left face-lock correction is allowed for compact mobile fit.
- The correction must not restore the old oversized mobile downshift.
- Eye and mouth overlay fit is a contract, not a visual preference.

### 5. Version Surfaces Move Together

- Package metadata, service-worker defaults, public version badges, verifier surfaces, release docs, and current-release pointers must agree at the final `v72.0.0` cut.
- Historical release documents preserve their own version truth.

## Important Non-Changes

- The verifier proof-format and producer-payload contract remain unchanged.
- Trusted-signature success still requires verified `signatureV4`.
- Effective anchor context remains required for verified outcomes.
- Groth16 fields remain required and only real `g16:` proof payloads are accepted.
- The `v64.0.0` value-loop invariants remain in force.
- The `v70.0.0` public alpha invariant register remains the baseline law.

## Non-Regression Rule

A change is out of bounds for `v72.0.0` if it:

- renders owner/private enrichment as a slower or different profile truth than public profile truth
- preserves route warmth across account uid changes
- shows market or private wallet state for a previous actor after account switch
- treats mobile twin overlay fit as optional decoration
- restores the old oversized mobile downshift to solve compact portrait fit
- allows version drift between package metadata, public badges, verifier surfaces, and release docs
