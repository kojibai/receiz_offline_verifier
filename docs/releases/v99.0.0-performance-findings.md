# Receiz v99.0.0 Performance Findings

`v99.0.0` locks the full post-`v97.0.0` working set while preserving Receiz scale law: rich developer, graph, ledger, sports, market, profile, webhook, archive, SDK, and MCP mechanics must reduce weaker work beneath visible primitives instead of delaying known proof truth.

## Findings

- Public proof graph performance depends on bounded graph data, mobile navigation, active panel selection, lighter styling, ledger pagination, market-cap downsampling, OHLC sampling, and only-open asset chart construction. These optimizations are allowed because `/proof-graph` remains a public proof surface rooted in real proof/economy/network data and live Kai pulse state.
- Ledger and transaction surfaces gained public pagination, proof-bundle extraction, transfer value snapshots, transaction prioritization, settlement rollups, and Kai/upulse display. The performance boundary is proof-first: route/API rows and chart state may project proof truth, but they cannot outrank proof bundles, settlement primitives, or verified appends.
- Sports Arena and vault performance work uses bounded rollup workers, deferred surfaces, active-job dedupe, image helpers, owner-current projections, route contracts, and DB performance contracts. The preserved primitive is known proof/media/ownership/closeout/settlement truth before idle/background work.
- SDK, MCP, OIDC delegated tokens, developer webhooks, app-state rows, public-store rows, and service-worker state improve developer speed and runtime continuity while staying beneath proof objects, verified local truth, identity proofs, ownership appends, settlement rows, public proof surfaces, and offline verification.
- Pitch Command day-proof archive catch-up adds large proof archive material. Generated manifests, projections, and gzip bundles are release-carried proof archive evidence, not disposable static assets.

## Performance Boundary

- Known verified truth must paint before hydration, analytics, SEO, session checks, lazy imports, global sync, chart hydration, ledger pagination, or server reconciliation.
- Public proof, profile, market, ledger, transaction, and Sports surfaces may append verified additions after known heads; they must not blank or rediscover broad state before first paint.
- Chart downsampling, worker dedupe, route extraction, cursor repair, and service-worker continuity are weaker mechanics beneath proof primitives.
- Visual evidence can block release even when static contracts pass.
