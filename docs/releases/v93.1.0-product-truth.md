`v93.1.0` preserves Receiz as a proof-native artifact system and freezes the durable-memory model learned from Sports history. The release does not create a new authority above proof objects. It makes admitted proof truth more durable, more shared, and faster to project.

## Frozen Primitives

| Primitive | v93.1.0 truth | Must not regress |
| --- | --- | --- |
| Durable proof memory | Once proof truth is admitted on a device, it is durable local truth and must project immediately. | Do not treat admitted truth as stale cache that must be rediscovered before display. |
| First admission, then append forever | Full-history hydration is allowed for first admission or recovery. After admission, network work discovers additions. | Do not rehydrate old history on every open when durable truth exists. |
| Shared proof object projection | Calendar points and Account brain nodes can project the same admitted event proof objects. | Do not fork the same event into competing Calendar truth and Account truth. |
| Account brain activity nodes | Account brain nodes are proof projections over admitted account/calendar actions. | Do not replace durable activity nodes with route-local hydration state. |
| Wallet Kai-ledger window | Wallet settlement rows are ordered by Kai pulse/upulse with Chronos as display/fallback context. | Do not make Chronos the backend ordering authority when Kai head state exists. |
| Sports append memory | Sports card appends and event history remain durable proof memory. | Do not make card history disappear while a route refetches known appends. |
| Bounded Sports projection | Sports vault, event proof, visible append, and showcase reads remain bounded beneath proof authority. | Do not restore broad unbounded reads to rediscover already-known proof rows. |
| Developer SDK convenience boundary | `@receiz/sdk` helps developers verify and project Receiz assets; it is mechanics beneath proof authority. | Do not let SDK responses outrank sealed artifacts, proof bundles, or verified registers. |
| Release identity coherence | Public labels, service worker, runtime storage, SDK version, and release docs move as one version. | Do not ship mismatched public version surfaces. |

## Frozen Invariants

- Known verified truth paints first.
- Verified stored truth is not cache and is not stale.
- A route is a projection over proof objects, not an authority competing with them.
- The same proof object may project into many surfaces without duplicating authority.
- Network, server, DB, RPC, and schema cache paths may discover verified additions only below admitted truth.
- A weaker partial payload must not erase or replace a stronger admitted payload.
- Wallet settlement asks for verified additions after the known Kai ledger head.
- Sports and Calendar history may hydrate only when the local node does not yet hold durable proof memory.
- Developer tooling must expose Receiz primitives without creating a new proof root.
- Tests must be upgraded to stronger proof contracts; production proof fields must not be made optional to satisfy stale fixtures.

## Residual Risks

- The full `pnpm test:release-freeze` wrapper can fail before product gates if pnpm attempts a non-TTY modules purge. Run the underlying gates directly when that happens and fix package-manager setup separately.
- Installed PWAs need the `93.1.0` service-worker update before stale `93.0.x` shells are guaranteed gone.
- New devices or cleared browser storage still require first admission from server/global proof sync before durable local projection exists.
- Production Supabase state must be observed after deployment for timeout recurrence, RPC visibility, and schema-cache freshness.
- Live sports feeds and media/CDN availability remain external runtime inputs. They can confirm projection, but they do not define proof truth.
