# Receiz v46.0.0

Release date: March 30, 2026

## Overview

`v46.0.0` is the lock-the-surface release for everything shipped after `v45.0.0`. The release turns the post-`45.0.0` product work into explicit product truth: richer original and document viewing, a more complete verified-original player, stronger Public Twin photoreal forge behavior, authority-aware face-feature alignment, better streaming and storage correctness, and a clearer release-freeze contract around the whole platform.

This release does not relax Receiz's core rule. The file remains the authority. Deterministic manifests remain the authority for historical route truth. Replayable append-only ledger math remains the authority for settlement. New viewers, players, photoreal shells, and twin runtime paths stay transport around those boundaries, not replacements for them.

The product-truth freeze for this release is defined in `docs/releases/v46.0.0-product-truth.md`.

## What Defines v46.0.0

- Original-file and document delivery now behave like deliberate product surfaces instead of attachment passthrough.
- The verified-original player is stronger as a user-facing destination: richer metadata, clearer purchase/download behavior, and better social/share framing.
- Public Twin photoreal generation now operates under an explicit portrait contract with feature-map authority and fallback rules.
- World stage rendering is harder to fake or accidentally degrade: stage and dock stay separate, and video only renders when a real delivery exists.
- Release truth is now codified as a reusable freeze layer rather than a loose collection of checks.

## Major Accomplishments Since v45.0.0

### 1. Original and Document Viewing Became First-Class

Receiz materially upgraded the original-file experience:

- richer document classification across PDF, DOCX, text, JSON, CSV, and code-like assets
- fullscreen document reading instead of attachment-only fallbacks
- inline viewer URLs that preserve truthful reading behavior
- explicit download precedence between original, derivative, and moment-record assets
- better profile/original and verified-player delivery coordination

This matters because a deterministic artifact platform should not force users back into blunt download-only flows for core reading behavior.

### 2. Verified Original Player Grew Into a More Complete Surface

The live verified-player surface picked up:

- clearer profile identity framing
- stronger purchase and paid-download gating behavior
- richer Kairos and pulse presentation
- better queue and timeline handling
- direct profile navigation
- dedicated social images for the live-player route

The result is that the verified-player surface feels more like a stable destination and less like a thin transport wrapper.

### 3. Public Twin Forge and Photoreal Shell Hardening Accelerated

This release meaningfully tightened Public Twin and world behavior:

- photoreal avatar generation now enforces a portrait contract instead of silently accepting weak renders
- feature-map metadata is persisted and surfaced with explicit authority information
- face-feature alignment now has backend, native, and heuristic fallback paths
- compact and mobile rendered-portrait overlays use deterministic fit primitives instead of ad hoc placement
- owner setup, embed, and inactive-twin handling became more deliberate

This makes the photoreal shell behave more like a governed embodiment surface and less like a best-effort visual effect.

### 4. Streaming and Delivery Correctness Improved

Supporting delivery infrastructure also hardened:

- signed storage proxy routes preserve byte-range streaming behavior
- `HEAD` requests keep partial-content semantics
- inline document viewing stays openable without collapsing important delivery rules
- service-worker versioning remains tied to release-version truth

These are not flashy changes, but they are the difference between a product that demos well and one that stays truthful under real media access.

### 5. Release Truth Is Now More Explicitly Locked

`v46.0.0` adds a stronger release-freeze layer:

- a release-scoped product-truth document
- a canonical product-surface inventory contract
- broader profile/world release-contract coverage
- a single `pnpm test:release-freeze` gate that groups the major local release blockers

This is the release where the repo becomes more explicit about which behaviors are allowed to evolve and which are now treated as locked product truth.
