# Receiz v106.0.0 Release Process

## Goal

Ship `@receiz/sdk`, MCP, AI skills, and the independent verifier as one enforcement chain without allowing a release contract to overrule stronger implemented proof behavior. Preserve **first admission only, then append forever**.

## What Worked

Repository doctrine, failing behavioral contracts, deterministic migration fixtures, independent replay, packed-package installation, and exact registry-digest comparison exposed real boundaries. The owner-local profile regression was fixed in the admission path before strengthening its contracts.

## What Did Not Work

An early return on canonical-row presence conflated global visibility with local admission. It allowed an older owner snapshot to win, dropping the PBI signature and Live Player projection. Development-server browser attempts were not valid evidence when Turbopack panicked or webpack did not settle.

## Practical Rule For Future Work

Fix the stronger truth path first. Then write a contract that fails on the original regression. Never change a contract to bless a weaker snapshot, missing signature, unavailable Live Player, or changed first paint. Run `@receiz/sdk` conformance, MCP conformance, skill validation, independent verification, packed installation, governance, performance, production build, and the full release freeze before attestation.
