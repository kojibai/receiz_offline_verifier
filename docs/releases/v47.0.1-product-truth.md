# Receiz v47.0.1 Product Truth

Release date: April 5, 2026
Status: Locked for `v47.0.1` release prep

## Purpose

This document defines the patch-level product-truth freeze for Receiz `v47.0.1`. It preserves the `v47.0.0` product truth while recording the specific stability fixes that are now treated as explicit non-regression behavior at release time.

For this offline verifier repository, the patch is represented as current-release documentation and version-surface alignment only. The verifier trust contract remains unchanged relative to `v47.0.0`.

## Core Authority Boundaries

The release does not relax the core Receiz authority model:

- The file remains the authority for artifact truth.
- Deterministic manifests remain the authority for historical route truth.
- Public readers, verified-player surfaces, storage proxies, world embodiment, and release metadata remain transport and governance around those boundaries, not replacements for them.

## Locked Patch Surface

### 1. Profile Composer Upload Handoff

The owner upload path on `/profile?compose=1` remains part of the governed product surface.

- Original video uploads must keep the intended resumable storage handoff available while upload progress is being reported.
- UI progress state must not suppress or break the real browser-safe upload completion path.
- Storage-confirmed originals must not be downgraded to failed uploads just because a public URL is not returned immediately.

### 2. Storage-Backed Sealed Video Resolution

Storage authority must win over incidental response-shape differences.

- Storage-backed sealed video handoffs must resolve to a real download target.
- Authoritative object presence in storage is sufficient to treat the upload as successful pending downstream URL resolution.
- The composer must not report failure after authoritative storage success has already occurred.

### 3. Profile Artwork Preload Discipline

Profile artwork warming remains allowed, but only under explicit priority rules.

- Warming must be constrained to the visible showcase window.
- Only explicit high-priority images may emit head preload links.
- Non-critical artwork must not be head-preloaded in a way that predictably generates unused-preload warnings.

### 4. Release Governance and Version Truth

The release contract remains locked across visible version surfaces.

- Release-scoped note and product-truth documents are required for the patch.
- Release-governed version surfaces must move together across package metadata, service worker defaults, public badges, and release documents.
- This repository records the public release documents and shipped HTML version markers for that lockstep requirement.

## Allowed Evolution

Implementation details may change after `v47.0.1` only if they preserve the authority boundaries and locked patch behaviors above. If a future change materially alters one of these truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v47.0.1` if it:

- breaks the real browser-safe profile composer upload path on `/profile?compose=1`
- treats storage-confirmed sealed video uploads as failed because a public URL is not returned immediately
- head-preloads non-critical profile artwork outside the visible showcase or explicit high-priority set
- changes release truth without updating the versioned release docs and aligned version surfaces
