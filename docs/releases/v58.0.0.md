# Receiz v58.0.0

Release date: April 17, 2026

## Overview

`v58.0.0` is the authoritative-market and release-surface coherence cut on top of the `v57.0.0` baseline. The core rule for this release is simple: if Receiz already knows the deterministic market truth and the live user overlay truth, the app should show that truth immediately instead of dropping positions, delaying wallet state, or rebuilding an oversized desk snapshot after paint.

This release restores signed-in market authority on first paint, unifies market and chart-buy wallet hydration around the same live snapshot path, converts market transport into a lean deterministic seed plus live user overlay, corrects tracked-position semantics so expired or non-open contracts do not count as live positions, and moves package metadata, service worker defaults, public badges, offline verifier artifacts, plans/upgrade badges, and release docs together to `v58.0.0`.

There is no dedicated `v57.0.1` through `v57.99` public release note or tag in the repository. This `v58.0.0` note therefore rolls up the full repository delta after `v57.0.0`, including the intervening internal prep commits and the final market fixes in the current release-prep tree, into one coherent release record.

The product-truth freeze for this release is defined in `docs/releases/v58.0.0-product-truth.md`.

## What Defines v58.0.0

- Signed-in `/market` and signed-in market asset entry are authoritative again on first paint instead of waiting on a late client snapshot before positions, history, and wallet state appear.
- Market transport now ships a deterministic seed plus live overlay instead of serializing the fully expanded desk snapshot over the wire.
- The chart buy popover and market wallet now hydrate through the same wallet-summary path, so buying power and wallet state do not lag behind the visible signed-in state.
- Tracked-position surfaces now count live shares plus only contracts that are still open; expired or non-open contracts no longer inflate the live position count.
- Market snapshot continuity, prediction history, and share-trade history are treated as durable user-visible truth rather than disposable background fetch state.
- Release-governed version surfaces now move together to `v58.0.0` across package metadata, service worker defaults, public verifier artifacts, home/make/footer/legal/trust/plans/upgrade badges, and the active release docs.

## Since v57.0.0

### 1. Signed-In Market Entry Became Authoritative Again

The release removes the regression where signed-in market entry could start from a light shell and show empty positions or history until a later snapshot fetch completed.

The current repository state now provides:

- authoritative signed-in market entry on `/market`
- authoritative signed-in market entry on `/market/assets/[receizId]`
- snapshot continuity logic that preserves newer wallet/history truth instead of throwing it away on a weaker refresh
- route hydration that treats signed-in market state as durable route truth rather than a disposable bootstrap hint

The result is that history, positions, and balance state no longer depend on a second-settle fetch before the user sees what they already own.

### 2. Wallet And Chart-Buy Hydration Now Share One Coherent Path

The chart buy popover was previously able to discover wallet data without stitching that data back into the shared market snapshot, which left buy surfaces showing wallet-loading copy after sign-in or after background refresh.

The current state now includes:

- a shared wallet-summary client for market surfaces
- wallet hydration that patches live signed-in market snapshots directly
- popover preloading that seeds wallet truth before open instead of waiting for a full desk refresh
- retry and continuation logic that keeps chart-buy funding, passkey re-entry, and snapshot refresh on one coherent wallet path

This makes the market wallet and the chart buy sheet describe the same live buying-power truth.

### 3. Market Transport Became Deterministic And Lean

This release removes the oversized market transport path that was shipping the expanded desk snapshot when the client could derive most of it deterministically.

The current state now provides:

- a dedicated market transport type carrying deterministic public inputs plus the live user overlay
- local snapshot materialization from transport on server and client
- lighter snapshot transport for `/market` and `/api/market/snapshot`
- route prewarm, market refresh, and chart-buy refresh paths that all rebuild from the same transport contract

That keeps the payload smaller while preserving authoritative wallet, position, history, and pricing truth.

### 4. Tracked Position Semantics Now Match Live Contract Truth

The visible tracked-position count can no longer include contracts that have already closed or moved beyond the open state.

The current state now locks:

- tracked positions = live share positions + still-open contracts
- open-contract wallet rail filtered through real open-contract semantics instead of any attached prediction position
- history counters and prediction-history rails separated from live tracked-position counters

This aligns the visible numbers with the product rule users actually experience: if a contract is not open, it is not a live tracked position.

### 5. Release Surfaces Were Re-Unified

The repository had drift between `package.json`, the service worker version, and public version badges. This release cuts one explicit `58.0.0` surface across the governed release boundary.

The current state now aligns:

- `package.json`
- `public/sw.js`
- `app/components/pwa/ServiceWorkerRegistration.tsx`
- public/offline verifier and offline record-studio artifacts
- home, make, trust, footer, legal, plans, and upgrade release badges
- active release docs
