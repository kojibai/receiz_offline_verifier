## Locked Product Truth

- Receiz remains a proof-native artifact system, not a generic app with proof features.
- Offline sealed files and offline moments are proof objects when they carry deterministic identity, proof material, verification behavior, durable state, and a stronger source of truth than generic display state.
- Compact `/v` proof token transport is not authority. Authority remains the sealed `receiz.proof_bundle`, Signature V4 bundle signature, Signature V4 device certificate, payload hash, artifact basis hash, claim identity, route-bound slug/code/pulse, and proof digests.
- `/v/[slug]/[code]/[pulse]` verifies compact offline-generated signed proof bundles as `Proof` when Signature V4 and route binding are valid.
- Profile composer publication, database rediscovery, session state, and server lookup are not permission for an offline proof object to be true.
- Coordinate-only evidence remains historical compatibility evidence and cannot be upgraded above a signed proof bundle.
- Malformed, mismatched, or invalid proof material is rejected or not elevated.
- Offline moment QR transport points to the proof-bearing compact `/v` URL after sealing, while sealed artifact truth remains bound to the artifact basis and proof bundle.
- Sports single-card QR/value surfaces remain public proof surfaces around Sports card proof objects.
- World Estate remains a World/public twin identity surface and cannot replace stronger preview, portrait, profile, or identity truth with weaker session or presentation state.
- Market/profile asset loading and first-paint contracts preserve public proof objects under RLS constraints.
- Browser storage, RLS cleanup, and service-worker release/runtime coherence remain contract-governed support surfaces beneath product truth.
- Release labels, package metadata, service-worker runtime version, offline verifier labels, README release pointer, release docs, and public markers all move together to `v87.0.0`.

## Forbidden Downgrades

A change is out of bounds for `v87.0.0` if it:

- downgrades offline proof to fallback mode.
- downgrades proof object to card.
- downgrades compact proof token to a new authority layer.
- downgrades signed proof bundle truth to server lookup, profile publication, route metadata, or SEO payload.
- upgrades coordinate-only evidence above historical compatibility.
- downgrades public proof surface to share page.
- downgrades Sports single-card QR cockpit to generic card page.
- downgrades Sports card value surface to decorative balance display.
- downgrades World Estate identity/public witness truth to session or presentation state.
- downgrades deterministic first paint to loading state.
- downgrades verified history to activity feed.
- downgrades service-worker release truth to incidental cache behavior.
- lets external systems define Receiz truth instead of providing implementation mechanics.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker runtime, badges, or release markers.
