## Locked Product Truth

- Receiz remains a proof-native artifact system, not a generic app with proof features.
- Sports event proof is a proof object with event identity, tiering, card impact, headline labels, rarity odds, link actions, and milestone preservation.
- Sports Vault is an artifact system. Owner-row projection, `SportsVaultSurface`, vault projection flags, team-code normalization, and batch reveal append verified vault truth rather than replacing it.
- Sports card ownership transfer is an ownership append. Claimed/uploaded card image transfer reuses the existing offline-transfer ownership append route before signed-in local vault commit.
- Sports value movement is a settlement primitive. Live value, market-depth sales, and market value seeds may project settlement evidence but may not become decorative balance display.
- Sports conformance is release evidence. The Sports conformance control, mode guide, issuance tests, and Sports conformance tests protect proof-native behavior.
- Competition rooms are proof-native game/market coordination surfaces, not generic chat, room, or lobby UI.
- Metadata projects public proof. Absolute URL metadata for Sports event proof sharing may help projection, but it does not define proof or outrank route-bound proof payloads.
- `/api/receiz/anchor` remains the foreground proof publication route and stays excluded from generic local signed-action outbox admission.
- Profile original publication still requires showcase truth, public profile market append truth, and visible local route snapshots.
- The sealed Sports Vault image/manifest remains the proof object. Server publish, showcase, and global sync may append beneath it, but may not replace it.
- Receiz ID, PBI, wallet settlement, wallet ledger receipts, and durable runtime projection remain identity and settlement primitives.
- Service-worker/cache mechanics may warm and activate, but they do not redefine signed-in proof continuity, settled first paint, or release truth.
- Regression lessons and commit-history boundaries are release truth and must stay required by `release:lock`.

## Forbidden Downgrades

A change is out of bounds for `v90.0.0` if it:

- downgrades Receiz from a proof-native artifact system to a generic app with proof features.
- downgrades Sports event proof to a card detail, hero component, activity feed, or share page.
- downgrades Sports Vault to a card grid, shelf, or upload payload.
- downgrades Sports card ownership transfer to visual vault movement or account display state.
- downgrades live value, market depth, or market value seed behavior to generic balance display.
- downgrades Sports conformance evidence to product copy.
- downgrades competition rooms to generic chat, room, lobby, or game UI.
- lets metadata define proof instead of projecting public proof.
- re-admits `/api/receiz/anchor` to a broad `/api/receiz` outbox classifier.
- treats a profile original DB row as globally complete without showcase, market append, and visible route snapshot projection.
- downgrades deterministic first paint to loading state.
- downgrades verified history to activity feed.
- downgrades service-worker release truth to incidental cache behavior.
- lets external systems define Receiz truth instead of providing implementation mechanics.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker runtime, badges, or release markers.
