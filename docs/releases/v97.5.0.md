## Title

Receiz `v97.5.0` - Public App-State, Enterprise SDK, and MCP Agent Rail Release

## Summary

`v97.5.0` locks the post-`v97.0.0` developer platform into one coherent release: durable public app-state projections, typed Commerce Cloud and enterprise runtime rails in `@receiz/sdk`, Receiz-issued delegated agent tokens, and the published `@receiz/mcp-server` agent interface.

The release makes Receiz easier for developers and AI agents to build with while preserving the authority hierarchy. SDK and MCP helpers are convenience and tooling beneath proof objects, sealed artifacts, verified local truth, ownership appends, settlement rows, Kai/Klok state, public projections, and verified appends. The database, server, SDK, and MCP can index, sync, inspect, publish, or project truth; they do not replace truth.

## What Changed Since v97.0.0

- Public app-state projections became a first-class rail for third-party apps. Apps can publish durable public projections by URL, host, namespace, ID, or creator and recover public storefront state after server cold starts without adding a separate database.
- `@receiz/sdk` gained typed `appState` and `publicStore` helpers for publish, resolve, restore, URL/host normalization, public store records, tenant records, and durable projection envelopes.
- `@receiz/sdk` gained tenant customer sessions, customer portal helpers, merchant onboarding helpers, one-click checkout, media upload and transform helpers, domain helpers, events, jobs, webhooks, permissions, audit, risk, compliance, portability, search, notifications, release pinning, sandbox helpers, and offline proof queues.
- `@receiz/sdk/react` exposes optional hooks and hosted components for identity, checkout, app-state, media, domains, wallet balance, and Twin writer surfaces.
- `@receiz/mcp-server` became the official MCP package for AI agents building and verifying Receiz apps through canonical SDK rails.
- The MCP server now supports SDK-backed diagnostics, capability inspection, required scopes, tenant sessions, app-state publish/resolve, public-store publish/resolve, public proof lookup, offline manifest verification, sports card memory, pitch proof witness lookup, transfer previews, pack-open previews, marketplace template generation, and `receiz://` resource templates.
- `/developers/connect` now creates scoped delegated agent tokens for MCP, CI, and trusted local agents. Tokens are issued by Receiz OIDC, shown once, revocable by token ID, and labeled with an `agent:*` audit label.
- The MCP binary startup path now resolves npm `.bin` symlinks with `realpathSync`, so `npx -y @receiz/mcp-server` starts the stdio server correctly.
- Developer surfaces now expose MCP from `/developers`, `/developers/mcp`, `/developers/openapi`, and Connect setup so humans and agents can find the correct integration path.

## Developer Outcome

Developers can now build a Receiz-powered app with the SDK as the application rail and MCP as the agent rail:

```ts
import { createReceizClient } from "@receiz/sdk";

const receiz = createReceizClient({ accessToken });

const store = await receiz.publicStore.resolve({
  host: "bjklock.receiz.app",
});

await receiz.publicStore.publish({
  tenantHost: "bjklock.receiz.app",
  merchantReceizId: "bjklock.receiz.id",
  state: store.state,
}, {
  idempotencyKey: "store:publish:bjklock:v1",
});
```

Agent hosts can add:

```json
{
  "mcpServers": {
    "receiz": {
      "command": "npx",
      "args": ["-y", "@receiz/mcp-server"],
      "env": {
        "RECEIZ_BASE_URL": "https://receiz.com",
        "RECEIZ_ACCESS_TOKEN": "<delegated agent token from /developers/connect>"
      }
    }
  }
}
```

## Product Law Preserved

- Proof objects remain the source of truth.
- Kai Klok remains the proof object state machine.
- Durable proof memory remains first admission only, then append forever.
- Public app-state projections remain recovery projections beneath proof objects.
- Connect remains delegated API access, not identity proof authority.
- Delegated agent tokens remain scoped Connect/OIDC access beneath Receiz identity proof and are revocable/auditable by token ID.
- `@receiz/sdk` remains a typed convenience layer beneath Receiz primitives.
- `@receiz/mcp-server` is agent tooling, not authority.
