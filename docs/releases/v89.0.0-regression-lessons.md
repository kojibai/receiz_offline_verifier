# Receiz v89.0.0 Regression Lessons

Release date: June 13, 2026
Status: Required release-lock artifact for `v89.0.0`

## Purpose

This document records the post-v88 regression lessons that `v89.0.0` turns into release gates. It is release truth, not an activity feed. Each lesson names the stronger primitive that must keep authority during foreground publication, projection, first paint, global sync, route snapshot refresh, runtime navigation, or database pressure repair.

## Lessons Locked

1. Foreground anchor publication cannot be routed through generic outbox replay.
   - Stronger primitive: `/api/receiz/anchor` foreground proof publication route truth and its readable anchor response contract.
   - Regression boundary: local signed-action outbox mechanics may replay/sync weaker signed actions, but may not admit, sign, inspect, wrap, consume, or replay the foreground anchor response path before the composer reads it.

2. A broad `/api/receiz` classifier cannot override a named proof route exclusion.
   - Stronger primitive: route-specific foreground anchor exclusion from local signed-action outbox admission.
   - Regression boundary: future generalized proof classifiers must preserve explicit route exclusions before broad path matching.

3. A profile post is not globally complete because a DB row exists.
   - Stronger primitive: profile original publication projection into canonical profile showcase truth, public profile market append truth, and visible local route snapshots.
   - Regression boundary: DB writes, sync jobs, and server reconciliation may persist data, but they do not satisfy the visible publication contract until projections are appended or refreshed.

4. Sports Vault public proof cannot be reduced to browser payload upload size.
   - Stronger primitive: sealed Sports Vault image/manifest proof object truth.
   - Regression boundary: compact QR publish, showcase publish, server upload, and global sync may append beneath the sealed vault proof, but may not replace it.

5. Brand-new accounts without saved profile usernames cannot lose public vault route truth.
   - Stronger primitive: public Sports Vault URL projection using saved usernames when present and the default profile username projection when no saved username exists yet.
   - Regression boundary: unsaved profile setup may not remove the public vault route or public event proof route.

6. Sports expansion surfaces remain proof objects.
   - Stronger primitive: Sports cards, packs, lineups, vault objects, event proofs, FIFA/UFC/fight cards, and public showcases.
   - Regression boundary: hydration, server reads, route redirects, and fallback media may append verified additions; they may not rename or erase proof collection truth.

7. Wallet/runtime continuity remains identity and settlement truth.
   - Stronger primitive: Receiz ID, PBI, wallet settlement, wallet ledger receipts, wallet first-paint cookie, and durable runtime projection.
   - Regression boundary: cache, service-worker warmup, profile reads, or session checks may not redefine signed-in proof continuity or settled first paint.

8. Ownership and settlement surfaces cannot be downgraded to generic listings.
   - Stronger primitive: lease lifecycle register, market ownership, sale, claim, and transaction room settlement truth.
   - Regression boundary: marketplace handoff and transaction room UI support ownership/settlement primitives; they do not define them as generic listing state.

9. Performance repairs are valid only when they preserve stronger proof truth.
   - Stronger primitive: proof/source payloads, known proof objects, profile originals, vault objects, active entries, and first-paint shells.
   - Regression boundary: faster SQL, cache, parallel reads, and append discovery may discover verified additions, but may not erase known stronger truth or substitute stale server state.

## Release-Lock Requirement

`pnpm release:lock` must require this document for `v89.0.0` and later release cuts. A release that omits regression lessons is incomplete because post-regression truth has not been made explicit.

## No Downgrade Statement

The lessons above preserve proof-native artifact system truth, foreground proof publication, deterministic first paint, public proof/witness surfaces, service-worker release truth, ownership and settlement primitives, and append-only verified history. They do not downgrade proof objects to cards, public proof surfaces to share pages, live proof surfaces to generic UI, settlement primitives to balance display, ownership state to listings, route truth to framework convention, or foreground anchor publication to generic outbox replay.
