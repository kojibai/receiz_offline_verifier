# Receiz v104.0.0 Regression Lessons

## What Regressed

The already-published MCP 103.0.0 manifest leaked `workspace:^` for
`@receiz/sdk`, so external installations could not resolve the monorepo-only
protocol.

## Why It Regressed

Source-workspace tests did not install the packed MCP artifact in a fresh
external project, and publication was not locked to pnpm's workspace-aware path.

## Locked Rules

- Publish SDK, MCP, and AI skills through pnpm workspace-aware commands.
- Pin coordinated v104 dependencies exactly.
- Reject `workspace:` in source, packed, and externally installed manifests.
- Keep compiler/MCP inspection beneath proof verification authority.
- Durable proof memory remains **first admission only, then append forever**.

## Contract Gates

`test:coordinated-package-release`, `test:mcp-release-contracts`, SDK release
contracts, AI-skills distribution, verification conformance, and release freeze
must pass before publishing `@receiz/sdk` or companion packages.

## Release-Lock Requirement

`pnpm release:lock 104.0.0` must require these regression lessons in the standalone offline-verifier checkout. Omitting them would sever the coordinated release from the externally reproduced `workspace:` dependency defect and the packed-artifact installation gate that prevents recurrence.
