`v98.0.0` preserves Receiz as a proof-native artifact system and locks all post-`v97.0.0` developer-runtime, public proof, settlement, sports, profile, and proof-archive work beneath implemented Receiz primitives.

## Frozen Primitives

| Primitive | v98.0.0 truth | Must not regress |
| --- | --- | --- |
| Proof object | Receiz proof objects carry deterministic identity, verifiable payload, provenance, inspection behavior, and Kai/Klok state. | Do not let SDK, MCP, webhooks, charts, DB rows, service worker state, or app-state projections treat proof truth as generic app data. |
| Kai Klok | Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit. | Do not use Chronos, DB insertion time, provider time, or chart sampling order as the primary state coordinate for admitted proof objects. |
| Durable proof memory | Verified local truth and admitted proof history remain first admission only, then append forever. | Do not call admitted proof memory cache, stale data, hydration state, or rediscovery state. |
| Public app-state and public-store projection rail | Public app-state, public-store, and storefront rows are recovery projections beneath proof objects and verified appends. | Do not let projection rows outrank sealed artifacts, identity proofs, ownership appends, settlement rows, verified local truth, or verified appends. |
| Merchant Identity Seal / Receiz Key publish | Merchant-owned public-store state can be signed by the merchant proof object and appended without delegated bearer-token authority. | Do not require `RECEIZ_ACCESS_TOKEN` where the merchant proof object carries authority. |
| SDK runtime | `@receiz/sdk` exposes typed developer rails for public proof, app-state, commerce, media, domains, jobs, webhooks, audit, compliance, React, sandbox, and offline proof queues. | Do not let SDK convenience become truth authority over proof objects or verified appends. |
| MCP agent rail | `@receiz/mcp-server` exposes Receiz SDK/API rails to agents and names its authority boundary. | MCP must never replace, rename, or outrank proof objects, identity primitives, ownership surfaces, settlement primitives, public proof surfaces, or verification primitives. |
| Delegated agent token | Receiz OIDC can issue scoped, revocable, one-time-visible agent tokens for MCP, CI, and trusted local agents. | Do not treat an agent token, MCP host, env var, client secret, or OIDC registration as identity proof authority. |
| Developer webhook rail | Developer webhooks sign delivery events and expose test/rotate/manage flows beneath delegated scopes. | Do not let webhook delivery, endpoint rows, test events, or secrets become proof authority. |
| Public proof graph | `/proof-graph` is a public proof surface for inspecting Receiz proof/economy/flow/asset/agent state with live Kai pulse. | Do not turn the graph into decorative marketing or static simulation; it must stay rooted in release/API/product proof data. |
| Ledger and transaction proof surfaces | Wallet ledger, public ledger, and transaction routes project proof-bundle, Kai, value, transfer, and settlement material. | Do not downgrade settlement primitives into generic balances or let route/API rows outrank proof-bundle truth. |
| Market proof and settlement projection | Market-cap, asset, transaction-volume, and settlement rollup views project deterministic proof/economy state beneath first paint. | Do not make cold proof surfaces wait for market snapshot reconstruction, broad scans, or chart hydration before showing known truth. |
| Sports vault and arena | Sports card/vault/arena surfaces remain proof objects and settlement/ownership surfaces with deterministic first paint and bounded background work. | Do not move known scores, card media, close markers, rollups, or ownership projections behind idle/background rediscovery. |
| Pitch Command day-proof archive | Pitch Command day proofs are release-carried proof archive material with manifests and projections. | Do not treat day-proof archives as disposable static content or loosen strict proof-register typing. |
| Receiz AI skills | Repo-carried Receiz AI skills teach proof-native construction, commerce, distribution, verification, MCP, offline proof, and sports card workflows. | Do not let generated skill guidance soften implemented primitives into generic app categories. |

## Frozen Invariants

- Use known verified truth immediately.
- Ask only for verified additions after the known proof/Kai head.
- Durable proof memory remains first admission only, then append forever.
- Public proof surfaces, market surfaces, profile surfaces, ledger surfaces, transaction surfaces, and sports surfaces must preserve deterministic first paint.
- SDK, MCP, OIDC, webhooks, service worker, database rows, charts, and generated skill guidance are mechanics beneath primitives.
- Merchant proof authority can publish merchant-owned public-store state without delegated bearer-token authority.
- Delegated tokens authorize scoped agent/server writes below Receiz identity proof and must remain revocable/auditable.
- Webhook events must remain signed delivery mechanics, not proof authority.
- Public proof graph data must carry actual proof/release/API/product network state and live Kai pulse state.
- Ledger and transaction displays must preserve Kai/proof coordinates, settlement language, and proof-bundle inspection.
- Market-cap chart work must reduce computation beneath the primitive without replacing deterministic proof/economy truth.
- Sports arena rollup, worker, and image changes must reduce pressure without removing proof, ownership, settlement, closeout, or first-paint truth.
- Profile/market write paths own public projection freshness; read paths must not restore speed by rediscovering all profile truth before paint.
- Pitch Command archives must remain strict proof archive material with day-proof manifest/projection contracts.
- App, SDK, MCP, service-worker, offline verifier, and public release labels move together at `98.0.0`.

## Residual Risks

- Full `pnpm test:release-freeze` is the umbrella release gate and can be long-running; if skipped or blocked, the exact failing/skipped command must be named.
- npm publication for `@receiz/sdk@98.0.0` and `@receiz/mcp-server@98.0.0` is an external registry action after local gates pass.
- Production database migrations, Vercel health, and live webhook delivery are runtime evidence after deploy, but they do not become proof authority.
- Browser visual verification should be repeated for `/proof-graph`, `/transactions/[slug]/[code]/[pulse]`, `/developers/connect`, `/developers/webhooks`, Sports Arena, `/market`, and live player routes before production promotion because visual evidence overrides passing tests.
- Existing Playwright and Lighthouse artifacts in `output/` are evidence from the reviewed working set; fresh release-candidate screenshots still matter for the final shipped browser state.

## Release-Lock Requirement

`pnpm release:lock 98.0.0` must require this document for `v98.0.0` and later release cuts. A release that omits the product-truth freeze is incomplete because post-`v97.0.0` proof object authority, durable proof memory law, SDK/MCP convenience limits, delegated token limits, webhook boundaries, public proof graph behavior, ledger/transaction/settlement projection truth, sports vault/arena first-paint law, Pitch Command archive material, and app/SDK/MCP version lock cannot remain explicit release truth.
