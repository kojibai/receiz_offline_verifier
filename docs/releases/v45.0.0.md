# Receiz v45.0.0

Release date: March 28, 2026

## Overview

`v45.0.0` is the production consolidation release for everything shipped after `v40.0.0`. The release takes the first world-runtime wave and locks it into a production-grade public twin platform with stronger bootstrap and route continuity, richer photoreal stage rendering, stricter voice and playback behavior, deterministic trail and Twin Mind packaging, calendar bundle hardening, and broader trust/docs surface maturity.

This release keeps the core Receiz rule unchanged: the file remains the authority. Public twins, world routes, calendar bundles, rendered media, and public delivery surfaces add embodiment and transport around deterministic artifacts rather than replacing that truth boundary.

## What Defines v45.0.0

- Public Twin is now a release-governed production surface, not an experimental shell around backend routes.
- Photoreal stage rendering, mouth articulation, face alignment, and voice playback are treated as locked product behavior.
- Owned and internal twin runtime lanes are stronger, more operational, and better separated from public presentation.
- Calendar bundles, trail bundles, and Twin Mind transports are more deterministic and portable.
- Public trust, governance, legal, developer, onboarding, and payments surfaces now match the maturity of the runtime beneath them.

## Major Accomplishments Since v40.0.0

### 1. Public Twin Became a Real Product Surface

The `40.x` to `45.x` span pushed Public Twin from an emerging world route into a coherent product path:

- separate chat-stage and dock rendering paths
- stronger panel placement and viewport recovery
- persistent session state across panel, reply, and performance transitions
- deterministic bootstrap and route continuity behavior
- better rendered portrait handling on the white-panel stage
- stronger close, reopen, and preview lock behavior

The result is that the public twin no longer feels like a thin wrapper around world APIs. It behaves like a stable user-facing surface with continuity and embodiment rules that can be version-locked.

### 2. Photoreal Embodiment and Playback Tightened Materially

This release deeply improved the live-stage shell:

- portrait feature-map alignment and authoritative rendered-portrait overlays
- blink and Kai breath envelopes
- photoreal mouth depth, palate, gums, upper and lower teeth, wetline, and tongue anatomy
- viseme-driven tongue articulation that stays legible at small size
- stricter playback routing between baked video, audio assets, browser speech fallback, and pulse fallback
- stage-shell motion now locked to real voiced playback and held for the full audio window

This matters because the public twin is only believable if the stage behaves consistently under real media transport, not just in ideal demos.

### 3. Owned and Internal Twin Runtime Infrastructure Matured

The runtime stack continued expanding:

- self-hosted/internal TTS worker behavior
- stronger production twin backend routing
- local model-pack and latent bundle handling
- rerender quality gating
- approval-gated public promotion
- improved synthetic provider and provenance behavior

The net effect is a cleaner separation between how twins are generated, approved, transported, and publicly presented.

### 4. Deterministic Trail, Twin Mind, and Calendar Bundles Expanded

Receiz added more deterministic bundle surfaces around the world stack:

- world trail bundle packaging and download routes
- Twin Mind import/export transport hardening
- canonical calendar event bundling
- calendar payload sanitization and ordering guarantees
- ledger-backed calendar projection behavior

These changes preserve the same trust model while making the runtime outputs more exportable and auditable.

### 5. Public Trust and Platform Surfaces Reached Better Parity with the Runtime

This release also broadened the outward-facing shell:

- refreshed `world`, `public-twin`, `payments`, `status`, `trust`, `governance`, `security`, `standards`, `faq`, `start-here`, `developers/receiz-connect`, and legal surfaces
- stronger wallet, ledger, media, storage, and profile-delivery plumbing
- continued PWA and service-worker hardening
- improved OG/Twitter assets and public framing

Receiz now presents the trust and runtime model with a more coherent platform-level surface rather than a set of uneven subsystems.
