# Receiz v113.0.0 Performance Findings

`v113.0.0` adds production global reconciliation for the explicitly named `receiz.com/global/v1` coordination domain while preserving exact sealed artifact bytes, independently verified causal history, and deterministic first paint as stronger truth.

## Findings

- Known artifact truth paints before remote-domain construction, authentication, accepted-head resolution, staging, commit, effects polling, or reconciliation requests.
- `client.coordination.remoteDomain()` keeps plan, Identity Seal authorization, neutral stage, commit, accepted-head resolution, attempt recovery, and effect status as distinct phases.
- Connect authenticates the account and tenant once but cannot satisfy plan capability authority.
- Private immutable staging avoids repeated candidate mutation and preserves exact bytes without advancing the accepted head.
- Atomic expected-head acceptance prevents last-write-wins and performs no authority-bearing writes on failed boundaries.
- Scoped idempotency keeps retries inside one tenant, domain, resource, and operation identity; an indeterminate response preserves the original attempt rather than creating a second mutation.
- Monotonic resolution rejects returned ancestors and treats divergent siblings as structural divergence rather than auto-merge work.
- Every returned artifact is independently reverified before the SDK exposes runtime-custodied artifact truth.
- Effects and outbox delivery are observable after acceptance and cannot reverse accepted artifact truth.
- Registry protocol limits bound exact artifact bytes, archive expansion, verified-history nodes, reconciliation additions, predecessor fetches, namespaces, reducer operations, and law evaluation.
- The private migration adds coordination-only rows and object storage; applying it is deployment work and does not affect deterministic product first paint.
- Static verifier behavior is unchanged; only release identity and evidence advance in this repository.

## Performance Boundary

- “Global” means accepted by `receiz.com/global/v1`, not universal consensus.
- Browser/local admission and remote coordination stores remain distinct.
- Remote rows, storage objects, receipts, audit references, effects, outbox state, and Connect tokens do not become artifact, identity, ownership, history, or operation authority.
- This repository validates release surfaces and constitutional records; it does not apply or benchmark the private production migration.
