- Receiz remains a proof-native artifact system, not a generic app with proof features.
- Offline proof remains real proof. Compact `/v` transport remains subordinate to sealed proof bundle truth.
- Live Sports game state is a live proof surface when it projects locked entry, player score, field, reward, and append truth.

- Locked entries, individual card scores, finalized reward recovery, and live cockpit totals preserve stronger proof/local truth through first paint, hydration, and game switching.

- Public Live Cockpit and `/live` rooms are public witness/live proof surfaces, not generic chat UI.
- Market/profile first paint must preserve public proof objects and canonical profile truth before owner/session overlays append controls.
- Service-worker version, package version, public release labels, runtime coherence defaults, and deterministic session/profile projection keys move together.
- Explicit session clear/logout clears runtime-scoped legacy keys and cannot resurrect stale signed-in first paint.
- Supabase timeout hardening remains a DB policy/index/migration surface beneath product truth; production application is a deployment and privilege boundary.
- Regression lessons are release truth and must stay required by `release:lock`.

## Forbidden Downgrades

A change is out of bounds for `v88.0.0` if it:

- downgrades offline proof to fallback mode.
- downgrades proof object to card.
- downgrades live Sports proof truth to scoreboard display.
- downgrades locked entry or player score truth to hydration placeholder.
- treats `/baseball` as a forked generic route instead of a direct entry to the canonical baseball arena.
- replaces known detailed Sports field truth with partial fallback data.
- downgrades Public Live Cockpit or `/live` rooms to generic chat UI.
- downgrades market/profile first paint to session, SEO, analytics, or rediscovery state.
- downgrades deterministic first paint to loading state.
- downgrades verified history to activity feed.
- downgrades service-worker release truth to incidental cache behavior.
- lets external systems define Receiz truth instead of providing implementation mechanics.
- ships version drift between package metadata, release docs, README/release notes, static entrypoints, service-worker runtime, badges, or release markers.
