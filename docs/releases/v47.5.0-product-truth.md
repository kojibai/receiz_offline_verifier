# Receiz v47.5.0 Product Truth

Release date: April 9, 2026
Status: Locked for `v47.5.0` release prep

## Purpose

This document defines the patch-level product-truth freeze for Receiz `v47.5.0`. It preserves the current `v47.x` product truth while recording explicit revenue-skill policy, outbound opportunity lanes, operator playbooks, proof-first record kits, unified action-card embeds, booking confirmation packs, and post-call closeout rails as explicit non-regression behavior at release time.

For this offline verifier repository, the patch is represented as current-release documentation and version-surface alignment only. The verifier trust contract remains unchanged relative to `v47.0.0`.

## Core Authority Boundaries

The release does not relax the core Receiz authority model:

- The file remains the authority for artifact truth.
- Deterministic manifests remain the authority for historical route truth.
- World revenue skills, outbound lanes, operator playbooks, proof kits, embeds, confirmation packs, and closeout packs must run on top of the existing trust and payment primitives, not weaken or replace them.
- Owner policy remains the authority for saved world behavior, including revenue skills and outbound channel configuration.
- Public twins, operator surfaces, action rails, embeds, and release metadata remain transport and governance around those boundaries, not replacements for them.

## Locked Patch Surface

### 1. World Must Support Explicit Revenue Skills And Action Lanes

- World revenue skills must be stored as explicit owner policy, normalized server-side, and grounded into runtime prompt context.
- Deterministic world reply fallbacks must understand quote, deposit, proof, follow-up, renewal, and team-routing lanes.
- Public twin action rails must surface `quote`, `deposit`, `proof`, and `renew` alongside intake, routing, and handoff controls.

### 2. `/world` Must Remain A Revenue-Operations Control Surface

- The agent tab must expose a compact revenue-skills section with a recommended conversion stack.
- Operator playbooks must load post-call closeout, warm follow-up, and renewal drafts directly into the outreach composer.
- Playbooks must carry the best available next-step route using existing payment, scheduling, proof, and public-twin paths.

### 3. Outbound Channels Must Remain Policy-Governed

- Receiz email must send a real outbound email and open a secure Receiz reply thread.
- SMS must send through the configured relay and route the recipient back into the twin.
- App-webhook delivery must push structured outreach into Slack, Discord, CRM, Zapier, Make, or similar systems.
- All outbound channels must remain tied to saved world policy rather than unsaved local state.

### 4. Surrounding Business Surfaces Must Stay Productized

- Proof-kit landing pages must open preloaded record flows instead of blank generic forms.
- The unified action-card embed must bring ask, book, and pay into one deployable surface.
- Booking confirmation packs and call closeout packs must keep follow-through operational instead of conversational.
- The developer site kit must package login, verify, pay, and schedule primitives into a clear adoption surface.

### 5. Release Governance And Version Truth

- Release-scoped note and product-truth documents are required for the patch.
- Release-governed version surfaces must move together across package metadata, service worker defaults, public badges, offline verifier surfaces, and release documents.
- This repository records the public release documents and shipped HTML version markers for that lockstep requirement.

## Allowed Evolution

Implementation details may change after `v47.5.0` only if they preserve the authority boundaries and locked patch behaviors above. If a future change materially alters one of these truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v47.5.0` if it:

- removes or weakens explicit revenue-skill policy, runtime grounding, or deterministic revenue-lane fallbacks on governed world surfaces
- hides or bypasses `/world` revenue-operations controls or stops operator playbooks from loading governed follow-up drafts
- routes outbound email, SMS, or app-webhook outreach through unsaved local state instead of saved owner policy
- regresses proof-kit, action-card, booking confirmation, call closeout, or developer site kit surfaces back into generic or manual flows
- changes release truth without updating the versioned release docs and aligned version surfaces
