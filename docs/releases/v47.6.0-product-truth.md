# Receiz v47.6.0 Product Truth

Release date: April 9, 2026
Status: Locked for `v47.6.0` release prep

## Purpose

This document defines the patch-level product-truth freeze for Receiz `v47.6.0`. It preserves the current `v47.x` product truth while recording the atlas-first world surface, concierge twin-run operations, standalone contacts CRM, truthful outreach persistence, and checkout completion verification as explicit non-regression behavior at release time.

For this offline verifier repository, the patch is represented as current-release documentation and version-surface alignment only. The verifier trust contract remains unchanged relative to `v47.0.0`.

## Core Authority Boundaries

The release does not relax the core Receiz authority model:

- The file remains the authority for artifact truth.
- Deterministic manifests remain the authority for historical route truth.
- The atlas shell, concierge task cards, contacts timelines, outreach integrity, and checkout success reporting must run on top of the existing trust and payment primitives, not weaken or replace them.
- Saved operator and account state remain the authority for world policy, contacts memory, outreach artifacts, and payment completion state.
- Public twins, operator surfaces, embedded checkout shells, and release metadata remain transport and governance around those boundaries, not replacements for them.

## Locked Patch Surface

### 1. `/world` Must Remain Map-First And Atlas-Grounded

- The district workspace must open as a richer atlas with landmass, route-pressure, and district focus rails.
- District selection must stay tied to one live world map instead of splitting attention between disconnected cards and detail panes.
- The cinematic atlas shell must remain additive to the existing world runtime rather than replacing the underlying world model.

### 2. Concierge Must Quarterback Structured Twin Work

- Detailed prompts must be able to start a structured twin run instead of only returning a static answer.
- Live task cards must expose intent, stages, logs, next unlocks, and recent progress inside the transcript shell.
- Active task state must feed back into concierge context so follow-up prompts stay grounded in the run already in motion.

### 3. Contacts Must Remain A First-Class Operating Surface

- Account must link into `/contacts` as a standalone CRM-style surface.
- Contacts import and export must support CSV, JSON, and vCard, plus browser-native contact picking where the Contacts Picker API is available.
- Notes, stage filters, last-meeting fields, next steps, invite history, and person-first timelines must remain available in one operating surface.

### 4. Outbound And Checkout State Must Fail Truthfully

- App-webhook outreach must reject localhost, private-network, link-local, and other non-public destinations before any server-side send.
- Email outreach must verify delivery readiness first and may only create chat persistence after the email send succeeds.
- Embedded checkout completion-only flows must verify Stripe session state before emitting success to the parent surface.

### 5. Release Governance And Version Truth

- Release-scoped note and product-truth documents are required for the patch.
- Release-governed version surfaces must move together across package metadata, service worker defaults, public badges, offline verifier surfaces, and release documents.
- This repository records the public release documents and shipped HTML version markers for that lockstep requirement.

## Allowed Evolution

Implementation details may change after `v47.6.0` only if they preserve the authority boundaries and locked patch behaviors above. If a future change materially alters one of these truths, it should ship under a new explicit release note and updated product-truth freeze.

## Non-Regression Rule

A change is out of bounds for `v47.6.0` if it:

- regresses `/world` from a single live atlas shell back into split or disconnected district surfaces
- strips concierge of structured twin-run launch or in-transcript progress visibility
- pushes contacts memory, notes, invite history, or next-step state back into scattered secondary surfaces
- allows private/internal app-webhook destinations, persists email-thread artifacts before delivery succeeds, or reports checkout success before Stripe session verification
- changes release truth without updating the versioned release docs and aligned version surfaces
