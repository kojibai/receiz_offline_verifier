# Receiz v54.0.0

Release date: April 15, 2026

## Overview

`v54.0.0` is the release that picks up from the first `v53.0.0` release-prep commit (`9b92586ed589220cc455a3438474ef48458a80c0`) and broadens that public-market baseline into a more complete operating system.

Since the first `v53.0.0` release commit, the repository has expanded the public product map, introduced a clearer Business surface, reshaped admin into an operational command deck, tightened Stripe-to-ledger transparency, pushed signed-in routes toward shared session bootstrap instead of cold identity refetches, made profile and asset value surfaces deterministic before live hydration, persisted world-twin continuity across the profile/artifact/live-player orbit, and eliminated a real SVG hydration mismatch inside the Kai sigil itself.

The core rules still do not change. The file remains authoritative for verification. Deterministic route state remains authoritative for first paint. Ledger-backed settlement remains authoritative for money state. Live deltas hydrate on top of that truth instead of replacing it.

The product-truth freeze for this release is defined in `docs/releases/v54.0.0-product-truth.md`.

## What Defines v54.0.0

- Admin is no longer a long scroll report. It is a command deck with one-view access to payments, users, wallet operations, growth, integrity, and backfill operations.
- Stripe-funded wallet deposits, payment-embed flows, and profile purchase flows now route into transparent wallet-ledger behavior with global ledger visibility and admin backfill controls.
- Signed-in workspace routes now prefer one shared session bootstrap graph over mount-time identity probing across account, wallet, calendar, contacts, upgrade, plans, profile entry, embed scheduler, and adjacent onboarding surfaces.
- Profile, artifact viewer, live player, and market now share deterministic market-preview state so price pills render immediately and hydrate live afterward instead of waiting for a click.
- World twin continuity is now carried across profile, artifact viewer, and live player with session reuse and cached twin runtime loading instead of replaying the twin load sequence on every handoff.
- Public product explanation is broader and cleaner: Business, the full outcomes map, the "what can you do" ladder, a simpler "what is a receiz," and a clearer start-here reference system now describe the product in plain language.
- Kai sigil SVG output is now serialization-stable across server and client so deterministic sigils do not produce hydration mismatches from tiny floating-point drift.
- Release-governed version surfaces now move together to `v54.0.0`.

## Since The First v53.0.0 Release Commit

### 1. The Public Product Map Became Much Broader

The first `v53.0.0` release centered the public ladder on Verify, Seal, Record, and Public Twin. `v54.0.0` keeps that beginner-first direction, but fills in the rest of the map.

The current repository state adds or materially expands:

- `/business` as a dedicated business-front-door narrative instead of a buried upgrade concept
- `/everything-receiz-lets-you-do` as a full outcomes map
- `/what-can-you-do-with-a-receiz` as a cleaner ladder from proof to access to value
- `/what-is-a-receiz`, `/start-here`, `/public-twin`, and `/receiz-world` as a more coherent plain-English system
- updated `/developers`, `/developers/receiz-connect`, `/faq`, `/powered-by`, and related public surfaces
- claim-ownership narrative material and generated system-brief collateral

This matters because the public story is no longer trying to compress a large system into one or two gateway pages. The system is now explained in layers without throwing away the plain-English entry point.

### 2. Admin Became An Operational Workspace Instead Of A Report

Admin is no longer one very long page that forces scrolling across unrelated control areas.

The current release state introduces:

- a docked admin command deck shell
- one-view access to payments, users, wallet operations, growth, connect, performance, and integrity panels
- action-center style controls for backfill, compaction, rollup drain, and snapshot refresh
- per-user Stripe USD lookups and clearer payment-source visibility

This is not just a layout improvement. It changes admin from an audit page into an operating surface.

### 3. Money Operations Are More Transparent

The repo now carries a stricter wallet and settlement model for admin and Stripe-backed flows.

The important change is that balance adjustment is no longer treated like raw mutable table state. Admin adjustments are modeled as explicit ledger events, and Stripe-funded flows are expected to land in the wallet/ledger path in a way that can be replayed and audited.

The current release-prep working tree carries:

- admin wallet adjustment behavior that records ledger entries instead of silently mutating balances
- clearer Stripe-funded wallet-credit rollup views
- admin visibility into who deposited what in USD via Stripe
- explicit backfill controls for Stripe-derived wallet/ledger state

This keeps money truth aligned with the rest of the product rule: state should be explainable after the fact, not merely visible in the moment.

### 4. Signed-In Workspaces Now Reuse One Session Bootstrap Graph

One of the biggest practical changes since the first `v53.0.0` release commit is architectural, not cosmetic.

A large set of signed-in routes now prefer a shared session/bootstrap snapshot before they reconcile live state:

- account
- wallet
- calendar
- contacts
- upgrade
- plans
- make
- seal
- affiliate
- profile entry
- embed scheduler
- payments builder
- analytics and related signed-in utility surfaces

The goal is simple: if the app already knows who the user is and what their baseline state is, route entry should render from that deterministic graph immediately and reconcile later, instead of restarting with `/api/session` and similar bootstrap probes on every mount.

### 5. The Asset Journey Is Closer To Deterministic First Paint

The profile -> artifact -> live-player -> market orbit is now much closer to the product rule you have been pushing for.

This release-prep state adds:

- deterministic `marketPreview` state onto profile originals
- profile, artifact viewer, and live player wiring that passes that preview through instead of re-fetching before paint
- route/viewer/live-player hydration guards that reject stale snapshots with missing deterministic market preview
- price pills that show deterministic value immediately and then background-hydrate live market state after the page settles

The important behavior change is that the price pill no longer waits for a click to "wake up" the whole market graph. The route can render the deterministic value first and then reconcile live value without blocking the page.

### 6. World Twin Continuity Is Now A Real Runtime Contract

The first `v53.0.0` release already had public twin continuity rules. `v54.0.0` pushes those rules into the profile/artifact/live-player orbit more concretely.

The current release-prep working tree carries:

- world twin session reuse that keeps the deterministic world graph across profile, artifact viewer, and live player transitions
- deferred twin loading that reuses the already loaded rig/module runtime across route remounts
- artifact routes moved onto the same deferred twin path as profile/live player instead of bypassing continuity logic
- removal of the square render-image fallback leak on the dock before the 3D rigged twin is ready

This matters because "the twin loaded already" now means something operationally. It is no longer only a data reuse story; it is also a runtime reuse story.

### 7. Interaction Discipline And Hydration Stability Improved

Several changes since the first `v53.0.0` release commit are about not doing unnecessary work and not letting UI details drift into instability:

- haptics policy/runtime support was added and integrated into more interaction points
- touch-safe interaction behavior continued to improve around market and profile controls
- the old bottom-bar swipe experiment path was removed rather than left half-active
- the Kai sigil geometry builders now quantize their SVG output so server and client emit the exact same path data

That last change fixes a real hydration mismatch, not by suppressing React warnings, but by making the sigil renderer deterministic at the string-serialization layer.

## Public Summary

Receiz `v54.0.0` extends the first `v53.0.0` public-market release into a broader operating release: a clearer public product map, a real Business surface, a command-deck admin, transparent Stripe-to-ledger admin controls, graph-first signed-in continuity, deterministic market-preview pricing on profile and player surfaces, persistent twin continuity across the asset journey, and deterministic sigil rendering that no longer drifts at hydration time.
