# Receiz v48.0.0

Release date: April 10, 2026

## Overview

`v48.0.0` is the operator-surface release for Receiz contacts and world. It upgrades contacts from a dense record viewer into a one-view operator cockpit, keeps linked-account import upgrades durable, and turns world into a cleaner mobile atlas with authoritative naming and separated signal layers.

For this offline verifier repository specifically, `v48.0.0` is a release-surface alignment release. The operator-first contacts shell, linked-account merge persistence, and mobile world atlas cleanup live in the broader Receiz application; this repository carries the canonical release note, product-truth freeze, and visible version-surface bump so the public release artifacts stay locked together.

The product-truth freeze for this release is defined in `docs/releases/v48.0.0-product-truth.md`.

## What Defines v48.0.0

- Contacts now default to an operator-first shell with best-move, continuity, latest-touch, queue-style prioritization, and compact mobile overlays instead of a long stacked record.
- Mobile contacts still preserve full access to overview, next-step, continuity context, recent activity, edit, invite, log, schedule, and full history.
- Imported contact merges still persist a `linked_user_id` upgrade so authenticated identity upgrades do not silently fail to write when the rest of the contact payload is unchanged.
- World now presents an authoritative atlas surface on mobile, with cleaner HUD layers, non-overlapping signal presentation, and a differentiated return-signal panel instead of redundant lower cards.
- Release-governed version surfaces move together to `v48.0.0` across package metadata, service worker defaults, public badges, offline verifier surfaces, and release docs.

## Targeted Release Outcomes

### 1. Contacts Behave Like An Operator Cockpit

The initial `/contacts` mobile frame should feel like an operator console, not a vertical stack:

- the main viewport keeps the command header, dock, and active relationship shell visible
- the roster is ranked by what needs action instead of just recent contact memory
- dense sections move into overlays instead of forcing immediate scroll

### 2. World Atlas Reads Cleanly On Mobile

The initial `/world` mobile frame should read as a durable atlas instead of an overloaded demo panel:

- atlas naming stays authoritative across the surface
- the map itself remains readable without stacked pills and repeated floating labels fighting for space
- lower panels do different work: atlas focus explains the district and return signal explains why to come back

### 3. Continuity Upgrades Persist Cleanly

Import-driven enrichment should not stop at guest or direct identity state:

- a merged contact now persists `linked_user_id` upgrades
- existing contact rows can upgrade into authenticated continuity without requiring some unrelated field change
- the merge contract remains covered by a source-level contacts release test
