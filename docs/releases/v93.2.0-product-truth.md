`v93.2.0` preserves Receiz as a proof-native artifact system and expands the public developer SDK so third-party apps can implement Receiz proof objects with fewer mistakes.

## Frozen Primitives

| Primitive | v93.2.0 truth | Must not regress |
| --- | --- | --- |
| Proof object | Asset manifests project deterministic identity, media, owner, proof, appends, settlement, and verification rows. | SDK helpers must not replace proof verification or sealed artifact truth. |
| Sports card proof object | Sports card manifests project collectible identity, claim hash, ownership, value basis, append summary, and event proof summary together. | Apps must not split card identity from its claim, ownership, value, or proof append context. |
| Durable proof memory | SDK proof registers admit first truth once, preserve it, and append verified additions. Duplicate transport fills missing fields only. | Newer partial payloads must not drop known admitted proof truth. |
| Public developer surface | `@receiz/sdk` exposes schemas, projections, clients, webhook helpers, and proof memory utilities. | Developer convenience must remain lower authority than sealed artifacts and verified appends. |

## Frozen Invariants

- The SDK is convenience, not authority.
- Runtime validators reject malformed manifests and webhook events.
- Schema exports are developer tooling, not a replacement for proof verification.
- Projections are deterministic display models from validated Receiz objects.
- The proof register is append-only memory for admitted truth, not newest-response-wins cache.
- Same-ID duplicate admission must preserve the first admitted payload and only add missing fields.
- Developers should render known proof truth immediately and ask for verified additions after the known head.

## Residual Risks

- Live npm publication is an external registry action and must publish `@receiz/sdk@93.2.0`, not overwrite `93.1.0`.
- SDK helpers do not remove the need to verify artifact bytes when bytes arrive from untrusted transport.
- The SDK exposes portable in-memory proof register primitives. Host apps still choose their durable storage layer.
