# Receiz v47.3.0

Release date: April 9, 2026

## Overview

`v47.3.0` is the custom-domain continuity and sign-in experience release. It keeps the current `v47.x` product truth intact while making business custom domains behave like Receiz-authenticated surfaces instead of isolated public mirrors, and it removes the dead-end passkey experience inside in-app browsers by opening the email path immediately with accurate guidance.

For this offline verifier repository specifically, `v47.3.0` is a release-surface alignment patch. The custom-domain auth bridge, embedded-browser sign-in fallback, and trusted-host hero-media handling live in the broader Receiz application; this repository carries the current release note, product-truth freeze, and visible version-surface bump so the public release artifacts stay locked together.

The product-truth freeze for this release is defined in `docs/releases/v47.3.0-product-truth.md`.

## What Defines v47.3.0

- Verified custom domains can reuse an existing Receiz session, mint a local domain session, and land the user back on the business host without reworking the canonical Receiz auth core.
- Custom-domain signing flows keep the canonical Receiz WebAuthn / PBI authority while feeling native on the business domain through the bridge flow.
- Sign-in surfaces detect likely embedded browsers such as Instagram, X/Twitter, Facebook, LINE, TikTok, and generic webviews, then open email fallback immediately instead of presenting passkey failure as a product error.
- First-load profile hero media on custom domains resolves with the same trusted-host treatment as the main Receiz domain.
- Release-governed version surfaces stay locked together across package metadata, service worker defaults, public badges, and release documents.

## Targeted Release Outcomes

### 1. Custom Domains Behave Like Receiz-Authenticated Surfaces

The main additive change is the new auth bridge:

- a signed-in Receiz user can arrive on a verified custom domain and reuse that existing Receiz authority
- the custom domain mints its own local session without weakening the canonical Receiz identity, passkey, or signing model
- logout and signing flows remain symmetric across the business host and `receiz.com`

### 2. In-App Browsers Stop Dead-Ending Passkey Sign-In

Receiz now handles embedded browser reality directly:

- sign-in popovers and pages detect likely in-app browsers before prompting for passkeys
- email fallback expands automatically instead of waiting for a WebAuthn failure
- the UI tells the truth: continue with email here, or open the page in Safari, Chrome, or the default browser to use passkeys

### 3. Custom-Domain First Paint Matches Receiz More Closely

The profile route on a custom domain now trusts its own branded host for proxied hero media fetches, which prevents the first avatar / header render from intermittently falling into a broken state until reload.

### 4. Release Truth Moves With The Patch

This release also rolls the visible release surfaces to `v47.3.0`, so the live app, offline surfaces, and release docs present one consistent version truth.

In this repository, that alignment shows up as updated release documentation plus the shipped version markers in the published verifier, studio, and settlement entrypoints.
