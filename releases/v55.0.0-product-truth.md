# Receiz v55.0.0 Product Truth

Release date: April 15, 2026

This document freezes the primitives and invariants that `v55.0.0` is allowed to extend but not weaken after the broader `v54.0.0` operating baseline.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Guided owner setup stays canonical | `/profile` must remain the guided owner setup surface, including sticky completion, inline username claim, fast-entry hydration, and the five-step activation rail. | `app/profile/page.tsx`, `app/profile/ProfileEntryClient.tsx`, `app/u/[username]/ProfileClient.tsx`, `app/lib/profile/workspaceHydration.ts`, `app/lib/profile/__tests__/profileActivationStickyContract.test.ts` |
| Profile media is sealed product state | Avatar and cover uploads must continue being processed, stored, and minted as Receiz-backed originals instead of only mutating profile rows. | `app/api/profile/me/media/route.ts`, `app/lib/profile/mediaMint.ts`, `app/lib/profile/proofManifest.ts`, `app/lib/profile/__tests__/proofManifest.test.ts` |
| World avatar freshness outranks stale preview state | The freshest saved avatar render must win across world/profile/public-twin handoffs, with preview locks and refresh fanout preserving that winner across tabs and routes. | `app/world/WorldExperience.tsx`, `app/lib/world/server.ts`, `app/lib/world/publicTwinPreview.ts`, `app/lib/world/publicTwinPreviewLock.ts`, `app/lib/world/__tests__/worldAvatarFreshnessContract.test.ts` |
| World hydration remains budget-aware | Desktop-class environments may background-hydrate the heavier twin module, but compact touch devices and low-memory touch devices must stay on on-demand activation. | `app/u/[username]/worldTwinBootstrap.ts`, `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/__tests__/worldTwinBackgroundHydrationContract.test.ts`, `app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts` |
| Stable route continuity spans market, profile, and player | Internal market exits, profile bottom-bar entry, and live-player handoff must keep using shared stable navigation and deterministic route seeds instead of reintroducing bespoke reload paths. | `app/components/layout/GlobalAuthChrome.tsx`, `app/lib/navigation/stableRouteNavigation.ts`, `app/market/MarketDeskExperience.tsx`, `app/lib/market/__tests__/marketRouteHydrationContract.test.ts`, `app/market/__tests__/marketNavigationContract.test.ts` |

| Prediction hydration remains bounded | Large market snapshots must continue loading prediction-book state in bounded batches instead of a single oversized query. | `app/lib/market/predictions.server.ts`, `app/lib/market/__tests__/predictions.server.test.ts` |
| Release-governed version surfaces | Package version, service worker version, public badges, offline verifier surfaces, and release docs must move together. | `scripts/check_release_lock.mjs` |

## Residual Risks

- Fast-entry profile hydration is much better, but it still depends on short-lived cookie and session-storage hints staying coherent across tabs and auth transitions.
- Avatar preview locking reduces stale face surfaces, but multi-tab timing can still briefly diverge until the storage or broadcast signal lands.
- Compact-device hydration gating protects mobile performance, but it also means some borderline tablets may feel slower to reach the fully loaded twin until explicit user intent.
- Profile media sealing is stronger, but it depends on storage availability and a configured signing key at upload time.
- Canonical route grouping improves performance analytics, but historical accuracy still depends on backfilling older rows that were written before the new route-group rules.
