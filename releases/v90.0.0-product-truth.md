## Frozen Invariants

- Receiz is a proof-native artifact system.
- Implemented primitives must be named directly.
- Stronger truth beats newer weaker transport.
- Known verified truth must paint immediately.
- Deterministic trust surfaces must not use fake loading theater.
- Offline proof is deterministic projection from verified truth, not fallback mode.
- Foreground proof publication is not a background outbox action.
- `/api/receiz/anchor` remains excluded from local signed-action outbox admission.
- A profile post is not globally complete because a DB row exists.
- Profile writes must refresh canonical profile snapshot truth, public profile market append source truth, and visible route snapshots.
- Sports Vault sealed manifest/image truth remains above server showcase, DB row, session, browser payload, and UI state.
- Sports single-game entry admission must verify the game and owned Sports Vault proof cards before rejecting stale client lineup transport.
- Sports ownership transfer must reuse the existing offline-transfer ownership append boundary before local custody projection.
- Sports event proof pages and metadata project proof truth; metadata does not define proof truth.
- Live value and market-depth surfaces may project settlement/value context beneath card and event proof truth.
- Competition rooms must remain proof-native coordination surfaces when tied to Sports, market, settlement, or ownership state.
- Release docs must preserve the full v80-to-current ledger for v90.
- Public release labels, offline verifier labels, service-worker version, runtime coherence defaults, and package version must all carry `90.0.0`.
