# Receiz v99.0.0 Regression Lessons

`@receiz/sdk` remains mechanics beneath proof objects. Durable proof memory remains **first admission only, then append forever**.

## What Regressed

- Release identity split across `98.0.0`, `98.0.1`, `98.0.2`, and `98.0.3` surfaces without a complete lock record.
- Service-worker contract assertions froze obsolete app-version and cache-epoch literals.
- Market/profile paths needed stronger guards against tombstoned or partial membership replacing held public proof truth.
- PBI signature history was briefly handled through a path that was reverted before permanent append preservation was restored.
- Identity publication and recovery paths risked letting cookie/session timing redefine locally admitted proof.
- Explore and live player risked rediscovering public truth rather than starting from deterministic/held proof prefixes.
- Market Chronos labels implicitly selected UTC on the server and device timezone in the browser, producing contradictory text on hydration even though the visible Market logic was otherwise correct.
- Explore forced a static document beneath request-specific CSP nonces, so WebKit blocked bootstrap scripts and left the two-card feed unhydrated; its live-player handoff also computed queue completeness and then discarded it.
- Image proof URLs were present in the deterministic bundle, but images loaded before first service-worker control were never admitted as durable proof media; the generic URL warmer also misclassified query-addressed storage APIs as document routes. Audio/video usually arrived after control through playback, masking the asymmetry.

## Why It Regressed

- Patch version bumps were used as runtime deployment mechanics without completing the repo’s release identity lock.
- A behavior-specific service-worker test encoded historical version state rather than package alignment and proof-persistence law.
- Mutable feed/cache assumptions treated newer transport as stronger than a complete admitted register.
- Session issuance, publication, and proof admission were allowed to blur into one another.
- Profile/public projection obligations were not always enforced at the write path, inviting read-path rediscovery.

## Locked Rules

- App, `@receiz/sdk`, MCP, runtime, service worker, public labels, verifier labels, and release records move together for every full release.
- `CACHE_EPOCH` is deployment mechanics. It can invalidate runtime assets but cannot redefine or erase durable proof memory.
- Complete whole-Market proof truth outranks partial, older, malformed, archive, or pending membership payloads.
- Removal requires a verified current-membership tombstone or governed public-proof revocation.
- Local PBI authorship is verified locally, retained without caps/eviction, and published as an idempotent append.
- Receiz Key portable state carries the complete verified prefix available at export and cannot be truncated by weaker projections.
- Local identity proof projects immediately; publication, bind, import, cookie, and global sync are additive follow-up work.
- Deterministic live player and public profile truth paint before session/API/private enrichment.
- Kai moment derives from sealed Kai/pulse truth, not Chronos transport state.
- Server-rendered Chronos labels on deterministic Market first paint use an explicit timezone; device-local formatting cannot redefine the settled shell during hydration.
- Explore first paint remains deterministic and repo-local, but the document is request-rendered so enforced CSP nonces bind to its bootstrap scripts; computed queue completeness must be carried into the player handoff.
- Loaded showcase images on the first uncontrolled page are re-admitted only through the existing public-proof-media response checks; attachment, private, no-store, authorization-varying, and non-media responses remain forbidden.
