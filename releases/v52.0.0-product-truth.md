# Receiz v52.0.0 Product Truth

Release date: April 13, 2026

This document freezes the primitives and invariants that `v52.0.0` is allowed to extend but not weaken.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| File-authoritative trust core | The file remains the authority. Deterministic manifests remain the authority for route truth, and ledger-backed settlement remains the authority for payment truth. Market execution operates around those boundaries rather than replacing them. | `app/lib/profile/document.ts`, `app/lib/profile/server.ts`, `app/lib/receiptView.ts`, `app/lib/wallet/payIntent.ts`, `app/api/wallet/ledger/public/route.ts` |
| Stable public ladder from `v51.0.0` | The public product ladder remains `Free -> Authorship -> Business -> Automation -> Enterprise`. Market may deepen the system, but it must not reintroduce public plan confusion. | `app/upgrade/page.tsx`, `app/plans/PlansClient.tsx`, `app/lib/premium/plan.ts` |
| Deterministic market quote engine | Asset execution must resolve through one canonical quote model that computes reference mid, quote mid, bid, ask, spread, and shift from deterministic inputs. | `app/lib/market/shareQuoteModel.ts`, `app/lib/market/shareTrade.server.ts`, `app/lib/market/shareFees.ts` |
| Spread as quote behavior, not additive UI fee | Market sustainability is achieved through the live quote surface and bounded spread math, not a separate visible runtime fee stacked on top of execution. | `app/lib/market/shareTrade.server.ts`, `app/lib/market/shareFees.ts`, `supabase/migrations/20260413153000_market_share_trade_fees.sql` |
| Replayable trade quote evidence | Every market trade must remain explainable after the fact through persisted quote snapshot fields and pressure inputs. | `app/lib/market/shareTrade.server.ts`, `supabase/migrations/20260413174500_market_quote_engine_snapshot.sql` |

| Dedicated market conformance | Market must remain governed by its own conformance standard instead of being treated as a hidden side effect of settlement. | `app/api/market/conformance/route.ts`, `app/api/market/conformance/badge/route.ts`, `app/market/conformance/page.tsx`, `scripts/test_market_conformance.ts`, `scripts/create_release_attestation.ts` |

| Release-governed version surfaces | Package version, service worker version, public badges, offline verifier surfaces, and release docs must move together. | `scripts/check_release_lock.mjs` |

## Release Gates

`v52.0.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `pnpm test:market-conformance`
- `pnpm test:release-freeze`
- `pnpm release:lock`
- `pnpm run build`
- `pnpm release:attest`
- `pnpm release:verify-attestation <path>`

The manual release procedure is defined in `docs/releases/v52.0.0-checklist.md`.

## Residual Risks

- The current market conformance history and rollups surfaces are live wrappers rather than a full persisted historical warehouse. They verify the model but do not yet represent a long-term archival evidence system.
- The lightweight market pill path is materially better than the previous full-snapshot dependency, but dense feeds with many live assets may still benefit from future batching or parent-provided pill hydration.
- The countdown path is now isolated, but any future component that imports the combined ticker hook unnecessarily can reintroduce avoidable rerenders if the split is not respected.
- Spread sustainability is materially stronger, but quote math alone is not a full solvency guarantee without operators respecting the hard-cap and exposure model already embedded in the runtime.
