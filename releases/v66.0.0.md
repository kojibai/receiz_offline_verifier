## Title

Receiz `v66.0.0` - Local Proof Continuity

## Summary

`v66.0.0` is the release that moves the post-`v64.0.0` system from value-loop completeness into fuller obedience to Receiz's actual authority model:

> Identity is local. Proof is local. State is deterministic. History is append-only. Memory is compact. Sync is propagation.

The server, database, and network remain important for discovery, publication, global propagation, appends, and online enrichment. They are no longer allowed to delay or downgrade stronger verified truth already held by the node.

## What Changed Since v64.0.0

### Local proof became the default authority

- Receipt rendering now classifies proof authority instead of treating all evidence as equal.
- Verified-register-backed proof outranks legacy offline evidence.
- Receipt details and proof popovers expose stronger local authority without letting older payloads reframe it.
- Wallet transaction receipts now verify deterministic ledger proof bundles before treating ledger claims as truth.

### Offline identity and account continuity moved forward

- Account surfaces preserve a valid bootstrap/keyfile/PBI-backed local identity when the session API is unavailable or weaker.
- Anonymous or missing network session payloads no longer erase locally held identity proof.
- The account route now treats network session as enrichment over a local identity root, not permission to exist.
- Brand-new local Receiz ID creation writes a local PBI enrollment block, projects the signed-in shell and profile seed immediately, and then publishes/binds to the global register as a best-effort verified append when online.

### Public twin gained a real local rail

- The public twin can project from the local twin mind register and local model path before the network stream is needed.
- Local twin register projection now wins for the visible reply. The online stream path remains available when no local register projection exists, but it no longer silently replaces a settled local projection.
- Browser local model support is integrated through the WebLLM rail with deterministic fallback when the model is unavailable.
- Twin mind PNG import now requires portable public-key verification instead of server-secret HMAC authority for canonical import.

### World, Explore, Wallet, and Market honor known state more strictly

- World profile/public/me API snapshots opt into service-worker offline persistence for deterministic or private route truth.
- Explore route snapshots are durable verified local truth and reconcile into the visible view without replacing useful already-visible content.
- Wallet now projects the durable wallet graph during first render instead of applying it in a later effect.
- Market snapshot continuity preserves known local positions and refuses lagging payloads that temporarily drop a buy or sell already known locally.
- Tracked positions now follow the same rule: a transport payload must prove a stronger state before it can remove visible local position truth.
- Signed-in Market first paint now carries active share and prediction positions in the entry transport; ownership is not delayed until the background market snapshot.

### The service worker became a verified-snapshot carrier

- Offline persistence is explicit through `x-receiz-offline-persist` response headers.
- Deterministic public snapshots and private account snapshots are stored as local truth carriers, not generic cache guesses.
- The app-shell fallback remains a resilience path; route truth is still carried by verified snapshots and deterministic payloads.

### Product language and release surfaces moved together

- Public version labels, offline verifier artifacts, package metadata, and service-worker defaults now move together to `v66.0.0`.
- The release docs now freeze the offline primitive as release truth, not just as engineering preference.

## What This Release Does Not Replace

- The existing online twin brain stays intact.
- The server runtime stays intact.
- The database stays useful for global register propagation, discovery, sync, and verified additions.
- Online AI stays useful when it can add capability, but it is not allowed to make known local truth provisional.
- Local PBI account creation now exists as local truth first; richer recovery, cross-device onboarding polish, and global publication UX remain later extensions.

## Production Requirements

Twin mind portable import/export requires the public-key signing environment to be present in production:

- `RECEIZ_WORLD_TWIN_MIND_SIGNING_PRIVATE_KEY_PEM`
- `RECEIZ_WORLD_TWIN_MIND_SIGNING_PUBLIC_KEY_PEM`
- `RECEIZ_WORLD_TWIN_MIND_KEY_ID`

No new database migration is required by the release docs themselves. Existing production schema requirements from the value-loop and world/profile releases still apply.

## Release Principle

Known local truth should move immediately.

Everything else must justify its existence.

If a future change asks the network to reconfirm a proof the node already holds, replaces a stronger local block with a weaker remote payload, or makes a settled screen visibly reload from background hydration, it is a regression against `v66.0.0`.
