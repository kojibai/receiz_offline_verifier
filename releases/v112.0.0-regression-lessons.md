# Receiz v112.0.0 Regression Lessons

Public developer surface: `@receiz/sdk` v112.0.0. Durable proof memory remains **first admission only, then append forever**.

## What Regressed

Pre-v112 MCP helper names remained in several human AI instructions after the machine manifests had been coordinated. The first clean-room package contract also treated public release-path constants as though they performed filesystem reads.

## Why It Regressed

Manifest parity and human instruction parity were initially checked by different inventories, while the package graph assertion matched a path string instead of an actual filesystem operation.

## Locked Rules

- Admission is eligibility, not operation authorization. Verified capability and atomic commitment remain separate required boundaries.
- Portable transition identity excludes idempotency and external effects. The plan, not the transition digest, binds those coordination facts.
- Effects must originate from registry-authorized operation law. Caller-supplied effects cannot expand authority.
- A receipt reports an actor; it must not expose a runtime-custodied verified actor type that can be fed back into authority APIs.
- The committer must independently resolve staged bytes. Trusting a staged reference, plan, or receipt would let weaker state replace artifact truth.
- Same-domain idempotency conflict and cross-domain mismatch are distinct failures.
- Durable local admission truth is not cache, but it also is not a proof-object store. Keeping proof bytes outside IndexedDB protects both the stronger truth hierarchy and known-artifact first paint.
- Unknown namespace preservation is part of object continuity. Reducers may interpret known namespaces but cannot erase unrecognized history.
- v111 exact-byte compatibility does not imply v111 runtime authority.
- The executable registry can be locked deterministically by canonical JSON, digest, predecessor chain, and generated package identity. v112 does not require or claim a native `.receiz` or signed wrapper.

No new product HTTP route or database migration was needed. Visual release identifiers may advance without altering hierarchy, media, hydration, routing, or service-worker behavior.
