## Frozen Primitives

| Primitive | v92.1.0 truth | Must not regress |
| --- | --- | --- |
| Receiz product identity | Receiz remains a proof-native artifact system. | Do not describe this release as generic sports UI work. |
| Deterministic first paint | Known event, pitch, play, card, and archived proof truth paints before weaker hydration. | Do not make known proof wait for DB refresh, video lookup, session state, or full-slate rediscovery. |
| Sports event proof | Event pages and card popovers project the same event proof object, including video and milestone truth when carried. | Do not let a public event page show less proof than the logged-in card popover when the event proof carries the media. |
| Play Command proof object | Play Command carries event identity, official outcome, play telemetry, geometry, and end-to-end field visualization. | Do not reduce Play Command to a dense metric panel or generic popover. The field of play is primary. |
| Play Command public proof surface | `/play` is the standalone Play Command route, matching the `/pitch` standalone pattern for proof navigation. | Do not keep play proof accessible only as a nested card UI. |
| Pitch Command proof object | Pitch Command opens exact pitch witnesses from event proof pills, including historical source dates. | Do not open the current live pitch when a historical event's pitch witness is available. |
| Event proof media | Playable video attaches beneath event proof truth and must not block the fast event shell. | Do not make video availability a prerequisite for event proof first paint. |
| Milestone proof | Cycle completion and other event-level milestones project from verified proof/game truth. | Do not miss milestone events because the event renderer only displays the base play result. |
| Sports live append truth | Active signed-in Sports Arena foreground views warm appends at visible cadence. | Do not hide live appends behind a stale background guard while the user is actively viewing the card. |
| Pack Derby proof game | Derby remains settled gameplay with stable opening, film-backed field view, scoring, and proof rewards. | Do not let stabilization work force-close or refresh the game entry path. |
| Rarity/value settlement surface | Card rarity, rookie status, and value project as proof/settlement truth with deterministic chart continuity. | Do not replace value truth with transient sampling UI or let event panes consume the chart. |
| Pitch day-proof archive | Archived pitch proofs and projections paint from public archive files before refresh. | Do not treat archived proof as stale cache or wait for full live reconstruction. |
| DB hot-path projection | Migrations and RPCs accelerate append projection beneath proof truth. | Do not let DB indexes, RPC results, or sync guards become stronger than proof payloads. |

## Frozen Invariants

- Stronger proof truth outranks fresher weaker transport.
- Event proof video and milestone projection are additive beneath event proof identity.
- Play Command field telemetry is primary; outcome geometry detail may slide out or inspect secondary data.
- Pitch Command historical event pills must carry exact witness identity, source date, and game binding.
- Foreground Sports Arena appends must not wait behind background stale guards when the user is actively viewing cards.
- Archived pitch proof is durable proof truth, not cache.
- Derby opening stability is part of the Sports proof game surface.
- Type safety around nullable proof/video inputs is release-blocking because Vercel build failure blocks proof deployment.
- Public release labels, offline verifier labels, runtime coherence, service worker app version, and package version must move together.
