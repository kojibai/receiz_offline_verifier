## Purpose

This checklist locks `v70.0.0` as the public alpha release candidate.

Do not release `v70.0.0` unless all of these are true:

1. release-governed version surfaces all say `v70.0.0`
2. `/signin` pass-through opens existing or new Receiz ID profile flow without becoming a parked page
3. `/<username>` remains canonical profile truth and `?manage=1` remains an overlay
4. local proof and verified route truth move before network rediscovery
5. Settlement/Reserve causality remains intact
6. stale runtime recovery preserves the current route
7. twin identity never falls back to profile image
8. mind trail ZIP bundles verify file-by-file and as a bundle

## Manual Launch Tests

### Receiz ID and signin

- Open `/signin` with an existing local Receiz ID.
- Confirm it routes directly to the managed profile.
- Open `/signin` on a fresh supported browser profile.
- Confirm it creates local Receiz ID truth and routes directly to the new managed profile.
- Confirm the user is not parked on a signin page.

### Profile law

- Open `/<username>` signed out and signed in.
- Confirm the first settled shell is the same identity, showcase, media, and layout.
- Open `/<username>?manage=1`.
- Confirm owner powers append without redefining the base shell.

### Runtime recovery

- Resume after a deploy or stale service-worker state.
- Confirm recovery returns to the current page route, not home, unless no safe route exists.
- Confirm the page does not visibly load twice after the shell settles.

### World and proof

- Open live player through an in-app browser path.
- Confirm selected media paints from route truth and twin identity never falls back to profile image.
- Download a mind trail ZIP and verify the root manifest plus every file bundle.

### Value loop

- Run buy, sell, certificate, note, send, and wallet paths.
- Confirm Settlement and Reserve labels, balances, and receipts preserve funded causality.

## Release Rule

Do not release `v70.0.0` if any gate or smoke shows:

- version drift
- parked signin page
- profile public truth being replaced by owner/session/workspace truth
- a settled surface visibly remounting or loading again
- local proof waiting on weaker server permission
- Settlement/Reserve collapse
- stale runtime recovery dumping a known route to home
- twin image fallback to profile image
- unverifiable files in proof packages
