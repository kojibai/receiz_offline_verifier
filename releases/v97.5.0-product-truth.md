## Frozen Primitives

| Primitive | v97.5.0 truth | Must not regress |
| --- | --- | --- |
| Proof object | Complete Receiz proof objects carry deterministic identity, sealed or verifiable payload, provenance, inspection behavior, and Kai/Klok state. | Do not let app-state, SDK helpers, MCP tools, server responses, or database rows treat proof truth as generic app data. |
| Kai Klok | Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit. | Do not use Chronos as the primary state machine for admitted proof objects. |
| Durable proof memory | Product, SDK, and MCP surfaces must use first admission only, then append forever. | Do not call admitted proof memory cache, stale data, hydration state, or rediscovery state. |
| Public app-state projection rail | App-state and public-store records are durable public projections for public app recovery and first paint. | Do not let app-state projections outrank sealed artifacts, identity proofs, ownership appends, settlement rows, verified local truth, or verified appends. |
| Enterprise SDK runtime | `@receiz/sdk` exposes typed convenience rails for commerce, identity, media, domains, events, jobs, RBAC, audit, risk, compliance, portability, search, notifications, release pinning, React UI, sandbox, and offline proof queues. | Do not let SDK convenience become authority over proof objects or verified appends. |
| MCP agent rail | `@receiz/mcp-server` exposes Receiz SDK/API rails to agents over MCP. | MCP must never replace, rename, or outrank Receiz proof objects, identity primitives, ownership surfaces, settlement primitives, public proof surfaces, or verification primitives. |
| Delegated agent token | Receiz OIDC can issue scoped agent tokens for MCP, CI, and trusted local agents. Tokens are shown once, revocable by token ID, and audit-labeled as `agent:*`. | Do not treat an agent token, MCP host, env var, or OIDC client secret as proof authority or identity proof root. |
| Offline proof queue | Offline SDK queues preserve signed local work until replay. | Do not treat offline replay as permission to replace known verified truth. Replay may only append or project verified additions. |
| Connect | Connect is delegated API access for third-party apps. | Do not let Connect become the identity proof root or replace Receiz ID, Identity Record, Identity Seal, Receiz Key, PBI, or artifact proof. |

## Frozen Invariants

- Use known verified truth immediately.
- Ask only for verified additions after the known proof/Kai head.
- Public app-state and public-store records are public recovery projections beneath proof objects and durable proof memory.
- SDK convenience stays beneath proof objects, verified appends, ownership appends, settlement rows, identity artifacts, Twin routes, World projections, and public app-state projections.
- MCP tools stay beneath SDK/API rails and report the authority boundary in tool responses.
- MCP resource templates address Receiz proof and projection surfaces; they do not create authority.
- Delegated agent tokens authorize scoped writes below Receiz identity proof and must remain revocable/auditable.
- Tenant sessions help app developers scope customer experience; they do not become identity proof authority.
- RBAC, audit, risk, compliance, search, notifications, release pinning, jobs, and media rails are enterprise rails beneath the proof system.
- Offline proof queues preserve local proof work and replay verified additions; they do not prove old truth stale.
- React hooks and hosted UI components are rendering convenience only.
- Every write helper that can replay must carry idempotency.
- `@receiz/sdk`, `@receiz/mcp-server`, and app release identity must move together for official releases.

## Release-Lock Requirement

`pnpm release:lock 97.5.0` must require this document for `v97.5.0` and later release cuts. A release that omits the product-truth freeze is incomplete because proof object authority, Kai/Klok state-machine doctrine, durable proof memory law, public app-state projection boundaries, enterprise SDK limits, MCP agent rail limits, delegated agent token limits, offline queue replay, Connect delegated-access limits, and app/SDK/MCP version lock cannot remain explicit release truth.

## Residual Risks

- npm publication is an external registry action and must be run after local gates pass.
- New delegated routes must exist in production before each optional helper can complete against live Receiz.com.
- MCP host behavior varies by client; the package contract covers stdio, tool list, tool calls, and resource templates.
- Production DB and Vercel health remain runtime evidence after deploy, but they do not become proof authority.
- Full `pnpm test:release-freeze` can be long-running; any skipped umbrella gate must be named before release.
