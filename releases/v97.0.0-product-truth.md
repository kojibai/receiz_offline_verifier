# Receiz v97.0.0 Product Truth

Release date: 2026-06-30

`v97.0.0` preserves Receiz as a proof-native artifact system and locks the public app-state projection rail plus `@receiz/sdk` app-state developer surface beneath the same primitive law as the product.

## Frozen Primitives

| Primitive | v97.0.0 truth | Must not regress |
| --- | --- | --- |
| Proof object | Complete Receiz proof objects carry deterministic identity, sealed/verifiable payload, provenance, inspection behavior, and Kai/Klok state. | Do not let SDK, docs, routes, app-state projections, or examples treat proof truth as generic app state. |
| Kai Klok | Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit. | Do not use Chronos as the primary state machine for admitted proof objects. |
| Durable proof memory | Product and SDK surfaces use first admission only, then append forever. | Do not call admitted proof memory cache, stale data, hydration state, or rediscovery state. |
| Public app-state projection rail | App-state records are durable public projections for public app recovery. | Do not let app-state projections outrank sealed artifacts, identity proofs, ownership appends, settlement rows, verified local truth, or verified appends. |
| Connect | Connect is delegated API access for third-party apps. | Do not let Connect become the identity proof root or a replacement for Receiz ID, Identity Record, Identity Seal, Receiz Key, PBI, or artifact proof. |
| SDK package | `@receiz/sdk` version must match the app release version and expose canonical convenience beneath Receiz primitives. | Do not release an SDK package whose package version, runtime constant, docs, or app release identity diverge. |
| OIDC consent | Consent routes authorize delegated application access through browser-safe redirect semantics. | Do not leave consent buttons unreadable, ambiguous, or visually disabled when authorization is available. |
| Public proof projection | Public proof responses must return canonical absolute verification URLs and owner-scoped writes. | Do not return weaker relative paths or let another owner claim an existing owner-scoped projection. |

## Frozen Invariants

- Use known verified truth immediately.
- Ask only for verified additions after the known proof/Kai head.
- App-state projections are public recovery projections beneath proof objects and durable proof memory.
- SDK convenience stays beneath proof objects, verified appends, ownership appends, settlement rows, identity artifacts, Twin routes, World projections, and public app-state projections.
- Connect scopes grant delegated route access; they do not create truth.
- A URL, host, namespace, creator, or id read may recover public app projection state, but it does not replace the proof object beneath the app.
- `publishRecord()` may reduce boilerplate, but it must generate the same feed path used by `publish(feed)`.
- `byHost()` may normalize a tenant/custom domain, but it must read through the same public URL projection route.
- A partial later payload may fill missing fields, but it must not downgrade stronger admitted truth.
- `@receiz/sdk` release version must match the app release version.

## Release-Lock Requirement

`pnpm release:lock 97.0.0` must require this document for `v97.0.0` and later release cuts. A release that omits the product-truth freeze is incomplete because proof object authority, Kai/Klok state-machine doctrine, durable proof memory law, Connect delegated-access limits, public app-state projection boundaries, SDK app-state convenience, OIDC consent behavior, public proof projection correctness, and app/SDK version lock cannot remain explicit release truth.

## Residual Risks

- npm publication is an external registry action and must be run after local gates pass.
- The production app-state migration must be present before the deployed app-state write path is used in production.
- Production DB and Vercel health remain runtime evidence after deploy, but they do not become proof authority.
- Full `pnpm test:release-freeze` can be long-running; any skipped umbrella gate must be named before release.
