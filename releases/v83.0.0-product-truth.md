## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint has priority over background enrichment.
- Stronger known deterministic truth is reused before network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base shell.
- `/game/sports` cannot show a white screen while waiting for data that can be hidden-hydrated.
- Sports Main Event and single-game contexts remain separate lock and score domains.
- Single-game tab selection and scoreboard selection use the same selected-game path.
- Live scoreboards cannot be rewritten by active entry lock state.
- A legitimate zero score is valid score truth.
- Pick 5 availability labels must not hide known starter, bench, backup, role, or injury status.
- Card reveal input state survives viewer open and close.
- Users can deselect all cards if they choose.
- Reward reveal cards follow the same interaction contract as purchased pack cards.
- Issue number, pack origin, proof object, owner, and card memory move together.
- Live-player route playback must remain usable from iMessage.
- SEO structured data is enrichment, not a first-paint dependency.
- Explore first paint is deterministic and not blocked by in-flight build work.
- Market first paint stays fast; conformance must protect that speed.
- Callback URL policy remains centralized and deployment-traced.
- Offline proof object truth can exist before chain reconciliation.
- Calendar and account pagination append known pages without blanking the surface.
- Release labels and service-worker defaults must match the package version.

## Residual Risks

- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
- Target Supabase environments must have the Sports, Explore, profile, calendar, and DB read-path hardening migrations applied.
- External MLB StatsAPI availability remains a live-data dependency, but unavailable feed data must never make active lock state replace selected-game truth.
- Player starter and injury intelligence depends on provider availability; unknown availability must be represented honestly, not guessed into starter truth.
- Stripe, Supabase, Vercel runtime cache, wallet settlement, storage, and browser/PWA behavior remain deployment dependencies, but they must not become permission to downgrade already-held deterministic truth.
- Sports files remain large and should continue to be modularized only behind behavior-preserving contracts.
