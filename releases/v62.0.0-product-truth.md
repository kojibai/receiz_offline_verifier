# Receiz v62.0.0 Product Truth

Release date: April 19, 2026

This document freezes the primitives and invariants that `v62.0.0` is allowed to extend but not weaken after the `v60.0.0` stable baseline, the `v60.2.0` continuity-fix line, and the explicit stabilization pass against the current `HEAD` experimentation tree.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Release-governed version surfaces stay canonical | `package.json`, service-worker defaults, public version badges, plans and upgrade labels, offline verifier artifacts, and active release docs must all expose `v62.0.0` together when the cut is made. | `package.json`, `public/sw.js`, `app/components/pwa/ServiceWorkerRegistration.tsx`, public page/version surfaces, `scripts/check_release_lock.mjs` |
| Settled twin shells remain explicit-intent only | Background warming may load code and hidden state, but it must not render the loaded world module until explicit activation. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts` |
| Home twin shell stays on the direct stable path | The home dock renders through the direct `DeferredWorldProfileModule` path without a second wrapper cache or handoff layer. | `app/components/HomeHeroTwinDock.tsx`, `app/u/[username]/DeferredWorldProfileModule.tsx` |
| Profile route truth stays on the direct strong path | The public profile route keeps the direct `ProfileClient` and trusted-stats merge path in the initial route tree instead of waiting on a second lazy wrapper. | `app/[username]/UserProfileRouteClient.tsx`, `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Hero market chart warms in the background without visible takeover | The home hero may warm the shared market asset after settlement so the back chart is live on flip, but the visible shell must not stage a second loader or visible upgrade. | `app/components/HeroLogoVideo.tsx`, `app/lib/profile/__tests__/heroHomeMarketWarmContract.test.ts` |
| Deterministic sigil scope IDs remain stable | Market pills keep caller-provided deterministic scope IDs across hero, profile, player, and artifact surfaces so hydration order cannot change the SVG namespace. | `app/components/market/MarketValuePill.tsx`, market sigil contract tests |
| Current-head service-worker reset experiments stay out of the release cut | The release must not reintroduce dev-time service-worker cleanup scripts or other forced reload behavior into the settled user path. | `app/layout.tsx`, `scripts/test_page_first_paint.ts` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Hidden twin preparation never promotes the visible shell | Hidden preload, remembered state, or idle preparation must not reopen or replace the visible dock shell without explicit user intent. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts` |
| Home shell continuity beats wrapper cleverness | The home twin path must not add a second loader/cache layer that can remount or visually redefine the already-settled shell. | `app/components/HomeHeroTwinDock.tsx`, release process record |
| Profile route truth is not deferred behind a second client wrapper | Already-known route truth must not wait on lazy `ProfileClient` or lazy trusted-stats merge loading before the main route tree can settle. | `app/[username]/UserProfileRouteClient.tsx`, `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Hero chart warming stays hidden | The home hero may warm market data, but it must not add a visible loading stage or flip-time shell correction. | `app/components/HeroLogoVideo.tsx`, `app/lib/profile/__tests__/heroHomeMarketWarmContract.test.ts` |
| Deterministic sigil identity beats hydration order | Market pill SVG IDs must stay deterministic and surface-scoped; caller-visible sigils must not depend on render timing or `useId()` order. | `app/components/market/MarketValuePill.tsx`, market sigil contract tests |
| Current-head service-worker reset behavior does not return by accident | The release must not ship dev cleanup scripts or force visible reload semantics as part of the settled experience. | `app/layout.tsx`, `scripts/test_page_first_paint.ts` |
| Release surfaces remain canonical | Public version badges, service-worker defaults, offline verifier surfaces, package metadata, and release docs must align with `package.json` when the cut happens. | `scripts/check_release_lock.mjs` |

## Release Gates

`v62.0.0` is considered locked only if these pass together:

- `pnpm test:page-first-paint`
- `npx tsx --test app/lib/profile/__tests__/profileRouteContinuityContract.test.ts app/lib/profile/__tests__/heroHomeMarketWarmContract.test.ts`
- `npx tsx --test app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts`
- `pnpm build`
- `pnpm release:lock`
- `pnpm test:release-freeze`

The manual release procedure is defined in `docs/releases/v62.0.0-checklist.md`.

## Residual Risks

- The stale-PWA service-worker fixes from the current `HEAD` line solved a real returning-install edge case, but they were intentionally kept out of `v62.0.0` because they also changed runtime behavior. If that bug family returns in production, it should be addressed in a separate governed release rather than folded back into this cut casually.
- The larger profile/media/original continuity stack removed from the current `HEAD` line may still hold value for narrower edge cases, but it was not a downside-free fit for this stabilization release.
- Hero chart warming improves likely-first-flip freshness, but it is still a background warm path, not a hard server-seeded guarantee for every cold first flip.
- `v62.0.0` is not fully release-cut until public version surfaces, service-worker defaults, offline verifier surfaces, package metadata, and docs all move together to the same version.
