`v72.0.0` is a stability freeze on top of the `v70.0.0` public alpha product truth.

## Frozen Primitives

| Concept | v72.0.0 truth | Must not regress |
| --- | --- | --- |
| Product law | Literal doctrine and exact user law still outrank implementation convenience. | Do not weaken v70 profile, local truth, settlement, or recovery law. |
| Account switching | The active account uid is the actor scope for warmed route truth. | Do not let old account route snapshots, wallet summaries, or market private state survive a uid switch. |
| Market continuity | Richer wallet and position graphs are reusable only when the viewer user id matches. | Do not preserve private market state across accounts because it is richer. |
| Hydration | Background hydration may refresh stale open-app state without redefining settled visible shells. | Do not fix stale hydration by making first paint slower or visibly load twice. |
| Profile first paint | Public and owner profile views share canonical public profile truth first. | Do not let owner/session projection replace an existing canonical public profile. |
| Twin overlay | Mobile eyes and mouth are face-locked to detected portrait features. | Do not move the overlay off-face with broad viewport offsets or cosmetic guesses. |
| Release surfaces | Package version, service worker defaults, public badges, verifier surfaces, and release docs move together. | Do not ship version drift. |

## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint beats richer rediscovery.
- Stronger known deterministic truth is reused before any network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base public shell.
- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains canonical profile truth plus owner permissions.
- Account switching must clear profile workspace seeds, wallet route seeds, market route snapshots, world route snapshots, market prepared transitions, fast-entry hints, and shared wallet summary cache before projecting the new account.
- Signed-in market snapshots must carry viewer user id when available.
- Market wallet continuity must return the incoming snapshot, not preserved old private state, when signed-in viewer principals do not match.
- Compact mobile rendered-portrait overlay corrections must be bounded, mobile-only, and remain on the detected face boxes.
- Service-worker and app-version defaults must match the release version.

## Release Gates

Run from the exact release candidate:

```bash
pnpm release:lock
pnpm test:continuity-release-contracts
pnpm test:world-release-contracts
pnpm test:release-freeze
pnpm release:attest
```

The full release-freeze script also runs product inventory, deterministic surfaces, page-first-paint, artifact binding, document seal, world trail package, golden path, contact contracts, signature, API security, chaos failure, value-loop, economy, market, verification, world, profile, chat, performance, and governance gates.

## Residual Risks

- The release is not fully locked until `pnpm test:release-freeze` passes after the `v72.0.0` surface bump.
- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
- Public signed-out browser routes in local dev do not expose an active moving Live Twin overlay, so the overlay fix is locked by fit math and world twin contracts unless an authenticated live-twin fixture is provided.
- Production database, Redis, Stripe, and external provider health remain deployment concerns, but they must not become permission to believe already-held local proof.
