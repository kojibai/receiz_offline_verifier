# Receiz v52.0.0 Release Lock Checklist

Release date target: April 13, 2026

## Purpose

This checklist locks `v52.0.0` to one exact release state without allowing regression in deterministic market execution, spread-governed sustainability, replayable quote evidence, partial share-certificate custody, market conformance, runtime hardening, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v52.0.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm exec tsc --noEmit --pretty false
pnpm test:market-conformance
pnpm release:lock
pnpm test:release-freeze
pnpm run build
```

If any of the existing world, settlement, verification, or contacts release gates are already part of the candidate deploy checklist for the same commit, they should also pass before tag creation even if they are not restated here.

## Production Environment + Market Preflight

The production database must already include the active market rails:

- `supabase/migrations/20260413041500_market_share_positions.sql`
- `supabase/migrations/20260413070500_market_prediction_books.sql`
- `supabase/migrations/20260413102000_market_execution_guards.sql`
- `supabase/migrations/20260413134500_market_share_certificates_partial_balance.sql`
- `supabase/migrations/20260413153000_market_share_trade_fees.sql`
- `supabase/migrations/20260413174500_market_quote_engine_snapshot.sql`

The main app production env must still include the normal app/session/database rails required for market execution:

- `NEXT_PUBLIC_SITE_URL` or the deployed canonical app URL
- Supabase anon/service-role configuration used by the existing market runtime
- wallet/session envs required for signed-in settlement and authenticated trade execution

Before release, confirm:

- market trade writes can persist the new quote snapshot fields
- share certificate import/export paths are deployed only after the partial-balance migration is live
- `GET /api/market/conformance` returns a passing result in the candidate environment
- `GET /api/market/conformance/badge` renders normally in the candidate environment
- Vercel/Next output tracing includes the market conformance source set so the suite does not fail from missing traced files

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- `/market` opens with the dark premium shell and does not feel like it is rerendering continuously while idle
- opening and closing desk panels does not trigger hydration mismatch warnings
- one asset can be bought and sold through the live market trade path with the resulting quote evidence present in persisted trade metadata/columns
- the market pill on profile feed, artifact viewer, and live player shows a live value without forcing the full desk snapshot before the popover opens
- opening a market pill popover can still escalate into full market context and signed-in trading
- a partial share-certificate withdrawal reduces the live position by the requested quantity rather than forcing a full export
- a partial share-certificate redeposit restores only the selected quantity rather than requiring an all-or-nothing import
- conformance center, status surfaces, and powered-by trust rows now acknowledge market conformance
- public version badges and offline verifier surfaces all show `v52.0.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v52.0.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact includes the market conformance result.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any quote-determinism regression, spread/evidence persistence regression, market-pill runtime regression, share-certificate partial-balance regression, market-conformance regression, version-surface mismatch, release-lock check, or attestation step fails, `v52.0.0` is not released from that commit.
