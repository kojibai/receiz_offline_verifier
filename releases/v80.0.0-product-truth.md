## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint beats richer rediscovery.
- Stronger known deterministic truth is reused before any network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base public shell.
- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains canonical profile truth plus owner permissions.
- Verified stored truth is permanent known truth, not stale mutable cache.
- Sync asks only for verified additions after the held prefix.
- PBI/passkey recovery proves a legacy account; Receiz ID proves a device; binding is an idempotent append, not session issuance.
- `/api/auth/receiz-id/bind` must not depend on Redis or replay-guard storage.
- Identity Record, Identity Seal, and Receiz Key restore complete visible login at proof admission.
- Import, bind, wallet refresh, global mirror, and portable-state sync are enrichment jobs after the active account projects.
- Stale old-cookie, anonymous, previous-account, or weaker server responses cannot downgrade an admitted local account.
- Settlement, Reserve, wallet holds, notes, certificates, shares, auctions, transfers, and battle stakes must preserve funded causality.
- Market and wallet private state is same-actor only.
- Sold or transferred-out local truth cannot be resurrected by older snapshots.
- World agent work must create visible artifacts, safe holds, or concrete next actions.
- Creator paid work must be charged idempotently or blocked by explicit shortfall.
- Signal Cards are Receiz assets, not mutable gameplay UI.
- Signal Card achievement truth, generated art evidence, transfer appends, battle appends, auction state, and current owner truth remain separate.
- Signal Card World discovers owner truth, not showcase eligibility.
- Vault Card export starts only after explicit user action.
- Vault Card export is complete only when archive bytes, manifest, hash, chunking, integrity, and restore rules are embedded.
- Battles append requests, settlements, ledger rows, transfer state, and card appends without mutating original claims.
- AI twin defense requires explicit owner authorization.
- Public Circuit, Exchange, Proof, Status, and conformance surfaces must match real route behavior and avoid private route exposure.
- Runtime stale recovery is hidden coherence work, not a visible second shell.
- Service-worker and app-version defaults must match the release version.
- World resident worker manifests must preserve numeric schema versions and carry the app release through a release marker.
- Release gates must catch visible behavior regressions, not merely match the latest code.

## Residual Risks

- The release is not fully locked until `pnpm lint`, `pnpm build`, `pnpm release:lock`, `pnpm test:release-freeze`, and `pnpm release:attest` pass after the `v80.0.0` surface bump.
- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
- Production database migrations for Public Proof, Market Twin authority, Creator OS, Signal Vault, Signal Packs, Signal Circuit conformance, Signal Pack Foundry, card markets, auctions, live Battle Arena, auction wallet holds, and million-card storage must be applied in the target environment.
- External services including Supabase, Redis, Stripe, Resend, OpenAI, storage, Vercel, GitHub, Vercel Sandbox, media/render providers, wallet settlement rails, and browser/PWA download behavior remain deployment concerns, but they must not become permission to downgrade already-held deterministic truth.
- Mobile Safari, installed PWA, and in-app browser download paths should continue to receive manual device verification before public rollout.
- Public status probes must remain curated as new proof, creator, card, exchange, battle, auction, and conformance routes are added.
