# Receiz v62.0.0 Release Lock Checklist

Release date target: April 19, 2026

## Purpose

This checklist locks `v62.0.0` to one exact release state without allowing regression in settled twin-shell continuity, direct profile-route truth, hero chart live parity, deterministic market sigil identity, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v62.0.0-product-truth.md`

Process retrospective:

- `docs/releases/v62.0.0-process.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm test:page-first-paint
npx tsx --test app/lib/profile/__tests__/profileRouteContinuityContract.test.ts app/lib/profile/__tests__/heroHomeMarketWarmContract.test.ts
npx tsx --test app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts
pnpm build
pnpm release:lock
pnpm test:release-freeze
```

If the candidate deploy also requires broader governance or attestation checks beyond the list above, those must still pass before tag creation.

## Production Environment Preflight

The production environment must already include the active `v60.0.0` baseline. `v62.0.0` adds no new database migrations on top of that cut.

Before release, confirm:

- home and profile twin shells still settle once and stay visually immutable until explicit open intent
- hidden twin warming does not auto-promote the visible shell
- the public profile route still resolves truthful content directly without a visible second client assembly stage
- the home hero back chart is already live or nearly live by the time the user flips the card
- market sigil surfaces still keep deterministic stable IDs across hero, profile, player, and artifact routes
- public and offline verifier surfaces, service-worker version, `package.json`, and release docs all show `v62.0.0`

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- load `/` and a public profile, let both pages fully settle, and confirm the visible shell does not perform a second visual reload or runtime takeover
- explicitly activate the home twin dock and confirm the richer twin opens only after that user action
- flip the homepage hero card and confirm the market chart is already live or catches up without a visible staged loading frame
- open profile, player, and artifact surfaces and confirm market pills still render without SVG ID drift
- public version badges, service-worker defaults, offline verifier surfaces, package metadata, and release docs all show `v62.0.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v62.0.0`.
2. Run the `Release Governance Attestation` workflow if the release process for this cut still requires a signed attestation.
3. Verify the generated artifact with `pnpm release:verify-attestation <path>` if an attestation is produced.
4. Confirm the attestation references the published key ID and the expected release version.
5. Store the attestation path and approval reference with the release record.

## Release Rule

If any twin-shell continuity regression, profile-route continuity regression, hero-chart parity regression, sigil-identity regression, version-surface mismatch, release-lock failure, or release-freeze failure occurs, `v62.0.0` is not released from that commit.
