## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint beats richer rediscovery.
- Stronger known deterministic truth is reused before any network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base public shell.
- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains canonical profile truth plus owner permissions.
- Signal Cards are Receiz assets, not mutable game UI state.
- Signal Card achievement truth, generated art evidence, battle appends, transfer appends, and current owner truth remain separate.
- Signal Card artwork must match sealed card truth and cannot be downgraded by older local cache.
- Signal Card World reads owner truth, not showcase eligibility.
- Global discovery may append verified rows but must not erase owner-scoped local truth.
- Vault Card export starts only from explicit user action.
- Vault Card export is complete only when archive bytes, manifest, hash, and restore rules are embedded.
- Transfer packages append ownership movement without rewriting the original card issue.
- Battles append requests, settlements, ledger rows, and card appends without mutating original claims.
- AI twin defense requires explicit owner authorization.
- Card auctions preserve wallet-hold and settlement truth.
- Public Circuit and Exchange pages must match conformance and route behavior.
- Public status must increase assurance without exposing privileged internals.
- Service-worker runtime coherence assets must move with release version.
- Release worker manifests must carry the v78 release marker without changing schema meaning.
- Release gates must catch visible behavior regressions, not merely match the latest code.

## Residual Risks

- The release is not fully locked until `pnpm lint`, `pnpm build`, `pnpm test:release-freeze`, and `pnpm release:attest` pass after the `v78.0.0` surface bump.
- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
- Production database migrations for live Battle Arena and card auction wallet holds must be applied in the target environment.
- External services including Supabase, Redis, Stripe, storage, PWA/browser download behavior, Vercel, GitHub, media/render providers, and wallet settlement rails remain deployment concerns, but they must not become permission to downgrade already-held deterministic truth.
- Mobile Safari, installed PWA, and in-app browser download behavior should continue to receive manual device verification before public rollout.
- Public status probes must remain curated as new card, exchange, battle, auction, and conformance routes are added.
