## Title

Receiz `v88.0.0` - Live Sports Truth, Runtime Coherence, Public Live Cockpit, And Post-v87 Regression Lock

## Summary

`v88.0.0` locks the complete arc from the `v87.0.0` tag through the `87.5.0` stable head and promotes it as the next major official release line.

The post-`v87.0.0` arc spans 90 commits, 195 changed files, 72,663 insertions, and 6,475 deletions before the final `v88.0.0` release edits. This release does not narrow the v87 offline proof primitive. It keeps offline proof, compact `/v` transport, Signature V4 verification, and public proof surfaces intact while locking the live product repairs that were made after v87:

- Sports Arena live field truth restored to the `87.0.3` behavior while keeping locked-entry first-paint and per-player scoring upgrades.
- `/baseball` is now a direct entry route for the same canonical baseball arena; `/game/sports/baseball` remains working and remains canonical.
- Single-game locked entries, individual player scores, live cockpit totals, and finalized reward state now preserve stronger proof/local truth during hydration and game switching.
- The center field layout is locked: score plate shows inning, score, and base state; center recap shows real pitch-by-pitch; bottom ticker shows last play.
- Public Live Cockpit, `/live` rooms, and sports append hydration are documented as live proof and public witness surfaces.
- Service-worker/runtime version coherence is hardened so deterministic session/profile projection keys follow the package release when `NEXT_PUBLIC_SW_VERSION` is omitted.
- Session bootstrap durable-key clear/migration behavior is hardened so logout/clear cannot resurrect stale signed-in first paint.
- Supabase timeout hardening is documented and covered by migration tests for the DB policies and indexes the repo can verify.
- Market/profile first-paint repairs, market value pill fixes, Win flow, exchange/signal-circuit framing, notification delivery, and chat push/call improvements are included in the official release story.
- Regression lessons are now a required release-lock artifact.

## v87.0.0 Baseline

`v87.0.0` froze offline proof as a real proof primitive:

- sealed offline proof objects carry proof truth
- compact `/v` URLs are transport for proof truth
- `/v` verifies Signature V4 before historical fallback
- coordinate-only evidence stays below signed proof
- offline verifier/studio HTML must match app proof semantics

`v88.0.0` preserves that baseline. It extends the release lock over live Sports, public live rooms, runtime coherence, market/profile continuity, and DB timeout hardening.

## Whole Post-v87 Arc

### Win, Exchange, Signal Circuit, And Operating Surfaces

The first post-v87 commits refined the public operating surfaces:

- exchange and Signal Circuit page framing and copy
- social economy and earned-card copy
- Win page sequence, payout kicker, automatic checkout handoff, and OG/Twitter image support

These are public proof and settlement-adjacent surfaces. They must remain direct product surfaces, not marketing placeholders.

### API Latency And Route Audit

The route audit added a reusable latency harness covering hundreds of route files and methods. Safe GET/HEAD/OPTIONS paths were probed without mutating production state, while mutating routes stayed skipped unless fixtures exist.

This release keeps that boundary:

```txt
measure safe read paths
do not mutate production as a side effect of audit tooling
turn missing fixtures into explicit uncertainty, not fake coverage
```

### Sports Media And Proof Moment Evidence

Sports event media moved from broad highlight matching toward exact moment evidence:

- real recorded video path added
- `mlb-statsapi-game-video-offset` accepted as exact recorded video when offset basis and event alignment match
- broad player highlights separated into related highlights instead of exact proof media
- deterministic proof-motion clip preserved
- event media tests lock exact moment evidence

The proof rule is direct: broad highlights may enrich, but they do not replace exact proof media.

### Sports Console Split And First-Paint Repair

The Sports Arena console was split below the Babel source deopt threshold and backed by model/first-paint tests. Post-midnight games and live feed hydration were repaired after real regressions.

Locked truth:

- selected live games must keep detailed field data
- `/baseball` may route users directly into the arena, but it must reuse the canonical `/game/sports/baseball` page logic
- locked entries must first-paint from stored proof/local truth
- public feed updates may append and refresh, not erase stronger selected-game truth
- single-game proof projection restores locked entries when selected game state is stale

### Market And Profile First Paint

Market and profile regressions after the market pill/live chart work were repaired:

- public profile originals and assets preserved under tightened RLS/security constraints
- market desk buy/sell continuation repaired
- market pill chart/live popover repaired without making `/market` rediscover known truth
- profile first-paint continuity restored

The profile law remains:

### Sports Live Cockpit And Single-Game Proof

The largest post-v87 arc repaired live Sports behavior end to end:

- per-player append hydration
- live cockpit total projection
- live header and cockpit sizing
- Sports appends no longer hard-limited per game
- live append sync covers every live/final MLB game for the source date
- single-game leaderboard hydration uses live game entries without demo fallback
- individual card rows use proof-backed card scores while appends hydrate
- active locked entry truth outranks unresolved finalized rewards

The exact visible field allocation is locked:

```txt
score plate: inning, score, base state or pregame countdown
center field recap: real pitch-by-pitch from arenaField.playByPlay
bottom ticker: last play from arenaField.lastPlay
player cards: real batter and pitcher when live data provides them
fallback: allowed only when the real feed is genuinely unavailable, never as replacement for known detailed truth
```

### Notifications, Chat, And Runtime Continuity

The release includes open-app and installed-PWA notification delivery:

- chat push subscription table and API
- push notification registration and server helpers
- chat call/message route behavior improvements

Runtime continuity was also hardened:

This protects deterministic first paint and prevents old runtime/session keys from becoming accidental authority.

### DB Timeout Hardening

The Supabase hardening arc covered multiple timeout classes:

- high-value FK indexes for hot chat-call and market tables
- live and push table timeout hardening
- RLS init-plan-safe `(select auth.uid())` rewrites for advisor-flagged policies
- chat RLS membership helper/policy hardening
- guard tests that prevent direct `auth.uid()` reintroduction in new hardening migrations
- repository guard preventing broad automated Supabase Advisor/linter scans that can flood production logs

The repo contains the migration and tests. Production application of the migration remains a deployment/privilege boundary and must target the exact Supabase project that backs production.

## Product Statement

`v88.0.0` exists because live truth must not be reinterpreted by hydration.

If Receiz already has stronger proof/local truth for a locked entry, live score, player card, reward state, profile shell, market proof object, or sealed artifact, that truth paints first. Network, server, DB, and appends may enrich beneath it. They may not replace it with weaker partial state, a fallback field, or a rediscovery frame.
