## Title

Receiz `v72.0.0` - Stable Continuity Release

## Summary

`v72.0.0` keeps the `v70.0.0` public alpha law intact and locks the post-v70 stabilization work that improved real user experience without changing product truth.

This release freezes two visible stability fixes:

- Account switching now clears actor-scoped route warmth and refuses to preserve market/private wallet state unless the viewer user id matches.
- Compact mobile Live Twin rendered-portrait overlays apply a bounded lower-left face-lock correction so eyes and mouth stay on the person's face without restoring the old oversized mobile downshift.

## What This Release Locks

- `/<username>` remains canonical profile truth; `?manage=1` remains owner permissions only.
- Owner/private enrichment must not make signed-in profile first paint slower than public profile truth.
- Useful hydration repairs stay: stale app-open state can refresh in the background without replacing a settled visible shell.
- Account switch on `/account` must immediately project the new account to every warmed route surface.
- Market, wallet, world, and profile route continuity are actor-scoped and must be cleared when the active account uid changes.
- Mobile twin eye and mouth overlay fit is a contract, not a visual preference.
- Package metadata, service-worker defaults, public version badges, verifier surfaces, and release docs move together to `v72.0.0`.

## Release Rule

Do not ship `v72.0.0` unless the release gates in `docs/releases/v72.0.0-checklist.md` pass from the exact release candidate and the governed surfaces all show `v72.0.0`.
