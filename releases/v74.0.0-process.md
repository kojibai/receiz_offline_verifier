# Receiz v74.0.0

## Goal

Ship the post-`v73.0.0` Receiz Key, identity artwork, profile media, and continuity work as an official `v74.0.0` release while preserving the v70 public alpha law and the v73 ownership/vault baseline.

## What Worked

- Measuring the release from the actual `v73.0.0` tag made the diff concrete: 45 commits plus release-candidate working-tree updates, 132 changed files, 11,462 insertions, and 714 deletions.
- Separating Receiz Key restore from PBI attestation made recovery match the product law: PBI can attest key creation, but an already-held private Identity Record restores through its embedded key-payload proof.
- Account key controls fit the existing Account protection surface when key status, export, import, refresh, and revocation were treated as owner security controls rather than hidden recovery plumbing.
- Deterministic profile identity artwork worked when it derived from stable account-creation Kai truth, decoded locally before signed PBI state changed the visible hero source, and kept the KaiSigil glyph as an untouched canonical layer above the Receiz chrome.
- Long-form originals, attached links, signed identity seals/records, and video playback derivatives stayed safe when each payload was classified as verified addition, media realization, or explicit experience state instead of base profile truth.
- Sold-position market continuity was tractable when local closed-position state was treated as stronger than weaker live market snapshots.

## What Did Not Work

- Treating Receiz Key restore as another PBI login path was too broad; it would make PBI a restore permission instead of optional attestation.
- Profile identity artwork became unstable whenever creation time or signed-state transitions could move after first paint.
- Identity record styling became unsafe when it treated the KaiSigil glyph as decoration; the correct split is record/proof chrome around the glyph and the real creation KaiSigil rendered with canonical geometry and size math.
- Attached URL fixes could not be solved with card text truncation because the exact href is transport truth.
- Video derivative playback would be unsafe if it replaced original proof downloads instead of adding a playback-only media source.
- Release readiness was blocked until all governed public version surfaces, service-worker defaults, verifier files, README pointer, and release docs moved together.

## Practical Rule For Future Work

Classify recovery and profile payloads before they reach the visible shell. Embedded Identity Record key-payload proof, verified portable state, deterministic identity artwork, original proof downloads, and sold local positions are truth. PBI receipts, long-form readers, playback derivatives, market labels, and owner controls may append or realize that truth, but they must not redefine first paint, replace canonical proof, or hide release version drift.
