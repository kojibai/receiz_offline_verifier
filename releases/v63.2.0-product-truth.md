# Receiz v63.2.0 Product Truth

Release date: April 19, 2026

This document freezes the value-state and certificate rules shipped in `v63.2.0`.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Release-governed version surfaces stay canonical | `package.json`, service-worker defaults, public version badges, offline verifier artifacts, and active release docs expose `v63.2.0` together. | `package.json`, `public/sw.js`, `app/components/pwa/ServiceWorkerRegistration.tsx`, public version surfaces, `scripts/check_release_lock.mjs` |
| Settled balance is the cash-like lane | Settled balance comes from funded deposits/payments or proceeds backed by settled value. | `app/lib/wallet/server.ts`, `app/api/wallet/ledger/route.ts`, `app/lib/wallet/__tests__/valueStateContract.test.ts` |
| Sovereign PHI is Receiz-native value | PHI can move and be used inside Receiz, but it is not externally withdrawable by wire and cannot mint settlement notes until funded. | `app/wallet/WalletAccountExperience.tsx`, `app/market/MarketDeskExperience.tsx`, `app/lib/wallet/__tests__/valueStateFramingContract.test.ts` |
| Notes stay settled-only | Notes do not carry PHI or unfunded live value. | `app/wallet/WalletAccountExperience.tsx`, `app/lib/wallet/__tests__/valueStateContract.test.ts` |
| Share certificates bridge live value to funded settlement | Certificate value becomes settled only when a funded buyer or backed liquidity event pays for it. | `app/api/market/share-certificates/[noteToken]/trade/route.ts`, `app/market/c/[noteToken]/MarketShareCertificateExperience.tsx`, `app/lib/market/__tests__/shareCertificateMarketFlowContract.test.ts` |
| Visible balances patch from known trade truth | Market sells update settled and Sovereign lanes immediately from the executed trade split; they do not wait for a page refresh. | `app/lib/market/shareTrade.server.ts`, `app/lib/market/shareTradeHydration.ts`, `app/lib/market/__tests__/shareTradeHydrationContract.test.ts` |
| Certificate share pages are action-first | Public certificate pages do not show a default login gate; buy/sell starts Receiz ID only when required by the action. | `app/market/c/[noteToken]/MarketShareCertificateExperience.tsx`, `app/lib/market/__tests__/shareCertificateViewportContract.test.ts` |
| Certificate proof surfaces feel live and verifiable | The certificate chart can flip into proof data showing token, status, shares, mark, value, basis, P/L, pulse, settlement rule, source, and verification links. | `app/market/c/[noteToken]/MarketShareCertificateExperience.tsx`, `app/market/c/[noteToken]/MarketShareCertificateExperience.module.css` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Settled and Sovereign lanes must not blur | UI must not show PHI as externally withdrawable cash or treat unfunded market growth as settled value. | Value-state and framing contracts |
| A sell is settled only when funded | Backed value settles; unbacked value returns as PHI. | Funded liquidity and share trade hydration contracts |
| A certificate mint does not credit the seller | Locking shares into a certificate is not a realization event. | Share certificate market-flow contract |
| Public certificate actions continue on the same surface | Auth, funding, and trade completion must not eject users into a detached flow. | Market continuation and certificate viewport contracts |
| Provider names stay out of buyer-facing liquidity framing | Funding copy should say secure checkout, card funding, settled value, and Sovereign PHI. | Wallet and market surface copy checks |
| Release surfaces remain canonical | Public version badges, service-worker defaults, offline verifier surfaces, package metadata, and release docs must align. | `scripts/check_release_lock.mjs` |

## Residual Risks

- Direct visual browser verification of the certificate proof flip was not rerun in this turn; the behavior is locked by source contracts and type-check.
- Full production payment settlement still depends on the deployed payment provider configuration and webhook/session-sync environment.
- This release does not implement creator-owned asset withdrawal into certificates beyond the existing creator reserve/certificate rules.
