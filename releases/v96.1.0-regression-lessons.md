# Receiz v96.1.0 Regression Lessons

Release date: 2026-06-30

## What Regressed

No runtime product behavior is intended to change in this release. The regression risk was release identity drift:

- SDK source gained Twin and World public capability while the package version still said `96.0.0`.
- Developer docs and API catalogs needed to describe the same surface as the SDK.
- Public release labels, service-worker/runtime version defaults, and SDK runtime constants needed to move together.

## Why It Regressed

The SDK moved faster than the official release lock. That is a mild process drift: the implemented public developer surface was real, but the version identity had not yet been promoted to match it.

This is not a reason to slow SDK work. It is a reason to lock package identity, release docs, OpenAPI/status docs, and contracts immediately when a developer surface becomes public.

## Locked Rules

- `@receiz/sdk` must be version-locked to the app release.
- `RECEIZ_SDK_VERSION` must match `packages/receiz-sdk/package.json`.
- Public app release labels must match `package.json`.
- Service-worker and runtime coherence defaults must match the release.
- Connect scopes must be documented anywhere developers discover Connect capability.
- SDK docs must explain Twin and World as convenience beneath existing Receiz primitives.
- OpenAPI/status catalog entries must stay aligned with exposed developer routes.
- The operating law remains first admission only, then append forever.

## Contract Gates

- `scripts/check_release_lock.mjs`
- `scripts/test_sdk_release_contracts.ts`
- `app/lib/__tests__/developersSdkPackageContract.test.ts`
- `app/lib/__tests__/developersOpenApiContract.test.ts`
- `app/lib/status/__tests__/publicStatusSurfaceContract.test.ts`
- `packages/receiz-sdk/test/sdk.test.ts`
- `scripts/test_release_freeze.ts`

These gates fail if the app and SDK version identity diverge, the SDK drifts from packaged evidence, or public developer route documentation stops matching the implemented surface.

## Release-Lock Requirement

`pnpm release:lock 96.1.0` must require this document for `v96.1.0` and later release cuts. A release that omits regression lessons is incomplete because SDK/package identity drift, Twin and World developer-surface visibility, Connect scope documentation, runtime version coherence, and app/SDK version lock lessons would not be explicit release truth.
