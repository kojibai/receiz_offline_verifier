# Receiz v93.2.0

Release date: 2026-06-26

## Goal

Ship `v93.2.0` as the SDK Proof Developer Experience release.

The goal is to make Receiz the easiest proof-native system for developers to implement: one SDK, typed clients, schema exports, deterministic projections, webhook helpers, Connect primitives, and append-only proof memory.

## What Worked

- The existing SDK already had strong typed clients for verification, public proof, Sports, wallet, Connect, identity, payments, and webhooks.
- The production Sports history breakthrough gave the correct SDK law: first admission only, then append forever.
- Runtime validators already provided the right base for developer conformance.
- Adding projections and proof memory was additive beneath existing proof authority.

## What Did Not Work

- `@receiz/sdk@93.1.0` had already been published, so new SDK capabilities could not honestly remain on `93.1.0`.
- The SDK made developers call validators and clients, but did not yet give them the complete render/admit/persist pattern.
- Without proof memory helpers, third-party apps could accidentally implement newest-response-wins feed logic.

## Practical Rule For Future Work

Every SDK feature must answer:

```txt
What proof primitive does this help developers use faster without becoming authority?
```

If the SDK feature cannot answer that, it does not belong in the SDK.
