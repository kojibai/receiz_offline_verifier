# Receiz v47.8.1 Release Lock Checklist

Release date target: April 10, 2026

## Purpose

This checklist locks `v47.8.1` to one exact release state without allowing regression in one-view mobile contacts operation, person-first follow-up access, linked-account import merges, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v47.8.1-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm exec eslint app/contacts/ContactsPageExperience.tsx
pnpm test:contacts-release-contracts
pnpm release:lock
pnpm test:release-freeze
```

Environment and deployment preflight for production lanes:

- the production database must already include `supabase/migrations/20260409163000_contacts_crm.sql`
- server-side Supabase admin/service-role credentials must remain available for authenticated contact sync and world action-ledger projection
- service worker and public offline verifier artifacts must all expose `v47.8.1`

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- `/contacts` loads on mobile with the command shell and primary controls visible without an initial scroll wall
- the mobile summary tabs open `Overview`, `Next`, `Context`, and `Recent` overlays cleanly and close cleanly
- the mobile `Tools` surface exposes edit, log, invite, and full-history access without clipping
- schedule, message, and call still work from the selected-contact hero
- importing data that upgrades an existing person to a linked account persists that identity upgrade
- public version badges and offline verifier surfaces all show `v47.8.1`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v47.8.1`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any mobile contacts regression, contact-tool access regression, import-merge identity regression, version-surface mismatch, release-lock check, or attestation step fails, `v47.8.1` is not released from that commit.
