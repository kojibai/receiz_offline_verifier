# Receiz v94.0.0 Regression Lessons

Release date: 2026-06-27

## What Regressed

The primary regression family across the `v93.x` arc was not visible product intent. It was implementation drift away from Receiz law:

- known proof truth was sometimes treated as cache that could be rehydrated or replaced
- Kai coordinates were sometimes derived from Chronos after the app had already supplied the correct proof coordinate
- route-local projections sometimes rediscovered truth instead of projecting admitted proof memory
- database pressure appeared where projection code asked the database to re-find old proof history
- developer integration risk remained when SDK helpers stopped at low-level clients instead of carrying proof-memory law

## Why It Regressed

The inherited app pattern is newest-response-wins hydration. Receiz law requires known-truth-first projection.

When the code followed the inherited pattern, it created pressure and instability:

- an old proof object could disappear from a route until the server rediscovered it
- a partial payload could look newer than a fuller admitted payload
- a DB timeout could look like missing truth
- multiple proof children born in one Chronos instant could collide if the trigger ignored the Kai child coordinate

The fix was not to add more generic caching. The fix was to honor the existing primitives.

## Locked Rules

- Proof memory is not cache.
- Known verified truth is not stale.
- First admission only; append forever.
- A weaker partial response may fill missing fields but must not erase admitted proof fields.
- Kai pulse/upulse is the ordering coordinate where present.
- Chronos is display/fallback, not the Receiz backend state machine.
- RPCs and indexes are bounded projection mechanics beneath proof objects.
- SDK convenience remains below sealed artifacts, proof bundles, verified appends, ownership appends, and settlement ledger rows.
- Calendar points, account activity nodes, wallet rows, Sports appends, and event proofs are projections of proof objects, not independent truth systems.
