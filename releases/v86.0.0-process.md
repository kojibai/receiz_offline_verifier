# Receiz v86.0.0

## Goal

Ship `v86.0.0` as the official locked baseline for the full `v80.0.0` to current product arc:

```txt
proof-native artifact system
literal product law
deterministic first paint
canonical profile proof append
deterministic Market public shell
signed-in ownership hydration behind the shell
instant Explore proof truth
Sports live proof economy
Main Event payment-confirmed entry
single-card QR public proof
World public twin continuity
coherent release identity across package, service worker, docs, and public surfaces
```

The goal is to freeze the current working baseline so future work starts from the laws and primitives that now exist, instead of relearning them.

## What Worked

- The release became stable when work followed Receiz truth hierarchy instead of generic app convention.
- Profile posting worked when visible proof objects appended to canonical public profile truth.
- Market became stable when visible profile proof objects were carried by the deterministic Market shell and hidden visibility removed them.
- Direct Market entry became fast when the route stopped waiting on server discovery before first paint.
- Signed-in Market ownership recovered when private graph hydration was treated as after-paint enrichment that cannot erase public shell truth, positions, owned assets, or history.
- Explore became instant when it reused deterministic proof bundle truth and preserved the identity image data its surface needs.
- Charts stabilized when the proof-back chart reused the proven pill chart instead of maintaining a weaker async loading path.
- Certificate/share value became correct when deterministic market transport stayed first and existing asset-pill recovery stayed beneath it.
- Sports user-live became reliable when warm recovery covered every relevant source date instead of a forced-today or capped subset.
- Pack variety improved by adding team-level collector spread without removing player eligibility.
- Single Sports card QR proof landed cleanly because it duplicated the proven public-proof route pattern and left vault links untouched.
- Main Event checkout became correct when cancellation cleared pending funding while confirmed payment kept the immediate entry continuation.
- Calendar wallet events became correct when rate display used the deterministic USD/PHI basis already present.
- Release law worked when the rule was written in docs and protected by focused contracts.

## What Did Not Work

- Treating DB/session/server state as first-paint authority created profile, market, and signed-in hydration regressions.
- Signed-in Market failed when anonymous or partial snapshot results were allowed to replace private ownership truth.
- New profile proof objects dropped from Market when the write/projection path was treated as a special repair instead of the normal append law.
- Hidden/unhidden profile proof objects failed when visibility did not update the Market projection symmetrically.
- Chart loading regressed where an async-only path existed even though deterministic chart math and proven chart rendering already existed.
- Sports user-live appends failed when recovery was capped or aimed at the wrong date set.
- Pack pulls felt wrong when team spread was not included in future issue pressure.
- Main Event payment could appear entered after dismissing checkout if pending hosted-checkout continuation was left alive.
- Release candidates became risky when service worker, package version, public labels, and docs did not move together.
- Any implementation that makes known truth wait for rediscovery remains a regression even when the endpoint eventually returns correct data.

## Practical Rule For Future Work

Use this order:

```txt
name the primitive
identify the strongest source of truth
paint known truth first
append verified additions after
hydrate private/server/global data behind the shell
never let weaker snapshots erase stronger local/deterministic truth
reuse implemented proof/identity/ownership/settlement primitives
write the contract that fails if the law regresses
```

For profile and Market:

```txt
public profile proof object
canonical profile truth
deterministic market projection
visible means in Market
hidden means out of Market
private owner graph hydrates after paint
older/null/anonymous snapshot cannot erase known truth
```

For Explore:

```txt
deterministic proof bundle
visible post identity
user image data for the pill
server discovery behind first paint
no Market proof payload regression
```

For Sports:

```txt
owned card
lineup context
score context
event proof
all relevant source dates
closeout truth
settlement truth
vault history
card QR proof
market/trade action
```

For Main Event payment:

```txt
insufficient wallet value
funding request
checkout
cancel clears continuation and does not enter
paid session verifies
join resumes immediately
```

For releases:

```txt
package version
service worker version
public labels
release note
product truth
checklist
process
release lock
focused contracts
production attestation
```

The `v86.0.0` rule is:

```txt
If Receiz knows stronger proof truth, it paints that truth now.
Everything weaker may append, hydrate, index, or distribute after.
```
