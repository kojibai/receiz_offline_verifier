## Frozen Primitives

| Concept | v68.9.0 truth | Must not regress |
| --- | --- | --- |
| Canonical profile route | `/<username>` is the only canonical profile truth. | Do not let owner/session/workspace state replace it. |
| Manage overlay | `/<username>?manage=1` is canonical profile truth plus owner append. | Do not let manage mode redefine the base shell. |
| Settled shell | Public and owner routes first-paint the same identity, showcase, and shell. | Do not create a richer signed-in shell that loads later. |
| Hidden inventory | Hidden items are known owner additions layered on top of the canonical profile. | Do not let hidden truth disappear, reappear incorrectly, or drive the public shell. |
| Signed-in speed | Authenticated enrichment may append but may not make signed-in users slower than signed-out users for the same shell. | Do not hold first paint hostage to server-only owner fetches. |
| Deterministic snapshots | Route snapshots accelerate navigation from already-known truth. | Do not let weaker or differently scoped payloads downgrade stronger known truth. |
| Local proof | Receiz ID and local verified state project immediately. | Do not wait for server permission before using held proof. |
| Binding law | PBI recovery plus Receiz ID binding is an append, not session issuance. | Do not make recovered binding depend on replay-guard storage. |
| Runtime recovery | Stale runtime/cookie/service-worker states recover through proof and resync. | Do not let stale runtime crash the app or strand the shell. |
| Release surfaces | Package version, service-worker version, public badges, and release docs move together. | Do not ship governed surface drift. |

## Frozen Invariants

- `/<username>` is canonical truth.
- `?manage=1` only appends owner truth.
- Stronger known truth is reused before any rediscovery.
- Weaker payloads do not downgrade stronger known truth.
- Signed-in users do not wait longer than signed-out users for the same truthful shell.
- Hidden warming is allowed; visible second-load behavior is not.
- Background work may append owner-only panels and server additions, but it may not redefine a settled visible profile shell.
- Verified local truth is permanent known truth, not cache.
- Receiz ID projection, PBI recovery, and binding obey local-first/server-always law.
- Stale runtime recovery is part of the product truth, not an optional hardening layer.

## Residual Risks

- The governed release packaging is only as strong as the release evidence and attestation inputs available in the local environment.
- Profile law is restored at the route and contract level, but any future generic snapshot abstraction that blurs public and owner scope would threaten this release.
- The auth/runtime line remains stronger than older baselines, but those recovery paths must keep being tested together with profile law instead of in isolation.
- Any future version-surface drift would make the release mechanically invalid even if behavior stayed good.
