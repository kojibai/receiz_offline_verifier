# Receiz v76.0.0

## Goal

Ship the full post-`v75.0.0` agentic World, Public Proof, Market Twin, wallet settlement, and public/developer trust-surface work as an official `v76.0.0` release while preserving the v70 public alpha law, v74 identity-artwork baseline, and v75 local-proof/file-restore baseline.

## What Worked

- Measuring from the real `v75.0.0` tag made the release scope concrete: the pre-bump candidate contains 46 commits, 177 changed files, 28,359 insertions, and 1,002 deletions after that tag.
- The Agents SDK became valuable only after it was bounded: direct Concierge streaming remains the normal reply path, while tools, handoffs, tracing, and orchestration are reserved for agentic work.
- World became more useful when tasks turned into missions with visible artifacts, revenue attribution, Sales Brain routing, daily briefs, payment/booking paths, and next commercial actions.
- Follow-up campaigns became less fragile when email was treated as the default destination and invalid legacy app-webhook state could heal instead of blocking the user.
- Market Twin became stickier when it opened fast, preserved saved authority, explained the best move, guarded the action, and showed a ledger of value, replay, and next action.
- Public Proof became product-real when observe, claim, verify, seal, dispute, rights, identity, OAuth, creator, lookup, and registry feed surfaces moved together with mobile UI and contracts.
- Wallet reliability improved when reserve credits and non-subscription settlement were treated as deterministic value state instead of incidental payment context.
- Status became more useful when it exposed all high-value public/developer routes and explicitly excluded private surfaces that would help attackers more than users.
- Release contracts became effective when they targeted the exact user-visible risks: Concierge not replying, Market Twin resetting, follow-up blocked by app destinations, status hiding public value, and proof/wallet/market loops losing deterministic truth.

## What Did Not Work

- Letting the Agents SDK take over normal Concierge responses caused reply regressions; the SDK had to be additive, not a replacement for direct streaming.
- Treating connected-app delivery as the default follow-up path blocked real campaigns when no valid webhook destination existed.
- Waiting for Market Twin mandate hydration before rendering changed the established fast-entry behavior and created a visible loading regression.
- Market Twin value language became unsafe if replay or simulated results sounded like guaranteed yield; the loop needed truthful labels and mandate gates.
- A status page that showed only a narrow public allowlist underrepresented product value for real users and developers.
- A status page that exposed everything would create attacker value; the right boundary is curated public/developer usefulness only.
- Public Proof could not be framed as a product surface unless read paths, mutation paths, identity proof, registry state, UI, and contracts moved together.

## Practical Rule For Future Work

Use agents, automation, and status exposure only where they increase truthful user value. A route belongs on Status when a real public user or developer needs confidence in it and exposing it does not materially help an attacker. A World or Market agent action belongs in the product when it produces a concrete artifact, a protected hold, or a next revenue/value action. Anything else is implementation plumbing and must stay behind the user-visible loop.
