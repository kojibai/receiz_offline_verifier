## Title

Receiz `v93.0.0` - Sports Proof Economy Release

## Summary

`v93.0.0` is the clean release break after `v92.0.0`. It turns the Sports proof arc into a full proof economy: Sports Economy is a live public proof surface, Sports cards carry current proof memory and deterministic value basis, event and card history remain ledger-backed and inspectable, Play Command and Pitch Command become command-grade proof surfaces, physical activation becomes a governed proof/settlement route, and the database read model is hardened around bounded proof projection instead of broad JSON scans.

The documented implementation baseline is `v92.0.0..f7f638af5`. That range spans `139` commits, `854` files changed, `114,773` insertions, and `3,371` deletions. The mechanical commit ledger is preserved in `docs/releases/v93.0.0-commit-history.md`.

The release-lock edit itself adds this v93 document set and moves the governed public version surfaces from `92.1.x` to `93.0.0`: package metadata, service worker version, runtime storage version, public labels, offline verifier labels, and release documents.

## What Changed Since v92.0.0

### Sports Economy Proof Surface

Sports Economy now reads live Sports card ownership, value, market, score, issue, and append history from the implemented Sports proof ledgers. It is not a generic dashboard. It is a public proof surface over Sports cards as owned artifacts, value objects, score-memory objects, market objects, and settlement-bearing objects.

The release adds the Sports Economy command model, deterministic first-paint proof object, live inventory route, score ledger projection, market-cap panels, inventory panels, proof panels, and mobile/desktop command surfaces. The surface now has enough proof structure to explain why a Sports card has value, how its value moved, which proof appends contributed to its live score, and which ownership/economy events carry its market state.

### Bounded Proof-Read Database Model

The DB work is a release primitive, not incidental tuning. The new pattern is:

1. Select a bounded page of row IDs through a shape-specific index or RPC.
2. Join back to the authoritative ledger payload only for those rows.
3. Keep `append_json`, `proof_json`, and `source_json` as source-of-truth payloads, not as broad covering-index cargo.
4. Preserve the same proof payload and visible experience while removing wide PostgREST aggregation pressure.

This pattern now covers Sports Economy live append reads, event proof reads, raw collectible proof reads, Signal Run battle queue reads, user-live game append hydration, visible card appends, entry append summaries, and append pressure guards.

### Command Surfaces

Play Command and Pitch Command are now command proof surfaces. Play Command carries official play telemetry and field geometry. Pitch Command carries exact pitch witnesses, archive projection, media, day-proof manifests, and current/review navigation.

The release moved proof command behavior away from current-state shortcuts. Historical proof actions must open the exact selected witness, event, game, source date, and archive state already proven by the object.

### Pack Derby Proof Game

Pack Derby is now locked as a proof game with film-backed mobile scenes, stable opening, compact HUD, proof reward flow, and faster entry/reward paths. The game is not a decorative add-on. It is a Sports pack proof route that must preserve settled gameplay continuity and proof reward truth.

### Physical Activation

The activation route creates a proof-native physical activation flow: intake, session, checkout, activation assets, settlement, public page, and contract coverage. It is a physical proof object route and a settlement route, not a generic ecommerce checkout. It must preserve the object being activated, the proof/asset relationship, and the funded settlement boundary.

### Sports Card Value, Lineage, And Proof Memory

The release corrected value and lineage drift. Sports card value is deterministic proof/settlement projection, not seller ask, UI estimate, or transient chart state. Value basis now explains deterministic math. Causal ownership lineage walks the full card-wrapped proof chain. Card history and event proof memory project across public, economy, logged-in, and export surfaces without degrading into route-local fragments.

### Visible Appends And User-Live Hydration

Visible-card append memory and user-live hydration now respect stronger known truth and active foreground use. The app learned that background stale guards are useful for pressure but wrong when they delay appends for a user actively viewing proof-bearing Sports cards. Foreground views may warm or append behind settled truth; they must not make known proof wait.

### Release-Speed Finding

The major speed improvement did not come from hiding data, dropping history, or caching weaker truth. It came from aligning performance with the product ontology:

- proof payloads stay authoritative
- first paint uses known deterministic truth
- background work appends behind settled surfaces
- hot reads are bounded before payload hydration
- UI overlays inspect proof instead of remounting surfaces
- exact proof projection replaces route-specific rediscovery

This is the core v93 finding: Receiz gets faster when the implementation stops treating proof truth as generic app state.
