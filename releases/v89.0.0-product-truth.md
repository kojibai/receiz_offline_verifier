## Frozen Primitives

| Primitive | v89.0.0 truth | Must not regress |
| --- | --- | --- |
| Receiz product identity | Receiz remains a proof-native artifact system. | Do not describe Receiz as a generic app with proof features. |
| Foreground anchor publication | `/api/receiz/anchor` is the foreground proof publication route with its own readable response contract. | Do not admit it to generic local signed-action outbox wrapping, replay, or response inspection. |

| Profile showcase/global sync | Visible profile originals must append or refresh canonical profile snapshot, public profile market append source, and visible local route snapshots. | Do not let the object disappear after refresh, owner close/reopen, or market/profile navigation. |
| Live player route | Profile original navigation must load the same live player object shape used by existing showcase posts. | Do not create a newer object shape that lacks the route/proof fields older objects carry. |

Saved profile usernames win; brand-new accounts without a saved username use the default profile username projection for `/vault/{sport}/{username}`. | Do not make public vault proof depend on private owner session state or manual profile setup. |
| Sports/FIFA/UFC/fight proof | Sports cards, packs, lineups, fight entries, soccer/FIFA players, and event proofs are proof objects. | Do not downgrade them into cards, UI widgets, or generic game inventory. |

| Runtime/SW coherence | Package version, service worker version, runtime build id, projection keys, and public release labels move together. | Do not bump one version surface without the others. |
| Stronger/weaker truth ladder | Receiz law, sealed proof, deterministic artifact state, verified local truth, verified register blocks, and foreground proof publication outrank server/session/cache/outbox/global sync. | Do not let a weaker layer replace, rename, delay, inspect, consume, or downgrade stronger truth. |

## Frozen Invariants

- `/api/receiz/anchor` is excluded from local signed-action outbox admission.
- No broad `/api/receiz` local signed-action classifier may re-admit the foreground anchor route.
- Foreground proof publication endpoints own their response contracts.
- Generic outbox, service worker, DB, server append, and global sync are weaker layers beneath foreground proof publication.
- A profile post is not globally complete because a DB row exists.
- Profile original writes must update canonical profile snapshot, public profile market append source, and visible local route snapshots.
- Existing showcase object shape is the compatibility law for new profile originals.
- Profile originals must not disappear after refresh, PWA close/reopen, or navigation back to the public profile.
- Profile original live-player navigation must use the same route/proof fields carried by older profile originals.
- Visible public profile originals belong in market.
- Sports Vault sealed manifest/image truth must exist before public showcase/server publication tries to project it.
- Sports Vault QR/showcase publication may use server helpers, but the sealed vault proof remains the stronger truth.
- Sports Vault public URLs must use saved profile username first and default profile username projection for new accounts with no saved username.
- Sports/FIFA/UFC/fight event proofs must remain proof objects and may not be demoted into generic cards.
- Wallet settlement and Receiz identity continuity must first-paint from stronger known truth before weaker session/runtime enrichment.
- Service worker version, runtime coherence, and public version labels must move with `package.json`.
- Regression lessons and full commit history are release artifacts for this release.
