## Frozen Invariants

- Receiz is a proof-native artifact system.
- Implemented primitives are named directly.
- A settled visible shell is primary reality.
- A settled visible shell is immutable unless the user explicitly interacts.
- Stronger truth may be decorated, synced, indexed, hydrated, displayed, or appended to by weaker truth.
- Weaker truth never replaces stronger truth.
- Weaker truth never renames stronger truth into a weaker category.
- Sealed artifact truth outranks server, DB, session, UI cache, and generic app convention.
- Durable local truth is not cache.
- Offline proof is not fallback mode.
- Public proof surfaces are not generic share pages.
- Ownership surfaces are not dashboard state.
- Settlement primitives are not balance displays.
- Deterministic first paint is not loading state.
- First paint must not serialize non-critical data to prove a visible shell that already has stronger truth.
- First paint must not paint empty/null when minimum visible truth is already available.
- Authenticated/private enrichment must never make signed-in users wait longer than signed-out users for the same truthful shell.
- SEO, analytics, metadata, personalization, DB refresh, session check, hydration, lazy import, dynamic JSON serialization, global sync, server reconciliation, tracking, marketing, and external framework convention never block or downgrade deterministic first paint.
- Background work may warm, append, hydrate, index, or refine only when it does not disturb the settled surface.
- If the app already knows stronger local/artifact truth, it reuses that truth immediately.
- Visible public profile proof objects must be present in public profile truth and Market proof truth.
- Hidden public profile proof objects must not remain visible in Market.
- Unhidden public profile proof objects must return to Market.
- Market direct entry and profile-to-Market navigation must converge on the same visible proof object set.
- Signed-in Market positions, history, and owned assets hydrate after paint without erasing public shell truth.
- A null, anonymous, stale, or partial authenticated snapshot cannot erase known owned state, public proof objects, local settlement, positions, or history.
- Buy and sell must visually execute from local settlement truth immediately, then hydrate global sync behind it.
- Executed sells must remove sold shares from the visible ownership surface immediately.
- Lagging snapshots cannot reopen a locally settled or claimed position.
- Explore must first-paint from deterministic proof truth.
- Explore identity pills must retain user image data needed by the visible surface.
- Sports user-live append warming must cover every relevant source date.
- Sports Main Event checkout dismissal or cancellation must not enter the user.
- Sports Main Event successful payment must verify paid settlement and enter immediately.
- Single-card QR proof pages must not replace existing vault card route/link authority.
- Series Passport local proof admission must work before weaker global sync.
- Series Passport payout math must match Main Event split: 80% prizes, 12% Receiz, 5% referral, 3% reserve.
- Series sharing must reuse the implemented affiliate/referral system.
- Profile OpenGraph and metadata enrichment cannot become a profile first-paint gate.
- Player/live share metadata must preserve playable social/iMessage behavior while route click-through uses the live player.
- Calendar wallet events must show correct deterministic rate when USD and PHI basis are present.
- Any change that makes a previously settled deterministic surface feel like it loads again is a regression.
- Agents run the primitive decision procedure before modifying primitive surfaces.
- Vague global uncertainty is forbidden. Exact uncertainty boundaries are required.
- External systems may inform how Receiz is built. They may never define what Receiz is.
- Generic convention is not neutral authority. Hedge weak inherited priors before strong implemented primitives.
- The user must not have to restate this hierarchy.

- Production Supabase targets must have every Sports, Series Passport, Signal Run vault, profile/public asset, original engagement, market, wallet, Explore, World, OTC, and refund/closeout migration applied.
- Live sports feeds remain external-service behavior; feed outages must not authorize stale locks, fake scores, or weaker DB/session state replacing known entry truth.

- Browser/PWA clients running an older service worker need the `86.0.0` service-worker update to take the new runtime shell.
- Final production tag and release attestation must be created from the checked candidate commit.
