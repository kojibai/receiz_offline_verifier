# Receiz v56.0.0

Release date: April 17, 2026

## Overview

`v56.0.0` is the release-cut and hardening release on top of the `v55.1.0` continuity baseline. The core profile, market, player, and route-continuity model introduced in `v55.1.0` remains the product floor; this release locks that floor in place, closes the fresh-account analytics regression, fixes logged-in world concierge entitlement churn, and moves every public version surface together to one canonical release number.

This release prepares one coherent `56.0.0` cut across package metadata, service workers, public verifier artifacts, footer/legal/public badges, plans/upgrade surfaces, and release docs.

The product-truth freeze for this release is defined in `docs/releases/v56.0.0-product-truth.md`.

## What Defines v56.0.0

- Release-governed version surfaces now move together to `v56.0.0` across `package.json`, the service worker, public verifier artifacts, and public release badges.
- The world route no longer trusts malformed or stale cached route snapshots; logged-in world hydration now protects itself before adopting cached truth.
- World concierge task polling is now entitlement-aware, so signed-in users without Automation access no longer hammer `/api/world/concierge/tasks` with repeating `403 world_automation_required` requests.
- Fresh-account profile analytics no longer fail the login path when the database is still on the older analytics event-name constraint; the server degrades cleanly while the release migration brings the schema back into sync.
- The `v55.1.0` continuity baseline remains intact: profile truth reuse, market continuation, USD-authoritative settlement, deterministic player presentation, and button-local route feedback stay as release requirements.

## Since v55.1.0

### 1. Release Surfaces Were Finally Cut Together

The repository had drift between live product badges, service worker defaults, offline verifier artifacts, and the package version. This release removes that split.

The current repository state now aligns:

- `package.json`
- `public/sw.js`
- `app/components/pwa/ServiceWorkerRegistration.tsx`
- public version badges on home, trust, make, footer, legal, upgrade, and plans surfaces
- public offline verifier and offline record seal artifacts
- release docs for the active version cut

The release number is now one value instead of several stale public echoes.

### 2. Logged-In World Reliability Was Hardened

Two logged-in world regressions were closed in this release-prep state:

- world route hydration now rejects malformed cached snapshots before the client can adopt and crash on incomplete bootstrap shape
- world concierge polling now checks the real Automation entitlement before polling or rendering live task controls

That means the world surface no longer breaks because of stale route cache shape, and signed-in non-Automation users no longer get a broken concierge lane backed by repeated 403s.

### 3. Fresh-Account Analytics No Longer Break First Login

Fresh owner accounts immediately fire owner-side profile analytics events such as `profile_activation_checkpoint`.

The app code already considered those events valid, but the database constraint set had not been updated to allow all of them. That created a real mismatch: first-login analytics could 500 on a new account even though nothing was wrong with the account itself.

This release does two things:

- adds the database migration that updates analytics event-name constraints and windowed rollups for the newer owner-side events
- hardens the server analytics path so stale schema drift degrades to `tracked: false` instead of throwing a 500 during login or first-use flows

That keeps first-account entry clean while still pushing the real schema into the release.

### 4. The v55.1.0 Continuity Model Remains The Baseline, Not An Experiment

`v56.0.0` does not replace the last release’s architecture. It formalizes it.

The locked baseline from `v55.1.0` remains:

- profile truth is loaded once and reused across `/profile`, public profiles, search handoff, and the global composer host
- market/chart `Buy` is one continuation path with passkey first, exact shortfall funding, and automatic order replay
- sell remains real-position-only and budget-first
- live-player fit and public twin precedence stay deterministic
- route feedback stays on the button surface itself with no artificial delay

## Public Summary

Receiz `v56.0.0` is the canonical release cut for the continuity architecture: version surfaces finally align, the world no longer exposes broken concierge polling to non-Automation accounts, and fresh-account analytics no longer throw server errors when the database is briefly behind the app’s event model.
