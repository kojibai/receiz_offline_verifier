# Receiz v50.0.0 Release Lock Checklist

Release date target: April 11, 2026

## Purpose

This checklist locks `v50.0.0` to one exact release state without allowing regression in durable Contacts truth, world revenue operations, campaign autopilot, rail readiness, provider telemetry, close-loop revenue attribution, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v50.0.0-product-truth.md`

Run from the exact release commit only:

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- `/world` loads on mobile with the compact operator shell, usable task-history popover, and no swipe-navigation conflicts inside world tabs
- the revenue launcher offers a money-first path with live rail/readiness truth
- a concierge run can source leads, import them into Contacts, and either send outreach or show a truthful blocked reason
- a run blocked on missing endpoints shows user-facing copy, not backend/database internals
- `Run again`, `Continue`, `Dismiss`, and `Clear done` work on live task history
- `/contacts` loads with the compact operator shell and preserves full action access without an immediate scroll wall
- importing a row that maps to an existing linked user merges into the existing Contact rather than failing
- a campaign with autopilot enabled shows due/blocked/reply state truthfully and advances after a heartbeat
- Resend/Twilio webhook delivery events appear in campaign history after a send
- public version badges and offline verifier surfaces all show `v50.0.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v50.0.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any contacts durability regression, world operator regression, campaign/autopilot regression, enrichment or provider-event regression, revenue-attribution regression, version-surface mismatch, release-lock check, or attestation step fails, `v50.0.0` is not released from that commit.
