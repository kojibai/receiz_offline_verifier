# Receiz v47.8.1

Release date: April 10, 2026

## Overview

`v47.8.1` is the contacts mobile compaction and import-merge patch release. It keeps the continuity hardening from `v47.8.0`, then tightens the contacts mobile shell into a one-viewpoint operating surface while fixing an import-merge gap where linked account upgrades were not always persisted onto existing contact records.

The product-truth freeze for this release is defined in `docs/releases/v47.8.1-product-truth.md`.

## What Defines v47.8.1

- The contacts mobile UI now keeps the primary shell compact by hiding the stat slab and deep record sections behind summary and tools overlays instead of stacking every section in the default viewport.
- Mobile operators can still reach overview, next-step, continuity context, recent activity, edit, invite, log, schedule, and full history without removing any existing contact functionality.
- Imported contact merges now treat a `linked_user_id` upgrade as a persisted change, so authenticated identity upgrades do not silently fail to write when the rest of the contact payload is unchanged.
- Release-governed version surfaces move together to `v47.8.1` across package metadata, service worker defaults, public badges, offline verifier surfaces, and release docs.

## Targeted Release Outcomes

### 1. Mobile Contacts Stay Operable In One View

The initial `/contacts` mobile frame should feel like an operator console, not a vertical stack:

- the main viewport keeps the command header, dock, and active relationship shell visible
- dense sections move into overlay panels instead of forcing immediate scroll
- contact actions remain reachable in one or two taps

### 2. No Contact Capability Is Removed To Gain Space

This release compresses, but does not cut:

- overview, next-step, continuity context, and recent activity stay available from summary tabs
- edit, invite, log, and full-history tools stay available from the mobile tools surface
- schedule, message, and call remain primary hero actions

### 3. Imported Identity Upgrades Persist Cleanly

Import-driven enrichment should not stop at guest or direct identity state:

- a merged contact now persists `linked_user_id` upgrades
- existing contact rows can upgrade into authenticated continuity without requiring some unrelated field change
- the merge contract is covered by a source-level contacts release test
