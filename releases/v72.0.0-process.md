# Receiz v72.0.0

## Goal

Ship a stable post-v70 release that keeps the hydration improvements that stopped stale open-app state while removing the user-experience regressions they introduced.

## What Worked

- Stale hydration fixes were useful because they refreshed app state after a user left the app open and returned later.
- Account switching did not need a rewrite; the deterministic bug was stale actor-scoped route continuity that could survive the switch.
- Viewer user id is the correct market continuity key because username can be temporarily blank during a local account projection.
- The mobile twin overlay needed a bounded primitive correction, not a shell-level viewport shove.

## What Did Not Work

- Treating blank session username as a wildcard let old signed-in market snapshots survive during account switching.
- Preserving richer wallet graphs without proving the same actor could leak old account market state into the new account.
- Removing all mobile overlay downshift fixed the old low-mouth failure but made eyes and mouth read too high on mobile.
- Broad visual claims are not enough for release; the fit math and contracts must fail if eyes or mouth drift again.

## Practical Rule For Future Work

Keep hydration additive and actor-scoped. If a change preserves continuity, it must first prove the same identity; if a visual fit changes on mobile, it must be bounded by detected face geometry and locked by a contract.
