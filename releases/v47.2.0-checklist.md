# Receiz v47.2.0 Release Lock Checklist

Release date target: April 9, 2026

## Purpose

This checklist locks `v47.2.0` to one exact release state without allowing regression in custom-domain root routing, forwarded-host handling, DNS setup execution clarity, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v47.2.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
npx tsx scripts/test_proxy_security_headers.ts
npx eslint proxy.ts app/account/page.tsx scripts/test_proxy_security_headers.ts
pnpm exec next build --webpack
pnpm release:lock
```

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- A previously connected and DNS-live custom hostname opens `https://custom.host/` on the bound public profile
- The same custom hostname still serves an existing non-root route without forced canonical redirect
- `/account` shows the DNS records for a Business user with direct copy controls for record name and value
- Public version badges and offline verifier surfaces all show `v47.2.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v47.2.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any routing regression, release-lock check, attestation step, or manual smoke check fails, `v47.2.0` is not released from that commit.
