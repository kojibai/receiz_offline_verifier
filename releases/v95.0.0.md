## Title

Receiz `v95.0.0` - Official Public Table Release

## Summary

`v95.0.0` is the clean public table release for Receiz as a proof-native artifact system. It carries forward the `v94.0.0` SDK launch and durable proof operating model, then removes the remaining language drift that made Kai/Klok sound optional, decorative, or SDK-specific.

The release locks the public developer contract to the internal canonical proof bundle contract: Kai Klok is the proof object state machine, and `kaiPulseEternal` / `kai_upulse` is the pulse unit. A complete Receiz proof object carries Kai. Without that living coordinate, it is not complete offline-verifiable Receiz proof truth.

## What This Release Makes Publicly Exact

- Receiz is not a generic app with proof features. It is a proof-native artifact system.
- The SDK is not a looser developer approximation. It exposes the canonical Receiz proof object contract.
- Kai Klok is not a vibe, badge, timestamp, or metadata. It is the state machine.
- `kaiPulseEternal` / `kai_upulse` is the unit carried by that state machine.
- Chronos is human display and external compatibility. It is not the backend authority for proof object order.
- Durable proof memory is not cache. First admission creates known truth; later truth appends.
- Server, DB, SDK, and API routes project, sync, index, and publish proof truth. They do not replace it.
