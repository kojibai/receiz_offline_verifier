# Receiz v98.0.0 Commit History

Baseline: `v97.0.0`

Commit count: `65 committed changes after v97.0.0 before the v98.0.0 release-lock edits`

Reviewed range: `v97.0.0..6cd0af58a`

This release document summarizes the complete working set after `v97.0.0`. The release-lock edits for `v98.0.0` add the final version alignment and documents on top of this reviewed range.

## High-Level Release Delta

- Developer runtime: `@receiz/sdk` gained app-state, public-store, commerce, media, domains, events, jobs, webhooks, permissions, audit, risk, compliance, portability, search, notifications, release pinning, sandbox, React, and offline proof queue rails.
- Durable proof memory remains first admission only, then append forever across `@receiz/sdk`, MCP, public proof, ledger, sports, market/profile, archive, and release identity work.
- Agent runtime: `@receiz/mcp-server` became the MCP package and agent interface over canonical SDK/API rails.
- Delegated access: OIDC agent-token routes and `/developers/connect` UI added scoped, revocable, one-time-visible delegated tokens for MCP/CI/trusted local agents.
- Merchant proof publish: signed public-store publishing uses Identity Seal / Receiz Key proof authority without delegated bearer-token authority.
- Developer webhooks: endpoint management, secret rotation, test events, signature helpers, OpenAPI, SDK/MCP support, and portal UI were added.
- Public proof graph: `/proof-graph` added live Kai pulse, proof/economy/flow/asset/agent views, detail panels, ledgers, market-cap, settlement, transaction, and mobile contracts.
- Ledger/transaction/settlement: wallet and public ledger routes added proof-bundle extraction, transaction pages, transfer value snapshots, Kai/upulse display, public pagination, cursor fixes, and settlement rollups.
- Market/profile first paint: profile publish append and first-paint fallback work preserved market membership as a write-side projection obligation.
- Sports vault/arena: owner-current projections, sport-aware image behavior, base issue labels, deferred surfaces, bounded rollup workers, active-job dedupe, transfer routes, value snapshots, and performance contracts were added.
- Pitch Command archive: day proofs through July 7, 2026, manifest/projection updates, strict archive parsing, workflow catch-up, and lint-safe proof-register typing were added.
- Receiz AI skills: builder, commerce, distribution, MCP agent, offline verifier, proof, skill bundle, and sports card skills were added with proof-native language.
- Release identity: app, SDK, MCP, service worker, offline verifier, public labels, and release documents are locked to `98.0.0`.

## Release-Lock Requirement

`pnpm release:lock 98.0.0` must require this document for `v98.0.0` and later release cuts when the release brief includes a commit-history artifact. A release that omits the commit-history boundary is incomplete because the `v97.0.0` baseline, 65-commit reviewed range, SDK/MCP developer runtime, delegated OIDC tokens, signed merchant public-store rail, developer webhooks, proof graph, ledger/transaction/settlement work, market/profile projection repairs, Sports Arena performance, Pitch Command archives, Receiz AI skills, and repo-local release-lock edits would not be explicit release truth.
