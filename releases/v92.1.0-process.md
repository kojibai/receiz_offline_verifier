# Receiz v92.1.0

Release date: June 22, 2026

## Goal

Ship `v92.1.0` as the Sports proof telemetry lock after `v92.0.0`.

The goal is not a generic UI polish release. The release locks exact proof behavior:

- Play Command visualizes full play telemetry and has a standalone public proof route.
- Pitch Command opens the exact event pitch witness from historical card/event proof pills.
- Event proof pages project the same video and milestone truth available in card popovers.
- Active Sports Arena cards receive foreground live appends immediately.
- Pack Derby keeps its settled opening and proof-game surface.
- Version, service-worker, runtime, offline verifier, and release docs move together.

## What Worked

- The event-panel Pitch proof issue had a concrete source: the action path could fall back to the current Pitch Command state instead of the selected event witness.
- The missing event video issue was a projection split: the logged-in card popover had playable-video proof material that the public event page did not reuse.
- The live append issue was cadence, not missing proof: a stale guard was correct for background pressure but wrong for active foreground viewers.
- The Vercel failures were real build blockers and were fixed with narrow nullable-type corrections.
- The `v92.0.0` release-lock structure gave this release an exact schema for product truth, process, regression lessons, checklist, and commit history.

## What Did Not Work

- Dense Play Command controls could dominate the mobile field surface, contradicting the product law that the field/play telemetry is primary.
- The Derby first-scene stabilization work touched the opening path and introduced forced-close behavior.
- Event proof public pages and logged-in popovers diverged even though they were projecting the same proof object.
- Foreground live Sports Arena activity inherited a background append stale guard, which let some events appear late even though they were eventually present.
- Archive workflow commits can fail deployment when Vercel cannot map the commit author to a GitHub account.

## Practical Rule For Future Work

When a Sports proof exists in one Receiz surface, the next surface must reuse that proof identity and projection directly.

Do not re-answer proof questions through current live state, DB freshness, route-local UI state, or background sync policy. Event proof, pitch witness, play proof, video proof, milestone truth, archived pitch proof, and card append truth must project immediately from the strongest proof material already held, with weaker transport only appending beneath it.

## Release-Lock Requirement

`pnpm release:lock 92.1.0` must require `docs/releases/v92.1.0-regression-lessons.md` and `docs/releases/v92.1.0-commit-history.md`. A `v92.1.0` release that omits either document is incomplete because the regression lessons and exact commit-history boundary would not be explicit release truth.
