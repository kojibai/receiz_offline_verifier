## Title

Receiz `v82.0.0` - Sports Arena World-Class Gameplay Release

## Summary

`v82.0.0` is the release where Receiz moves from proof-native platform plus Signal Circuit into a full live sports game economy.

The release base before the final `v82.0.0` lock edits is `edb39fa11a`, 174 commits after tag `v80.0.0` and 138 commits after tag `v81.0.0`. With the `v82.0.0` release-lock worktree applied, the complete `v80.0.0` to `v82.0.0` arc spans 242 changed files, 125,626 insertions, and 508 deletions including captured QA evidence. Excluding Playwright and screenshot evidence artifacts, the code and documentation arc spans 185 changed files, 56,576 insertions, and 508 deletions. The post-`v81.0.0` release line spans 140 code and documentation files, 47,858 insertions, and 134 deletions.

`v80.0.0` made Receiz a complete proof, identity, value, creator, world, and Signal Circuit operating platform.

`v81.0.0` made Signal Cards feel like durable Receiz assets with DNA, funded reward sealing, card activity, card transfer history, and mobile card polish.

`v82.0.0` makes Sports the next primary network: live MLB games, owned cards, pack issuance, rarity odds, Pick 5 skill, daily tournaments, per-game play, live scoring, card careers, market depth, trading, conformance, and scale hardening now move through one proof/value/game loop.

The product outcome is direct:

```txt
owned cards become playable inventory
live games become scoring truth
scoring becomes rank, proof, reward, memory, value, and repeat play
```

This is not a sports skin. It is a proof-backed live competition economy.

## What Is New

### Sports Arena Is A Real Product Surface

Receiz now has a full Sports experience rather than a partial game panel.

The arena has sport navigation, MLB live rows, Pick 5, live field, live cockpit, leaders, proofs, cards, profile access, mobile bottom chrome, and Sports-specific routes. The `/game` surface now points to Sports with a dedicated multi-sport SVG icon, and `/game/sports` carries global navigation without covering the game.

The outcome: Sports is now a first-class Receiz destination.

### MLB Cards Became Playable Inventory

The MLB card set moved from static catalog to owned, issued, playable inventory.

Cards now support:

- base-set display without fake rarity labels
- owned rarity variants only when the user actually owns them
- deterministic card issue numbers
- legacy issue backfill and issue assignment
- rookie registry support
- exact pack origin on downloaded cards
- rarity-specific visual treatments
- card viewer rarity switching across owned versions
- highest-rarity then lowest-issue owned-card ordering
- full downloaded card metadata and premium card export

The outcome: every card can have a birth, issue number, owner, history, pack origin, live career, and market path.

### Rarity And Player Strength Are Separate

`v82.0.0` locks the Sports economy law:

```txt
player strength is a player scarcity gate
card rarity is a variant issuance gate
both can compound, but neither replaces the other
```

A low-rated player can have a rare card. A high-rated player can appear as a base card. A Mythic high-rated card is hardest because both gates stack.

The UI and simulator now respect that separation:

- base cards do not show fake rarity labels
- rarity filters operate only on owned rarity variants
- published odds reflect deterministic issuance math
- observed simulation output validates published odds
- pack health catches bad player concentration and bad rarity availability
- free packs, starter packs, premium packs, rewards, and chase packs all keep the same conceptual model

The outcome: rarity feels rare, not cosmetic.

### Pack Odds Became Auditable

Sports pack simulation now reports launch-grade health, published odds, observed odds, player scarcity, spread, duplicate pressure, top-player concentration, and registry basis hashes.

The simulator now supports large samples and exposes enough detail to catch misleading odds before users do.

The outcome: odds are inspectable, repeatable, and governed by code instead of copy.

### Pick 5 Became Clearer And More Strategic

Pick 5 now separates the two game modes:

- per-game play uses owned cards from the selected matchup teams
- Main Event uses the user's best 5 across the slate
- per-game play can accept fewer than five owned matchup cards
- selected cards rise to the top
- unselected cards read differently without breaking card identity
- card picker scores are scoped to the selected game/date/context
- the button language changes with game mode
- final games cannot be entered as live games

The outcome: users know what they are entering, what cards count, and why the button says what it says.

### Live Cockpit Became Watchable

The live cockpit moved Sports from static cards into a screen users can stare at.

It now shows:

- selected scoring cards
- scoped live deltas
- score total with correct positive/negative color
- rank and payout position for Main Event
- in-the-money or distance-to-payout language
- best card, next spark, momentum, and scoring story
- compact panels that preserve mobile space
- collapsible behavior alongside Pick 5
- line-chart movement instead of dead bar charts
- stock-style value cues and live pointers

The outcome: the user can watch cards earn, lose, move, and matter in real time.

### Live Game State Is Deterministic

The latest Sports patch locks a key law:

```txt
entry lock state is not field state
selected game field data must match the selected game
active entries cannot rewrite live scoreboards
finalized proof fallback cannot hijack live play
```

That fixes the class of bugs where a selected game could display a stale field, a first live game, or a misleading "locked" last-play line.

The outcome: live rows, field, cockpit, and entry state no longer fight for authority.

### Pregame Became A Real Matchup Surface

Pregame is no longer a dead state.

When the selected game has not started, the field now shows the selected matchup, venue, first-pitch readiness, and the highest-rated team star cards from both clubs. The surface shows rating, projected score, stat line, and card access without pretending a live play has happened.

The outcome: the game feels alive before first pitch, and users can scout the matchup without leaving the field.

### Daily Tournaments Became A Real Economy

The Main Event tournament path now carries the daily prize arena loop:

- entry ticket and wallet funding
- Stripe delta recovery through wallet checkout
- minimum participant logic
- top-percent payout model
- daily tournament view
- joined state on first paint
- finality and result pages
- settlement payout paths
- underfilled refund support
- last-winner and result share surfaces
- play-again reset logic

The outcome: the Main Event is no longer a mock tournament. It is a repeatable daily arena architecture.

### Per-Game Play And Main Event Are Decoupled

Single games and Main Event are now separate lock domains.

A user can:

- lock into a single matchup for free play
- join the Main Event separately
- see the correct cockpit for the selected context
- claim final rewards without accidentally re-entering old games
- move to upcoming games after claimed finals clear

The outcome: "this game" and "today's event" no longer collapse into the same state.

### Finality Became A Product Moment

Finality now has dedicated paths instead of hidden state transitions.

Sports entries can finalize, rank, issue rewards, open reward packs, update proofs, append card career memory, and move the UI forward. Final games stay selectable when rewards need claiming, and claimed proof surfaces clear instead of re-locking old entries.

The outcome: users can finish the day, claim the result, and start the next one without refresh rituals.

### Card Memory Became Long-Term

Sports card memory now has dedicated append infrastructure.

Cards can receive:

- arena entry memory
- rank memory
- reward memory
- score totals
- source date
- game key
- lineup scope
- append hashes
- provenance rows
- value and ownership context

The outcome: cards can be used for years without losing their career.

### Sports Market Became A Real Market Surface

The Sports card market now includes:

- sales pages
- purchase flow with wallet-first and Stripe delta funding
- provenance routes
- public card stages
- value panels
- bids, asks, floors, spread, volume, and value movement
- stock-style line charts
- compact market panels
- card listings
- card sale history
- market conformance isolation

The outcome: Sports cards can be played, inspected, listed, bought, valued, and shared from one product system.

### Trading Became A Serious User Flow

Sports trading now supports card proposals, trade action routes, guarded accept/decline/cancel behavior, trade baskets, counterproposal direction, and robust status handling.

The release also fixes a race condition: declining or cancelling a trade must return `409 sports_trade_not_open` when the row is no longer open, and unexpected lookup failures must stay `500 sports_trade_lookup_failed`.

The outcome: users can negotiate real card movement without silent stale-state success.

### Sports Conformance Became Official

Sports now has its own conformance suite as a child of Game:

`v82.0.0` includes the first real Sports scale layer:

### PWA And First Paint Stability Improved

The Sports page now hydrates account state more aggressively on first paint without waiting for the user to press Arena. Entered state, cockpit state, card catalog hydration, and logged-in projection all move earlier.

Profile signing and newly posted player routes also received surgical continuity improvements:

- PBI signing updates local original state immediately after signature
- live player hydration can accept optimistic post anchors
- signing controls prewarm before first action
- newly posted profile content no longer needs a leave-and-return cycle to show the stronger state

The outcome: the app uses truth it already has instead of making users refresh.

### Sports Share, Sales, And Downloaded Cards Became Premium

Sports card pages gained a mobile-first sales/share stage, premium card stage, tap/swipe flip, tighter buy panel, full metadata, pack ID, issue number, provenance, rarity treatment, and export alignment with the viewer image window.

The outcome: a Sports card looks and behaves like a premium Receiz object, not an inventory row.
