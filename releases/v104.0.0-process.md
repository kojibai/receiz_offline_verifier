## Goal

Ship the Receiz App Contract Compiler across `@receiz/sdk`, MCP, CLI, Next.js
adapter, templates, and `@receiz/ai-skills` as one production release.

## What Worked

Test-first focused modules kept one deterministic SDK rules engine beneath every
interface. Packed tarballs installed into an empty external project and proved
exact 104.0.0 alignment with zero `workspace:` dependencies. Durable proof
memory remains **first admission only, then append forever**.

## What Did Not Work

The abandoned MCP 103.0.1 patch was not released because v104 became the
coordinated feature boundary. The first external smoke attempted registry
resolution before v104 existed; local tarball overrides corrected the
pre-publication harness without altering packed manifests.

## Practical Rule For Future Work

Keep app integration rules in `@receiz/sdk`; expose them through thin CLI/MCP
interfaces; inspect the packed artifact and install it outside the workspace
before publication.
