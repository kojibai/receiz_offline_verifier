# Receiz v47.0.1 Product Truth

Release date: April 5, 2026

This document freezes the primitives and invariants that `v47.0.1` is allowed to extend but not weaken.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Authoritative file bytes | Verification authority remains the original file bytes plus versioned proof material. Upload transport and storage handoff logic may change, but the stored bytes and proof chain remain the truth. | `docs/receiz-standard-v1.md`, `scripts/test_receiz_artifact_binding.mjs`, `scripts/test_receiz_signature_v4.ts` |
| Deterministic historical manifests | Canonical history resolves from published deterministic manifests, not mutable client state or optimistic decoration. | `docs/verified-history-first-principles.md`, `app/lib/profile/__tests__/deterministicFeed.test.ts` |
| Profile original upload handoff | Profile composer uploads must keep the authoritative storage handoff intact for videos and must not treat a storage-confirmed seal as failed just because a public URL is deferred. | `app/lib/profile/__tests__/profileOriginalSignedUploadContract.test.ts`, `app/lib/profile/__tests__/profileOriginalUploadRouteContract.test.ts` |
| Profile artwork preload discipline | Non-critical profile artwork warming must not escalate into broad head-preload spam; only explicit high-priority images may emit head preload links. | `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Release-governed version surfaces | Package version, release docs, public version badges, and service worker defaults must move together on each release. | `scripts/check_release_lock.mjs` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Video upload progress cannot disable the real handoff path | Showing upload progress must not force the composer onto a weaker upload branch that regresses real browser behavior. | `app/lib/profile/__tests__/profileOriginalSignedUploadContract.test.ts` |
| Storage-backed seals remain publishable | A successful storage handoff must still produce a valid publish path even when no immediate public URL is returned. | `app/lib/profile/__tests__/profileOriginalSignedUploadContract.test.ts` |
| Profile route continuity stays primed without preload waste | Profile route warming stays focused on visible showcase items and avoids non-critical head preload spam. | `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Canonical history cannot be downgraded | Weaker partial payloads, augmentation, or live deltas must not replace a stronger canonical block. | `app/lib/profile/__tests__/deterministicFeed.test.ts` |
| Release surface inventory stays canonical | The committed release surfaces and version badges must align with the package and service worker release. | `scripts/check_release_lock.mjs` |

## Release Gates

`v47.0.1` is considered locked only if these pass together:

- `npx tsx --test app/lib/profile/__tests__/profileOriginalSignedUploadContract.test.ts app/lib/profile/__tests__/profileOriginalUploadRouteContract.test.ts app/lib/profile/__tests__/profileRouteContinuityContract.test.ts`
- `pnpm exec next build --webpack`
- `pnpm release:lock`
- `pnpm release:attest`
- `pnpm release:verify-attestation <path>`

The manual release procedure is defined in `docs/releases/v47.0.1-checklist.md`.

## Residual Risks

The following areas are materially stronger after this release but are not yet absolute:

- The authenticated browser smoke proves the real profile composer path locally, but production-like smoke still depends on valid auth fixtures and Supabase-backed environment availability.
- Profile preload discipline is tightened for current showcase warming behavior, but future new warming entry points still need to preserve the same head-preload restraint.
