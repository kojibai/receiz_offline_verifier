# Receiz v60.1.0 Product Truth

Release date: April 18, 2026

This document freezes the primitives and invariants that `v60.1.0` is allowed to extend but not weaken after the `v60.0.0` stable baseline.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Release-governed version surfaces stay canonical | `package.json`, service worker defaults, public version badges, offline verifier artifacts, and active release docs must all expose `v60.1.0` together when the cut is made. | `package.json`, `public/sw.js`, `app/components/pwa/ServiceWorkerRegistration.tsx`, public page/version surfaces, `scripts/check_release_lock.mjs` |
| Public market witness truth is holder-aware | Public profile asset markets must derive holder count and latest purchase witness from real public positions and buy activity when available, not just creator timestamp fallback. | `app/lib/market/publicProfileAssets.ts`, `app/lib/market/compose.ts`, `app/market/MarketDeskExperience.tsx`, `app/lib/market/__tests__/marketWitnessContract.test.ts` |
| Compact market presentation stays readable without hiding truth | Public and signed-out desk surfaces, chart dock fit, timestamps, captions, and manifest metadata must remain dense but legible. | `app/market/MarketDeskExperience.tsx`, `app/components/valuation/chart/LiveChart.tsx` |
| Settled twin shells are visually immutable until explicit intent | Background preload may warm code and data, but it must not swap or remount the visible home or profile shell before explicit open. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts`, `app/lib/world/__tests__/worldTwinSessionReuseContract.test.ts`, `app/lib/world/__tests__/worldTwinStageSeparationContract.test.ts` |
| Market sigil namespaces stay deterministic across hydration | Market pills on profile, player, hero, and artifact surfaces must use stable scope IDs instead of render-order-sensitive namespaces. | `app/components/market/MarketValuePill.tsx`, `app/u/[username]/ProfileClient.shared.tsx`, `app/[username]/verified/[receizId]/player/VerifiedOriginalPlayerExperience.tsx`, `app/[username]/originals/[receizId]/OriginalArtifactViewer.tsx`, `app/components/HeroLogoVideo.tsx` |
| Public profile route truth stays lighter without shell drift | Lazy-loaded route assembly may reduce initial bundle work, but it must not introduce client-only truth gaps or imply silent full-surface completion after settlement. | `app/[username]/UserProfileRouteClient.tsx`, `app/u/[username]/ProfileClient.tsx`, `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Market continuation remains same-surface and position-aware | Buy continuation must stay passkey-first and same-surface, and sell must stay hidden when the viewer has no position. | `app/market/MarketDeskExperience.tsx`, `app/components/market/MarketValuePill.tsx`, `app/lib/market/__tests__/marketTradeContinuationContract.test.ts` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Public holder count prefers real positions | Public profile asset cards must not fall back to generic envelope holder counts when live public position count is known. | `app/lib/market/publicProfileAssets.ts`, `app/lib/market/compose.ts`, `app/lib/market/__tests__/marketWitnessContract.test.ts` |
| Latest witness time prefers holder activity | Latest witness timing must not prefer creator seal or post time over known public holder purchase activity. | `app/lib/market/publicProfileAssets.ts`, `app/market/MarketDeskExperience.tsx`, `app/lib/market/__tests__/marketWitnessContract.test.ts` |
| Witnessed duration cannot stay flat when the system already knows activity | Real purchase, position-open, or engagement activity must not still render as zero witness time. | `app/market/MarketDeskExperience.tsx`, `app/lib/market/__tests__/marketWitnessContract.test.ts` |
| Background twin preload never auto-activates the full module | Remembered twin session state or preload completion must not auto-promote the full twin module on load. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts`, `app/lib/world/__tests__/worldTwinSessionReuseContract.test.ts` |
| Visible twin shell never swaps after settlement | The home or profile dock shell must stay visually stable until the user explicitly opens the richer twin surface. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/__tests__/worldTwinStageSeparationContract.test.ts` |
| Market pill call sites always provide stable sigil scope | New `MarketValuePill` surfaces must not reintroduce `useId()`-driven SVG namespace drift. | `app/components/market/MarketValuePill.tsx` plus current pill call sites |
| Profile surface does not silently full-hydrate itself after settlement | Profile must not schedule a background full-showcase hydration path that makes the surface feel like it reloads itself after it already looks complete. | `app/u/[username]/ProfileClient.tsx`, `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Release surfaces remain canonical | Public version badges, service worker version, offline verifier surfaces, and release docs must align with `package.json` when the version cut happens. | `scripts/check_release_lock.mjs` |

## Release Gates

`v60.1.0` is considered locked only if these pass together:

- `npx tsx --test app/lib/market/__tests__/marketTradeContinuationContract.test.ts app/lib/market/__tests__/marketWitnessContract.test.ts`
- `npx tsx --test app/lib/profile/__tests__/profileRouteContinuityContract.test.ts`
- `npx tsx --test app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts app/lib/world/__tests__/worldAvatarFreshnessContract.test.ts app/lib/world/__tests__/worldTwinSessionReuseContract.test.ts app/lib/world/__tests__/worldTwinStageSeparationContract.test.ts`
- `pnpm build`
- `pnpm release:lock`
- `pnpm test:release-freeze`

The manual release procedure is defined in `docs/releases/v60.1.0-checklist.md`.

## Residual Risks

- Holder-activity rollups degrade to zero on timeout or fetch failure. If public position or trade reads are unavailable, witness surfaces can flatten instead of failing loud.
- Sigil hydration stability now depends on every new `MarketValuePill` call site continuing to pass a stable `sigilScopeId`.
- Twin-shell continuity is explicit in the shared deferred module, but any future deferred visible surface can reintroduce late promotion if activation gets tied to preload or remembered session state again.
- `v60.1.0` is not fully release-cut until public version surfaces, service worker defaults, offline verifier surfaces, package metadata, and docs all move together to the same version.
