# Receiz v52.5.0 Release Lock Checklist

Release date target: April 13, 2026

## Purpose

This checklist locks `v52.5.0` to one exact release state without allowing regression from the `v52.0.0` market baseline in deterministic market execution, spread-governed sustainability, replayable quote evidence, partial share-certificate custody, market conformance, premium runtime hardening, draggable market-value tooling, inline market launch creation, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v52.5.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm exec tsc --noEmit --pretty false
pnpm test:market-conformance
pnpm test:release-freeze
pnpm release:lock
pnpm run build
```

If the candidate deploy also requires world, settlement, verification, or contacts release gates, those should still pass before tag creation even if they are not restated here.

## Production Environment + Market Preflight

The production database must already include the active market rails:

- `supabase/migrations/20260413041500_market_share_positions.sql`
- `supabase/migrations/20260413070500_market_prediction_books.sql`
- `supabase/migrations/20260413102000_market_execution_guards.sql`

- `supabase/migrations/20260413153000_market_share_trade_fees.sql`
- `supabase/migrations/20260413174500_market_quote_engine_snapshot.sql`

The deployed app environment must still include the standard app/session/database rails required for signed-in market execution.

Before release, confirm:

- market trades can persist quote snapshot fields successfully
- share certificate import/export paths are live only after the partial-balance migration is active
- `GET /api/market/conformance` returns `pass`
- `GET /api/market/conformance/badge` renders normally
- Vercel/Next tracing includes the market conformance source set required by the suite

## Manual Smoke

Confirm these flows on the candidate without changing code:

- `/market` opens with the dark premium shell and does not feel like it is rerendering continuously while idle
- opening/closing desk panels does not trigger hydration mismatch warnings
- one asset can be bought and sold through the live market path with quote evidence present afterward
- partial share-certificate withdrawal reduces the live position by the selected quantity
- partial share-certificate redeposit restores only the selected quantity
- profile, artifact viewer, and live player each render the refined market value pill correctly
- the market value pill opens without backdrop blur and can be dragged around the viewport without escaping it
- the value-pill trade surface still routes cleanly into the full market desk
- the launch panel sections open as slide-down disclosures and can record-and-seal a memory, seal a file-backed asset, and create a venture inline without routing out to another composer
- market audio playback can crossfade into the next market track without dropping out after one song
- the market-only bottom player skins to the active asset instead of using the default blue player shell
- `/market/assets/[receizId]` renders a focused market route with a valid Open Graph/Twitter card
- public version badges and offline verifier surfaces all show `v52.5.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v52.5.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the attestation artifact includes the market conformance result.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any quote-determinism regression, spread/evidence persistence regression, certificate-balance regression, market-pill runtime regression, draggable-overlay regression, inline-launch regression, market-conformance regression, version-surface mismatch, release-lock check, or attestation step fails, `v52.5.0` is not released from that commit.
