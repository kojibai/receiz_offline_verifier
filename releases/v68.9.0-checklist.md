## Purpose

This checklist locks `v68.9.0` to the restored `v68.8.1` profile-law behavior plus the already-landed auth/runtime hardening on the current line.

Do not release `v68.9.0` unless the product does all of the following at once:

1. paints `/<username>` as canonical public truth
2. treats `?manage=1` as additive owner truth only
3. keeps signed-in users at least as fast as signed-out users on the same base shell
4. preserves proof-first auth/runtime recovery and stale-runtime resilience

## Manual Launch Tests

### Profile law

- Open `/<username>` signed out and signed in.
- Confirm the first settled shell is the same profile identity and showcase in both cases.
- Open `/<username>?manage=1`.
- Confirm owner controls append without redefining the base shell.
- Hide and unhide older profile items.
- Confirm previously hidden inventory stays present and does not reappear on the public shell.

### Navigation speed

- Tap Profile from the bottom bar while signed in.
- Confirm navigation resolves immediately to the username-managed route instead of waiting on discovery.
- Confirm the page does not feel loaded and then loaded again.

### Auth/runtime

- Resume from stale session/service-worker state.
- Confirm the app recovers through local proof or server resync without a visible crash loop.
- Confirm Receiz ID continuation and PBI recovery still bind and project locally first.

## Release Rule

Do not release `v68.9.0` if any smoke or gate shows:

- canonical public profile truth being replaced by owner/session/workspace truth
- `?manage=1` redefining instead of appending
- hidden profile inventory disappearing or reappearing incorrectly
- signed-in profile entry waiting longer than public entry
- a settled screen visibly loading again
- stale runtime recovery, Receiz ID continuation, or binding flows regressing
