# Receiz v55.1.0 Release Lock Checklist

Release date target: April 16, 2026

## Purpose

This checklist locks `v55.1.0` to one exact release state without allowing regression in profile truth continuity, search/profile hydration parity, same-surface market continuation, USD-authoritative settlement, live-player fit, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v55.1.0-product-truth.md`

## Production Environment + Governance Preflight

The production database must already include the active market, profile, and world rails from earlier releases.

For the `v55.1.0` release-prep state, confirm the target environment also includes:

- `supabase/migrations/20260416025923_market_nonentitled_usd_execution.sql`
- `supabase/migrations/20260416041200_market_prediction_nonentitled_usd_execution.sql`

Before release, confirm:

- passkey auth, session bootstrap propagation, and magic-link continuation all resolve on the same chart and desk trade-continuation path
- the trade-funding modal still uses the approved embedded checkout pattern and returns through the pending-order continuation path
- the market database is running the USD-authoritative non-entitled execution functions expected by the current API and settlement layer
- public/offline verifier surfaces, service worker version, `package.json`, and public version badges all show `v55.1.0` when the final version cut is made
- `public/governance/governance-keyring.v1.json` still contains the active release-attestation key ID if release attestation is generated for this cut

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- `/profile` direct load stays fully hydrated for signed-in users and does not show an extra shell or second settle
- logged-out `/profile` still routes correctly to signin instead of exposing the owner surface
- floating people search into a public profile lands fully hydrated, including header image and joined metadata, without needing a reload
- opening the bottom-right profile composer, posting, closing, and reopening keeps the surrounding profile current instead of feeling stale
- artifact-viewer and live-player buttons from profile open on the first press and transition immediately with button-local pending feedback
- account to dashboard transitions immediately and shows route-pending feedback on the button surface itself
- chart `Full` and desk entry reuse already-loaded market truth instead of feeling like a cold reload
- signed-out `Buy` runs passkey directly; if funding is needed it opens the exact-shortfall funding modal and auto-resumes the order after payment
- cancelling or closing funding after a successful passkey still leaves the user signed in to the created or authenticated account
- passkey unsupported, cancelled, or failed falls back to the magic-link path and resumes the exact pending buy on return
- chart and desk `Sell` only appear when the viewer actually holds that asset, accept USD intent, and derive shares from the live bid
- live-player back-card copy fits inside frame and public twin presence is deterministic when route conditions are valid
- public version badges, service worker version, offline verifier surfaces, and release docs all show `v55.1.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v55.1.0`.
2. Run the `Release Governance Attestation` workflow if the release process for this cut still requires a signed attestation.
3. Verify the generated artifact with `pnpm release:verify-attestation <path>` if an attestation is produced.
4. Confirm the attestation references the published key ID and the expected release version.
5. Store the attestation path and approval reference with the release record.

## Release Rule

If any profile-truth regression, search/profile hydration regression, route-feedback regression, market continuation regression, USD-settlement regression, sell-lane regression, live-player fit regression, public-twin regression, version-surface mismatch, or release-lock failure occurs, `v55.1.0` is not released from that commit.
