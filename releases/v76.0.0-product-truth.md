## Frozen Primitives

| Concept | v76.0.0 truth | Must not regress |
| --- | --- | --- |
| Product law | Literal doctrine, v70 profile law, v74 identity-artwork law, and v75 local-proof law still outrank implementation convenience. | Do not weaken first paint, local proof, file restore, profile canonical truth, or value-loop continuity. |
| Release diff | The release diff is measured from tag `v75.0.0` to the `v76.0.0` candidate. | Do not describe v76 as a label-only bump or omit the post-v75 product changes. |
| Concierge transport | Normal Concierge replies use direct Responses streaming. Agents SDK is additive for agentic work. | Do not make the SDK transport a blocker for ordinary replies or tool failures. |
| Agent architecture | Tools, handoffs, tracing, and orchestration are valuable only when they produce visible user artifacts and safe next actions. | Do not expose raw traces, fake work, or suppress the assistant response while tools run. |
| World value loop | World tasks become missions that create leads, contacts, drafts, sends, follow-ups, revenue attribution, and payment/booking paths. | Do not end agent work with vague summaries when a concrete revenue action can be produced. |
| Campaign delivery | Follow-up delivery defaults to email unless a valid connected-app destination was explicitly configured. | Do not block campaigns on missing app destinations or treat invalid webhook state as user intent. |
| Market Twin | Market Twin opens fast from cached market truth, hydrates saved authority in place, explains moves, gates actions, and records value. | Do not reintroduce loading gates, reset saved settings, fake returns, or bypass mandate/risk checks. |
| Market replay | Replay is deterministic, local to held market snapshot truth, and labels simulated results honestly. | Do not fetch hidden market truth before first paint or present replay as guaranteed yield. |
| Public Proof | Public Proof is a governed observe, claim, verify, seal, dispute, rights, identity, creator, and registry surface. | Do not expose protected mutation routes as public status, and do not hide public proof reads from the public/developer contract. |
| Wallet settlement | Reserve credits, non-subscription wallet settlement, and running reserve balances remain deterministic value truth. | Do not double count reserve credits or let payment context redefine wallet value state. |
| Status inventory | Status is a public/developer trust surface, not an internal route dump. | Do not publish admin, webhook, internal-worker, private session-only, or privileged issuer routes. |
| Account surfaces | Account frames operating surfaces and exposes Market Twin and Public Proof as first-class user tools. | Do not regress to generic product framing or hide core value surfaces. |
| Release surfaces | Package version, service worker defaults, public badges, verifier surfaces, README release pointer, and release docs move together. | Do not ship version drift. |

## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint beats richer rediscovery.
- Stronger known deterministic truth is reused before any network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base public shell.
- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains canonical profile truth plus owner permissions.
- Concierge must answer even when agentic tools are unavailable, blocked, or not needed.
- Agent tools must produce explicit user-facing artifacts, safe holds, or next actions.
- Follow-up campaigns use email by default unless the user explicitly chose a valid app webhook.
- Market Twin settings are user mandate truth and survive navigation.
- Market Twin does not claim profit that is not realized, replayed, or explicitly simulated.
- Public Proof public reads belong in public/developer trust surfaces; protected mutation routes require session authority.
- Status should increase trust for real users and developers without increasing attacker value.
- Wallet value, reserve credits, and ledger events must remain deterministic and auditable.
- Release gates must catch visible behavior regressions, not merely match the latest code.

## Residual Risks

- The release is not fully locked until `pnpm lint`, `pnpm build`, `pnpm test:release-freeze`, and `pnpm release:attest` pass after the `v76.0.0` surface bump.
- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
- Production database migrations for Public Proof, wallet reserves, identity key compatibility, and Market Twin authority must be applied in the target environment.
- External services including OpenAI, Stripe, Resend, Twilio, Supabase, Redis, storage, and media/render providers remain deployment concerns, but they must not become permission to downgrade already-held deterministic truth.
- Status probes must remain safe and curated as new public/developer routes are added.
