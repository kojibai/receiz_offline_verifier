## Frozen Primitives

| Primitive | v97.2.0 truth | Must not regress |
| --- | --- | --- |
| Proof object | Complete Receiz proof objects carry deterministic identity, sealed/verifiable payload, provenance, inspection behavior, and Kai/Klok state. | Do not let SDK helpers, React components, routes, docs, or examples treat proof truth as generic app state. |
| Kai Klok | Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit. | Do not use Chronos as the primary state machine for admitted proof objects. |
| Durable proof memory | Product and SDK surfaces use first admission only, then append forever. | Do not call admitted proof memory cache, stale data, hydration state, or rediscovery state. |
| Enterprise app runtime SDK | Customer, merchant, commerce, media, event, job, permission, audit, risk, compliance, portability, search, notification, release, and offline proof queue helpers compose existing primitives. | Do not let SDK convenience become issuer, verifier, database authority, identity root, settlement authority, or proof authority. |
| Idempotent writes | New write helpers accept idempotency material so interrupted or offline app flows can safely replay writes. | Do not let replay behavior duplicate ownership, settlement, inventory, audit, media, or proof append effects. |
| React SDK subpath | `@receiz/sdk/react` exposes hooks and hosted component factories for developer ergonomics. | Do not let hosted UI controls outrank proof objects, identity primitives, ownership appends, settlement rows, public projections, or verified appends. |
| Connect | Connect is delegated API access for third-party apps. | Do not let Connect become the identity proof root or a replacement for Receiz ID, Identity Record, Identity Seal, Receiz Key, PBI, or artifact proof. |
| SDK package | `@receiz/sdk` version must match the app release version and expose canonical convenience beneath Receiz primitives. | Do not release an SDK package whose package version, runtime constant, docs, React subpath, or app release identity diverge. |

## Frozen Invariants

- Use known verified truth immediately.
- Ask only for verified additions after the known proof/Kai head.
- App runtime helpers are typed convenience rails, not a new truth authority.
- App-state projections are public recovery projections beneath proof objects and durable proof memory.
- Commerce helpers must preserve ownership append and settlement row authority.
- Customer session and portal helpers are tenant-scoped access mechanics, not identity proof roots.
- Merchant onboarding helpers must not outrank identity primitives, ownership state, settlement state, or proof objects.
- Audit, event replay, jobs, search, notifications, and release pinning expose or recover verified history. They do not replace verified history.
- Offline proof queues replay verified additions with idempotency material. They must not rediscover or overwrite admitted truth.
- React hooks and hosted component factories must not rename primitives as generic UI, cards, share pages, balance displays, or loading states.
- `@receiz/sdk` release version must match the app release version.

## Release-Lock Requirement

`pnpm release:lock 97.2.0` must require this document for `v97.2.0` and later release cuts. A release that omits the product-truth freeze is incomplete because proof object authority, Kai/Klok state-machine doctrine, durable proof memory law, Connect delegated-access limits, enterprise app runtime SDK boundaries, idempotent write replay, React subpath limits, offline proof queue behavior, and app/SDK version lock cannot remain explicit release truth.

## Residual Risks

- npm publication is an external registry action and must be run after local gates pass.
- Enterprise SDK package contents, React subpath exports, and generated App Runtime docs are broader platform evidence outside this checkout.
- Production DB and Vercel health remain runtime evidence after deploy, but they do not become proof authority.
- Full `pnpm test:release-freeze` can be long-running; any skipped umbrella gate must be named before release.
