## Goal

Make it impossible for `@receiz/sdk` or MCP integrations to admit anything less than a complete sealed Receiz proof object that retains payload, ownership, provenance/appends, and settlement continuity across platforms.

Durable proof memory remains **first admission only, then append forever**.

## What Worked

- Reusing the existing sealed bundle and offline verifier kept authority in the Receiz artifact.
- A canonical portable continuity document made cross-platform round trips deterministic.
- Keeping the existing verifier verdict and strengthening only SDK admission preserved Receiz.com behavior.
- Re-verifying at the SDK, API, server-admission, and database boundaries closed convenience-path bypasses without introducing a stronger external authority.
- Tests rejected the public manifest admission API before implementation changed it.

## What Did Not Work

- A self-hashable manifest could describe an asset without carrying the complete asset through the enclosing sealed artifact.
- Artifact verification alone proved file integrity, not complete asset continuity.
- The v1 database RPC could admit an asset record without the canonical continuity document.
- Calling both outcomes “verified” without separate verdicts hid the missing boundary.
- A first v101 draft exposed a second public verification verdict; review correctly narrowed the change back to SDK admission and restored Receiz.com’s single verdict before release.

## Practical Rule For Future Work

1. Verify the enclosing Receiz artifact first.
2. Parse continuity only from bytes carried by that artifact.
3. Require payload, ownership, provenance/appends, and settlement together.
4. Keep asset admission an idempotent append beneath the verified file.
5. Never restore manifest-only asset admission under a compatibility alias.
6. Extend `receiz.portable_asset.v1` compatibly or consume a deliberate major boundary.

Promotion follows `docs/releases/v101.0.0-checklist.md`; no external action is complete until its command and read-back verification succeed.
