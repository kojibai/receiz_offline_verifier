# Receiz v70.0.0 Growth From v60.0.0

Release date: April 24, 2026

This document records the plain-language growth from `v60.0.0` to the `v70.0.0` public alpha release candidate.

The code-history scope measured before the v70 documentation cut was `v60.0.0..3651d3986`: `637` files changed, `57,656` insertions, and `10,219` deletions across app code, docs, public verifier artifacts, scripts, package metadata, and release infrastructure.

## Plain-Language Product Arc

`v60.0.0` made the front door live product. The homepage stopped being a static explanation and started showing proof, world, and market together.

`v61.0.0` made history governed. Older pre-v4 receiz became an explicit historical cohort instead of an accidental legacy exception, and historical packages preserved raw bytes inside verified transport.

`v62.0.0` stabilized the visible shell. Background work could warm hidden code and data, but it could not visibly replace a page that already looked loaded.

`v63.0.0` and `v63.2.0` hardened runtime recovery, certificates, wallet liquidity, and the settled/unsettled value split. This is where the economy became more honest about what was funded now and what was proof-native value waiting for a buyer-funded event.

`v64.0.0` made the value loop complete. Settlement became the proof-native value lane. Reserve became funded action capital. Notes stayed Reserve-only. Certificates became the bridge from live proof-native value to buyer-funded Reserve.

`v66.0.0` moved authority to the local node. Receiz stopped treating server session as the root when the device already held stronger proof. Local proof, verified route snapshots, wallet graphs, market positions, and twin truth became first-class authority.

`v68.0.0` made that model production-safe. It preserved the stable `v64.6.0` server and UX baseline while moving known local truth earlier. Receiz ID became the primary one-click login rail. PBI, keyfile, email, and magic link remained recovery and interoperability rails.

`v68.9.0` restored and governed profile law. `/<username>` became the canonical profile truth again, and `?manage=1` became an owner overlay instead of a separate base profile source.

The `v69.0.0` through `v69.9.x` line finished release-prep hardening: existing account login, market prediction execution, conformance isolation, custom-domain and in-app browser mobile normalization, logout as a truthful final state, follow-login continuation, route-preserving stale-runtime recovery, twin image locking, proof-sealed world trail ZIP bundles, PWA-only hero proof-card polish, and the v70 invariant register.

## Domain-by-Domain Scope

### Identity and Login

- Receiz ID became the normal login path.
- Existing device identity opens immediately from local proof.
- New devices create a local Receiz ID first and publish globally after.
- PBI and keyfile remain account recovery and account-binding proof.
- Magic links remain interoperability and recovery, not the primary everyday model.
- Logout now clears the active session while preserving Receiz ID continuity and presenting a final logged-out/keyfile step.
- `/signin` is now a pass-through route so users do not land on a manual login page.

### Profile

- Public and owner profile views are governed by the same canonical route truth.
- Owner powers append to the public shell instead of replacing it.
- Hidden inventory stays owner truth and cannot redefine public first paint.
- Existing and new account login routes resolve directly to the managed profile.
- Profile route hydration uses canonical public truth first and stronger local truth only as permitted fallback.

### Market and Wallet

- Market reads holder-side witness truth when available.
- Buy, sell, prediction, certificate, wallet, and funding flows preserve action continuity.
- Settlement and Reserve are separate value lanes with separate rules.
- Sells split funded Reserve from remaining Settlement.
- Certificates package shares/value into buyer-funded liquidity events.
- Wallet panels and add-funds panels were tightened for actual user action instead of decorative state.

### Proof, Artifacts, and Verification

- Document seal, artifact binding, ZK document seal, golden path, signature v4, offline verifier, and proof package checks became release gates.
- Historical packages preserve raw bytes while adding governance proof.
- Mind trail ZIP downloads now include Receiz proof material for the root manifest and each file.
- Verification surfaces are deterministic trust surfaces, not loading theater.

### World and Live Player

- Home and profile twin docks keep visible shells stable until explicit activation.
- The live player paints route media truth before enrichment.
- Twin image identity is locked per user and twin.
- Profile-picture fallback for twin identity is forbidden.
- Runtime recovery preserves the current route when stale chunks or service-worker state force a hard recovery.

### Runtime, PWA, and Service Worker

- Service-worker snapshots carry explicit deterministic/private truth instead of generic cache guesses.
- Online runtime chunk paths prefer fresh runtime assets.
- Stale runtime recovery moves through local proof, server resync, and safe route preservation.
- PWA-only CSS corrections stay scoped to standalone display mode.
- Package, service-worker, public badge, verifier, and release-doc versions move together.

### Governance and Conformance

- Product law docs became binding statute.
- The v70 invariant register enumerates primitives that should never be violated.
- Conformance scripts were kept local by default so repo checks do not accidentally depend on production.
- Release-freeze now aggregates inventory, deterministic surfaces, first paint, proof, auth, market, world, chat, performance, and governance gates.

## What Users Get

- A one-click identity model that works because the device holds proof.
- A profile that feels already there instead of rediscovered.
- A market and wallet that tell the truth about funded value versus proof-native value.
- Proof files and downloaded bundles that can verify offline.
- Runtime recovery that keeps the user on the thing they were doing.
- Custom domains, in-app browsers, mobile browsers, and PWA mode aligned to the same product feel where scoped fixes allow it.
- A release line whose docs, tests, and visible version surfaces say the same thing.

## What v70.0.0 Refuses To Do

- It does not make the server the root of identity when local proof exists.
- It does not let owner state redefine public profile truth.
- It does not call a visible second load acceptable.
- It does not collapse Settlement into Reserve.
- It does not let profile pictures stand in for twin identity.
- It does not ship unverifiable proof ZIPs.
- It does not treat release docs, version badges, and service-worker defaults as optional cleanup.
