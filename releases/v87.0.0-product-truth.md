`v87.0.0` freezes the post-v86 product truth: offline sealed proof objects and offline moments are proof objects when they carry a valid signed proof bundle. A compact `/v` URL is transport for that proof object, not a new authority. The public `/v` surface must verify the embedded proof and display `Proof` when Signature V4 and route binding pass.

## Frozen Primitives

| Primitive | v87.0.0 truth | Must not regress |
| --- | --- | --- |
| Receiz product identity | Receiz remains a proof-native artifact system. | Do not describe Receiz as a generic app with proof features. |
| Offline sealed proof object | An offline sealed artifact carrying `receiz.proof_bundle`, artifact basis, and Signature V4 is a proof object. | Do not downgrade offline proof into historical fallback, metadata, cache, or coordinate evidence when signed proof exists. |
| Offline moment proof object | An offline moment sealed by the actual sealer carries a proof object link that `/v` must verify as proof. | Do not reconstruct coordinate-only links after the proof-bearing link already exists. |
| Compact `/v` proof token | The `c=` token is compact transport for the signed proof bundle. | Do not treat the token as a new issuer, allowlist, registry, or server permission layer. |
| `/v` public proof surface | `/v/[slug]/[code]/[pulse]` verifies signed proof bundle transport and displays `Proof` when Signature V4 and route binding pass. | Do not fall through to `Historical` when a valid signed offline proof bundle is present. |
| Historical compatibility | Coordinate-only offline evidence remains historical compatibility evidence. | Do not upgrade coordinate-only evidence to the same authority as Signature V4 proof. |
| QR proof transport | Live offline card QR points to the compact proof-bearing `/v` URL. | Do not scan to a coordinate-only historical URL after proof sealing has produced the signed proof link. |
| Offline verifier/studio | Offline HTML seal and verifier surfaces produce and preserve compact proof-bearing links. | Do not let static offline surfaces diverge from app `/v` proof semantics. |
| Sealed artifact truth | The sealed proof bundle and artifact basis remain stronger truth than server, profile composer, DB, session, or UI state. | Do not require profile publication or server lookup before recognizing offline proof. |
| Service-worker release truth | Release version, SW runtime version, public labels, offline verifier labels, and release docs move together. | Do not ship a version that leaves installed PWAs serving stale verifier/studio HTML for the release. |
| Sports single-card QR proof | The Sports single-card QR page is a public proof surface around the Sports card proof object. | Do not replace existing vault card/export/link authority while improving the QR surface. |
| Sports value surface | Card value popovers and QR page value panels display settlement/value truth with readable mobile behavior. | Do not let portal mechanics or mobile layout hide or visually corrupt value truth. |
| World Estate | `/world/estate` is a World identity/public witness surface with its own immersive estate presentation. | Do not let the estate surface inherit generic footer/state behavior that downgrades its visual identity. |
| Market/profile public assets | Public profile assets and market proof objects must remain available under tightened RLS/security constraints. | Do not let security hardening remove public proof objects from deterministic first paint. |
| Stronger/weaker truth ladder | Receiz law, sealed proof, deterministic artifact state, verified local truth, and verified appends outrank server/session/cache/global sync. | Do not let a weaker layer replace, rename, delay, or downgrade stronger truth. |
| Deterministic first paint | Proof, ownership, settlement, market, vault, World, Sports, and offline verification surfaces preserve truthful first paint. | Do not add fake loading or rediscovery before known proof truth. |

## Frozen Invariants

- Offline proof is not fallback mode.
- A signed offline proof bundle is a proof object.
- A compact URL is transport for proof truth, not proof truth by itself.
- `/v` must verify signed offline proof object transport before historical fallback.
- Coordinate-only historical evidence remains below Signature V4 proof authority.
- The profile composer is not permission for a proof object to be real.
- Server, DB, session, and profile publication may index, hydrate, publish, or append to offline proof truth; they do not define it.
- Live card QR must point to the proof-bearing link after sealing.
- Offline verifier and offline studio HTML must produce the same proof-bearing link semantics as the app.
- Service-worker cache version must move when release surfaces and offline verifier/studio HTML move.
- Sports card QR pages are public proof surfaces around Sports card proof objects.
- Sports card value popovers must keep value and settlement truth visible and readable on mobile.
- World Estate must preserve its identity/public witness surface without inherited global footer interference.
- Market/profile proof objects must remain public proof truth beneath RLS/security hardening.
- Stronger sealed artifact truth must never be replaced by weaker server/session/UI state.
- Deterministic first paint must never wait on rediscovery of already-known proof truth.
- Visual evidence that shows `Historical` for a valid signed offline proof link is failing evidence.
- Visual evidence that shows QR scanning to a historical coordinate link after sealing is failing evidence.

## Residual Risks

- Production deployment/cache state must be verified after release promotion.
- Installed PWAs running the old service worker need the `87.0.0` service-worker update to fetch the new offline verifier/studio HTML.
- Release attestation requires the production signing environment.
- Production Supabase targets must include the post-v86 RLS migration before DB-backed Signal Run and conformance paths are considered release-ready.
- Browser QR scan behavior should be confirmed on actual mobile devices after deploy.
- TypeScript checking may be blocked by generated `.next/dev/types` corruption until the generated Next types are cleaned or regenerated.
- Visual regression risk remains for any surface not re-screenshotted after the final v87 release lock.
