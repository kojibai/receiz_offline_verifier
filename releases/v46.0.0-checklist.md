# Receiz v46.0.0 Release Lock Checklist

Release date target: March 30, 2026

## Purpose

This checklist locks `v46.0.0` to one exact release state without allowing regression in authoritative file verification, deterministic profile/original delivery, Public Twin/world runtime behavior, or release-governed trust surfaces.

Product-truth reference:

- `docs/releases/v46.0.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm install --frozen-lockfile
pnpm test:release-freeze
pnpm test:release-golden-path
pnpm test:signature-v4-server-remote
```

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- Record
- Seal
- Verify
- Public `/v` verification route
- Offline verifier open/import/verify flow
- Offline studio install, signer provisioning, record, seal, and verify flow
- Profile original delivery and verified-original player routes
- Rich document fullscreen reading and inline viewer behavior
- Original, derivative, and moment-record download precedence
- Sign in, passkey, and Receiz Key flows
- Wallet note status, claim, and admin adjustment surfaces
- Payments embed, login embed, and Connect payment-session entry paths
- Calendar event bundle and appointment confirmation flows
- World home, district, avatar/twin, public twin, and public-world navigation surfaces
- Public twin bootstrap, stage playback, browser fallback, and close/reopen continuity
- Photoreal shell overlay mouth behavior during real voiced playback
- Avatar forge success, contract failure messaging, and preview freshness behavior

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v46.0.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Post-Deploy Observation

After deployment:

- Run `synthetic:uptime`
- Run `prod:slo-check`
- Observe production for 24-72 hours
- Hold a stabilization window with additive fixes only

## Release Rule

If any release-freeze gate, signature/conformance gate, attestation step, or manual smoke check fails, `v46.0.0` is not released from that commit.
