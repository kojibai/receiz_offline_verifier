# Receiz v89.0.0

## Goal

Ship `v89.0.0` as the release that locks post-v88 foreground proof publication and the full commit arc that led to it.

The release goal is precise:

```txt
preserve v88 live proof and runtime truth
lock foreground /api/receiz/anchor above generic outbox mechanics
restore profile-original publish/global sync to the June 6 working behavior
keep profile showcase, live player, public profile, and market projections coherent
preserve Sports Vault sealed-manifest public proof
document every commit since v88.0.0
make the anchor regression impossible to miss in future release checks
```

## What Worked

- Comparing the broken behavior against the June 6 baseline focused the investigation on object shape, profile projection, and foreground publication instead of inventing a new authority.
- The production route evidence showed the server could still return readable JSON, which prevented misclassifying `/api/receiz/anchor` itself as the broken source of truth.
- Reading the local signed-action outbox boundary identified the actual regression: `/api/receiz/anchor` was allowed into weaker generic outbox admission.
- Excluding `/api/receiz/anchor` from local signed-action admission fixed the root without changing profile object truth, showcase projection, market projection, or server route authority.
- The outbox contract now proves the anchor route is excluded before generic signed-action handling can inspect or wrap it.
- The same contract prevents a broad `/api/receiz` classifier from re-admitting foreground anchor publication later.
- Profile publish contracts lock the projection obligation: visible writes must update canonical profile snapshot, public profile market append source, and visible route snapshots.
- The Sports Vault URL contract now proves brand-new accounts without a saved username still get a human public vault route through the default profile username projection.
- The full commit-history document gives the release a repo-local ledger of everything shipped since `v88.0.0`, not only the final anchor repair.

## What Did Not Work

- Treating the error text as proof that the anchor server route was broken was too weak. The route could return readable JSON while the foreground response boundary was still being consumed by the wrong local layer.
- Earlier patches around response parsing and error readability did not remove the weaker authority from the path.
- Local-first behavior alone was not enough. The post still needed the same global showcase/market projection obligations as older June 6 profile originals.
- A DB row, owner-visible local post, or temporary local route snapshot did not prove global publication.
- Broad generic offline/outbox mechanics were allowed to get too close to a foreground proof publication route.
- Version and release documentation were behind the amount of proof/market/vault/profile/runtime work that had landed since `v88.0.0`.

## Practical Rule For Future Work

For foreground anchor publication:

```txt
/api/receiz/anchor owns the anchor response contract
exclude it from generic local signed-action outbox admission
read the route response once as route-owned foreground proof output
readable server errors are allowed
unreadable foreground response caused by weaker wrapping is a regression
```

For profile original publication:

```txt
old showcase objects define the compatibility shape
new posts must carry the same live-player route/proof fields
DB insert is not global completion
owner local view is not global completion
write path must append profile showcase truth
write path must refresh public profile market append source
write path must update visible local route snapshots
refresh and app close/reopen must not drop the post
```

For Sports Vault public proof:

```txt
sealed image and sealed manifest are stronger truth
public showcase publishing projects beneath that proof
full browser payload upload is not required to own vault proof truth
server/DB/session may append or index
server/DB/session may not replace the sealed vault proof object
```

For releases:

```txt
release note
product truth
process
checklist
regression lessons
full commit history since previous major release
version surfaces
release lock
focused contracts
full promotion gates
production/deploy/session evidence boundaries named exactly
```

The `v89.0.0` rule is:

```txt
Foreground proof publication is not a background outbox action.
```
