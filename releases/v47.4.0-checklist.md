# Receiz v47.4.0 Release Lock Checklist

Release date target: April 9, 2026

## Purpose

This checklist locks `v47.4.0` to one exact release state without allowing regression in native Business entitlement continuity, Receiz-owned upgrade scheduling surfaces, public embedded payment checkout continuity, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v47.4.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
npx tsx --test app/lib/premium/__tests__/plan.test.ts
npx tsx --test app/lib/world/__tests__/worldDistrictRouteContract.test.ts app/lib/world/__tests__/worldMeRouteContract.test.ts
pnpm exec tsc --noEmit --pretty false
pnpm exec next build --webpack
pnpm release:lock
```

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- A Business-entitled account reports Business correctly on native session-derived surfaces and does not appear as merely Premium in Business-aware UI or API payloads
- The affiliate surface still opens normally for paid users and does not regress when the effective entitlement is Business
- Upgrade and WireProof scheduling CTAs open the Receiz calendar embed rather than any Calendly-branded surface
- The public payments embed opens secure card checkout inside the Receiz popup surface and does not navigate the buyer directly to a raw Stripe-hosted checkout URL
- Redirect-required Stripe methods return to the Receiz-hosted checkout surface cleanly
- Public version badges and offline verifier surfaces all show `v47.4.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v47.4.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any entitlement-continuity regression, scheduler-surface regression, public checkout-surface regression, release-lock check, or attestation step fails, `v47.4.0` is not released from that commit.
