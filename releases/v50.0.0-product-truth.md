# Receiz v50.0.0 Product Truth

Release date: April 11, 2026

This document freezes the primitives and invariants that `v50.0.0` is allowed to extend but not weaken.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| File-authoritative trust core | The file remains the authority. Deterministic manifests remain the authority for route truth, and ledger-backed settlement remains the authority for payment truth. World, outreach, and campaign layers are execution around those boundaries, not substitutes for them. | `app/lib/profile/document.ts`, `app/lib/profile/server.ts`, `app/lib/receiptView.ts`, `app/lib/wallet/payIntent.ts`, `app/api/wallet/ledger/public/route.ts` |
| Durable person-first Contacts CRM | Contacts must remain the durable operating memory for people, history, notes, stage, identity, and outreach continuity rather than falling back to a local-only shell or detached world-only memory. | `app/contacts/ContactsPageExperience.tsx`, `app/api/contacts/route.ts`, `app/api/contacts/import/route.ts`, `app/api/contacts/[contactId]/route.ts`, `app/api/contacts/[contactId]/events/route.ts`, `app/lib/contacts/server.ts`, `supabase/migrations/20260409163000_contacts_crm.sql` |
| World as operator workspace | `/world` must remain a live operator surface with atlas continuity, concierge execution, campaign memory, and next-best-move presentation rather than collapsing back into a static prompt shell. | `app/world/WorldExperience.tsx`, `app/world/WorldExperience.module.css`, `app/lib/world/server.ts`, `app/lib/world/types.ts` |
| Campaign-first revenue OS | Campaigns, campaign contacts, runs, grade, queue state, events, and autopilot schedule are first-class system objects and must remain durable, queryable, and connected to Contacts. | `app/lib/world/campaigns.ts`, `app/api/world/campaigns/route.ts`, `app/api/world/campaigns/autopilot/route.ts`, `app/api/world/campaigns/[campaignId]/autopilot/route.ts`, `supabase/migrations/20260410113000_world_campaigns.sql`, `supabase/migrations/20260410143000_world_campaign_autopilot_scheduler.sql`, `supabase/migrations/20260410170000_world_campaign_events.sql` |
| Truthful rail readiness and delivery state | Outreach policy, provider configuration, deliverable endpoints, provider events, and blocked rails must remain explicitly distinguishable. | Self-healing research and enrichment loop | Lead research and dispatch preparation must broaden, enrich, retry, and continue campaign continuity when possible instead of hard-failing on the first thin search or missing endpoint. | `app/lib/world/leadResearch.ts`, `app/lib/world/conciergeTasks.ts`, `app/lib/world/campaignAutomation.ts`, `world-enrichment-service-template/lib/enrichment.js` |
| User-facing operator truth | Concierge and campaign surfaces must present user-facing blocked/failure explanations, not raw backend or database internals, and active runs must remain controllable through restart, continue, dismiss, and clear flows. | `app/lib/world/conciergeTasks.ts`, `app/api/world/concierge/tasks/route.ts`, `app/world/WorldExperience.tsx` |
| Commercial close-loop attribution | Positive replies, bookings, checkout settlement, and Receiz wallet transfers must be able to flow back into campaign contact state and revenue reporting rather than ending as detached activity. | `app/lib/world/campaigns.ts`, `app/lib/world/runtime.ts`, `app/lib/wallet/payIntent.ts`, `app/wallet/WalletAccountExperience.tsx`, `app/api/chat/conversations/[conversationId]/transfers/route.ts`, `app/api/connect/transfers/route.ts` |
| Release-governed version surfaces | Package version, service worker version, public badges, offline verifier surfaces, and release docs must move together on each release. | `scripts/check_release_lock.mjs` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Contacts stay server-truth and person-first | Signed-in contacts must persist through the server-backed contacts API and survive refresh, import, merge, and operator handoff. | `app/contacts/ContactsPageExperience.tsx`, `app/api/contacts/route.ts`, `app/lib/contacts/server.ts` |
| Import merges cannot silently drop identity upgrades | A `linked_user_id` upgrade or duplicate-linked-user merge must resolve into durable contact state instead of surfacing as a dead-end conflict. | `app/lib/contacts/server.ts`, `app/lib/contacts/__tests__/serverImportMergeContract.test.ts` |
| No false send success | Campaign or concierge dispatch cannot report success unless a real outreach delivery occurred on a usable rail. | `app/lib/world/conciergeTasks.ts`, `app/lib/world/outreachDelivery.ts`, `app/lib/world/__tests__/conciergeLeadOpsContract.test.ts` |
| Blocked rails must stay blocked | Missing provider config, missing explicit webhook targets, missing imported Contacts, and missing deliverable endpoints must show as blocked operator state rather than completed work. | `app/lib/world/conciergeTasks.ts`, `app/lib/world/railReadiness.ts`, `app/lib/world/__tests__/railReadiness.test.ts` |
| The twin must keep trying viable public paths | Research and enrichment must widen the brief and retry viable public contact paths before giving up, while remaining truthful when no public endpoint exists. | `app/lib/world/leadResearch.ts`, `app/lib/world/campaignAutomation.ts`, `world-enrichment-service-template/lib/enrichment.js`, `app/lib/world/__tests__/leadResearchFallback.test.ts`, `app/lib/world/__tests__/worldEnrichmentServiceTemplateContract.test.ts` |
| Provider telemetry must feed campaign state | Delivery failures, replies, and other provider events must update campaign/contact state instead of being left outside the campaign loop. | `app/api/world/providers/resend/webhook/route.ts`, `app/api/world/providers/twilio/status/route.ts`, `app/lib/world/campaigns.ts`, `app/lib/world/__tests__/campaignRevenueEngineContract.test.ts` |
| Reply and close signals must not stop at visibility | Reply, booking, and settlement signals must remain actionable and attributable back into campaign metrics, queue state, and next-best moves. | `app/lib/world/campaigns.ts`, `app/lib/world/runtime.ts`, `app/lib/wallet/payIntent.ts`, `app/lib/wallet/__tests__/payIntentContext.test.ts` |
| User-facing errors stay user-facing | Concierge history, campaign cards, and blocked summaries must not leak raw SQL, Postgres, provider, or internal exception strings as primary operator copy. | `app/lib/world/conciergeTasks.ts`, `app/world/WorldExperience.tsx` |
| Release surface inventory stays canonical | The committed release surfaces and version badges must align with package and service-worker release truth. | `scripts/check_release_lock.mjs` |

## Release Gates

`v50.0.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `pnpm test:contacts-release-contracts`
- `pnpm test:world-release-contracts`
- `npx tsx --test app/lib/world/__tests__/campaignRevenueEngineContract.test.ts app/lib/world/__tests__/conciergeLeadOpsContract.test.ts app/lib/world/__tests__/leadResearchFallback.test.ts app/lib/world/__tests__/outreachRouteContract.test.ts app/lib/world/__tests__/railReadiness.test.ts app/lib/world/__tests__/worldConciergeTaskControlsContract.test.ts app/lib/world/__tests__/worldEnrichmentServiceTemplateContract.test.ts app/lib/contacts/__tests__/serverImportMergeContract.test.ts app/lib/wallet/__tests__/payIntentContext.test.ts`
- `pnpm release:lock`
- `pnpm test:release-freeze`
- `pnpm release:attest`
- `pnpm release:verify-attestation <path>`

The manual release procedure is defined in `docs/releases/v50.0.0-checklist.md`.

## Residual Risks

The following areas are materially stronger after this release but are not yet absolute:

- Provider-backed enrichment is still bounded by public evidence quality and the external enrichment deployment staying healthy.
- Deliverability truth is much stronger, but production sender reputation still depends on real-domain DNS, sending policy, and provider setup outside the repo.
- Off-platform settlement attribution is strongest for Stripe-backed and Receiz-wallet flows; manual wires or off-system invoicing still require explicit operator reconciliation.
- Campaign autopilot is now durable, but production scheduling still depends on a real cron/heartbeat deployment path invoking the autopilot route with the correct secret.
- Dense mobile world operator states still benefit from manual smoke on a populated account before tagging the release.
