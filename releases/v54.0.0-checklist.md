# Receiz v54.0.0 Release Lock Checklist

Release date target: April 15, 2026

## Purpose

This checklist locks `v54.0.0` to one exact release state without allowing regression from the first `v53.0.0` market/public baseline into the current admin-operating, signed-workspace-continuity, deterministic-asset-journey, world-twin-continuity, and rendering-determinism release state.

Product-truth reference:

- `docs/releases/v54.0.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm exec tsc --noEmit --pretty false
npx tsx --test app/components/KaiSigil/__tests__/geometryDeterminism.test.ts
npx tsx --test app/lib/market/__tests__/marketValuePillContract.test.ts
npx tsx --test app/lib/world/__tests__/worldTwinSessionReuseContract.test.ts
npx tsx --test app/lib/session/__tests__/sessionBootstrapConsumersContract.test.ts
npx tsx --test app/calendar/__tests__/calendarActivityProjectionContract.test.ts app/contacts/__tests__/contactsBootstrapContract.test.ts
pnpm test:release-freeze
pnpm release:lock
```

If the candidate deploy also requires broader market, world, or governance verification beyond the list above, those must still pass before tag creation.

## Production Environment + Governance Preflight

The production database must already include the active market/world rails from earlier releases.

For the wallet/admin transparency paths present in this release-prep state, confirm the target environment also includes:

- `supabase/migrations/20260319153000_wallet_stripe_credit_rollups.sql`
- `supabase/migrations/20260319170000_wallet_admin_post_adjustment.sql`

Before release, confirm:

- `public/governance/governance-keyring.v1.json` still contains the active release-attestation key ID if release attestation is generated for this cut
- admin Stripe/payment rollups can read from their intended reporting path
- the target environment can backfill wallet/ledger Stripe rollups without SQL/function drift
- public/offline verifier surfaces, service worker version, and public version badges all show `v54.0.0`

## Manual Smoke

Confirm these flows on the candidate without changing code:

- `/admin` opens as the command-deck shell and does not fall back to the old long-scroll report behavior
- the admin payments panel exposes backfill/compaction/action-center controls and Stripe per-user USD visibility
- admin wallet adjustments append ledger entries instead of directly overwriting balances
- profile price pills render a deterministic value before click and then upgrade to live market state after settle
- profile -> artifact -> live-player transitions keep the world twin loaded instead of replaying the twin loading phase
- the twin dock never flashes the square 4K render image before the rigged twin becomes visible
- artifact routes use the same deferred twin continuity path as profile and live player
- calendar, contacts, account, upgrade, plans, make, seal, and related signed-in surfaces open from session bootstrap when the app already knows the signed-in user
- no Kai sigil hydration mismatch appears on profile/price-pill surfaces
- `/business`, `/everything-receiz-lets-you-do`, `/what-can-you-do-with-a-receiz`, `/what-is-a-receiz`, `/start-here`, `/public-twin`, and `/receiz-world` present the intended plain-English product map
- `/start-here` does not list the same featured “Everything Receiz lets you do” page twice in the canonical references block

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v54.0.0`.
2. Run the `Release Governance Attestation` workflow if the release process for this cut still requires a signed attestation.
3. Verify the generated artifact with `pnpm release:verify-attestation <path>` if an attestation is produced.
4. Confirm the attestation references the published key ID and the expected release version.
5. Store the attestation path and approval reference with the release record.

## Release Rule

If any admin-command-deck regression, ledger-transparency regression, signed-workspace-bootstrap regression, deterministic-price-pill regression, twin-continuity regression, twin-dock-image leak regression, Kai-sigil hydration regression, version-surface mismatch, or release-lock failure occurs, `v54.0.0` is not released from that commit.
