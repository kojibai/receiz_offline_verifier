## Title

Receiz `v77.0.0` - Signal Circuit, Creator OS, Vault Settlement, And Runtime Trust Release

## Summary

`v77.0.0` locks the post-`v76.0.0` product candidate as the Signal Circuit and Creator OS release. The release preserves v70 profile law, v74 identity-artwork law, v75 local-proof/file-restore law, and v76 World/Market/Public Proof law while adding a serious game-grade collectible economy, creator production operating system, sealed vault workflow, public conformance assurance, and runtime-coherence recovery layer.

This is not a label-only bump. The diff from `v76.0.0` to the pre-bump v77 candidate includes Signal Circuit gameplay, verified Signal Cards, card art, sealed downloads, pack purchase and reveal continuity, account-scoped Vault sync, card transfer/listing/auction/purchase/battle surfaces, Signal Pack Foundry, Signal Circuit conformance, Creator Workspace, durable Creator OS queues, paid creator asset and publish jobs, managed launch rails, Vault ZIP restore and native preview, old-Safari Signature V4 document sealing, public/developer status expansion, runtime-stale recovery removal, and release-contract coverage for the new game, creator, vault, document, status, wallet, market, and runtime behavior.

## What This Release Locks

- Signal Circuit is now a dedicated `/game` product surface with no fake loader shell, a mobile game chrome, deterministic local game logic, and Signal Card issue paths that use existing Receiz proof primitives.
- Signal Cards are portable verified collectibles with sealed claims, separated achievement truth and transferable collectible truth, deterministic rarity classes, card kind, character/utility mechanics, battle profiles, lineage, visual profile, and standard verification paths.
- Signal Card artwork is generated, stored, replayed, and exported as part of the card contract. Purchased or recovered cards must not blindly downgrade to older cached art or weaker partial card payloads.
- Downloaded verified card fronts use `☤KAI` in steward, valuation, and related visible copy instead of old Pulse framing.
- Signal Pack purchases are wallet-aware, exact-rate, resumable, and settlement-safe. A pack opening is not complete until the sealed cards are issued, stored, and recoverable from the durable opening record.
- Signal Vault is account-scoped and paginated. It can hydrate local card truth in batches, retry partial sync, avoid account bleed, transfer ownership, list cards, append market/audit rows, and preserve cards across navigation without a one-card bottleneck.
- Card public pages, share pages, OpenGraph/Twitter images, proof panels, reserve-first purchase flows, listings, auctions, and battle discovery render from sealed card truth without liquid loader or placeholder replacement paths.
- Signal Arena and card battle surfaces keep battle history append-only, require owner opt-in for AI twin defense, and never mutate the original card claim when a battle result or progression record is appended.
- Signal Pack Foundry introduces tiered brand pack campaigns with visible odds, supply caps, pack limits, arena/AR options, manifest seals, campaign records, and verification isolation while keeping the existing Signal Card primitive unchanged.
- Signal Circuit conformance becomes public assurance: requirements, claim-to-code checks, source digests, scarcity proof, bounded simulation, badges, history, rollups, and status inventory move together.
- Public/developer Status includes Signal Circuit and high-value public card/conformance endpoints while keeping private vault, pack purchase, admin, webhook, worker, and legacy keyring surfaces out of the public trust surface.
- Creator Workspace is a first-class `/creator` route, not a reuse of `/make`. It first-paints a settled local workspace, uses Streamdown for creator output, and keeps private AI, wallet, assets, publish jobs, and launch jobs behind explicit action paths.
- Creator OS adds durable project, brand, run, message, event, output, asset, version, usage-ledger, publish-job, and safety-review storage with signed-in API boundaries and worker-drain authorization.
- Creator paid production debits wallet value idempotently for AI runs, asset lanes, and managed publishing before background production keeps running. Shortfalls open embedded wallet funding instead of pretending work is free.
- Creator managed launch can queue GitHub and Vercel launch jobs, use Vercel Sandbox for code execution when enabled, preserve provider proof, and record durable launch output without moving users away from Receiz.
- Verified Data Vault downloads linked bytes before sealing, exports verifiable ZIP/vault packages, restores original files, and opens image/audio/video/PDF/native previews from restored bytes.
- Old Safari document sealing no longer falls back to legacy V3. When native Ed25519 is unavailable online, it uses Signature V4 document sealing; offline old Safari tells users to upgrade Safari 17+ or go online.
- Runtime stale recovery no longer mounts an automatic root interstitial or redirects from the stale guard. Legacy recovery assets clear stale runtime state silently and return to a stable path without fake loading theater.
- Market share quote bootstrap batches long asset lists before Supabase `in` filters, keeping public market/quote hydration from creating avoidable database pressure.
- Wallet, Account, global chrome, bottom chrome, game chrome, Creator, Vault, Status, Developers, Payments, Public Proof, Seal, and verified player surfaces are updated as product surfaces without weakening profile canonical truth or local proof continuity.
- Release contracts now include `test:game-release-contracts` and `test:signal-circuit-conformance` inside `test:release-freeze`, plus targeted creator, vault, document, status, runtime, market, wallet, profile, and Signal Card contracts.

## Developer And Public Value

- Developers get public Signal Circuit assurance with source digests, bounded simulation, status badges, history, and clear public/private route boundaries.
- Creators get an operating surface that can generate, meter, queue, publish, and launch real production assets while preserving first-paint continuity and wallet settlement truth.
- Users get a game-grade Signal Circuit loop where earned cards become sealed, shareable, transferable, marketable, battle-ready Receiz assets instead of disposable game state.
- Operators get release gates that catch visible regressions in game issue flow, pack reveal continuity, card share proof, Creator OS queues, vault restore, Safari sealing, runtime recovery, status inventory, wallet settlement, and conformance assurance.
