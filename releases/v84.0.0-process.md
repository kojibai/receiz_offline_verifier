# Receiz v84.0.0

## Goal

Ship `v84.0.0` as the stable proof-native artifact-system release.

The release locks the v83 stability arc and the v84 primitive-law hardening pass into one coherent release identity:

```txt
Receiz is a proof-native artifact system.
Implemented primitives are named directly.
Stronger truth never yields to weaker truth.
Deterministic first paint is protected.
External systems provide mechanics only.
```

## What Worked

- The v83 Sports work succeeded when it treated first paint as product law instead of a loading phase.
- Main Event closeout became correct when settlement ran on the launch/read path and no longer depended only on a worker eventually catching up.
- Refunded tournament entries stopped relocking when the entry bootstrap treated cancelled/void tournament truth as stronger than active stale entry state.
- Signal Run vault stability improved when proof/appends sync stopped hiding behind idle scheduling and the write path gained bounded pressure guards.
- DB pressure dropped when hot-path append writes, original engagement rollups, public profile asset reads, and admin analytics received bounded indexes and contracts.
- Live player stayed fast when the route used the smallest visible truth and kept OG/share/SEO work out of the interactive hot path.
- The rollback of the heavy live-player seed proved the correct architecture: strongest visible truth, smallest server payload, background enrichment after the shell.
- Payment lock behavior became correct when paid access stopped depending on original/derivative download permission.
- Strict payload utility hardening kept deterministic hashing and encoding explicit without changing route behavior.
- Release law became operational when acceptance criteria, truth hierarchy, first-paint protection, decision procedure, forbidden downgrades, language rules, exact evidence boundaries, and final-report requirements were placed in repo law.
- Symmetric caution closed the remaining language drift: generic convention is the weaker prior, not the authority over implemented Receiz primitives.
- The World twin dock contract caught the `3731a3fea9` over-freeze before release; `stageWorld` now stays on `effectiveWorld` so fresh stronger twin truth is visible without waiting for activation.
- The perf budget stayed active and was recalibrated to the measured Next 16 root runtime floor; app root payload growth remains visible instead of hidden.

## What Did Not Work

- Deferring proof/appends sync behind idle scheduling damaged the vault path. Proof persistence is not decorative background work.
- Letting Main Event closeout depend only on worker/queue timing left launch/read paths showing stale locks after settlement truth already existed.
- Keeping Sports entries active after tournament entries were cancelled/void let a stale recovery path relock users into closed events.
- Treating live-player cold entry as either null-first-paint or heavy server payload created the wrong binary. The correct route carries only minimal public-anon visible truth.
- A live-player payment lock tied to download permission hid paid proof surfaces for the wrong reason.
- SEO, OG, and metadata work can break public proof surfaces when they move into the hot path or when they stop carrying the deterministic public witness image.
- Vague language around primitives creates architecture drift. When the implementation satisfies the primitive, softening the name is a product regression.
- Freezing a visible dock to initial server truth is a truth downgrade when `effectiveWorld` already carries fresher session, preview, or prepared World truth.

## Practical Rule For Future Work

Apply this order before every primitive change:

```txt
name the primitive
identify the source of truth
check stronger/weaker truth direction
protect deterministic first paint
use external systems for mechanics only
hedge weak inherited priors before strong primitives
state the exact uncertainty boundary
edit only after the law is preserved
```

For live routes:

```txt
minimal server truth
truthful first paint
no null shell when truth exists
no heavy payload to prove visible truth
background hydration
append only
```

For Sports:

```txt
selected context
settlement truth
entry truth
score truth
lock truth
visible shell
background feed and append work
final proof
```

For vault and public proof surfaces:

```txt
proof identity
media identity
ownership/custody
provenance
verification
action affordance
public witness
background enrichment
```

For agent language:

```txt
implemented primitive -> direct name
unknown evidence -> exact boundary
external convention -> mechanics only
weak inherited prior -> hedged first
stronger truth -> never downgraded
```

The `v84.0.0` release rule is:

```txt
Do not ask whether Receiz primitives are real after implementation satisfies the law.
Apply the primitives, preserve the hierarchy, and ship the surface already there.
```
