# Receiz v51.0.0 Release Lock Checklist

Release date target: April 12, 2026

## Purpose

This checklist locks `v51.0.0` to one exact release state without allowing regression in the new plan architecture, public ladder clarity, tier gating truth, full Contacts visibility, world campaign operator truth, user-facing activity history, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v51.0.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm exec tsc --noEmit --pretty false
npx tsx --test app/lib/premium/__tests__/plan.test.ts
pnpm test:contacts-release-contracts
pnpm test:world-release-contracts
npx tsx --test app/contacts/__tests__/contactsBootstrapContract.test.ts app/lib/world/__tests__/campaignRevenueEngineContract.test.ts
pnpm release:lock
pnpm test:release-freeze
```

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- signed-out `/upgrade` shows the full ladder clearly: Free, Authorship, Business, Automation, Enterprise
- signed-out users can understand plan differences before authentication
- the pricing page does not lead with WireProof, receipts, invoicing, or legacy `Premium/Pro` framing
- plan cards, comparison table, upgrade path, FAQ, and final CTA all read in the same voice
- signed-in `/upgrade` still supports embedded checkout and billing center behavior
- account and adjacent upgrade surfaces no longer leak `Premium` / `Pro` as the main public plan names
- Authorship-gated prompts still require the correct entitlement and do not grant free access to paid proof controls
- Business-gated prompts still require the correct entitlement and do not leak custom-domain or ownership-only controls to free users
- Automation-gated prompts still require the correct entitlement and do not leak automation-only controls to lower tiers
- `/contacts` shows the real owner CRM beyond the old first-page ceiling if the account has more than `1000` contacts
- placeholder `Receiz contact` junk rows with no real identity no longer appear in active CRM lists
- a world campaign contact opened from World resolves into the owner CRM record instead of disappearing into a generic contacts shell
- Contacts history and Calendar notes show user-facing copy and actual sent message text where persisted
- campaign/world activity feeds do not show provider names, route paths, raw schema event names, or provider-plumbing phrasing
- public version badges and offline verifier surfaces all show `v51.0.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v51.0.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any plan-language regression, pricing/gating mismatch, contacts visibility regression, placeholder cleanup regression, guest/profile merge regression, world operator-truth regression, activity-copy regression, version-surface mismatch, release-lock check, or attestation step fails, `v51.0.0` is not released from that commit.
