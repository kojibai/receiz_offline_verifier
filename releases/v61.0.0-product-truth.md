# Receiz v61.0.0 Product Truth

Release date: April 19, 2026

This document freezes the primitives and invariants that `v61.0.0` is allowed to extend but not weaken after the `v60.0.0` stable baseline and the cumulative post-`v60.0.0` continuity line.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Release-governed version surfaces stay canonical | `package.json`, service worker defaults, public version badges, plans and upgrade labels, offline verifier artifacts, and active release docs must all expose `v61.0.0` together when the cut is made. | `package.json`, `public/sw.js`, `app/components/pwa/ServiceWorkerRegistration.tsx`, public page/version surfaces, `scripts/check_release_lock.mjs` |
| Historical continuity is governed, not inferred | Pre-v4 receiz can only present as `Historical` by exact frozen cohort membership and pinned governance payload, while current `Verified` semantics remain strict. | `app/lib/security/receizHistoricalRecords.ts`, `docs/governance/historical-continuity-standard.v1.md`, `app/components/receipt/ProofPopover.tsx`, historical continuity tests |
| Historical downloads preserve original bytes inside verified transport packages | Historical download packaging may add sealed manifests and a governance bundle, but it must preserve untouched raw artifact bytes and must not silently mint a new current proof identity. | `app/api/profile/originals/download/route.ts`, `app/lib/security/receizHistoricalPackage.ts`, `scripts/test_receiz_historical_package.ts`, `scripts/test_receiz_legacy_repair.ts` |
| Public market truth and exit navigation stay on stronger known paths | Public asset witness truth must continue preferring holder activity, and deterministic internal market exits must continue using the stable client-navigation path where required. | `app/lib/market/publicProfileAssets.ts`, `app/market/MarketDeskExperience.tsx`, `app/lib/market/__tests__/marketWitnessContract.test.ts`, `app/market/__tests__/marketNavigationContract.test.ts`, `app/lib/market/__tests__/marketRouteContinuityContract.test.ts` |
| Settled twin shells remain visually immutable until explicit open intent | Background preload may warm hidden code and data, but it must not auto-promote the home or profile dock shell, and compact touch devices keep full twin hydration on-demand. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/u/[username]/worldTwinBootstrap.ts`, `app/lib/world/__tests__/publicTwinBootstrapReliabilityContract.test.ts`, `app/lib/world/__tests__/worldTwinBackgroundHydrationContract.test.ts`, `app/lib/world/__tests__/worldTwinStageSeparationContract.test.ts` |
| The home dock stays lightweight until explicit activation | `HomeHeroTwinDock` may warm the deferred module in the background, but the closed-shell audit path remains the visible dock until explicit activation hands off into the deferred module. | `app/components/HomeHeroTwinDock.tsx`, `app/u/[username]/DeferredWorldProfileModuleAudit.tsx`, `app/lib/world/__tests__/homeTwinDockColdPathContract.test.ts` |
| Compact verified-player controls stay legible | Mobile verified-player audio controls must remain compact enough to keep identity, queue, pulse, claim, and transport details visible without visual crowding. | `app/[username]/verified/[receizId]/player/VerifiedOriginalPlayerExperience.module.css` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Historical status cannot be granted by broad legacy shape | Missing `signatureV4`, localhost verify URLs, or old timestamps alone must never classify a record as `Historical`. | `app/lib/security/receizHistoricalRecords.ts`, `docs/governance/historical-continuity-standard.v1.md`, historical continuity tests |
| Current verifier semantics stay strict | Historical continuity must not loosen the active verifier or cause historical records to appear as current `Verified`. | historical continuity code paths plus receipt/proof presentation |
| Historical packaging cannot rewrite raw identity | The raw file inside a historical download package must remain untouched, and later packaging/attestation must not impersonate a new current issuance. | `app/lib/security/receizHistoricalPackage.ts`, `app/api/profile/originals/download/route.ts` |
| Public holder truth beats creator fallback | Public market witness timing and holder count must not fall back to weaker creator-side clocks or envelope counts when holder-side truth is known. | `app/lib/market/publicProfileAssets.ts`, `app/market/MarketDeskExperience.tsx`, market witness tests |
| Deterministic market exits stay continuous | Market exits to `/v/...`, `/r/...`, verified-player, and original-viewer routes must not silently drop back to weaker navigation behavior. | `app/market/MarketDeskExperience.tsx`, `app/market/__tests__/marketNavigationContract.test.ts`, `app/lib/market/__tests__/marketRouteContinuityContract.test.ts` |
| Hidden twin preparation never promotes the visible shell | Hidden preload, remembered state, or compact-device hydration must not reopen or replace the visible dock shell without explicit user intent. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/u/[username]/worldTwinBootstrap.ts`, world continuity tests |
| Home dock warming stays hidden until click intent | Warm-on-hover, warm-on-focus, or warm-on-touch may prepare the deferred module, but those paths must not visually replace the audit shell before explicit activation. | `app/components/HomeHeroTwinDock.tsx`, `app/u/[username]/DeferredWorldProfileModuleAudit.tsx`, `app/lib/world/__tests__/homeTwinDockColdPathContract.test.ts` |
| Release surfaces remain canonical | Public version badges, service worker defaults, offline verifier surfaces, and release docs must align with `package.json` when the version cut happens. | `scripts/check_release_lock.mjs` |

## Residual Risks

- The governed historical cohort currently covers one explicit public pre-v4 continuity set. Any future cohort expansion is a high-risk governance change and must not be inferred at runtime.
- Historical download packaging is stronger now, but it still depends on pinned governance artifact bytes and healthy storage/read paths for the raw historical asset.
- Twin-shell continuity now depends on every new dock or deferred twin entry point preserving the same explicit activation rule used by the home dock and profile dock.
- Compact verified-player control polish still benefits from manual smoke on real narrow devices, especially where long handles or claims compete with transport controls.
- `v61.0.0` is not fully release-cut until public version surfaces, service worker defaults, offline verifier surfaces, package metadata, and docs all move together to the same version.
