# Receiz v101.0.0 Regression Lessons

These lessons preserve `@receiz/sdk` beneath proof authority and durable proof memory as **first admission only, then append forever**.

## What Regressed

- SDK callers could submit a shape-valid asset manifest directly to proof memory.
- Offline verification could truthfully verify the artifact while callers incorrectly inferred complete asset continuity.
- Server and database admission did not require the same canonical continuity object carried by the file.
- MCP reported an older runtime version than its package release.

## Why It Regressed

- Artifact integrity and asset completeness were represented as one broad “verified” idea.
- A projection manifest was permitted to cross an authority boundary reserved for the enclosing sealed artifact.
- Convenience rails were not forced to repeat the same continuity invariant end to end.
- Version surfaces were not all derived from one coordinated release lock.

## Locked Rules

1. Receiz.com and the offline verifier keep one verification verdict.
2. SDK admission accepts only complete sealed proof objects reconstructed from verified artifact bytes.
3. Manifest admission methods stay closed and fail with `complete_sealed_proof_object_required`.
4. Server admission reuses the existing artifact verifier and authenticated owner boundary.
5. The v2 database RPC requires complete continuity and v1 execution stays revoked.
6. App, frontend, runtime, service worker, SDK, MCP, public verifiers, and release records share the release identity.

## Contract Gates

- Portable asset SDK and classifier tests.
- Server proof/admission and migration contract tests.
- Public OpenAPI/schema and downloadable offline-verifier tests.
- SDK release and primitive-binding contracts.
- Release lock, verification conformance, full release freeze, governance, attestation, and exact-build visual evidence.

## Release-Lock Requirement

`pnpm release:lock 101.0.0` must require these regression lessons in the standalone offline-verifier checkout. Omitting them would sever Portable Asset Continuity release identity from the correction history that closes manifest-only admission while preserving the existing artifact verdict.
