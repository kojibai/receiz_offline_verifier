# Receiz v58.0.0 Product Truth

Release date: April 17, 2026

This document freezes the primitives and invariants that `v58.0.0` is allowed to extend but not weaken after the `v57.0.0` release-cut baseline.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Release surfaces stay canonical | `package.json`, service worker defaults, public version badges, plans/upgrade badges, offline verifier artifacts, and active release docs must all expose `v58.0.0` together when the cut is made. | `package.json`, `public/sw.js`, `app/components/pwa/ServiceWorkerRegistration.tsx`, public page/version surfaces, `scripts/check_release_lock.mjs` |
| Signed-in market entry is authoritative | Signed-in `/market` and signed-in market asset entry must render wallet, positions, and history from authoritative truth on first paint instead of requiring a second settle before the user sees what they own. | `app/lib/market/server.ts`, `app/market/page.tsx`, `app/market/assets/[receizId]/page.tsx`, `app/market/MarketEntryClient.tsx` |
| Market transport stays deterministic and lean | Market snapshot transport must carry the deterministic public seed plus live user overlay rather than shipping the fully expanded desk snapshot payload over the wire. | `app/lib/market/types.ts`, `app/lib/market/compose.ts`, `app/lib/market/server.ts`, `app/api/market/snapshot/route.ts`, `app/lib/market/routeHydration.ts` |
| Wallet and chart-buy surfaces share one live wallet truth | Market wallet state and chart buy-popover wallet state must hydrate through the same summary and snapshot patch path so buying power does not diverge from the signed-in market snapshot. | `app/lib/market/walletSummary.ts`, `app/lib/market/walletSummaryClient.ts`, `app/components/market/MarketValuePill.tsx`, `app/market/MarketDeskExperience.tsx` |
| Tracked positions reflect only live market truth | Live tracked positions must count held shares plus only contracts that are still open; expired, resolving, claimable, claimed, lost, or cancelled contracts cannot inflate live tracked-position counts. | `app/market/MarketDeskExperience.tsx`, market wallet-history contracts |
| Snapshot continuity preserves stronger truth | Later weaker market snapshots must not erase newer wallet, owned-item, prediction-history, or share-trade-history truth that the client already has. | `app/lib/market/snapshotContinuity.ts`, `app/lib/market/routeHydration.ts`, `app/market/MarketDeskExperience.tsx` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Version surfaces do not drift | Public pages cannot advertise one release while `package.json`, the service worker, plans/upgrade badges, or offline verifier artifacts advertise another. | `scripts/check_release_lock.mjs` |
| Market does not drop signed-in truth on entry | If the signed-in market snapshot already exists on the server, the client must not start from an empty position/history shell and wait for a later client fetch to become correct. | server entry loaders, market entry client, market route hydration contracts |
| Market transport does not re-bloat | The transport boundary must remain the lean deterministic seed plus live user overlay; expanded desk composition belongs at the edge that consumes the transport. | transport types plus market snapshot route contracts |
| Buy-popover wallet does not diverge from market wallet | The chart buy surface must not remain in a wallet-loading state while the surrounding market already knows the live signed-in wallet truth. | wallet summary client + market value pill contracts |
| Tracked positions do not count closed contracts | If a contract is no longer open, it must not be counted as a live tracked position. | `app/market/MarketDeskExperience.tsx`, `app/lib/market/__tests__/walletHistoryContract.test.ts` |
| History rails remain durable | Prediction history and share-trade history must remain present across refresh/continuity paths instead of dropping when a later weaker snapshot arrives. | snapshot continuity and route-hydration contracts |

## Release Gates

`v58.0.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `npx tsx --test app/lib/market/__tests__/marketRouteHydrationContract.test.ts app/lib/market/__tests__/marketSnapshotWalletContract.test.ts app/lib/market/__tests__/marketTradeContinuationContract.test.ts app/lib/market/__tests__/marketValuePillContract.test.ts app/lib/market/__tests__/walletHistoryContract.test.ts`
- `pnpm test:market-conformance`
- `pnpm build`
- `pnpm release:lock`
- `pnpm test:release-freeze`

The manual release procedure is defined in `docs/releases/v58.0.0-checklist.md`.

## Residual Risks

- `v58.0.0` rolls up the full post-`v57.0.0` delta, including supporting schema/runtime work outside the market surface. Production still needs to apply the migration set before treating the release as fully live.
- The deterministic transport boundary is smaller now, but any future route that bypasses the transport composer and starts serializing the expanded desk snapshot again can reintroduce payload bloat.
- The tracked-position rule is now explicit in market experience code and contracts, but any new wallet summary surface that reuses raw prediction counts instead of the open-contract rule can reintroduce visible drift.
- `v58.0.0` is not fully release-cut until package metadata, service worker defaults, public badges, plans/upgrade labels, offline verifier surfaces, and release docs all move together to the same version.
