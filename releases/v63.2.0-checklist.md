## Production Environment Preflight

- Wallet settled balance and Sovereign PHI are visible as separate lanes only when the user has PHI value.
- Direct market sell preview tells users what settles now and what returns as Sovereign before the sell is submitted.
- Direct market sell completion patches both visible balances without requiring page refresh.
- Public certificate pages show buy/sell action controls without a default login wall.
- Public certificate buy starts Receiz ID only when needed and uses secure card funding only for shortfall.
- Public certificate sell is available to signed-in non-owner holders who own live shares.
- Certificate chart proof flip shows live proof object data and verification links.
- Funding copy does not frame user-facing liquidity as a provider-specific lane.
- Public version badges, service-worker defaults, offline verifier surfaces, package metadata, release docs, and process docs all show `v63.2.0`.

## Manual Smoke

- Open `/wallet` with a PHI-bearing user and confirm settled balance and Sovereign PHI are separate.
- Open `/market`, perform or simulate a sell, and confirm settled and Sovereign balances update in the same visible surface.
- Open a public certificate link signed out, enter an amount, and confirm the first explicit buy/sell action starts Receiz ID.
- Open the same certificate as a share holder and confirm sell uses owned live shares.
- Flip the certificate chart and confirm the back reads as a live proof object, not a static document card.
- Open `/` online and offline to confirm the service worker still prefers fresh online navigation and falls back only when offline.

## Release Rule

If any value-state framing regression, certificate checkout regression, immediate sell-balance regression, certificate proof regression, stale-shell navigation regression, version-surface mismatch, release-lock failure, or type-check failure occurs, `v63.2.0` is not released from that commit.
