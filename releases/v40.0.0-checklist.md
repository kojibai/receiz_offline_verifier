# Receiz v40.0.0 Release Lock Checklist

Release date target: March 27, 2026

## Purpose

This checklist locks `v40.0.0` to one exact release state without changing product behavior.

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
pnpm install --frozen-lockfile
pnpm release:lock
pnpm lint
pnpm test:deterministic-surfaces
pnpm test:page-first-paint
pnpm build
pnpm test:artifact-binding
pnpm test:document-seal
pnpm test:document-seal-zk
pnpm test:signature-v4
pnpm test:signature-v4-server-remote
pnpm test:api-security
pnpm test:chaos-failures
pnpm test:economy-conformance
pnpm test:verification-conformance
pnpm test:world-conformance
pnpm test:chat-call-quality
pnpm test:chat-call-topology-policy
pnpm perf:budgets
pnpm governance:check
pnpm governance:evidence-check
pnpm governance:verify-artifact-signature
```

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- Record
- Seal
- Verify
- Public `/v` verification route
- Offline verifier open/import/verify flow
- Offline studio install, signer provisioning, record, seal, and verify flow
- Profile original delivery and verified artifact routes
- Sign in, passkey, and Receiz Key flows
- Wallet note status, claim, and admin adjustment surfaces
- WireProof issuance and dashboard surfaces
- Payments embed, login embed, and Connect payment-session entry paths
- Account/business custom-domain surfaces
- World home, district, avatar/twin, and public-world navigation surfaces

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v40.0.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Post-Deploy Observation

After deployment:

- Run `synthetic:uptime`
- Run `prod:slo-check`
- Observe production for 24-72 hours
- Hold a stabilization window with hotfixes only

## Release Rule

If any release lock command, conformance gate, attestation step, or manual smoke check fails, `v40.0.0` is not released from that commit.
