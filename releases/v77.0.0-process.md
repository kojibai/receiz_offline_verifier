# Receiz v77.0.0

## Goal

Ship the full post-`v76.0.0` Signal Circuit, Creator OS, Vault settlement, public conformance, document-seal compatibility, runtime recovery, and public/developer trust-surface work as an official `v77.0.0` release while preserving the v70 public profile law, v74 identity-artwork baseline, v75 local-proof/file-restore baseline, and v76 World/Market/Public Proof baseline.

## What Worked

- Measuring from the real `v76.0.0` tag made the release scope concrete: the pre-bump candidate contains 128 commits, 244 changed files, 103,328 insertions, and 1,373 deletions after that tag.
- Signal Circuit became product-real when the route, mobile chrome, game logic, card claim, card art, sealed download, vault storage, share page, market, packs, battle, and conformance surfaces moved together.
- Signal Cards became durable only when earned achievement truth, transferable collectible truth, ownership, lineage, art, battle profile, rarity gate, and proof bundle were separated instead of overloaded into one mutable game object.
- Pack purchase and reveal became safer when wallet settlement, pending opening records, card issue progress, stored claims, completed-card IDs, and account vault hydration all protected the same user path.
- Vault sync became credible when it became account-scoped, paginated, retryable, and able to preserve local card truth without leaking cards across accounts or hard-stopping at one visible page.
- Card market and share pages became trustworthy when public reads, share images, reserve-first purchase, listings, auctions, and proof panels used sealed card truth and removed liquid-loader replacement paths.
- Signal Pack Foundry became viable when brand control, supply limits, odds, campaign manifests, and campaign records were added around the existing card primitive instead of redefining that primitive.
- Creator OS became useful by being a new operating route with deterministic local first paint, signed-in durable APIs, wallet metering, asset queues, publish queues, worker authorization, and managed launch rails.
- Creator payments became safer when runs, assets, and publish jobs charge wallet value idempotently before production continues, and shortfalls use the existing embedded wallet funding path.
- Public Signal Circuit conformance made the serious parts inspectable: requirements, checks, source digests, simulation bounds, history, rollups, badges, and status inventory.
- Runtime recovery improved when the app stopped mounting a visible stale-runtime interstitial and treated stale runtime cleanup as invisible coherence work, not a second product shell.
- Old Safari document sealing became clearer when it used online Signature V4 fallback and refused to pretend legacy V3 was equivalent trusted sealing.
- Release contracts improved when `test:game-release-contracts` and `test:signal-circuit-conformance` were added to the release-freeze umbrella instead of relying on broad build success.

## What Did Not Work

- A game surface cannot feel like a temporary arcade shell. The Signal Circuit route needed no fake loader, no placeholder first paint, and no product language that made the asset loop feel disposable.
- The card share page could not use a liquid loader path. A public card proof route must open from deterministic card truth and keep the proof panel stable.
- Treating the final downloadable card asset as an afterthought created the wrong success boundary. The flow is not done until the sealed card is issued, stored, and recoverable.
- Purchased cards could not blindly reuse stale locally cached art. Stored deterministic art is useful only when it matches the sealed card truth.
- Account Vault hydration could not be a singleton. Cards must be scoped by owner, paginated by cursor, and safe when a user switches accounts.
- Pack purchases could not rely on best-effort client state. Durable opening records, pending recovery, completed-card IDs, and account-side completion all had to agree.
- Unbatched market quote reads and missing database repair paths created avoidable pressure. Release-critical value surfaces need bounded query shape and repair migrations.
- Legacy V3 signer surfaces could not remain framed as the trusted public path for modern document sealing. Signature V4 is the current trust surface; V3 is compatibility.
- A runtime recovery screen that visibly takes over the app violates settled-shell law even when it is trying to be helpful. Recovery can clear stale assets, but it cannot become fake loading theater.
- Public status is not an internal route dump. Signal Circuit public assurance belongs there; vault private writes, pack purchase internals, worker routes, admin, webhook, and privileged key material do not.

## Practical Rule For Future Work

Treat Signal Circuit cards, Creator OS outputs, Vault packages, document seals, and runtime recovery as Receiz truth surfaces, not ordinary app features. Anything already verified must project immediately, anything newly learned must append, and anything private or operational must stay behind the user action or authority boundary that created it. A release is not locked until the exact public surfaces, version labels, release docs, and contracts prove the same law.
