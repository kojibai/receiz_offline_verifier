## Release Arc

The v93.1.0 arc turns the v93 Sports proof economy breakthrough into a broader Receiz operating model:

- proof history is admitted once, stored durably, and appended forever
- Calendar and Account project the same event proof objects instead of maintaining separate truth
- Wallet settlement windows use Kai head state beneath Chronos display
- Sports vault, showcase, event proof, and visible append reads are bounded beneath proof authority
- the SDK and OpenAPI console expose proof-native developer mechanics without becoming authority
- release hygiene catches stale proof fixtures before they weaken the model
