# Receiz v97.0.0 Invariant Register

Release date: 2026-06-30

`v97.0.0` carries forward the proof-native artifact system and SDK ecosystem invariants from `v96.1.0` and adds the public app-state projection rail invariants needed for developer apps that recover public state from Receiz.

## Locked Invariants

1. Receiz remains a proof-native artifact system. Product truth is carried by files, identities, vault objects, ownership states, settlement states, public witness surfaces, verification routes, Twin surfaces, World projections, and public app-state projections.
2. Complete proof objects carry deterministic identity, sealed/verifiable payload, provenance, inspection behavior, and Kai/Klok state.
3. Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit.
4. Durable proof memory is not cache. First admission creates known truth, and later work appends verified additions.
5. Public app-state projections are durable public projection/index rows for public app recovery. They must not become proof authority.
6. `receiz.appState.publishRecord()` is convenience over the same delegated registry feed route used by `publish(feed)`.
7. `receiz.appState.byHost()` is convenience over public URL lookup with canonical host/URL normalization, not a separate state model.
8. `receiz.publicProof.registryFeed(feed)` remains an alias for existing integrations.
9. Connect is delegated API access. It must not become the identity proof root or replace Receiz ID, Identity Record, Identity Seal, Receiz Key, PBI, or artifact proof.
10. OIDC consent behavior must use browser-correct redirects and readable authorization controls.
11. Public proof projection responses must preserve canonical absolute verification URLs and owner-scoped behavior.
12. Admin performance analytics must measure the user-visible app experience, not misleading internal/admin surfaces.
13. SDK clients, docs, CLI, OpenAPI, status catalog, server, database, SEO, analytics, session, and framework routing conventions remain weaker than admitted proof truth.
14. `@receiz/sdk` release version, app release version, SDK runtime constants, release documents, public labels, and service-worker/runtime version must not diverge before promotion.
15. The operating law remains first admission only, then append forever.

## Forbidden Regressions

- proof object to card
- artifact system to UI experience
- durable proof memory to cache
- public app-state projection to proof authority
- Connect delegated scope to identity proof root
- SDK app-state helper to database authority
- URL/host lookup to a parallel state model
- OpenAPI/status docs to route or proof authority
- admin performance analytics to misleading internal-only measurement
- known-head sync to rediscovery-first loading
- release evidence to optional package hygiene

## Release-Lock Requirement

`pnpm release:lock 97.0.0` must require this document for `v97.0.0` and later release cuts. A release that omits the invariant register is incomplete because proof-native system identity, complete proof object Kai/Klok state, durable proof memory, public app-state projection rail, SDK app-state helpers, Connect delegated access, OIDC consent, public proof projection correctness, admin performance cleanup, version lock, and broader package evidence would not be explicit release truth.
