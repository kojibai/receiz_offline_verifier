Receiz `v92.1.0` - Sports Proof Telemetry Release

## Summary

`v92.1.0` locks the Sports proof arc after `v92.0.0`: Play Command becomes a full 3D play proof surface, Pitch Command remains directly bound to exact pitch witnesses from event panels, event proof video and milestone truth project onto public event pages, live Sports Arena appends return to foreground cadence, Pack Derby keeps its settled game surface, and the release surfaces move together to `92.1.0`.

## What Changed Since v92.0.0

### Play Command Proof Surface

Play Command is now a Sports proof object and public proof surface. It has a standalone `/play` route, card/event popover entry points, shared play-proof projection, and a 3D play space that uses the event's official geometry and telemetry to show the play end to end.

The play surface is required to prioritize the field of play. Outcome geometry and proof details are secondary inspection surfaces, not the dominant first viewport. Controls must stay compact, and telemetry navigation must preserve the immersive field surface.

### Pitch Command Event Binding

Pitch pills in card event panels now resolve the exact pitch witness for the selected historical event instead of opening the current live Pitch Command state. The binding uses event proof, snapshot, witness id, game pk, source date, and selected pitch state so a historical event opens its pitch directly.

The event panel no longer needs separate Open buttons beside proof pills. The proof pills are the action surface.

### Event Proof Video And Milestones

The event page now reuses the same playable-video proof material available in the logged-in card popover, so a public event proof page can show real video when the proof object carries it.

Cycle completion is now projected as milestone truth when the event proof and game state prove it. The Bryce Harper cycle-completion triple is the motivating case: the event page must not miss the milestone when the proof payload and historical game facts establish it.

This is additive beneath the event proof object. The event page must preserve its instant shell and attach media/milestone proof without blocking deterministic first paint.

### Live Sports Arena Appends

Active signed-in Sports Arena views now warm MLB appends on the visible foreground poll cadence instead of waiting behind the stale 75-second guard. The foreground path uses the active user-live read to request append warming immediately, with a scoped foreground sync lease to avoid duplicate work.

This explains the observed production behavior: events that were not visible later appeared because the slower background/server path eventually caught up. The patch remains necessary because future live events must append during the active foreground cadence rather than relying on delayed catch-up.

### Pack Derby

Pack Derby keeps the film-backed baseball game surface, mobile framing, compact HUD, and forced-close regression fix. The bug came from the recent first-scene/drop stabilization work touching the Derby opening path. The release locks that opening Derby should not refresh, close, or lose its first scene when the user enters the game.

### Rarity, Value, And Rookie Truth

The release includes value correction, premium rarity elevation, rookie status projection, rarity audit sampling, unique-pack-pull sampling, live rarity scoring on mount, and updated Sports card export/value contract coverage.

Rarity and value are settlement/proof projections. They may be recomputed or sampled through lower layers, but the visible card value surface must not lose the deterministic chart or replace proof-backed value with transient UI state.

### Pitch Day-Proof Archive

Pitch Command day proofs are served from public archive files with manifest and projection support. The range adds archived day proofs and projections from March 25, 2026 through June 20, 2026, plus a GitHub workflow and backfill script for continued archive appends.

Archived proof truth is not rediscovered before first paint. When local archived proof is held, it paints first and any live/date refresh appends behind it.

### Sports DB Hot Paths

The range includes compact append read paths, entry-append summary RPCs, projection fast-path indexes, append pressure guards, remaining slow-query hot-path hardening, and foreground live-append cadence migration work.

These migrations are performance mechanics beneath proof truth. They accelerate server append/index projection and must not outrank sealed proof payloads, card appends, event proofs, or durable local truth.

### Build And Deployment Fixes

The Vercel type failures reported during the release arc were fixed locally before the release lock:

- `selectedPitchCommandGamePk` is normalized to `string | null`.
- `selectedEventPanel.playableVideo` is narrowed before video metadata handlers read it.
- `useSelectedEventProofCommands` no longer passes nullable event proof into the pitch witness resolver.

The release also records the Vercel author-mapping failure for archive workflow commits and keeps archive append work separate from product runtime truth.
