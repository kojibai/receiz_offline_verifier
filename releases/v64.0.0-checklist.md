Release date: April 21, 2026

Product-truth reference:

## Purpose

This checklist locks `v64.0.0` as the Complete Value Loop release without allowing regression in Reserve/Settlement separation, notes that stay Reserve-only, buyer-funded certificates, mixed-lane sends and buys, split market sells, proof-native Settlement framing, wire-transfer funding, live chart quote math, stale-PWA safety, or release-governed version surfaces.

## Required Automated Checks

Run from the exact release candidate:

```bash
pnpm test:value-loop-guard
npx tsx --test app/lib/wallet/__tests__/valueStateContract.test.ts app/lib/wallet/__tests__/valueStateFramingContract.test.ts app/lib/market/__tests__/fundedLiquidityContract.test.ts app/lib/market/__tests__/shareCertificateMarketFlowContract.test.ts app/lib/market/__tests__/shareTradeHydrationContract.test.ts app/lib/market/__tests__/marketTradeContinuationContract.test.ts app/lib/market/__tests__/liveChartContract.test.ts app/components/pwa/__tests__/serviceWorkerNavigationContract.test.ts
pnpm test:economy-conformance
pnpm test:market-conformance
pnpm exec tsc --noEmit --pretty false
pnpm run build
pnpm release:lock
git diff --check
```

`pnpm test:value-loop-guard` is mandatory. It fails if public copy drifts from the funded Reserve / Settlement / certificate language, if product surfaces collapse the value lanes, if notes stop being Reserve-only, if certificate issuing looks like seller proceeds, or if release freeze stops running the value-loop guard.

## Manual Launch Tests

### Reserve-only user

- Wallet shows Reserve without confusing Settlement UI.
- Send uses Reserve.
- Market buy uses Reserve.
- Market sell funded proceeds post to Reserve.
- Note issue works from Reserve.

### Settlement user

- Wallet shows Settlement balance as the primary account value.
- Settlement balance displays USD first with `Φ` equivalent underneath.
- Reserve appears beneath Settlement as the funded external-conversion lane.
- Settlement and Reserve chart popovers use the account quote.
- Wire / note actions show Reserve as the funding source.

### Mixed user

- Send consumes Reserve first, then fills with Settlement.
- Buy consumes Reserve first, then fills with Settlement.
- Receipt displays Settlement + Reserve.
- Receiver preserves the same lane truth.

### Market sell

- Preview shows what settles to Reserve and what returns as Settlement.
- Full funded sell posts Reserve.
- Partial funded sell posts Reserve slice and returns remainder as Settlement.
- Visible wallet and market balances update without refresh.

### Certificate seller

- Certificate issue locks capacity.
- Funded Settlement share sale credits seller Reserve.
- Available Settlement shares decrease correctly.
- Duplicate capacity cannot be issued twice.

### Certificate buyer

- Public certificate page acts as ownership checkout.
- Buy action starts identity/funding only after the user acts.
- Buyer can fund with Reserve or card/checkout shortfall.
- Buyer receives shares/claim after funded purchase.

### Notes

- Note issue works.
- Note issue from Settlement is blocked and points to Settlement shares as the correct path.
- Existing notes continue to read as Reserve instruments.

### PWA/browser continuity

- Installed stale browser opens fresh online shell.
- Offline root opens canonical cached root shell.
- Wallet, Market, certificate, login, and passkey fallback work in a regular browser with cache.

## Release Rule

Do not release `v64.0.0` if any test or manual smoke shows:

- unfunded value credited as Reserve
- Settlement treated as eligible for note issue
- certificate issue pays seller before sale
- Settlement demoted beneath fiat conversion
- mixed-lane metadata lost
- Φ displayed by symbol-swapping dollars instead of quote conversion
- visible balance requiring refresh after known trade truth
- stale PWA/browser returning to a bad shell
