# Receiz v78.0.0

## Goal

Ship the full post-`v77.0.0` Signal Card World, sealed Vault Card export, mobile/PWA download, Battle Arena, public Exchange, dashboard first-paint, conformance repair, and release-worker coherence work as an official `v78.0.0` release while preserving the v70 public profile law, v74 identity-artwork baseline, v75 local-proof/file-restore baseline, v76 World/Market/Public Proof baseline, and v77 Signal Circuit/Creator OS/Vault baseline.

## What Worked

- Measuring from the real `v77.0.0` tag made the release scope concrete: the pre-bump candidate contains 57 commits, 88 changed files, 11,211 insertions, and 2,600 deletions after that tag.
- Signal Card World became credible when `/game/cards` stopped acting like a local-only inventory and started reading synced owner rows, local verified cards, listings, hidden state, showcased state, transferred-out state, and battle appends as one ownership-discovery surface.
- Global Vault sync closed the discovery gap by publishing server-known Vault rows into the discovery layer without requiring a showcase post first.
- The Vault Card export became a real product primitive when the PNG carried a chunked archive, hash, manifest, Merkle integrity, KAI timing, card paths, and restore rules instead of behaving like a screenshot.
- The mobile/PWA download path improved when export was tied to explicit click intent and no longer relied on hidden iframes, synthetic blank anchors, or panel-open side effects that could navigate or refresh Safari and in-app browsers.
- Transfer downloads became safer when ownership movement had its own explicit package instead of implying that sender and receiver should both keep the same current owner claim.
- Signal Arena became release-worthy when battle requests, queues, loadouts, stake modes, AI twin authorization, settlements, ledger events, transfer integration, and card append verification moved together.
- Auctions became safer when wallet holds were added and route behavior stayed attached to owner/reserve truth.
- `/signal-circuit` and `/exchange` gave the public system a better map: Circuit explains card ownership law, and Exchange connects that law to market and settlement rails.
- Signal Circuit conformance improved when source files, public badges, status catalog rows, and route checks were repaired around actual public code paths.
- Dashboard first paint improved when a large page implementation was split into a smaller page boundary, a client dashboard, an instant route contract, and a dashboard API route.
- PWA continuity improved when service-worker versioning, update checks, private-browser behavior, and header/layout patches moved together.
- Release coherence improved when package version, service-worker version, public release labels, offline verifier labels, README pointer, release docs, and resident surface-worker release markers were moved in one release bump.

## What Did Not Work

- Card discovery could not remain a personal Vault-only read. A global card world must show synced ownership, or it creates a false private-inventory gap.
- A Vault export cannot start because a user opened a panel. That creates fake work, surprise downloads, and visible instability before user intent.
- Hidden iframe and synthetic `_blank` download paths were too fragile in mobile Safari, installed PWAs, and in-app browsers. Download behavior must respect the settled app shell.
- Card export cannot be called complete without restore bytes. If the image cannot carry the archive and manifest needed to rebuild the card set, it is not the release primitive.
- Battle state cannot modify the original card. Requests, settlements, XP, money, transfers, and evolution all belong in append records and ledger rows.
- Public conformance cannot be a stale route inventory. It must point to the real public files and keep private/admin/worker paths out.
- A dashboard cannot be fast if the route first-paint path carries the whole operational surface. The route boundary has to stay small enough to feel settled.
- PWA release fixes cannot leave service-worker defaults on an older patch version. Runtime coherence assets are release surfaces.
- Worker manifest schema numbers are not app release numbers. The release belongs in a release marker so schema compatibility and release identity do not get confused.

## Practical Rule For Future Work

Treat Signal Cards as owned proof-bearing assets across every surface: local Vault, global discovery, export, transfer, battle, auction, market, and public conformance. The original claim stays permanent, current ownership is explicit, every later event appends, and user-visible work starts only after user intent. Release preparation must move public labels, service-worker defaults, release docs, and worker release markers together without weakening schema meaning or first-paint law.
