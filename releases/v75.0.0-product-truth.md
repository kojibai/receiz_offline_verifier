## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint beats richer rediscovery.
- Stronger known deterministic truth is reused before any network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base public shell.
- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains canonical profile truth plus owner permissions.
- Receiz ID device proof is enough to project local identity continuity; server propagation is follow-up publication and session convenience.
- Local account bindings are append-only account relationship truth and can safely repeat or refresh the same relationship.
- A restored file proof is the foreground login event; import, bind, publish, wallet refresh, and global sync are enrichment jobs.
- The recovered file account becomes active/default unless the user explicitly switches away.
- Stale old-cookie, anonymous, or previous-account responses cannot downgrade an admitted recovered account.
- Unsupported Receiz ID crypto gets a backup access path; it does not change the normal one-press Receiz ID sign-in primitive.
- Profile identity downloads, signed artwork, and owner-only release actions append proof and media truth without redefining canonical profile first paint.
- Market continuity preserves stronger same-actor signed-in truth and refuses cross-actor private preservation.
- Sold-position local truth outranks weaker active-position snapshots.
- Explore and market navigation must feel already-there, not like route rediscovery or mobile input churn.
- Service-worker and app-version defaults must match the release version.

## Residual Risks

- The release is not fully locked until `pnpm test:release-freeze` passes after the `v75.0.0` surface bump.
- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
- The post-v74 diff includes generated continuity one-pager PDF/PNG artifacts; those are release evidence outputs, not runtime product dependencies.
- Production database, Redis, Stripe, storage, media tooling, and external provider health remain deployment concerns, but they must not become permission to believe already-held local proof.
