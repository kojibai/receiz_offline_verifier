# Receiz v53.0.0 Product Truth

Release date: April 14, 2026

This document freezes the primitives and invariants that `v53.0.0` is allowed to extend but not weaken after the market line established in `v52.0.0`, hardened in `v52.5.0`, and broadened in the current release candidate.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| File-authoritative trust core | The file remains the authority for verification. Deterministic manifests remain the authority for route truth, and ledger-backed settlement remains the authority for payment truth. Market execution and public sharing still operate around those boundaries instead of replacing them. | `app/lib/profile/document.ts`, `app/lib/profile/server.ts`, `app/lib/receiptView.ts`, `app/lib/wallet/payIntent.ts`, `app/api/wallet/ledger/public/route.ts` |
| Stable public ladder with real Enterprise support | The public product ladder remains `Free -> Authorship -> Business -> Automation -> Enterprise`, and Enterprise must remain a first-class entitlement in shared runtime logic instead of falling through lower plan handling. | `app/lib/premium/plan.ts`, `app/lib/premium/getPlan.ts`, `app/plans/PlansClient.tsx`, `app/upgrade/page.tsx`, `app/lib/world/access.ts` |
| Deterministic market execution | Asset execution must continue resolving through one canonical quote model with replayable quote evidence, bounded spread math, and preview/execution alignment. | `app/lib/market/shareQuoteModel.ts`, `app/lib/market/shareTrade.server.ts`, `app/lib/market/shareFees.ts`, `app/lib/market/server.ts` |
| Public market data surfaces | Public market desk and asset routes must remain capable of exposing cacheable, machine-readable market data rather than only human-facing page shells. | `app/api/market/public-snapshot/route.ts`, `app/api/market/assets/[receizId]/route.ts`, `app/market/assets/[receizId]/seoData.ts`, `app/market/page.tsx` |
| Historical market conformance | Market conformance must persist snapshot history and daily rollups rather than living only as a disposable live response. | `app/api/market/conformance/route.ts`, `app/api/market/conformance/history/route.ts`, `app/api/market/conformance/history/rollups/route.ts`, `app/lib/marketConformanceHistory.ts`, `supabase/migrations/20260414012000_market_conformance_history.sql` |
| Prediction resolution precision | Prediction contracts must resolve from precise reference/final price values and deterministic expiry logic instead of rounded-cent shortcuts or ambiguous close-state behavior. | `app/lib/market/predictionMath.ts`, `app/api/market/predictions/wager/route.ts`, `app/api/market/predictions/finalize/route.ts`, `supabase/migrations/20260414004000_market_prediction_finalize_ambiguity_fix.sql` |

| Public proof and player metadata discipline | Public proof, live-player, and market asset pages must remain structured-data-aware, machine-readable, and indexable when the underlying public asset is meant to be public. | `app/lib/profile/proofStructuredData.ts`, `app/lib/profile/proofManifest.ts`, `app/[username]/verified/[receizId]/page.tsx`, `app/[username]/verified/[receizId]/player/live/page.tsx`, `app/sitemap.ts` |
| Touch-safe market interaction model | Market pills, desk actions, and chart tooling must remain usable on touch-first hardware without accidental duplicate presses or unreadable overlay density. | `app/components/market/ResponsiveTapButton.tsx`, `app/components/market/MarketValuePill.tsx`, `app/components/market/MarketValuePill.module.css`, `app/components/valuation/chart/LiveChart.tsx`, `app/market/MarketDeskExperience.tsx` |

Legacy HMAC verification may remain only as backward compatibility for old artifacts. | `scripts/create_release_attestation.ts`, `scripts/verify_release_attestation.ts`, `docs/governance/release-attestation-standard.v1.md`, `public/governance/governance-keyring.v1.json` |

## Release Gates

`v53.0.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `npx tsx --test app/lib/premium/__tests__/plan.test.ts`
- `pnpm exec tsx --tsconfig tsconfig.scripts.json scripts/test_send_receiz_auth_e2e.ts`
- `pnpm test:market-conformance`
- `pnpm test:release-freeze`
- `pnpm release:lock`
- `pnpm run build`
- `pnpm release:attest`
- `pnpm release:verify-attestation <path>`

The manual release procedure is defined in `docs/releases/v53.0.0-checklist.md`.

## Residual Risks

- Market conformance history now has SQL and local-file fallback paths; operational clarity still depends on production preferring the durable SQL path.
- Public market JSON and SEO surfaces materially improve crawlability and sharing, but they increase the number of places where market metadata and rendered market state can drift if not kept aligned.
- Passkey connection is materially safer now, but users may still misinterpret device-specific passkey availability when their passkeys do not sync across browsers or platforms.
- The beginner-first simplification is the right direction, but older bookmarks and mental models around `/receiz-world` and generic `/login` behavior now depend on redirect and messaging continuity.
- `Ed25519` release attestation is stronger, but releases now depend more explicitly on keyring hygiene: stale or unpublished key IDs should correctly block the release path.
