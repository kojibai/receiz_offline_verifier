# Receiz v47.6.0 Product Truth

Release date: April 9, 2026

This document freezes the primitives and invariants that `v47.6.0` is allowed to extend but not weaken.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Atlas-first world districts | The district workspace must remain a map-first world-reading surface with one coherent atlas rather than regressing back into a flat stacked-card shell. | `app/world/WorldExperience.tsx`, `app/world/WorldExperience.module.css` |
| Concierge twin-quarterback runtime | The existing concierge shell may start live twin work, but that work must stay operator-triggered, visibly tracked, and grounded back into the same transcript context. | `app/api/world/concierge/stream/route.ts`, `app/api/world/concierge/tasks/route.ts`, `app/lib/world/conciergeTasks.ts`, `app/lib/world/intelligence.ts`, `app/world/WorldExperience.tsx` |
| Standalone contacts operating surface | Contacts must remain a first-class, person-first operating view with robust import/export, notes, and interaction memory rather than a thin account subpanel. | `app/contacts/ContactsPageExperience.tsx`, `app/contacts/page.tsx`, `app/account/page.tsx` |
| Outbound delivery integrity | Webhook outreach must reject private/internal destinations, and email-thread persistence must remain contingent on a real successful send. | `app/lib/world/outreach.ts`, `app/api/world/outreach/route.ts`, `app/world/WorldExperience.tsx` |
| Checkout completion truth | Completion-only embedded checkout flows must verify real paid session state before reporting success to the parent surface. | `app/payments/embed/checkout/EmbedCheckoutPageClient.tsx`, `app/api/payments/embed/checkout/session/route.ts` |
| Release-governed version surfaces | Package version, service worker version, public badges, offline verifier surfaces, and release docs must move together on each release. | `scripts/check_release_lock.mjs` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| District workspace cannot flatten back into a passive list | The districts surface may evolve visually, but it must still read as one atlas-led world view rather than a disconnected scroll stack. | `app/world/WorldExperience.tsx`, `app/world/WorldExperience.module.css` |
| Task dispatch must stay visible and grounded | The concierge may direct twin work, but the user must be able to see that a run started and track its current state from the same UI. | `app/api/world/concierge/stream/route.ts`, `app/lib/world/types.ts`, `app/world/WorldExperience.tsx`, `app/lib/world/__tests__/conciergeQuarterback.test.ts` |
| Contacts must stay person-first and portable | Contacts cannot become a dead-end local list; import/export and per-person notes/history must remain core behavior even when native phone-contact APIs are unavailable. | `app/contacts/ContactsPageExperience.tsx`, `app/account/page.tsx` |
| Outbound failures must fail closed | Private/internal webhook targets must not be reachable through world outreach, and failed email sends must not leave behind false conversation state. | `app/lib/world/outreach.ts`, `app/api/world/outreach/route.ts`, `app/lib/world/__tests__/outreachDraft.test.ts`, `app/lib/world/__tests__/outreachRouteContract.test.ts` |
| Checkout success cannot be synthetic | Redirect-required or incomplete checkout returns must not emit `checkout-complete` unless the verified Stripe session is actually paid. | `app/payments/embed/checkout/EmbedCheckoutPageClient.tsx`, `app/api/payments/embed/checkout/session/route.ts`, `app/payments/embed/checkout/__tests__/completionVerificationContract.test.ts` |
| Release surface inventory stays canonical | The committed release surfaces and version badges must align with package and service worker release truth. | `scripts/check_release_lock.mjs` |

## Release Gates

`v47.6.0` is considered locked only if these pass together:

- `npx tsx --test app/lib/world/__tests__/conciergeQuarterback.test.ts app/lib/world/__tests__/outreachDraft.test.ts app/lib/world/__tests__/outreachRouteContract.test.ts app/payments/embed/checkout/__tests__/completionVerificationContract.test.ts`
- `pnpm exec tsc --noEmit --pretty false`
- `pnpm exec next build --webpack`
- `pnpm release:lock`
- `pnpm test:release-freeze`
- `pnpm release:attest`
- `pnpm release:verify-attestation <path>`

The manual release procedure is defined in `docs/releases/v47.6.0-checklist.md`.

## Residual Risks

The following areas are materially stronger after this release but are not yet absolute:

- The concierge task layer is now a real operator surface, but it is still not a durable multi-worker background orchestration system with persisted job recovery.
- The contacts app is robust and portable, but native mobile address-book access still depends on browser/device support and the current surface remains local-first rather than server-synced.
- The outbound webhook guard now blocks non-public destinations, but a strict destination allowlist and stronger destination signing would further tighten production use.
- Discovery-feed cache revalidation can still log transient upstream fetch failures during production builds; current behavior does not block the build, but it remains an operational signal worth watching outside the release lock itself.
