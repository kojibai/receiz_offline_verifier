## Title

Receiz `v80.0.0` - Major Stable Release

## Summary

`v80.0.0` is the stable major release that turns the `v70.0.0` public alpha law into a complete operating platform.

This release is not a feature bundle and it is not a marketing rename. It is the moment the full Receiz system coheres: identity, proof, profile, wallet, market, world, creator production, public proof, Signal Circuit, vault export, conformance, and runtime continuity now sit behind one release law.

The product is built around one unusually strict idea:

```txt
known verified truth moves first
server and network paths append verified additions
visible surfaces do not visibly rediscover what the system already knows
```

That law is now expressed across the whole product. A user can create identity, recover identity, publish proof, move value, hold cards, export vault state, inspect public conformance, run creator work, and use World/Market loops without the product pretending verified local truth is provisional cache.

## Release Trail From v70.0.0

The full trail from `v70.0.0` to this candidate spans 460 commits and 755 changed files. The immediate post-`v78.0.0` stabilization candidate is `997418abb`, 17 commits after tag `v78.0.0`, with 28 changed files, 3,439 insertions, and 1,334 deletions.

### v70.0.0 - Public Alpha Law

`v70.0.0` established the public alpha contract: Receiz ID is the everyday login rail, `/<username>` is canonical profile truth, `?manage=1` is an owner permission overlay, Settlement and Reserve preserve funded causality, local proof is real held truth, and release surfaces must move together.

This was the point where Receiz stopped behaving like an ordinary web app and became a proof-native product. The device can hold truth. A file can carry proof. A route can project canonical history. The server publishes, mirrors, recovers, syncs, and mints sessions, but it does not get to downgrade stronger proof already held by the user.

### v72.0.0 - Stable Continuity

`v72.0.0` hardened the visible continuity model after public alpha. Account switching became actor-scoped. Warmed wallet, market, profile, and world state had to clear when the active user changed. Owner/private data remained enrichment, not a replacement for public profile truth. Mobile Live Twin face features became bounded fit contracts instead of viewport guesses.

The practical result was a calmer product: a signed-in user does not pay a first-paint penalty for being signed in, and a stale warmed route cannot bleed one actor's private truth into another actor's screen.

### v73.0.0 - Ownership, Vault, And Account Authority

`v73.0.0` made ownership a first-class product boundary. Business account entity ownership, holdings, certificates, receipt provenance, transfer proof, vault portability, account page continuity, and account-brain proof detail all became governed release truth.

This moved Receiz from "a proof surface" toward an operating environment for provable ownership. Receipts could carry provenance. Vault state could preserve portable records. Account surfaces started acting like authority surfaces, not settings pages.

### v74.0.0 - Receiz Key And Deterministic Identity

`v74.0.0` made Receiz Key restore, Identity Record/Seal behavior, deterministic profile artwork, long-form originals, attached-link preservation, video playback derivatives, sold-position continuity, and portable profile state first-class.

The key product shift was sovereignty. A user holding a valid private Identity Record or Receiz Key can prove the account. PBI can attest new key export, but it is not permission to restore from an already-held key. Profile identity artwork also became deterministic account truth, not timing luck, upload state, or runtime randomness.

### v75.0.0 - Local Proof And File Restore Parity

`v75.0.0` made local proof the normal user experience. Receiz ID login projects local device identity immediately. Identity Record, Identity Seal, and Receiz Key restore project the recovered account as the active session once proof is accepted. Import, bind, wallet refresh, and global sync are enrichment jobs after the account is already real on screen.

This release also added old-browser fallback access, account/wallet/upgrade restore parity, stronger same-actor market continuity, sold-position protection, stable market navigation, Explore packaging, and Plans/Upgrade parity. The product stopped asking users to refresh to make already-proved identity visible.

### v76.0.0 - Value Production And Public Proof

`v76.0.0` expanded Receiz from proof continuity into value production. World gained agentic business actions, public-safe task trace, Sales Brain, autopilot briefs, campaigns, revenue attribution, and concrete next commercial actions. Market Twin became a fast-entry, mandate-safe money loop with persisted authority and deterministic replay. Public Proof became a governed observe, claim, verify, seal, dispute, rights, identity, and registry surface.

Wallet reserve settlement, public/developer Status, payment paths, and account surfaces also moved forward. The release made Receiz legible as an operating platform: users can produce business artifacts, prove public claims, move value, and inspect public system health without exposing private attack surfaces.

### v77.0.0 - Signal Circuit, Creator OS, Vault, And Runtime Trust

`v77.0.0` added the largest new product surface in the train: Signal Circuit. Signal Cards became sealed Receiz assets with separated achievement truth, transferable collectible truth, deterministic art, rarity, mechanics, lineage, battle profiles, public/share/market pages, pack purchase/reveal, and vault persistence.

Creator Workspace and Creator OS became production rails with durable projects, runs, messages, events, outputs, assets, versions, publish jobs, safety reviews, managed launch, and wallet-backed AI work. Verified Data Vault packages began restoring original bytes. Old Safari document sealing moved to Signature V4 online fallback. Runtime stale recovery stopped showing automatic interstitials or visible route takeovers.

The platform now had proof-native identity, proof-native value, proof-native creator production, and proof-native collectible ownership.

### v78.0.0 - Signal Card World, Export, Arena, Exchange, And PWA Continuity

`v78.0.0` turned Signal Circuit from a newly shipped collectible loop into an owned-asset network. `/game/cards` became Signal Card World, backed by local verified card truth and synced global owner rows. Vault Card export became a real receized PNG carrier with archive bytes, manifest, archive hash, chunking, Merkle integrity, KAI timing, paths, and restore rules.

Battle Arena, auctions with wallet holds, explicit transfer packages, public `/signal-circuit`, public `/exchange`, real conformance/source/status alignment, dashboard first-paint work, service-worker/PWA fixes, and value-loop copy contracts all moved together.

The key law was intent: opening a panel is not download intent, export work starts only from explicit user action, and mobile/PWA download paths must not navigate away from the settled product shell.

## The Remarkable Additions

These are the additions that make `v80.0.0` a major stable release instead of another numbered cut.

### A Local-Proof Identity Operating System

Receiz now has a complete identity model across Receiz ID, PBI/passkey, Identity Record, Identity Seal, Receiz Key, email, old-browser fallback, server sessions, and local account binding registers.

The important part is not that there are many login methods. The important part is that they now obey one law:

```txt
accepted proof -> project the correct account now
background work -> import, bind, mirror, sync, refresh, enrich
weaker stale echo -> cannot take the account away
```

That is why the v74/v75 sequence matters so much. Receiz Key restore, non-PBI record/seal restore, old-iPhone fallback, PBI-style file restore, restored-account precedence over stale cookies, plans/upgrade parity, and fallback account access all turned recovery from a technical path into a real product guarantee.

### Canonical Profile Truth With Deterministic Identity

The profile route is not a mutable social page. It is canonical truth. The release train added deterministic profile artwork, identity seals, identity records, creation-sigil stability, PBI-signed profile media, long-form originals, attached-link preservation past 280 characters, full-text read experiences, video playback derivatives, signed original downloads, and owner-only release controls.

The remarkable product outcome is that a profile can carry authorship, artwork, originals, proof, media, and ownership without letting owner mode redefine the public shell.

### Proof-Native Value And Market Continuity

The market and wallet work is more than trading UI. The train fixed Settlement/Reserve framing, account balance labeling, Reserve payment credits, running reserve balances, sold-position resurrection, share trade hydration, market route continuity, stable internal navigation, mobile buy/sell access, no-zoom inputs, certificate panels, auctions, wallet holds, and value-loop copy.

That means Receiz can talk about value without pretending all value is the same. Settlement remains proof-native. Reserve remains funded action capital. Shares, cards, notes, wires, auctions, transfers, purchases, and battles must keep funded causality intact.

### World Became A Revenue Surface

World moved from a live twin experience into a business operating loop: direct Concierge continuity, Agents SDK boundaries, lead research, contact enrichment, CRM sync, outreach drafting/sending, follow-up campaigns, reply classification, payment and booking paths, proof ledger entries, handoffs, Sales Brain, revenue attribution, autopilot briefs, campaign delivery repair, and public-safe task summaries.

The "wow" is that the twin is no longer decorative. It can produce concrete commercial artifacts while still respecting first-paint continuity and hidden enrichment law.

### Public Proof Became A Public Registry Product

Public Proof added observe, claim, verify, seal, dispute, rights, identity, creator lookup, OAuth channel control, registry feed discovery, graph/source views, tier framing, mobile proof command UI, and public/developer assurance.

It is a full public proof surface, not a verifier modal. It lets Receiz prove URLs, claims, identities, rights, and records while keeping protected mutation paths behind session authority.

### Creator OS Became Durable Production Infrastructure

Creator Workspace is now a real operating route with local first paint, Streamdown output, projects, brand kits, runs, messages, events, outputs, assets, versions, usage ledgers, publish jobs, safety reviews, worker authorization, wallet-backed AI runs, asset lanes, managed launch, GitHub/Vercel job rails, and Vercel Sandbox execution boundaries.

The product implication is large: Receiz can meter, produce, publish, and launch work inside the proof/value system instead of sending creators out to a disconnected toolchain.

### Signal Circuit Became An Owned-Asset Network

Signal Circuit is the largest new product category in the release trail. It grew from Signal Run into sealed Signal Cards, premium deterministic art, rarity, levels, power, arena utility, proof bundles, achievement seals, card anchors, full sealed proof storage, binder image durability, official card framing, append-only card records, Vault roster, pack purchase/reveal, pack failure recovery, global vault sync, Signal Card World, public share pages, deterministic OG/Twitter images, QR refinements, reserve-first purchases, listings, auctions, transfer packages, battle arena, AI twin defense authorization, battle appends, and Signal Circuit conformance.

The key product fact is that Signal Cards are not game state. They are Receiz assets. They can be earned, owned, found, exported, transferred, sold, auctioned, battled, restored, and verified without rewriting the original card claim.

### Vault Export Became Portable Proof, Not Download Decoration

Vault work moved from download convenience into portable proof infrastructure: verified data vault packages, restored original bytes, native previews, sealed ZIP flow, Vault Card PNG carriers, embedded archive bytes, archive hashes, chunking, Merkle integrity, KAI timing, restore rules, explicit transfer evidence, user-intent-only export, and mobile/PWA-safe download paths.

That is a serious product primitive. A user can carry assets and proof out of the app without losing the verification story.

### Exchange, Status, And Conformance Made The System Legible

`/signal-circuit` explains card law. `/exchange` ties Circuit, Market, settlement, Wallet, Payments, Transfers, Ledger, Offline notes, card discovery, and conformance into one value map. Status and conformance publish public-safe endpoints, source digests, route inventories, badges, simulations, history, and rollups without exposing admin, worker, webhook, privileged issuer, private vault write, or pack purchase internals.

The system became inspectable. That is part of stability.

### Runtime And PWA Continuity Became Product Law

The release trail removed visible stale-runtime recovery, patched stale service-worker behavior, kept runtime recovery hidden, fixed PWA headers, repaired Safari and in-app-browser download paths, preserved route return after recovery, and made service-worker defaults part of release lock.

This matters because a proof product cannot feel stable if it visibly reloads, redirects, or swaps shells after the screen already felt settled.

## What v80.0.0 Locks

- Receiz is now a stable proof-native operating platform, not a single receipt tool.
- Local identity proof, file restore, profile truth, market/wallet state, card ownership, vault export, and creator production all follow the same verified-truth-first law.
- `/<username>` remains canonical profile truth; `?manage=1` remains permission overlay only.
- Identity Record, Identity Seal, Receiz Key, PBI, email, server session, and Receiz ID continuity all converge into the same signed-in product experience after proof acceptance.
- Wallet, Settlement, Reserve, market shares, cards, auctions, packs, transfers, and battle stakes preserve funded causality and append-only value truth.
- Signal Cards are owned Receiz assets across game, world, vault, discovery, transfer, auction, battle, market, export, share, and conformance surfaces.
- Creator OS and World agent loops produce durable artifacts, charge or hold wallet value explicitly, and keep first paint settled while private work enriches behind it.
- Public Proof, Signal Circuit conformance, Status, and Exchange make the public system inspectable without publishing private/admin/worker routes.
- Runtime and service-worker coherence are release surfaces. A stale worker, stale header, stale chunk, or stale session must not create a second-load feeling.
- Release labels, service-worker defaults, offline verifier labels, README release pointer, release docs, and world resident worker markers all move together to `v80.0.0`.

## Developer And Public Value

- Users get one coherent product: prove identity, publish proof, recover from files, move value, hold assets, export vault state, play and trade Signal Cards, use World/Market loops, and inspect public proof without relearning the truth model.
