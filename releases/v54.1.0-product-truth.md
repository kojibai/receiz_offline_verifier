# Receiz v54.1.0 Product Truth

Release date: April 15, 2026

This document freezes the primitives and invariants that `v54.1.0` is allowed to extend but not weaken after the broader `v54.0.0` operating baseline.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Deterministic world first paint | `/world` must continue rendering through `WorldRouteSnapshot` with deterministic bootstrap fallback data instead of falling back to a bespoke loading-only shell. | `app/world/loading.tsx`, `app/world/WorldRouteSnapshot.tsx`, `app/world/page.tsx` |
| Authenticated loading shell continuity | When a valid session identity exists, the world loading shell must preserve authenticated shell chrome before the richer world bootstrap payload arrives. | `app/world/loading.tsx`, `app/world/WorldExperience.tsx`, `app/api/world/me/route.ts` |
| Market/profile bottom-bar parity | The bottom-bar profile action must use the same client-side navigation path from market that wallet and the other signed-in routes already use, rather than forcing a document reload. | `app/components/layout/GlobalAuthChrome.tsx`, `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| World route snapshot hydration contract | Richer world snapshots still outrank weaker fallbacks, and `/api/world/me` remains the post-first-paint reconciliation source of truth. | `app/world/WorldRouteSnapshot.tsx`, `app/world/WorldExperience.tsx`, `app/lib/world/routeHydration.ts`, `app/lib/world/__tests__/worldRouteHydrationContract.test.ts` |
| Anonymous preview remains public | Unauthenticated visitors must continue seeing the public world preview instead of accidentally inheriting signed-in shell state. | `app/world/loading.tsx`, `app/world/page.tsx`, `app/world/WorldExperience.tsx` |
| Release-governed version surfaces | Package version, service worker version, public badges, offline verifier surfaces, and release docs must move together. | `scripts/check_release_lock.mjs` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Signed-in `/world` does not flash visitor state | A valid session identity must not briefly render the logged-out world shell during route loading. | `app/world/loading.tsx`, `app/world/WorldExperience.tsx`, `app/lib/world/__tests__/worldRouteHydrationContract.test.ts` |
| Anonymous `/world` stays anonymous | The loading path must not overstate auth state for public visitors. | `app/world/loading.tsx`, `app/world/page.tsx` |
| Loading and page shells stay aligned | `/world/loading.tsx` and `/world/page.tsx` must continue feeding the same snapshot-driven shell rather than drifting into separate UI contracts. | `app/world/loading.tsx`, `app/world/page.tsx`, `app/world/WorldRouteSnapshot.tsx` |
| Market does not hard-reload profile | Bottom-bar profile navigation from `/market` must not bypass client-side routing with `window.location.assign`. | `app/components/layout/GlobalAuthChrome.tsx`, `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Release surfaces remain canonical | Public version badges, service worker version, offline verifier surfaces, and release docs must align with `package.json`. | `scripts/check_release_lock.mjs` |

## Release Gates

`v54.1.0` is considered locked only if these pass together:

- `pnpm exec eslint app/world/loading.tsx app/lib/world/__tests__/worldRouteHydrationContract.test.ts`
- `npx tsx --test app/lib/world/__tests__/worldRouteHydrationContract.test.ts`
- `npx tsx --test app/lib/profile/__tests__/profileRouteContinuityContract.test.ts`
- `pnpm test:page-first-paint`
- `pnpm exec tsc --noEmit --pretty false`
- `pnpm release:lock`
- `pnpm test:release-freeze`

The manual release procedure is defined in `docs/releases/v54.1.0-checklist.md`.

## Residual Risks

- The loading shell now reflects session truth earlier, but it still intentionally waits for `/api/world/me` before showing the richer profile, launchpad, and runtime state.
- Market/profile navigation now stays client-side, but continuity still depends on the broader signed-in route prefetch path staying healthy across future bottom-bar changes.
- Session identity can still change between first paint and refresh, so invalidation discipline remains important for logout and expired-session paths.
- The patch is small, but the clean release still depends on the broader release-freeze suite and build path remaining healthy in the target environment.
