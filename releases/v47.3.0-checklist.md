# Receiz v47.3.0 Release Lock Checklist

Release date target: April 9, 2026

## Purpose

This checklist locks `v47.3.0` to one exact release state without allowing regression in custom-domain auth continuity, embedded-browser sign-in fallback, custom-domain profile media continuity, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v47.3.0-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
npx tsx scripts/test_custom_domain_auth_bridge.ts
npx tsx --test app/signin/__tests__/passkeyBrowserSupport.test.ts app/api/media/image/__tests__/route.test.ts app/lib/profile/__tests__/livePlayerShareContract.test.ts
pnpm exec tsc --noEmit --pretty false
pnpm exec next build --webpack
pnpm release:lock
```

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- A previously connected and DNS-live custom hostname opens `https://custom.host/` on the bound public profile and the header / avatar artwork load correctly on first paint
- A user already signed in on `receiz.com` can sign in from a verified custom domain without re-authenticating manually
- A custom-domain signing flow returns from the canonical Receiz signer back to the business host successfully
- In Instagram, X/Twitter, TikTok, Facebook, LINE, or another embedded browser, the sign-in surface opens email fallback automatically and explains that passkeys may require opening the page in the device browser
- Public version badges and offline verifier surfaces all show `v47.3.0`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v47.3.0`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any auth-bridge regression, embedded-browser fallback regression, media continuity regression, release-lock check, or attestation step fails, `v47.3.0` is not released from that commit.
