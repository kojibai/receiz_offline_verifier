# Receiz v97.5.0 Performance Findings

Release date: 2026-06-30

`v97.5.0` keeps the `v97.2.0` enterprise app runtime SDK law intact and extends the developer surface to the MCP agent rail. Developers and agents can recover public app state, compose enterprise SDK rails, and inspect Receiz capability through MCP while proof objects, sealed artifacts, verified local truth, ownership appends, settlement rows, durable proof memory, Kai/Klok state, public projections, and verified appends remain stronger truth.

## Findings

- Public app-state and public-store projection reads improve cold-start recovery for storefronts and app surfaces without adding a separate public-state database as authority.
- SDK `appState` and `publicStore` helpers reduce integration time by encoding URL/host normalization and durable projection envelopes.
- Enterprise SDK rails for customer sessions, portals, merchants, checkout, media, domains, events, jobs, webhooks, permissions, audit, risk, compliance, portability, search, notifications, release pinning, sandbox, React hooks, hosted UI helpers, and offline proof queues improve developer throughput when they remain convenience beneath proof authority.
- MCP diagnostics, capability inspection, scope discovery, app-state/public-store calls, proof lookup, offline verification, sports memory, pitch witnesses, transfer previews, pack previews, marketplace templates, and `receiz://` resource templates improve agent build loops without creating authority.
- Receiz-issued delegated agent tokens improve setup safety because agents receive scoped, revocable, audited access instead of cookies, OIDC client secrets, Vercel secrets, or MCP host state.
- Resolving npm `.bin` symlinks through real paths improves MCP startup reliability for `npx -y @receiz/mcp-server`.

## Preserved Rules

- First paint must project known proof truth before hydration, analytics, SEO, session checks, lazy imports, global sync, server reconciliation, SDK convenience work, MCP calls, or hosted React controls.
- Hidden work may append verified additions only.
- Public app-state and public-store projections are public recovery/index rows beneath proof objects and durable proof memory.
- SDK helpers are typed convenience beneath proof objects, identity primitives, ownership appends, settlement rows, public projections, and verified appends.
- MCP tools are agent tooling beneath SDK/API rails and must report `mcpAuthority: false`.
- Delegated agent tokens authorize scoped Connect/OIDC access and do not become identity proof roots.
- Enterprise app runtime and MCP recovery must not teach developers or agents to add Supabase, Redis, Vercel KV, Shopify, custom databases, cookies, deployment secrets, or MCP host state as proof authority.
