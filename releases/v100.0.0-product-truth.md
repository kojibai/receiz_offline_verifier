## Release Law

`v100.0.0` is The Living Proof Release. Proof objects live, move, act, and compose without surrendering deterministic identity, sealed truth, custody, verification, or first-paint continuity to weaker runtime mechanics.

`@receiz/sdk` composes these primitives beneath their authority, and durable proof memory remains **first admission only, then append forever**.

## Frozen Primitives

| Primitive | v100.0.0 implementation truth | Forbidden downgrade |
| --- | --- | --- |
| Sealed Twin identity | Character projection, genome, mind artifact, portrait source, and showcase identity remain proof-bound. | Atlas, route, DB, or session state becoming identity authority. |
| Living proof media | Portrait atlases, animation regions, and Phi/Kai motion project deterministically from admitted Twin truth. | Treating proof media as optional decoration or rediscovered transport state. |
| Existing-object continuity | Current-portrait guards admit post-paint atlas appends into existing Twins. | Blocking first paint, replacing the current portrait, or inventing a parallel Twin. |

| Operational proof rails | Authenticated SDK routes append state, admit assets, project events, and exchange admitted proof objects through custody-aware rails. | SDK/API response becoming proof, ownership, custody, or settlement authority. |

## Frozen Invariants

1. Stronger sealed artifact and deterministic proof truth outrank server, DB, session, SDK, MCP, cache, and UI projection.
2. Backfill and hydration append behind settled first paint and cannot redefine the visible primitive.
3. A Twin atlas is admitted only against the enclosing Twin proof object and current portrait source.
4. Proof/custody registers are atomic and deterministic; escrow cannot exchange an unadmitted object.
5. Known Profile, Market, Sports, and Twin truth is projected immediately rather than rediscovered.
6. SDK and MCP compose existing Receiz primitives and do not introduce a competing authority.
7. Kai/Klok state orders proof objects; Chronos remains a compatibility/display projection.
8. Version identity moves across app, packages, runtime, service worker, public labels, verifiers, and release records as one lock.

## Residual Risks

- Production DB state is not proved by local contracts.
- Session-dependent owner/backfill behavior requires authenticated browser evidence where credentials are available.
- Package registry publication, annotated tag creation, push, deployment, and production smoke verification are external promotion actions.
- A local pass does not authorize production mutation.

## Versioning Law

From `100.0.0` onward, patch versions correct backward-compatible behavior, minor versions add backward-compatible capability, and the next major version is reserved for an intentional breaking public contract or artifact boundary. Product magnitude is expressed through a named release, not by weakening version meaning.
