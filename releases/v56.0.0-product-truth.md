# Receiz v56.0.0 Product Truth

Release date: April 17, 2026

This document freezes the primitives and invariants that `v56.0.0` is allowed to extend but not weaken after the `v55.1.0` continuity baseline.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Release surfaces stay canonical | `package.json`, service worker defaults, public version badges, offline verifier artifacts, and active release docs must all expose the same release number when the cut is made. | `package.json`, `public/sw.js`, `app/components/pwa/ServiceWorkerRegistration.tsx`, public page/version surfaces, `scripts/check_release_lock.mjs` |
| Profile/world/market continuity remains the product floor | The `v55.1.0` load-once/reuse-everywhere continuity model remains authoritative for profile entry, public profile navigation, market continuation, and live-player handoff. | `app/profile/page.tsx`, `app/[username]/UserProfileRouteClient.tsx`, `app/components/layout/GlobalPostComposerHost.tsx`, `app/market/MarketDeskExperience.tsx`, `app/components/market/MarketValuePill.tsx` |
| World route hydration rejects malformed cached truth | The world client may reuse prepared route truth, but only if cached snapshot shape is complete and safe to adopt. | `app/lib/world/routeHydration.ts`, `app/lib/world/__tests__/worldRouteHydrationContract.test.ts` |
| World concierge is entitlement-aware | Concierge task polling, live task controls, and the concierge live surface must only run for Automation-entitled accounts. | `app/world/WorldExperience.tsx`, `app/api/world/concierge/tasks/route.ts`, `app/lib/world/__tests__/worldConciergeTaskControlsContract.test.ts` |
| Fresh-account analytics never break owner entry | Profile analytics tracking may be best-effort, but stale database event constraints must never turn fresh-account login or activation into a visible server failure. | `app/api/profile/analytics/track/route.ts`, `app/lib/profile/analytics.ts`, `app/lib/profile/__tests__/profileAnalyticsTrackContract.test.ts` |
| Analytics schema matches emitted owner events | Database constraints and rollups must include the owner-side analytics events the product emits during activation and sharing flows. | `supabase/migrations/20260417094500_profile_analytics_activation_event_support.sql` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Version surfaces do not drift | Public pages cannot advertise one release while `package.json` or the service worker advertises another. | `scripts/check_release_lock.mjs` |
| World does not crash from stale route cache | Logged-in world entry must not fail because a stale cached snapshot is missing required bootstrap arrays or objects. | `app/lib/world/routeHydration.ts` |
| Non-Automation accounts do not hammer concierge task APIs | Signed-in users without Automation access must not repeatedly hit `/api/world/concierge/tasks` and fill the console with expected 403s. | `app/world/WorldExperience.tsx`, `app/lib/world/__tests__/worldConciergeTaskControlsContract.test.ts` |
| Fresh-owner analytics stay non-breaking | New account activation and first profile load must not produce `500` analytics route failures just because the DB constraint set is older than the app code. | `app/lib/profile/analytics.ts`, `app/lib/profile/__tests__/profileAnalyticsTrackContract.test.ts` |
| Continuity baseline remains intact | `/profile`, public profiles, market/chart continuation, sell gating, and live-player fit remain at least as strong as the `v55.1.0` baseline. | `docs/releases/v55.1.0-product-truth.md` plus current profile/market/world contracts |

## Release Gates

`v56.0.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `npx tsx --test app/lib/world/__tests__/worldConciergeTaskControlsContract.test.ts`
- `npx tsx --test app/lib/profile/__tests__/profileAnalyticsTrackContract.test.ts`
- `pnpm build`
- `pnpm release:lock`

The manual release procedure is defined in `docs/releases/v56.0.0-checklist.md`.

## Residual Risks

- If production does not receive the analytics constraint migration, owner-side analytics events will be skipped rather than tracked. That avoids broken UX, but it is still a deploy gap until the migration lands.
- World concierge gating now protects non-Automation users from repeated 403s, but any future concierge surface added outside the shared gate can reintroduce the same bug class.
- `v56.0.0` is not fully release-cut until package metadata, service worker defaults, offline verifier surfaces, public badges, and release docs all move together to the same version.
