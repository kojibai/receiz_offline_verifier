Release date: 2026-06-30

`v96.1.0` preserves Receiz as a proof-native artifact system and locks the expanded `@receiz/sdk` Twin and World surface beneath the same primitive law as the product.

## Frozen Primitives

| Primitive | v96.1.0 truth | Must not regress |
| --- | --- | --- |
| Proof object | Complete Receiz proof objects carry deterministic identity, sealed/verifiable payload, provenance, inspection behavior, and Kai/Klok state. | Do not let SDK, docs, routes, or examples treat proof truth as generic app state. |
| Kai Klok | Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit. | Do not use Chronos as the primary state machine for admitted proof objects. |
| Durable proof memory | SDK and product surfaces use first admission only, then append forever. | Do not call admitted proof memory cache, stale data, hydration state, or rediscovery state. |
| Connect | Connect is delegated API access for third-party apps. | Do not let Connect become the identity proof root or a replacement for Receiz ID, Identity Record, Identity Seal, Receiz Key, PBI, or artifact proof. |
| Twin | Twin routes expose delegated Twin read/write capability beneath the user's authorized Receiz account and proof memory. | Do not let Twin convenience outrank identity proof, durable local truth, or verified appends. |
| World | World clients expose public World/Twin projection surfaces for developer integrations. | Do not make public World reads a new proof issuer or DB authority. |
| OpenAPI/status catalog | Developer docs name the available routes and scopes. | Do not treat docs as authority above route contracts and proof verification. |
| SDK package | `@receiz/sdk` version must match the app release version. | Do not release an SDK package whose package version, runtime constant, docs, or app release identity diverge. |

## Frozen Invariants

- Use known verified truth immediately.
- Ask only for verified additions after the known proof/Kai head.
- SDK convenience stays beneath proof objects, verified appends, ownership appends, settlement rows, identity artifacts, Twin routes, and World projections.
- Connect scopes grant delegated route access; they do not create truth.
- Twin and World API helpers must preserve existing server route boundaries.
- Developer examples must teach local proof memory and known-head append sync.
- A partial later payload may fill missing fields, but it must not downgrade stronger admitted truth.
- `@receiz/sdk` release version must match the app release version.

## Release Gates

## Release-Lock Requirement

`pnpm release:lock 96.1.0` must require this document for `v96.1.0` and later release cuts. A release that omits the product-truth freeze is incomplete because proof object authority, Kai/Klok state-machine doctrine, durable proof memory law, Connect delegated-access limits, Twin and World SDK boundaries, OpenAPI/status visibility, and app/SDK version lock cannot remain explicit release truth.

## Residual Risks

- npm publication is an external registry action and must be run after local gates pass.
- Production DB and Vercel health remain runtime evidence after deploy, but they do not become proof authority.
- Full `pnpm test:release-freeze` can be long-running; any skipped umbrella gate must be named before release.
