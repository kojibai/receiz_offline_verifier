## Release Decision

Ship `v92.0.0` only when:

- version surfaces are coherent
- release lock passes
- focused reasoning, sports, value, first-paint, and documentation checks pass
- full promotion gates pass or exact failures are documented as release blockers
- production deployment/cache state is confirmed after promotion
- live production DB and authenticated-session visual evidence boundaries are named exactly
- no stronger proof, local, artifact, ownership, settlement, identity, verified append, or public proof truth is replaced by weaker server/session/API/hydration/UI state
