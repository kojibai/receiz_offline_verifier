## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint beats richer rediscovery.
- Stronger known deterministic truth is reused before any network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base public shell.
- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains canonical profile truth plus owner permissions.
- Signal Circuit cards are Receiz assets, not mutable game UI state.
- Signal Card achievement truth and current owner truth remain separate.
- Signal Card artwork must match the sealed card truth and cannot be downgraded by older local cache.
- Pack openings remain pending until every sealed card is issued, stored, and recoverable.
- Signal Vault reads and writes are owner-scoped and append-only.
- Card battles append verified records and never mutate the original sealed claim.
- Public card pages and share images must render sealed card truth without fake loader replacement.
- Signal Pack Foundry may add brand controls but cannot weaken the underlying card proof primitive.
- Creator Workspace first paint must be local and settled; signed-in AI/workspace/wallet data enriches behind it.
- Creator paid work must be charged idempotently or explicitly blocked by a funding shortfall.
- Creator worker and managed launch routes require authorized queue boundaries.
- Vault packages bind readable files and sealed files by hash; restored previews use restored bytes.
- Old Safari document sealing uses Signature V4 online fallback, not trusted V3 fallback.
- Runtime recovery may clear stale runtime state, but it must not become a visible second shell.
- Public status should increase trust for real users and developers without increasing attacker value.
- Release gates must catch visible behavior regressions, not merely match the latest code.

## Residual Risks

- The release is not fully locked until `pnpm lint`, `pnpm build`, `pnpm test:release-freeze`, and `pnpm release:attest` pass after the `v77.0.0` surface bump.
- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
- Production database migrations for Creator OS, Creator wallet charges, managed launch, Signal Vault, Signal Packs, Signal Circuit conformance history, Signal Pack Foundry, card market, auctions, and battle discovery must be applied in the target environment.
- External services including OpenAI, Stripe, Resend, Supabase, Redis, storage, Vercel Sandbox, GitHub, Vercel deployments, media/render providers, ElevenLabs, and Mubert remain deployment concerns, but they must not become permission to downgrade already-held deterministic truth.
- Signal Circuit scarcity, pack, and auction economics require ongoing monitoring for abuse, database pressure, and settlement drift.
- Status probes must remain safe and curated as new public/developer card, creator, and conformance routes are added.
