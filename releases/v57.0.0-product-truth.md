# Receiz v57.0.0 Product Truth

Release date: April 17, 2026

This document freezes the primitives and invariants that `v57.0.0` is allowed to extend but not weaken after the `v56.0.0` release-cut baseline.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Release surfaces stay canonical | `package.json`, service worker defaults, public version badges, offline verifier artifacts, and active release docs must all expose `v57.0.0` together when the cut is made. | `package.json`, `public/sw.js`, `app/components/pwa/ServiceWorkerRegistration.tsx`, public page/version surfaces, `scripts/check_release_lock.mjs` |
| Managed-owner navigation is canonical | Runtime owner entry must resolve to the managed owner route `/${username}?manage=1` when that route is knowable; generic `/profile` fallback must not be the primary runtime owner path. | `app/lib/profile/managedRoute.ts`, `app/lib/profile/managedRoute.server.ts`, `app/components/navigation/ManagedProfileRouteLink.tsx`, `app/components/layout/GlobalAuthChrome.tsx`, `app/[username]/UserProfileRouteClient.tsx` |
| World and market fast-entry reuse warmed route truth | World and market entry should reuse prepared route snapshots before background upgrade, matching the strongest wallet/profile continuity pattern instead of blocking on full rebuild. | `app/lib/world/clientNavigation.ts`, `app/world/WorldEntryClient.tsx`, `app/world/page.tsx`, `app/lib/market/clientNavigation.ts`, `app/market/MarketEntryClient.tsx`, `app/market/page.tsx` |
| Share certificates are native custody surfaces | Certificate public surfaces must render certificate-native history, cost-basis semantics, owner-only trade controls, and in-place auth/fallback behavior consistent with the current certificate product. | `app/market/c/[noteToken]/MarketShareCertificateExperience.tsx`, `app/components/valuation/chart/LiveChart.tsx`, certificate contracts |
| Certificate custody mutations are ledgered | Mint, restore, buy, and sell certificate mutations must be representable through the dedicated note-event ledger and recoverable into certificate-native history. | `supabase/migrations/20260417123000_market_share_note_events.sql`, `app/lib/market/shareCertificateNoteEvents.server.ts`, `app/lib/market/shareCertificates.server.ts` |
| Market balances and inventory stay live after custody/visibility changes | Export/import certificate flows and profile visibility mutations must update market-visible state without requiring page refresh to discover the real result. | `app/lib/market/shareTradeHydration.ts`, `app/market/MarketDeskExperience.tsx`, `app/lib/market/publicProfileAssets.ts`, `app/api/profile/originals/visibility/route.ts`, `app/api/receiz/route.tsx` |
| Market conformance matches the runtime boundary | Market conformance source manifests and route evidence must cover certificate-native history, note-event ledger guarantees, fast-entry continuity, and deployed trace availability for the current release boundary. | `app/lib/marketConformanceSources.ts`, `app/api/market/conformance/route.ts`, `app/lib/market/__tests__/marketConformanceRouteContract.test.ts`, `next.config.ts` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Version surfaces do not drift | Public pages cannot advertise one release while `package.json`, service worker defaults, or offline verifier artifacts advertise another. | `scripts/check_release_lock.mjs` |
| Owner entry does not rediscover profile identity after click | When owner route truth is already available, profile entry must not route through a weaker generic destination and wait for a second settle. | managed route helpers plus current profile continuity contracts |
| World and market do not discard warmed truth | Prepared route snapshots must be reused on entry; navigation cannot regress back to a blank shell while the app rediscovers data it already has. | world/market route hydration and fast-entry contracts |
| Certificate chart stays certificate-native | Certificate charts must show certificate-native history, basis semantics, and overlay controls instead of falling back to generic market-chart semantics. | `app/market/c/[noteToken]/MarketShareCertificateExperience.tsx`, `app/components/valuation/chart/LiveChart.tsx`, certificate viewport/share contracts |
| Certificate mint/import/export stays balance-coherent | Minting into a certificate removes shares from the live market view immediately, and importing or restoring shares adds them back immediately without a required refresh. | `app/lib/market/shareTradeHydration.ts`, `app/lib/market/__tests__/shareTradeHydrationContract.test.ts` |
| Public market inventory tracks public profile truth | Archiving or publishing public assets must propagate into market/prediction inventory deterministically instead of waiting for a later self-heal. | `app/lib/market/publicProfileAssets.ts`, `app/lib/market/__tests__/publicProfileAssetsContract.test.ts` |
| Market conformance evidence remains deployable | Required market conformance source files, including dynamic-segment certificate surfaces, must resolve in deployed runtime traces. | `app/lib/marketConformanceSources.ts`, `next.config.ts`, market conformance route/contract tests |

## Release Gates

`v57.0.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `npx tsx --test app/lib/profile/__tests__/managedRoute.test.ts app/lib/profile/__tests__/profileRouteContinuityContract.test.ts`
- `npx tsx --test app/lib/market/__tests__/marketConformanceRouteContract.test.ts app/lib/market/__tests__/shareCertificateMarketFlowContract.test.ts app/lib/market/__tests__/shareCertificateViewportContract.test.ts app/lib/market/__tests__/shareTradeHydrationContract.test.ts`
- `pnpm build`
- `pnpm release:lock`

The manual release procedure is defined in `docs/releases/v57.0.0-checklist.md`.

## Residual Risks

- Existing certificates created before the note-event ledger migration still rely on fallback reconstruction until production receives the migration and any optional backfill.
- Fast-entry route continuity is now much stronger across profile/world/market, but any newly added runtime entry surface that bypasses the shared route helpers can reintroduce the same dead-time bug class.
- `v57.0.0` is not fully release-cut until package metadata, service worker defaults, offline verifier surfaces, public badges, and release docs all move together to the same version.
