## Frozen Invariants

- A settled visible shell is immutable unless the user explicitly interacts.
- Truthful first paint beats richer rediscovery.
- Stronger known deterministic truth is reused before any network rediscovery.
- Weaker partial payloads never downgrade stronger known truth.
- Authenticated/private enrichment may append only; it may not slow or replace the base public shell.
- `/<username>` remains canonical profile truth.
- `/<username>?manage=1` remains canonical profile truth plus owner permissions.
- Receiz Key restore is not PBI login; PBI may attest key creation/export, but restore is embedded Identity Record key-payload proof plus active registration.
- Revocation is the global denial path for a stolen or retired Receiz Key, not a reason to require PBI before restore.
- A verified portable state block may seed first-time offline account/profile projection; editable Identity Record metadata or owner text may not.
- Profile identity artwork must remain deterministic across route hydration, workspace hydration, remounts, and signed-state transitions.
- The KaiSigil glyph on identity seals/records is the same creation glyph; trust chrome may sit behind or around it, but must not mask, replace, thicken, recolor, or simplify the glyph.
- PBI signing may append signed identity seal/record state only after artwork has decoded and been admitted locally.
- Downloading a profile Identity Seal or Identity Record is an owner-only operation and must complete PBI first; importing an already-held Identity Record to restore the account remains Receiz Key proof and must not require PBI.
- Profile original links, long-form text, video playback metadata, and signed download state are verified additions or media realization, not first-paint shell replacements.
- Sold market positions and closed local state must outrank weaker live snapshots.
- Service-worker and app-version defaults must match the release version.

- `pnpm release:attest` depends on release attestation signing environment outside the codebase.
