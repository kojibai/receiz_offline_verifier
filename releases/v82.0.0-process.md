# Receiz v82.0.0

## Goal

Ship `v82.0.0` as the Sports Arena gameplay and economy release that locks the full arc from `v80.0.0` platform completeness through `v81.0.0` Signal Card asset maturity into a live sports competition network.

The release candidate is `edb39fa11a`. The release result is Sports as a first-class Receiz system: cards, packs, odds, live scoring, tournaments, rewards, market, trades, conformance, scale, and card memory all operating under the same proof/value law.

## What Worked

- Separating player strength from card rarity kept the economy honest. Strong players can be scarce without pretending every high-stat base card is a rarity variant.
- Making rarity owned-only on card surfaces reduced cognitive load. Base cards stayed clean; owned variants became the surprise premium layer.
- Publishing observed odds beside published odds changed pack balance from copy to evidence.
- Per-game Pick 5 and Main Event became clear only after they were separated as different lock domains.
- The live cockpit became compelling when it stopped being only numbers and started showing rank, payout proximity, card movement, scoring story, and market-style lines.
- Finality improved when results, reward claim, card memory, and next-entry reset became one closed loop.
- Sports cards became durable when issue numbers, pack origin, downloaded-card metadata, career appends, and provenance all moved together.
- Market trust improved when value panels became compact, opaque, line-chart driven, and grounded in bid/ask/floor/sales context.
- Trading became serious when actions became guarded and stale open-state races returned conflicts instead of false success.
- Sports conformance became credible when it moved under Game with its own route, badge, history, odds, rarity, and source checks.
- Scale moved forward when hot snapshots and account-overlay throttling reduced repeated live/tournament pressure.
- PWA and first-paint behavior improved when Sports projected entered/account/cockpit state without requiring a click.

## What Did Not Work

- Letting active entry state replace live field state was wrong. A locked entry is a user state; it is not the MLB play state.
- Falling back from a selected final game to the first live game was wrong. Selected-game truth must be exact or neutral, never borrowed from another matchup.
- Treating Main Event and per-game play as the same lock created confusion. They are separate games with separate rewards and scoring scope.
- Showing rarity on all base cards created weight and confusion. Rarity belongs to issued owned variants.
- Showing internal funding language in purchase surfaces weakened trust. Users need outcome language, not implementation labels.
- Delayed pack reveal sync after issuance showed that issue numbers had to be part of the original reveal payload, not a later recovery step.
- Claimed proof actions that could re-enter old games violated finality. Claimed state must clear, not lock again.
- Sports conformance could not inherit Market conformance. It needed its own official evidence path.
- Bars made value panels feel static. Line charts with live pointers matched the market feel better.
- "More data" did not automatically mean better UX. The win came from reducing ambiguity: one selected context, one score, one field, one next action.

## Practical Rule For Future Work

Sports must stay a live proof loop.

The correct order is:

```txt
selected context
owned lineup
verified live data
scoped score
rank movement
final proof
card memory
market value
next play
```

Any future feature that breaks that order makes the game worse, even if it adds more data.

Every Sports addition should make the user feel one of these outcomes faster:

- I know what game I am in.
- I know which cards count.
- I know why my score moved.
- I know what I need next.
- I know what I earned.
- I know what this did to my card.
- I know why this card has value.
- I can play again without friction.

That is the gameplay bar for Receiz Sports.
