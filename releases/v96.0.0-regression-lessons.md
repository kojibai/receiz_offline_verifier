# Receiz v96.0.0 Regression Lessons

Release date: 2026-06-29

## What Regressed

No product runtime behavior regressed in this release. The gap was release evidence and developer ergonomics:

- SDK CLI conformance was not yet part of the release-freeze umbrella.
- SDK package dry-run evidence was not yet release-blocking.
- Developers had validators and docs, but not a single copy-paste integration suite for common app surfaces.
- Proof memory admission had no typed passive observation event.

## Why It Regressed

The SDK had reached the correct primitive boundary, but release evidence lagged behind product truth. That created an A/A- state: technically strong, but not yet undeniable to an outside developer because the package did not prove itself from install to conformance to starter.

## Locked Rules

- `@receiz/sdk` must be version-locked to the app release.
- SDK local conformance must run without network or database authority.
- SDK pack dry-run must include the CLI, docs, fixtures, and declarations.
- `test:sdk-release-contracts` must remain in `test:release-freeze`.
- Copy-paste docs must keep durable proof memory and knownHead sync as the default integration pattern.
- Passive observation must stay passive: evidence only, never proof authority.
- The operating law is first admission only, then append forever.

## Contract Gates

- `scripts/test_sdk_release_contracts.ts`
- `app/lib/__tests__/developersSdkPackageContract.test.ts`
- `packages/receiz-sdk/test/sdk.test.ts`
- `scripts/test_release_freeze.ts`

These broader platform gates fail if the SDK drifts back into docs-only integration, loses CLI conformance, drops package evidence, imports database authority, or stops teaching known-head append sync.

## Release-Lock Requirement

`pnpm release:lock 96.0.0` must require this document for `v96.0.0` and later release cuts. A release that omits regression lessons is incomplete because SDK CLI conformance, package dry-run evidence, copy-paste integration order, passive observation, version lock, and no-authority-leak lessons would not be explicit release truth.
