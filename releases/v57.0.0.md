# Receiz v57.0.0

Release date: April 17, 2026

## Overview

`v57.0.0` is the route-coherence, certificate-custody, and market-conformance release on top of the `v56.0.0` version-surface baseline. The work since the last tagged release concentrated on one product rule: if Receiz already knows the route truth, custody truth, or certificate truth, the app should stop rediscovering it after the click, after the login, or after the trade.

This release removes the remaining runtime dependence on the weak `/profile` owner fallback, extends wallet-style fast-entry into world and market, turns share certificates into a first-class live custody surface with note-native history and passkey-first management, keeps market balances coherent across certificate mint/import flows without reload, hardens market inventory refresh after profile visibility changes, and expands market conformance so the release boundary matches the runtime boundary.

There is no dedicated `v56.1.0` through `v56.95` release note or tag in the repository. This `v57.0.0` note therefore rolls up the full repository delta after `v56.0.0`, including the intervening internal prep commits, into one coherent release record.

The product-truth freeze for this release is defined in `docs/releases/v57.0.0-product-truth.md`.

## What Defines v57.0.0

- Runtime profile entry now resolves through the canonical managed owner route `/${username}?manage=1` instead of falling through a generic `/profile` destination when the owner route is already knowable.
- World and market now enter like wallet and the best live-player/profile paths: route intent is prepared first, warmed truth is reused immediately, and heavier enrichment happens after paint instead of blocking navigation.
- Share certificates are now treated as a live custody product, not a thin export artifact: the certificate page is compact, chart-first, passkey-first, USD-first, and tied to certificate-native history and basis semantics.
- Certificate mint/import/sell flows now keep market balances, certificate balances, and certificate note history coherent without requiring a page refresh to discover the real state.
- Public market inventory now revalidates against authoritative profile visibility changes instead of waiting for a lucky later self-heal.
- Market conformance now locks the actual market/runtime boundary, including certificate-native history, note-event ledger guarantees, fast-entry route continuity, and required trace coverage.
- Release-governed version surfaces now move together to `v57.0.0` across package metadata, service worker defaults, public verifier artifacts, legal/footer/home/trust/make/plans/upgrade badges, and the active release docs.

## Since v56.0.0

### 1. Profile And Runtime Navigation Became Canonical Instead Of Ambiguous

The app no longer treats `/profile` like a generic owner landing page that can be sorted out after navigation.

The current repository state now adds:

- one shared managed-owner route resolver on both client and server
- canonical `/${username}?manage=1` navigation from auth flows and runtime profile entry points
- proactive owner-route warming from the global auth chrome and chart/market continuation paths
- manage-route client reuse of deterministic owner snapshots instead of discarding them and forcing another settle
- user-facing copy reframed away from `/profile` language toward managed profile / managed workspace wording

The result is that profile entry behaves like the best live-player handoff everywhere else in the runtime: resolve exact destination first, carry route truth forward, then navigate.

### 2. World And Market Now Reuse Prepared Route Truth Instead Of Rebuilding On Entry

Wallet already felt instant because it reuses warmed route truth. This release brings world and market much closer to that same model.

The current state now provides:

- shared world and market client-navigation helpers that prime route snapshots before navigation
- route-link wrappers for runtime world and market entry surfaces instead of raw href or push paths
- bottom-bar, plans, upgrade, profile/twin, market, and marketing-surface entry points routed through the warmed path
- a world fast-entry client shell that reuses prepared world truth before background upgrade
- authenticated world snapshot preference over stale anonymous world truth
- cold `/market` entry that seeds a light entry snapshot first and upgrades in the background instead of blocking on the full desk bootstrap
- market route hydration that promotes already-hydrated chart/pill truth into market route truth so navigation does not throw that work away

This removes a large amount of dead time between click and settled page without reducing page scope or hiding data.

### 3. Share Certificates Became A First-Class Live Custody Surface

The largest product expansion after `v56.0.0` is the share certificate system.

The current repository state now provides:

- a compact mobile certificate surface with the chart as the dominant primary element
- in-place passkey / Receiz ID sign-in on the certificate page with magic-link fallback only after passkey actually fails
- certificate buy and sell controls that are USD-first and aligned to live quote previews
- exact owner-only trade gating and lane visibility derived from real certificate balance rather than weak warmed state
- a slimmer top panel with direct copy/share actions and animated feedback
- dynamic certificate-specific OG and Twitter images plus live route metadata for shared certificate URLs
- canonical public certificate links that rewrite away localhost/dev origins for exported and public-facing use

The certificate is now its own live product surface rather than a mostly static artifact.

### 4. Certificate History, Basis, And Trade Accounting Became Native

The certificate chart and custody logic now operate on certificate-native truth instead of loosely projected market truth.

The current state now adds:

- certificate-native history reconstruction tied to note events instead of only projecting from the market curve
- per-point cost-basis semantics in the chart and tooltip layer
- a dedicated `market_share_note_events` ledger migration plus server helpers for mint, restore, buy, and sell events
- authoritative post-import delta recovery from note metadata instead of assuming imported quantities ahead of the RPC result
- exact imported-quantity sell execution for certificate sells instead of re-deriving a second quantity from USD after import
- certificate chart overlay controls, brighter chart overlays, right-edge label protection, and domain handling that keep the chart readable on mobile

This makes the certificate page more defensible technically: the chart reflects certificate history, the basis line reflects certificate basis, and custody mutations are recorded at the note layer itself.

### 5. Market Balances And Inventory Now Stay Coherent Without Reload

This release also removes several places where market state was correct on the server but stale in the client until refresh.

The current state now includes:

- immediate export-side market snapshot hydration so minting a certificate removes shares from the live desk position immediately
- continued immediate import-side hydration so restoring shares from a certificate updates market balance without reload
- public profile asset inventory self-healing against stale published manifests
- explicit market-inventory invalidation on profile archive/unarchive and new public receiz creation
- chart/market auth continuation that clears sign-in CTAs immediately after successful passkey login and keeps wallet entry usable without refresh

The visible market state now follows the actual custody and visibility mutations as they happen.

### 6. Market Conformance Was Brought Up To The Real Runtime Boundary

The conformance work in this release is substantive, not cosmetic.

The current state now locks:

- certificate note-event ledger requirements
- authoritative import-delta recovery guarantees
- exact-quantity certificate sell execution
- certificate-native history and basis semantics on the public certificate surface
- market/world fast-entry route continuity requirements
- suite-level conformance surface drift checks
- trace coverage that keeps dynamic-segment market certificate source files available in deployed conformance runtimes

This closes the gap between what the market release actually depends on and what the conformance dashboard was previously checking.

## Public Summary

Receiz `v57.0.0` turns the strongest runtime patterns into the default system behavior: profile entry is canonical instead of ambiguous, world and market reuse warmed route truth instead of rebuilding from scratch, share certificates behave like a live custody product with native history and in-place auth, and market balances plus market inventory update immediately when certificate or visibility state changes.
