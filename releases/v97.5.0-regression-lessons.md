# Receiz v97.5.0 Regression Lessons

Release date: 2026-06-30

`v97.5.0` records the main release lesson from the public app-state, `@receiz/sdk`, and MCP work: developer convenience and agent tooling must make Receiz easier to use without inventing a new authority layer.

## What Regressed

- MCP direct-file execution worked, but npm binary execution exited silently through the `.bin` symlink path.
- MCP docs initially showed `npx receiz-mcp` instead of the package invocation `npx -y @receiz/mcp-server`.
- The developer MCP page initially lacked the later offline, sports, preview, and URI template tools.
- Agent setup initially had no first-class Receiz-issued token path, which could push developers toward copying browser cookies, OIDC client secrets, or deployment env secrets.

## Why It Regressed

- The entrypoint guard treated a filesystem symlink path and the resolved module file path as if they were the same string.
- Package execution was tested manually through the real file path before the npm symlink path was contract-covered.
- The MCP surface expanded quickly from SDK diagnostics into agent app-building tools, so docs had to be updated after the implementation hardened.
- MCP is tooling only; delegated write access belongs to Receiz Connect/OIDC and must be issued/revoked/audited there.

## Locked Rules

- CLI entrypoint checks must compare real paths, not raw argv path strings.
- Package docs and developer pages must use `npx -y @receiz/mcp-server`.
- MCP tests must cover initialize, tools/list, tools/call, resource templates, offline verification, and local-first sports helpers.
- MCP tools must report `mcpAuthority: false`.
- MCP may inspect, resolve, preview, and call delegated SDK rails. MCP must not replace, rename, or outrank proof objects.
- Agent tokens must be Receiz-issued, scoped, revocable by token ID, shown once, and audit-labeled with `agent:*`.
- Do not use browser cookies, OIDC client secrets, Vercel env secrets, or MCP host state as delegated proof authority.
- Durable proof memory remains first admission only, then append forever.

## Release-Lock Requirement

`pnpm release:lock 97.5.0` must require this document for `v97.5.0` and later release cuts. A release that omits regression lessons is incomplete because MCP symlink startup, package invocation, MCP docs expansion, delegated agent token safety, MCP authority boundaries, and durable proof memory lessons would not be explicit release truth.
