# Receiz v47.0.1 Release Lock Checklist

Release date target: April 5, 2026

## Purpose

This checklist locks `v47.0.1` to one exact release state without allowing regression in profile composer video upload continuity, deterministic profile/original delivery, profile-page preload discipline, or release-governed version surfaces.

Product-truth reference:

- `docs/releases/v47.0.1-product-truth.md`

## Pre-Tag Requirements

Run from the exact release commit only:

```bash
npx tsx --test app/lib/profile/__tests__/profileOriginalSignedUploadContract.test.ts app/lib/profile/__tests__/profileOriginalUploadRouteContract.test.ts app/lib/profile/__tests__/profileRouteContinuityContract.test.ts
pnpm exec next build --webpack
pnpm release:lock
```

## Manual Smoke

Confirm these flows on the release candidate without changing code:

- Signed-in owner opens `/profile?compose=1`
- Owner uploads an MP4 through the composer drop zone / picker
- Composer completes without `Could not upload the file for sealing.`
- New published original appears on the owner profile and resolves to a canonical `/v/...` verification path
- Profile load no longer produces browser warnings for non-critical artwork being preloaded and not used immediately
- Public version badges and offline verifier surfaces all show `v47.0.1`

## Tag + Attestation

After the pre-tag requirements pass:

1. Create the release tag from the locked commit: `v47.0.1`.
2. Run the `Release Governance Attestation` workflow with the required approval identities and signing secrets.
3. Verify the uploaded release attestation artifact.
4. Store the attestation path and approval reference with the release record.

## Release Rule

If any targeted upload/preload gate, release-lock check, attestation step, or manual smoke check fails, `v47.0.1` is not released from that commit.
