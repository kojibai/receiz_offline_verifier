# Receiz v54.0.0 Product Truth

Release date: April 15, 2026

This document freezes the primitives and invariants that `v54.0.0` is allowed to extend but not weaken after the first `v53.0.0` release-prep baseline.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| File-authoritative trust core | The file remains the authority for verification. Deterministic manifests remain the authority for route truth, and ledger-backed settlement remains the authority for money truth. | `app/lib/profile/server.ts`, `app/lib/profile/document.ts`, `app/lib/wallet/payIntent.ts`, `app/api/wallet/ledger/public/route.ts` |
| Deterministic-first route rendering | Once a deterministic route payload already exists, the UI should render from that graph first and reconcile live deltas afterward instead of cold-booting on every mount. | `app/lib/session/bootstrapHydration.ts`, `app/account/page.tsx`, `app/calendar/page.tsx`, `app/contacts/page.tsx`, `app/upgrade/page.tsx`, `app/plans/PlansClient.tsx` |
| Shared signed-session bootstrap | Signed-in workspace consumers must continue preferring the shared session/bootstrap snapshot before probing route-owned identity endpoints. | `app/lib/session/bootstrapHydration.ts`, `app/components/layout/GlobalAuthChrome.tsx`, `app/lib/session/__tests__/sessionBootstrapConsumersContract.test.ts` |
| Deterministic asset journey state | Profile, artifact viewer, live player, and market preview state must be able to render a deterministic first paint, including market preview value, before any live refresh. | `app/lib/profile/server.ts`, `app/lib/profile/routeHydration.ts`, `app/lib/profile/viewerHydration.ts`, `app/lib/profile/livePlayerHydration.ts`, `app/components/market/MarketValuePill.tsx` |
| World twin session continuity | Once a world twin is loaded for a user session, the asset journey must keep reusing that deterministic world graph and loaded runtime instead of replaying full twin boot on every route handoff. | `app/u/[username]/worldTwinSession.ts`, `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/publicTwinRouteState.ts`, `app/lib/world/__tests__/worldTwinSessionReuseContract.test.ts` |
| Transparent admin ledger operations | Admin balance adjustment and Stripe-funded wallet visibility must remain ledger-transparent and explainable after the fact instead of treating balance as an opaque mutable table value. | `app/admin/page.tsx`, `app/admin/AdminCommandDeck.tsx`, `app/api/admin/wallet/balance-adjust/route.ts`, `supabase/migrations/20260319153000_wallet_stripe_credit_rollups.sql`, `supabase/migrations/20260319170000_wallet_admin_post_adjustment.sql` |
| Public product map stays plain-English | The public ladder remains intelligible through Record, Seal, Verify, Live Twin, Business, and the canonical explanatory surfaces without collapsing back into product jargon first. | `app/start-here/page.tsx`, `app/business/page.tsx`, `app/everything-receiz-lets-you-do/page.tsx`, `app/what-can-you-do-with-a-receiz/page.tsx`, `app/what-is-a-receiz/page.tsx`, `app/public-twin/page.tsx` |
| Deterministic SVG serialization | Kai sigil geometry and color serialization must remain stable across server and client so the same deterministic input produces the same DOM output at hydration time. | `app/components/KaiSigil/constants.ts`, `app/components/KaiSigil/Art.tsx`, `app/components/KaiSigil/ZKGlyph.tsx`, `app/components/KaiSigil/__tests__/geometryDeterminism.test.ts` |
| Release-governed version surfaces | Package version, service worker version, public badges, offline verifier surfaces, and release docs must move together. | `scripts/check_release_lock.mjs` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| Signed workspace consumers do not cold-probe first | When the app already has a valid signed bootstrap snapshot, signed routes must not lead with `/api/session` or equivalent bootstrap fetches. | `app/lib/session/__tests__/sessionBootstrapConsumersContract.test.ts` |
| Price pills do not wait for click to show value | Public profile/player/artifact price pills must render deterministic `marketPreview` before any live market interaction and then hydrate in background. | `app/components/market/MarketValuePill.tsx`, `app/lib/market/__tests__/marketValuePillContract.test.ts` |
| Twin does not reload across the asset orbit | Profile, artifact, and live-player routes must continue sharing the same twin continuity path and loaded runtime reuse semantics. | `app/[username]/verified/[receizId]/VerifiedArtifactRouteClient.tsx`, `app/[username]/verified/[receizId]/VerifiedArtifactRouteSnapshot.tsx`, `app/[username]/verified/[receizId]/player/live/PlayerLiveRouteClient.tsx`, `app/lib/world/__tests__/worldTwinSessionReuseContract.test.ts` |
| Dock does not leak the square render image | The twin dock must not flash the square render fallback before the rigged twin appears. | `app/u/[username]/DeferredWorldProfileModule.tsx`, `app/lib/world/__tests__/worldTwinSessionReuseContract.test.ts` |
| Admin adjustments remain ledger-backed | Admin money operations must not regress into direct-balance mutation without ledger evidence. | `app/api/admin/wallet/balance-adjust/route.ts`, `app/admin/page.tsx` |
| Canonical references do not duplicate the featured page | The featured “Everything Receiz lets you do” surface must not be listed again in the compact canonical-reference grid below it. | `app/start-here/page.tsx` |
| Kai sigil hydration remains stable | SSR and client Kai sigil output must remain string-identical for equivalent deterministic inputs. | `app/components/KaiSigil/__tests__/geometryDeterminism.test.ts` |
| Release surfaces remain canonical | Public version badges, service worker version, offline verifier surfaces, and release docs must align with `package.json`. | `scripts/check_release_lock.mjs` |

## Residual Risks

- Shared session/bootstrap continuity is correct for first paint, but stale invalidation policy can still cause temporarily old signed-in state if mutation flows fail to refresh the shared graph.
- Deterministic market preview behavior improves first paint, but preview/live divergence still depends on good post-settle reconciliation and correct invalidation when market state moves quickly.
- Twin runtime reuse is materially better, but full-session continuity still depends on route handoffs staying inside the intended profile/artifact/live-player family.
- Admin ledger transparency is stronger, but deploys that miss the expected SQL/function surfaces can still produce drift between UI controls and actual reporting/state behavior.
- The public product map is clearer, but it also spreads the story across more pages, which means canonical-link discipline and navigation consistency matter more than before.
