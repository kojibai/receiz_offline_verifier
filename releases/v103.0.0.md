## Title

Receiz `v103.0.0` — Native Record And Seal Continuity Correction

## Summary

Receiz v103.0.0 is the corrective native-continuity release for the official app, `@receiz/sdk`, `@receiz/mcp-server`, and `@receiz/ai-skills`.

The v102 verifier incorrectly required a parallel portable carrier from native Profile Composer, Record, and Seal proof objects. v103 removes that gate. Existing and newly created native Receiz proof bundles verify through the same proof-object architecture.

Production rollback evidence confirms v101 as the valid baseline: the same saved assets verify and Profile showcase downloads work on v101, then fail only under the v102 verifier gate. v103 preserves v101 native verification while correcting SDK composition.

SDK `assets.createProofObject` now resolves the authenticated account's real Receiz ID, creates its native Record projection before sealing, seals the source bytes at the same coordinates, and verifies the returned native claim and verify path. Developers cannot submit an owner or construct parallel continuity.

Durable proof memory remains **first admission only, then append forever**.

Release evidence is defined by the v103 checklist and full release-freeze gate.

## Offline Verifier Release-Lock Requirement

`pnpm release:lock 103.0.0` must pass in the standalone offline-verifier checkout before its deterministic verifier and public witness surfaces are tagged. The lock restores native proof-bundle authority and must not reintroduce the rejected parallel portable carrier as a universal verification requirement.
