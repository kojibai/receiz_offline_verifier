# Receiz v97.5.0 Invariant Register

Release date: 2026-06-30

`v97.5.0` carries forward the proof-native artifact system, public app-state projection, and enterprise SDK invariants from `v97.2.0` and adds the MCP agent rail invariants needed for production developer and agent workflows.

## Locked Invariants

1. Receiz remains a proof-native artifact system. Product truth is carried by files, identities, vault objects, ownership states, settlement states, public witness surfaces, verification routes, Twin surfaces, World projections, public app-state projections, MCP-addressed resources, and verified appends.
2. Complete proof objects carry deterministic identity, sealed/verifiable payload, provenance, inspection behavior, and Kai/Klok state.
3. Kai Klok is the proof object state machine. `kaiPulseEternal` / `kai_upulse` is the pulse unit.
4. Durable proof memory is not cache. First admission creates known truth, and later work appends verified additions.
5. Public app-state and public-store projections are durable public projection/index rows for public app recovery. They must not become proof authority.
6. Enterprise SDK helpers are typed convenience beneath proof objects, identity primitives, ownership appends, settlement rows, durable proof memory, public projections, Kai/Klok state, and verified appends.
7. MCP tools are agent tooling beneath SDK/API rails and must report the authority boundary.
8. `mcpAuthority: false` remains part of the MCP response law wherever tool responses summarize authority.
9. MCP resource templates address proof and projection surfaces; they do not create authority.
10. Delegated agent tokens are scoped Connect/OIDC access, shown once, revocable by token ID, and audit-labeled with `agent:*`.
11. Delegated agent tokens, MCP hosts, env vars, cookies, OIDC client secrets, and deployment secrets must not become identity proof roots.
12. Offline proof queues replay verified additions after known heads and must not duplicate side effects.
13. React hooks and hosted component factories remain UI/developer convenience beneath Receiz primitives.
14. Package execution must use `npx -y @receiz/mcp-server`; the binary is `receiz-mcp`.
15. CLI entrypoint checks must compare real paths so npm `.bin` symlinks start the stdio server correctly.
16. `@receiz/sdk`, `@receiz/mcp-server`, app release labels, runtime constants, release documents, public labels, and service-worker/runtime version must not diverge before promotion.
17. The operating law remains first admission only, then append forever.

## Forbidden Regressions

- proof object to card
- artifact system to UI experience
- durable proof memory to cache
- public app-state projection to proof authority
- enterprise SDK helper to issuer, verifier, database authority, or settlement authority
- MCP tool to proof authority
- delegated agent token to identity proof root
- cookie, OIDC client secret, Vercel secret, or MCP host state to delegated proof authority
- settlement primitive to balance display
- React hosted component to proof authority
- offline proof queue to rediscovery-first loading
- idempotent replay to duplicate append behavior
- audit log to verified history replacement
- search surface to proof authority
- release evidence to optional package hygiene

## Release-Lock Requirement

`pnpm release:lock 97.5.0` must require this document for `v97.5.0` and later release cuts. A release that omits the invariant register is incomplete because proof-native system identity, complete proof object Kai/Klok state, durable proof memory, public app-state projections, enterprise SDK helpers, MCP agent tooling, delegated agent-token limits, offline proof queues, React subpath limits, Connect delegated access, version lock, and broader package evidence would not be explicit release truth.
