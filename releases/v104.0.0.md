## Title

Receiz `v104.0.0` — Application Contract Compiler

## Summary

Receiz v104.0.0 coordinates `@receiz/sdk`, `@receiz/mcp-server`, and
`@receiz/ai-skills` at exact version 104.0.0. A typed app contract now compiles
into deterministic scopes, environments, framework wiring, findings, safe
change previews, conformance, and upgrade guidance.

The SDK owns the compiler core. CLI and MCP are thin interfaces. MCP inspection
is not verification, writes require exact preview confirmation, and generated
changes remain inside the explicit repository root. The Next.js App Router
adapter generates small modules without database proof authority or browser
secrets.

The MCP package dependency defect is closed: packed and externally installed
manifests contain exact registry versions and no `workspace:` protocol.

Durable proof memory remains **first admission only, then append forever**.

## Offline Verifier Release-Lock Requirement

`pnpm release:lock 104.0.0` must pass in the standalone offline-verifier checkout before its deterministic verifier and public witness surfaces are tagged. The lock records the application compiler as a convenience rail and must not promote inspection, planning, generation, framework wiring, or package state above sealed artifact truth.
