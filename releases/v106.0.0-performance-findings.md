# Receiz v106.0.0 Performance Findings

`v106.0.0` preserves Receiz scale law while coordinating SDK command admission, MCP constitutional context, AI-skill contracts, migration, and independent replay around one pinned registry digest. Constitutional enforcement remains deterministic local work beneath sealed proof objects and verified local truth.

## Findings

- Constitutional evaluation is bounded by an explicit operation budget and cannot introduce network or database calls into local SDK conformance.
- Command admission, causal replay, atomic projection, signed receipt checks, and independent verifier replay remain deterministic for the same registry, command, proof state, and Kai coordinates.
- The independent verifier is separately pinned at `106.0.0`; replay evidence cannot promote registry payloads, MCP output, AI-skill output, migration reports, or release evidence into public proof authority.
- A canonical global profile row may enrich owner-local truth but cannot short-circuit PBI signature admission or Live Player projection for the newest accepted proof object.
- Known verified proof objects, PBI authorship, Showcase, and Live Player state paint from stronger known truth; background reconciliation may append only.
- Coordinated package and registry-digest checks fail closed on version skew, unsupported schemas, silent fallback, direct writes, missing tenant binding, missing idempotency, missing expected revision, and absent audit receipts.
- Publication, deployment, production database state, and authenticated production-browser evidence remain external actions and are not inferred from local qualification.

## Performance Boundary

- Known verified truth must paint before hydration, analytics, SEO, session checks, lazy imports, global sync, chart hydration, ledger pagination, or server reconciliation.
- Registry evaluation, SDK compilation, MCP context injection, AI-skill validation, migration planning, and independent replay remain weaker mechanics beneath proof identity, ownership, settlement, and append-only history.
