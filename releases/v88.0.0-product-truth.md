## Frozen Invariants

- Offline proof remains proof.
- Compact `/v` transport remains transport, not authority.
- Sports selected-game field state is not disposable UI cache.
- `/baseball` is a direct entry alias to the canonical baseball arena, not a second implementation.
- A selected live game with detailed field truth must not drop to fallback when switching games or hydrating.
- Score plate subtext is base state or countdown.
- Pitch-by-pitch belongs in the center field recap and must come from the real detailed live field state.
- Last play belongs in the bottom ticker.
- Batter and pitcher cards must be real live participants when known.
- Single-game locked entries must paint locked on first paint when durable proof/local truth exists.
- Individual player scores must not be zeroed, hidden, or understated by partial append hydration.
- Reward-ready/final text must not replace active live field state.
- Public Live Cockpit is a public witness/live proof surface.
- Public profile canonical truth remains `/<username>`; `?manage=1` is permission overlay only.
- Market/profile proof objects must not disappear because a weaker pill, chart, owner, or RLS path refreshes.
- Runtime version coherence must not rely only on a manually supplied `NEXT_PUBLIC_SW_VERSION`.
- Explicit session clear must not leave stale runtime-scoped durable session keys.
- Supabase timeout hardening is part of release readiness; broad advisor/linter automation against production remains forbidden unless explicitly scoped and approved.
- Regression lessons are release-lock material, not optional notes.
