# Receiz v107.0.0 release notes

## Coordinated release

`@receiz/sdk`, `@receiz/mcp-server`, and `@receiz/ai-skills` ship at exactly `107.0.0`. Receiz.com, runtime projection storage, public version labels, the service worker, capability ranges, ruleset, registry, manifests, and generated integrations use the same release identity.

## Identity and profile

- `identity.getProfile()` reads the canonical identity/profile proof head.
- `identity.checkUsernameAvailability()` is advisory and never reserves a name.
- `identity.updateProfile()` binds the same Receiz identity, expected head, idempotency key, profile fields, and optional avatar into one operation.
- Renaming preserves the original identity. A successful commit releases the prior username globally.
- Avatar input supports a raw `Blob`/`File` or an existing `receiz.media.proof-reference.v107` object.
- The frontend verifies the canonical receipt and matching head before reading the public profile projection.

## Portable continuity and app state

- `identity.restoreAccount()` verifies a Receiz Key, Identity Record, Identity Seal, or verified Vault enclosure and immediately projects carried truth without a network call.
- `identity.appendAccountState()` appends allowed app namespaces after an expected proof head and returns updated exportable artifact bytes.
- `continuity.reconcile()` produces a read-only digest-bound plan; `continuity.commit()` applies the exact owner-scope transition idempotently.

## Generic bearer ownership

`ownership.claimBearerAsset()` handles every qualifying Receiz bearer artifact. The enclosing artifact is verified first, claimant key control is verified, the ownership head is compared, one ownership append commits, new sealed artifact bytes are returned, and the prior owner projection becomes inactive only after commitment.

## Offline commands

`offline.executeOrQueue()` signs the exact command digest with the carried identity. Queues preserve FIFO causal order per proof object, permit independent proof-object lanes to resume in parallel, stop a conflicted lane deterministically, and verify committed receipts before projection. Queued work is never described as globally committed.

Every skill is bound to the exact v107 registry digest.
