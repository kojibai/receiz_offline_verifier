# Receiz v47.0.0 Product Truth

Release date: April 5, 2026

This document freezes the primitives and invariants that `v47.0.0` is allowed to extend but not weaken.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| Authoritative file bytes | Verification authority remains the original file bytes plus versioned proof material. Online and offline verification are convenience surfaces around the same artifact truth. | `docs/receiz-standard-v1.md`, `scripts/test_receiz_artifact_binding.mjs`, `scripts/test_receiz_document_seal.mjs`, `scripts/test_document_seal_zk.mjs`, `scripts/test_receiz_signature_v4.ts` |
| Deterministic historical manifests | Canonical history resolves from published deterministic manifests, not mutable client state or augmentation overlays. | `docs/verified-history-first-principles.md`, `app/lib/profile/__tests__/deterministicFeed.test.ts` |
| Truthful deterministic routes | Deterministic trust surfaces never paint fake loading state; they render real server truth, a real snapshot, or `null` until truth commits. | `docs/deterministic-surfaces.md`, `scripts/test_deterministic_surfaces.ts` |
| Document and original delivery | Profile/original delivery keeps inline document reading open, preserves explicit original-vs-derivative behavior, and prefers the stronger delivery path when both exist. | `app/lib/profile/__tests__/document.test.ts`, `app/lib/profile/__tests__/documentDownloadPrecedenceContract.test.ts` |
| Byte-range object delivery | Signed object delivery must preserve range and `HEAD` semantics so large audio/video/media flows stay truthful under streaming access. | `app/api/storage/object/__tests__/route.test.ts` |
| Public Twin preview lock | Public Twin presentation resolves from the freshest render-locked preview and canonical source hierarchy instead of stale source-only payloads. | `app/lib/world/__tests__/worldAvatarFreshnessContract.test.ts`, `app/lib/world/__tests__/worldProfileRouteContract.test.ts` |
| Face-feature authority | Photoreal shell alignment uses explicit authority metadata and server fallback paths instead of silent face-rig guesswork. | `app/lib/world/__tests__/worldFaceFeatureAuthorityContract.test.ts`, `app/lib/world/__tests__/worldTwinPhotorealFitContract.test.ts`, `app/lib/world/__tests__/worldTwinPhotorealShellContract.test.ts` |
| Release-governed version surfaces | Package version, release docs, public version badges, and service worker defaults must move together on each release. | `scripts/check_release_lock.mjs` |

## Frozen Invariants

| Invariant | What must not regress | Locked by |
| --- | --- | --- |
| No fake deterministic loaders | Spinner, skeleton, placeholder-card, and correction-frame UI stay banned on deterministic routes. | `scripts/test_deterministic_surfaces.ts` |
| Canonical history cannot be downgraded | Weaker partial payloads, augmentation, or live deltas must not replace a stronger canonical block. | `app/lib/profile/__tests__/deterministicFeed.test.ts` |
| Profile route continuity stays primed | Viewer and live-player routes warm deterministic hydration and route prefetch before navigation. | `app/lib/profile/__tests__/profileRouteContinuityContract.test.ts` |
| Document readers stay inline-first | Rich document surfaces open inline and do not silently collapse to attachment-only delivery. | `app/lib/profile/__tests__/document.test.ts` |
| Derivative-before-moment-record precedence stays stable | When both are available, derivative document downloads stay ahead of moment records. | `app/lib/profile/__tests__/documentDownloadPrecedenceContract.test.ts` |
| Stage and dock stay separate | Public Twin keeps chat-stage rendering and dock-rig rendering on separate paths. | `app/lib/world/__tests__/worldTwinStageSeparationContract.test.ts` |
| Live video stays real | The live stage only renders video when a real worker delivery or live session exists. | `app/lib/world/__tests__/worldTwinLiveSessionContract.test.ts`, `app/lib/world/__tests__/worldTwinStageSeparationContract.test.ts` |
| Photoreal shell stays authority-aware | Mouth, blink, breath, overlay fit, and rendered-portrait alignment stay driven by explicit shell primitives and feature-map authority. | `app/lib/world/__tests__/worldTwinPhotorealShellContract.test.ts`, `app/lib/world/__tests__/worldTwinPhotorealFitContract.test.ts`, `app/lib/world/__tests__/worldTwinRigPhotoHeadContract.test.ts` |
| Public world views stay sanitized | Public world/profile payloads must not leak internal persona, owner, treasury, or route internals. | `app/lib/world/__tests__/publicProfileContract.test.ts` |
| Release surface inventory stays canonical | The README product-surface inventory and the committed canonical route files must remain aligned. | `scripts/test_product_surface_inventory.ts` |
| Canonical artifact truth loop stays closed | A fixed artifact must preserve one canonical record -> seal -> verify chain, and a tampered artifact must fail trusted verification. | `scripts/test_receiz_release_golden_path.ts` |

## Release Gates

`v47.0.0` is considered locked only if these pass together:

- `pnpm test:release-freeze`
- `pnpm test:release-golden-path`
- `pnpm test:signature-v4-server-remote`
- `pnpm release:attest`
- `pnpm release:verify-attestation <path>`

The manual release procedure is defined in `docs/releases/v47.0.0-checklist.md`.

## Residual Risks

The following areas are materially stronger after this release but are not yet absolute:

- Not every first-class surface has the same anti-downgrade depth already present in profile history.
- Release attestation remains a release-workflow gate rather than a local `release:lock` requirement.
