## Title

Receiz `v92.0.0` - Receiz Logic Release

## Summary

`v92.0.0` locks the post-`v91.0.0` reasoning shift: Receiz pages, vault surfaces, public proof routes, settlement surfaces, and game surfaces now optimize by starting from implemented Receiz primitives instead of legacy app layers.

The documented committed range is `v91.0.0..45287cbd5`, ending before the v92 release-lock edits. That range spans `55` commits, `239` files changed, `21,980` insertions, and `3,883` deletions. The mechanical ledger is preserved in `docs/releases/v92.0.0-commit-history.md`.

The release statement is:

```txt
Receiz gets faster when it becomes truthful earlier.
Primitive first.
Strongest truth first.
Weaker layers append, index, hydrate, sync, decorate, or project beneath that truth.
They do not negotiate product reality.
```

`v92.0.0` is the adoption release for that logic.

## What v92 Locks

### Receiz Reasoning Kernel

`v92.0.0` adds `docs/receiz-reasoning-kernel.md` and links it into `AGENTS.md` and `docs/literal-product-law.md`.

The kernel makes the default loop explicit:

```txt
name the primitive
name the strongest truth
name weaker layers
name the forbidden downgrade
name the append path
name the contract or visual evidence
```

This is not a new authority layer. It compresses existing Receiz law so future work starts from proof, artifact, ownership, settlement, identity, public witness, verified append, durable local truth, and deterministic first paint before it touches route, DB, API, session, cache, component state, or hydration.

### Profile And Live Player First Paint

The post-v91 arc found and removed signed-in-only first-paint regressions.

The wrong behavior was:

```txt
signed-in route work -> session/private profile/API/world hydration -> public player shell
```

The correct behavior is now:

```txt
deterministic public proof/player truth -> settled public surface -> session/private/world enrichment
```

The live player no longer waits for private signed-in enrichment, player API hydration, route loading boundaries, or world twin preparation before showing the public proof/media surface. Signed-in and signed-out viewers must first-paint the same public proof truth.

That lesson applies to profile routes too:

```txt
/<username> is canonical public profile truth
/<username>?manage=1 adds owner permissions
owner state cannot redefine the base profile shell
```

### Vault And Ownership Surfaces

Sports vault work in this range made ownership truth more coherent without making DB state outrank artifact truth.

The release locks these rules:

- vault objects remain artifacts with identity, media, provenance, custody, history, and action affordances
- transfer-aware filtering prevents stale owner rows from projecting custody
- deferred vault admission can run after reveal without blocking the proof reward surface
- fast-entry snapshots and route hydration may accelerate vault entry but cannot replace stronger artifact truth
- inspected seal append rows merge scoped event proofs, transient entries, and scoped appends before seal projection
- local save is the direct blob path; server staging is not the proof authority for user-held artifacts

Ownership rows, route hydration, and deferred admission are mechanics beneath the vault object.

### Sports Value And Settlement Surfaces

The value panel work established a precise settlement-surface rule:

```txt
events may append beneath or beside value truth
events may scroll
events may pop above when explicitly opened
events may not consume or obscure the chart bottom
```

The single-card value panel now protects the deterministic value chart while allowing the full append/event memory pane to scroll. The final release-lock edit raises only the card-viewer event proof desk above the fixed value panel so the selected event proof remains clickable and visible when opened from the value panel.

The release keeps the event proof desk itself unchanged. Only the card-viewer stacking relationship changes.

### Append Truth And DB Indexing

The `source_date` incident is the cleanest v92 logic example.

The bad path selected `sports_arena_card_appends.source_date`, but the table does not have that physical column. The source date lives in append proof payload truth:

```txt
append_json.sourceProof.sourceDate
```

The correct fix was not a migration. The correct fix was to select the append proof payload and project source date from the existing proof material.

Rule:

```txt
DB columns may index append truth.
DB columns do not become append truth.
```

### Source Age And Live Game Truth

Source age now has coherent no-live-game behavior. When no game is live, the pulse can show `NEXT GAME` with a countdown from the existing live-feed scoreboard and next-game hint instead of rendering incoherent age state.

Rule:

```txt
freshness UI must project the next truthful source state
not a meaningless live-only label
```

### Visual Evidence As Product Truth

This release arc was shaped by visual evidence:

- the Derby plate rendered as a misaligned second plate
- the pitcher appeared to stand on second base instead of the mound
- the value chart bottom was consumed by events
- the event proof desk rendered behind the value panel
- signed-in live-player routes could show the wrong first-paint boundary

Those were not styling opinions. They were product-truth contradictions. The correct response was to strengthen code and contracts around the primitive, not to reinterpret the evidence.

## What We Learned

### Receiz Logic Beats Legacy Engineering

Legacy app logic starts with:

```txt
route -> session -> DB -> API -> cache -> component state -> hydration -> UI
```

Receiz logic starts with:

```txt
primitive -> strongest truth -> weaker layers -> append path -> contract -> visual evidence
```

That second chain produced smaller, stricter fixes:

- no DB migration for `source_date`
- no new value panel for event proof stacking
- no session wait before public proof first paint
- no chart shrink to solve event overflow
- no route-private truth replacing public profile truth
- no local proof being treated as stale cache

### Speed Is Authority Routing

The pages became faster because they stopped asking weaker layers for permission.

Profile and live player became faster when deterministic public proof truth painted before session/private work.

Vault became faster when snapshots and route hydration projected artifact truth instead of rebuilding full public proof context before entry.

Pitch Command became faster when admitted historical day proof painted immediately and live refresh could only append or refine.

Value became stable when event memory became scrollable beneath value truth.

Pack Derby became clearer when the visible geometry matched the gameplay truth.

### The Cleaner Logic Reduces Cognitive Load

The strict hierarchy removes false choices:

- proof payload beats missing DB field
- public proof beats signed-in private enrichment
- deterministic chart beats timer/accumulator
- admitted historical proof beats fresh rediscovery
- vault object beats ownership row
- visual evidence beats passing tests that do not name the primitive

When the hierarchy is explicit, the correct fix is easier to see.

## Release Boundary

`v92.0.0` does not replace `v91.0.0` scale law. It locks the reasoning layer that made the v91 hardening line coherent.

Future live streaming work belongs after this release boundary unless it is purely additive beneath existing proof surfaces. If live streaming becomes its own proof/live witness primitive, it should get its own release train.

`v92.0.0` is the clean break:

```txt
Receiz is not optimized by legacy app layering.
Receiz is optimized by submitting implementation mechanics to proof-native product truth.
```
