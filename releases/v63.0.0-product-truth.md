# Receiz v63.0.0 Product Truth

Release date: April 19, 2026

This document freezes the primitives and invariants that `v63.0.0` is allowed to extend but not weaken after the `v55.1.0` continuity constitution, the `v58.0.0` stable historical baseline, the `v60.2.0` calm home/profile correction line, and the current `62.0.2` runtime that this official release adopts.

## Release Gates

`v63.0.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `pnpm test:page-first-paint`
- `pnpm test:continuity-release-contracts`
- `pnpm test:deterministic-surfaces`
- `npx tsx --test app/lib/wallet/__tests__/valueStateContract.test.ts app/lib/wallet/__tests__/valueStateFramingContract.test.ts app/lib/market/__tests__/shareCertificateMarketFlowContract.test.ts app/lib/market/__tests__/marketConformanceRouteContract.test.ts`
- `pnpm build`
- `pnpm release:lock`
- `pnpm test:release-freeze`

The manual release procedure is defined in `docs/releases/v63.0.0-checklist.md`.

## Residual Risks

- `v58.0.0` remains the cleaner historical release before the later homepage/profile reload-bug family. `v63.0.0` is not claiming that history away; it is the governed current release built after that bug family was corrected.
- `v63.0.0` keeps more intentional machinery than `v58.0.0` or `v60.2.0`, especially in the hero open path and service-worker navigation path. That machinery is only acceptable while it stays under the continuity gates above.
- The server currently exposes settled wallet value as first-class value-state truth. Full withdrawable-now eligibility remains intentionally null until a real eligibility policy exists; this release must not fake that number in UI.
- Creator-owned asset issuance and creator reserve withdrawals are intentionally not included in this cut. They need a separate cap and reserve accounting model so creators cannot repeatedly withdraw the same live asset mark into new certificates.
- The production stale-PWA fix still relies on the new worker installing and activating naturally. `v63.0.0` does not use forced `skipWaiting` or mid-session reload behavior.
- `v63.0.0` is not fully release-cut until public version surfaces, service-worker defaults, offline verifier surfaces, package metadata, release docs, and the process record all move together to the same version.
