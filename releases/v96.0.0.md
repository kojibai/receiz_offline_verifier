## Title

Receiz `v96.0.0` - SDK A+ Release Evidence Lock

## Summary

`v96.0.0` takes the `v95.0.0` public table release and locks the developer surface to the same standard as the product: proof truth first, executable evidence second, release-blocking contracts always.

The core change is that `@receiz/sdk` is no longer only a typed client and documentation package. It now ships a `receiz` CLI that can inspect proof payloads, run local fixture conformance, and generate a local-first proof memory starter. The CLI uses existing SDK validators, projections, and durable proof memory. It does not create a new conformance authority, a new issuer, a new verifier, a new server confirmation layer, or a new database authority.

The release also promotes SDK package evidence into the umbrella release freeze. SDK tests, typecheck, build, pack dry-run, CLI conformance, published package contents, and no-authority-leak checks are now release-blocking in the broader platform workspace.

## What Changed Since v95.0.0

- `@receiz/sdk@96.0.0` ships the `receiz` CLI.
- `receiz conformance` validates packaged proof fixtures locally and reports zero network calls and zero DB calls.
- `receiz inspect <file>` projects a proof payload and prints its known Kai/proof head.
- `receiz init <dir>` writes a local-first durable proof memory starter.
- SDK proof memory exposes passive `ReceizSdkObservation` events through `onObservation`.
- Copy-paste developer integrations now cover React proof memory, browser proof admission, append-sync routes, webhook receivers, and Sports card renderers.
- `test:sdk-release-contracts` is wired into `test:release-freeze`.
- `@receiz/sdk` stays locked to the same release version as the app.

## Product Law Preserved

Receiz truth remains unchanged:

- Proof objects carry the truth.
- Kai Klok is the proof object state machine.
- Durable proof memory is not cache.
- The operating rule is first admission only, then append forever.
- Connect is delegated API access, not the identity proof root.
- Server, database, SDK, and CLI surfaces are weaker than admitted proof truth.
