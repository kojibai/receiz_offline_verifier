# Receiz v52.5.0 Product Truth

Release date: April 13, 2026

This document freezes the primitives and invariants that `v52.5.0` is allowed to extend but not weaken after the first market-governed release in `v52.0.0`.

## Frozen Primitives

| Primitive | Locked truth | Primary enforcement |
| --- | --- | --- |
| File-authoritative trust core | The file remains the authority. Deterministic manifests remain the authority for route truth, and ledger-backed settlement remains the authority for payment truth. Market execution operates around those boundaries rather than replacing them. | `app/lib/profile/document.ts`, `app/lib/profile/server.ts`, `app/lib/receiptView.ts`, `app/lib/wallet/payIntent.ts`, `app/api/wallet/ledger/public/route.ts` |
| Stable public ladder | Public plan language remains stable. Market may deepen the product, but it must not reintroduce pricing or ladder confusion. | `app/upgrade/page.tsx`, `app/plans/PlansClient.tsx`, `app/lib/premium/plan.ts` |
| Deterministic market quote engine | Asset execution resolves through one canonical quote model that computes reference mid, quote mid, bid, ask, spread, and shift from deterministic inputs. | `app/lib/market/shareQuoteModel.ts`, `app/lib/market/shareTrade.server.ts`, `app/lib/market/shareFees.ts` |
| Spread as quote behavior, not additive UI fee | Market sustainability is achieved through the live quote surface and bounded spread math, not a separate visible runtime fee stacked on execution. | `app/lib/market/shareTrade.server.ts`, `app/lib/market/shareFees.ts`, `supabase/migrations/20260413153000_market_share_trade_fees.sql` |

## Release Gates

`v52.5.0` is considered locked only if these pass together:

- `pnpm exec tsc --noEmit --pretty false`
- `pnpm test:market-conformance`
- `pnpm test:release-freeze`
- `pnpm release:lock`
- `pnpm run build`
- `pnpm release:attest`
- `pnpm release:verify-attestation <path>`

The manual release procedure is defined in `docs/releases/v52.5.0-checklist.md`.

## Residual Risks

- Market conformance history and rollups are still thinner than a full persisted long-term warehouse even though the standard surface is now live.
- The lightweight market-pill path is materially better than the prior full-snapshot dependency, but dense feeds may still benefit from additional batching in a later release.
- Quote math materially improves sustainability, but operator safety still depends on the hard-cap/exposure model staying active in runtime and production data.
- Inline launch now uses real server primitives directly from the market panel, so regressions in those server routes would now show up inside market launch immediately instead of being masked by route handoff.
