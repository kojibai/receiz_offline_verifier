# Receiz v97.0.0 Commit History

Release date: 2026-06-30

Baseline: `v96.1.0`

Commit count: `4 committed changes after v96.1.0 plus the v97.0.0 release-lock edits`

This `v97.0.0` release-lock state captures the developer ecosystem work after `v96.1.0`:

- Fixed idempotent Receiz projection behavior so canonical absolute verification URLs and owner-scoped lookup remain aligned.
- Updated admin performance analytics to measure user-visible app experience instead of misleading internal/admin route scores.
- Repaired OIDC consent redirect behavior and button readability so Connect authorization remains usable in third-party app flows.
- Added the app-state public projection rail for durable public app-state publishing and recovery.
- Added `@receiz/sdk` app-state clients for publish and public reads.
- Added `publishRecord()` and `byHost()` SDK convenience so builders and AI agents can launch advanced apps without hand-assembling the common public app-state envelope.
- Documented app-state projection quickstarts, backend behavior, OpenAPI/status catalog routes, and release law.
- Bumped public app, service-worker/runtime, public labels, and SDK release identity to `v97.0.0`.
- Updated this offline verifier checkout's release docs, version markers, service-worker release truth, and local release-lock surfaces for `v97.0.0`.

The release lesson from `v96.1.0` remains intact: public developer contracts must never understate internal proof truth. `v97.0.0` extends that lesson into public app-state recovery. Developers receive typed convenience. Receiz truth remains proof-native, append-only, locally durable, and governed by Kai/Klok state. The durable proof memory law remains first admission only, then append forever.

## Release-Lock Requirement

`pnpm release:lock 97.0.0` must require this document for `v97.0.0` and later release cuts when the release brief includes a commit-history artifact. A release that omits the commit-history boundary is incomplete because the v96.1 baseline, public app-state projection rail, SDK app-state clients, OIDC consent repair, public proof projection fixes, admin performance cleanup, package evidence, and repo-local release-lock edits would not be explicit release truth.
