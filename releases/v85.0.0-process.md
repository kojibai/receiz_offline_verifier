# Receiz v85.0.0

## Goal

Ship `v85.0.0` as the official release for the complete v80-to-v85 arc:

```txt
proof-native artifact system
deterministic first paint
Sports live proof economy
Market and wallet deterministic ownership
instant local buy/sell settlement projection
Series Passport proof object and annual skill final access
affiliate sharing through the implemented referral system
coherent release identity across package, service worker, docs, and public surfaces
```

The goal is not only to bump a number. The goal is to freeze the product truth that now exists in the code and contracts.

## What Worked

- The release got stronger when the work followed Receiz truth hierarchy instead of generic app convention.
- Signal Card history became useful once card DNA, transfers, rewarded purchases, wallet activity, and calendar activity were treated as provenance.
- Sports became coherent when card ownership, issue number, rarity, player score, event proof, vault, trade, and settlement were kept in one artifact loop.
- Main Event became reliable when closeout, refund, payout, and stale-lock cleanup moved onto launch/read paths instead of waiting only for workers.
- Live scoring became legible when Pick 5, Main Event, single-game, cockpit, and leaderboard scores used scoped context.
- Profile and Market speed recovered when canonical/deterministic first paint stopped waiting for owner/session/server enrichment.
- Market ownership recovered when positions and owned assets came from deterministic ownership/settlement truth instead of waiting for a weaker snapshot.
- Buy and sell speed recovered when local execution projected immediately and global sync hydrated afterward.
- Wallet popovers became correct when they reused the same ownership/history sources as the rest of the Market and wallet surfaces.
- Locally settled claimed positions stayed correct when lagging snapshots were prevented from reopening them.
- Series Passport landed cleanly because it reused existing proof object, settlement, local register, verification, and affiliate primitives instead of inventing a second authority.
- The Series payout model stayed credible when it reused the Main Event split and made the pool math visible.
- The final Arena strip fix worked because it corrected only the leaked labels and contract, leaving the established surface intact.
- Release lock is valuable because package version, service worker version, public labels, release docs, and product truth now move together.

## What Did Not Work

- Treating DB/session/server state as a source of truth caused speed and continuity regressions on Market and Profile.
- Signed-in views became slower when private enrichment was allowed to outrank the same truthful shell signed-out users could see.
- Snapshot lag could reopen settled state when the merge order did not protect newer local settlement truth.
- Executed sells looked wrong when sold shares remained visible until endpoint hydration caught up.
- Buttons got stuck when UI busy state stayed tied to remote completion after local execution had already changed truth.
- Series language drifted when internal product terms, leaked prize-pool labels, or cryptic seal copy appeared where direct Passport outcome language was needed.
- The Series simulator felt wrong until the payout split, 100,000 Passport default, top-30% distribution, and existing affiliate system were treated as one system.
- Mobile Series density was too tall until the same layout was compressed instead of redesigned.
- Input focus zoom on mobile was a browser behavior problem; product language could not solve it.
- Any implementation that makes known truth wait for rediscovery creates a product regression even when the endpoint eventually returns correct data.

## Practical Rule For Future Work

Use this order:

```txt
name the primitive
identify the strongest source of truth
paint known truth first
append verified additions after
let global sync hydrate behind the shell
never let weaker snapshots reopen stronger local settlement
reuse implemented affiliate/proof/identity/settlement primitives
write the contract that fails if the law regresses
```

For Market and wallet:

```txt
deterministic ownership
local settlement execution
visible shares/assets/history update
button state release
global sync hydration
equal-or-newer authoritative snapshot
```

For Sports:

```txt
owned card
lineup context
score context
event proof
closeout truth
settlement truth
vault history
market/trade action
```

For Series Passport:

```txt
Passport proof object
local admission
annual skill final access
rank path
payout math
affiliate-linked sharing
global sync append
```

For releases:

```txt
package version
service worker version
public labels
release note
product truth
checklist
process
release lock
focused contracts
production attestation
```

The `v85.0.0` rule is:

```txt
If Receiz already knows stronger truth, display it immediately. Sync can follow. It cannot make truth wait.
```
