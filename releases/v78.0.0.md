## Title

Receiz `v78.0.0` - Signal Card World, Vault Export, Battle Arena, Exchange, And PWA Continuity Release

## Summary

`v78.0.0` locks the work after `v77.0.0` into a release centered on turning Signal Circuit from a newly shipped collectible system into a smoother owned-asset network. The release keeps the v70 profile law, v74 identity-artwork law, v75 local-proof/file-restore law, v76 World/Market/Public Proof law, and v77 Signal Circuit/Creator OS/Vault law intact while hardening the card world, sealed export, mobile PWA download path, battle arena, public exchange page, conformance surfaces, dashboard route, and release-worker version coherence.

This is not a label-only bump. The diff from `v77.0.0` to the pre-bump v78 candidate contains 57 commits, 88 changed files, 11,211 insertions, and 2,600 deletions. The work includes Signal Card front/download refinements, QR and external share repairs, real OG/Twitter card imagery, global Vault discovery sync, account-scoped card export, transfer-download packages, user-intent-only mobile download behavior, Signal Arena requests and settlements, auction wallet holds, public `/signal-circuit` and `/exchange` routes, dashboard first-paint simplification, PWA header/service-worker fixes, conformance source repairs, and a release-surface worker marker for `v78.0.0`.

## What This Release Locks

- Signal Card downloaded fronts, share pages, share actions, QR payloads, and OG/Twitter image routes now describe and carry the sealed card more directly without detaching proof from ownership.
- Signal Card World at `/game/cards` no longer depends only on a user's local Vault rows. It can read synced Vault ownership, merge local IndexedDB/localStorage cards, preserve owner scope, and publish eligible local batches without rewriting card truth.
- Global Signal Vault sync moves server-known owner rows into the public discovery layer so card discovery can show current ownership, sale panels, auction paths, challenge paths, and proof state without a showcase gate.
- Signal Vault Card export is a real receized PNG carrier for a card archive. It embeds a manifest, archive hash, chunked archive payload, Merkle integrity, KAI timing, hidden/active roster counts, card claim paths, art paths, and restore rules.
- Vault card restore and download behavior is explicit user intent. Opening the Vault panel must not start sealing, show a timer, begin background export, or navigate the user through a hidden frame.
- Mobile and installed PWA card downloads use safer click/download paths so Safari and in-app browsers do not turn card export into a page navigation, refresh, or header collision.
- Transfer-download packages move ownership through an explicit transfer package instead of leaving sender and receiver with ambiguous duplicate ownership state.
- Signal Arena adds battle request, queue, loadout, AI twin authorization, settlement, XP/money/fun/keeps stake modes, action-ledger append, transfer integration, and card append verification without mutating the original card claim.
- Card auctions add wallet-hold backing and route repairs so auction settlement cannot float away from reserve/account state.
- `/signal-circuit` becomes a public standard page for earned cards becoming owned assets. It makes the card law inspectable: earn, own, find, move, hold, and use all resolve against the same card record.
- `/exchange` becomes the public routing page that ties Circuit, Market, settlement, wallet, payments, transfers, ledger, offline notes, card discovery, and conformance into one ownership/value story.
- Signal Circuit conformance sources, badges, route inventory, and public status entries were repaired so public assurance covers the real files and excludes high-risk private/worker/admin paths.
- Dashboard route rendering is split so the route can first-paint through a smaller server/page boundary while dashboard data loads through a dedicated client surface and API route.
- Service-worker registration, PWA header behavior, and runtime versioning were patched so installed or private-browser sessions do not keep stale worker/header state in the user's way.
- Bottom chrome, global auth chrome, chat dock, composer host, profile/card CSS, discovery controls, first-paint card artwork, and public card popover/footer placement were tightened around mobile and PWA use.
- Value-loop copy contracts were added so public value claims stay tied to reserve, settlement, ownership, and proof instead of drifting into generic marketing copy.
- Release surfaces now move together: package version, service-worker defaults, public badges, offline verifier pages, README pointer, v78 release docs, and world surface-worker release markers all identify `v78.0.0`.

## Developer And Public Value

- Users get a Signal Card World that behaves like an owned-asset network, not a local inventory page: global sync, owner rows, discovery, sale, auction, battle, and transfer all stay attached to sealed card truth.
- Card holders get explicit, recoverable export and transfer paths that do not start work without a click and do not turn mobile/PWA download into a visible app reload.
- Developers get conformance and status surfaces that now reflect the actual Circuit, Exchange, card discovery, and public/private boundary code paths.
- Operators get stronger release contracts around card discovery, mobile download intent, battle arena, dashboard first paint, service-worker navigation, value-loop claims, auction holds, and public surface inventory.
